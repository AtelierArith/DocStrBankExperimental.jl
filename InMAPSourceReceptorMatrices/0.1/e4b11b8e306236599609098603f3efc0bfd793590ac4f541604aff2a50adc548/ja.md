```
SRM(emis_type) -> srm::Array{Float32, 3}
```

`fs`から取得したソースレセプターマトリックスを返します。`emis_type` ∈ ("PrimaryPM25", "SOA", "pNO3", "pSO4", "pNH4")

これらは非常に大きく（52411 x 52411 x 3）、保持するために約60-70GBのRAMを必要とします。インデックスは`(receptor, source, layer)`を介して行います。メモリに複数保存する場合は注意が必要です。AWSから取得するのに5-10分かかることがあります。

特定のソースロケーションのグループのみのスパースSRMを作成するには、[`make_sparse_srm`](@ref)を参照してください。（これはまだ`SRM`を呼び出しますが、スパースマトリックスを返す前にガーベジコレクションを呼び出します）
