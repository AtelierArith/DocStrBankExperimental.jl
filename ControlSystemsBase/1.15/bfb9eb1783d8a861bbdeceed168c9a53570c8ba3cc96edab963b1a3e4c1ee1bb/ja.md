```
C = pid(param_p, param_i, [param_d]; form=:standard, state_space=false, [Tf], [Ts], filter_order=2, d=1/√(2))
```

PIDコントローラを計算して返します。

`form`は次のいずれかを選択できます（引数`param_p, param_i, param_d`の解釈方法を決定します）

  * `:standard` - $K_p(1 + 1/(T_i s) + T_d s)$
  * `:series` - $K_c(1 + 1/(τ_i s))(τ_d s + 1)$
  * `:parallel` - $K_p + K_i/s + K_d s$

`state_space`が`true`に設定されている場合、`Kd`はゼロであるか、フィルタを作成するために正の`Tf`を提供する必要があります。バランスの取れた状態空間の実現が返されますが、`balance = false`の場合は除きます。

## フィルタ

`Tf`が供給されると、フィルタが追加され、使用されるフィルタは次のいずれかです。

  * `filter_order = 2`（デフォルト）：$1 / ((sT_f)^2/(4d^2) + sT_f + 1)$がコントローラと直列に接続されます。注意：このフィルタのパラメータ化は、一般的なパラメータ化$1/(s^2 + 2dws + w^2)$とは異なる動作をします。前者は、`d`が変化してもほぼ固定された*帯域幅*を維持しますが、後者は極の原点からの固定距離を維持します。
  * `filter_order = 1`：$1 / (1 + sT_f)$が微分項のみに適用されます。

$$
T_f
$$

は通常、PIコントローラの場合は$T_i/N$、PIDコントローラの場合は$T_d/N$として選択され、`N`は一般的に2から20の範囲です。2次フィルタでは、`d`が減衰を制御します。`d = 1/√(2)`は極のバターワース構成を提供し、`d=1`は臨界減衰フィルタ（オーバーシュートなし）を提供します。`d`が1を超える場合も使用できますが、`d > 1`はますますオーバーダンプされたフィルタを生成します（このパラメータ化は、極を原点に送ることはありません$d → ∞$、$(ω,ζ)$パラメータ化のようにはなりません）。

## 離散時間

離散コントローラの場合、正の`Ts`を供給できます。この場合、連続時間コントローラはタスティン法を使用して離散化されます。

## 例

```
C1 = pid(3.3, 1, 2)                             # Kd≠0はtf形式でフィルタなしで動作します
C2 = pid(3.3, 1, 2; Tf=0.3, state_space=true)   # 状態空間ではフィルタが必要です
C3 = pid(2.,  3, 0; Ts=0.4, state_space=true)   # 離散
```

関数`pid_tf`と`pid_ss`もエクスポートされています。これらは同じパラメータを取り、`state_space`パラメータに基づいて`pid`で実際に呼び出されるものです。入力`[r; y]`と出力`u`を持つ2DOFコントローラについては、[`pid_2dof`](@ref)も参照してください。
