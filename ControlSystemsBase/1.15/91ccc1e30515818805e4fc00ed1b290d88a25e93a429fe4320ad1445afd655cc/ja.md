```
C = pid_2dof(param_p, param_i, [param_d]; form=:standard, state_space=true, N = 10, [Ts], b=1, c=0, disc=:tustin)
```

2DOF形式のPIDコントローラを計算して返します。入力は`[r; y]`、出力は`u`で、`r`は参照信号、`y`は測定出力、`u`は制御信号です。

以下に、コントローラの2つの異なる描写を示します。左側は2入力システムとして、右側はコントローラの2つの内部SISOシステムを示しています。

```
                                ┌──────┐                      
                             r  │      │                      
                            ───►│  Cr  ├────┐                 
r  ┌─────┐     ┌─────┐          │      │    │    ┌─────┐      
──►│     │  u  │     │ y        └──────┘    │    │     │ y    
   │  C  ├────►│  P  ├─┬─►                  +───►│  P  ├─┬───►
 ┌►│     │     │     │ │        ┌──────┐    │    │     │ │    
 │ └─────┘     └─────┘ │      y │      │    │    └─────┘ │    
 │                     │     ┌─►│  Cy  ├────┘            │    
 └─────────────────────┘     │  │      │                 │    
                             │  └──────┘                 │    
                             │                           │    
                             └───────────────────────────┘     
```

`form`は以下のいずれかを選択できます（引数`param_p, param_i, param_d`の解釈方法を決定します）

  * `:standard` - $K_p(br-y + (r-y)/(T_i s) + T_d s (cr-y)/(T_f s + 1))$
  * `:parallel` - $K_p(br-y) + K_i (r-y)/s + K_d s (cr-y)/(Tf s + 1)$
  * `b`は比例項のセットポイント重み付けです。
  * `c`は微分項のセットポイント重み付けで、デフォルトは0です。
  * `b`と`c`の両方がゼロに設定されている場合、コントローラのフィードフォワードパスは厳密に適切になります。
  * `Tf`は微分項のフィルタの時間定数で、デフォルトは`Td/N`で、`N`は10に設定されています。`Tf`を渡す代わりに、`N`を直接渡すこともできます。比例項はこのフィルタの影響を受けません。**注意してください**: この微分フィルタは、`pid`関数で使用されるものとは異なり、フィルタは2次であり、コントローラと直列に適用されるため、すべての3つのPID項に影響を与えます。
  * PDコントローラは、`param_i`をゼロに設定することで構築されます。
  * バランスの取れた状態空間実現が返されますが、`balance = false`の場合は除きます。
  * `Ts`が提供されると、コントローラは`disc`メソッドを使用して離散化されます（デフォルトは`:tustin`です）。

このコントローラには負のフィードバックが組み込まれており、`r`から`y`への閉ループシステムが形成されます。

```
Cr, Cy = C[1, 1], C[1, 2]
feedback(P, Cy, pos_feedback=true)*Cr                    # 代替案 1
feedback(P, -Cy)*Cr                                      # 代替案 2
feedback(P, C, U2=2, W2=1, W1=[], pos_feedback=true) # 代替案 3、見た目は劣りますが効率的で、より小さな実現を返します
```
