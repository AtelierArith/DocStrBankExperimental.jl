多次元データ（通常は行列）と関連する次元の境界を保持するための一般的な型です。その主な目的は、内部でワールド座標とグリッドインデックスの間の変換を処理することです。これらの2つの表現間の変換は、行を最初の変数（x軸）、列を2番目の変数（y軸）として扱います。

その典型的な使用法は、サンプリング可能な値の2Dマップとして機能することです。GridMapは、与えられた点が含まれるグリッドセルの値を返します。言い換えれば、マップの値は各セル内で一定です。これは最近傍近似として考えることもできます。

各セルインデックスは、そのセルの中心として扱われます。したがって、マップの下限は最初のセルの中心にあり、マップの上限は最後のセルの中心にあります。

また、基本メソッドを実装してサブタイプ化することにより、組み込みのN次元配列のように直接機能するように作られています。

フィールド:

  * `data::AbstractArray`: データのN次元配列
  * `bounds::@NamedTuple{lower::Vector{Float64}, upper::Vector{Float64}}`: 下限と上限のベクトル、デフォルトはゼロと1

GridMapは、データの多次元配列と次元の境界を渡すことによって構築されます。境界が渡されない場合、下限はゼロ、上限は1にデフォルト設定されます。

# 例

```julia
data = reshape(1:25, 5, 5)
bounds = (
    lower = [0.0, 0.0],
    upper = [1.0, 1.0]
)
gmap = GridMap(data, bounds)
gmap2 = GridMap(data) # bounds will be zero to one
```
