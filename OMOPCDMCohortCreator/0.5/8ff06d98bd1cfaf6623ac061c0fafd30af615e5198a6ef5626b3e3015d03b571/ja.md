GetPatientAgeGroup(     ids, conn;     minuend = :now,     age*groupings = [         [0, 9],         [10, 19],         [20, 29],         [30, 39],         [40, 49],         [50, 59],         [60, 69],         [70, 79],         [80, 89],     ],     tab = person,     ungrouped*label = "Unspecified" )

`age_groupings`で指定された年齢層のすべての個人を見つけます。

# 引数:

  * `ids` - `person_id`のリスト; 各IDは`Integer`のサブタイプでなければなりません
  * `conn` - DBInterfaceを使用したデータベース接続

# キーワード引数:

  * `age_groupings` - 年齢層のベクターで、形式は`[[10, 19], [20, 29],]`であり、それぞれ10 - 19歳および20 - 29歳の年齢層を示します; 年齢値は`Integer`のサブタイプでなければなりません
  * `minuend` - 患者の`year_of_birth`変数から引かれる年; デフォルトは`:now`です。設定できるオプションは2つあります:

      * `:now` - コードが実行される日のUTC時間での年
      * ユーザーが提供する任意の年（2022年、1998年など）で、`Integer`である限り
  * `tab` - Personテーブルを表す`SQLTable`; デフォルトは`person`
  * `ungrouped_label` - 提供された一致する年齢層に適合しない人に割り当てるラベル; デフォルトラベルは"Unspecified"

# 戻り値

  * `df::DataFrame` - `:person_id`と`:age_group`の列からなる2列の`DataFrame`

# 注意

年齢は一貫して計算するのが難しい場合があります。この場合、一貫性を確保するためにいくつかの仮定がなされています:

1. OMOP CDM v5.4によれば、特定の患者に対して保証されているのは`year_of_birth`変数のみです。これは年齢計算における minuend として使用される3つのオプションの1つです。
2. 減数は`minuend`キーワード引数に対して選択したものに基づいています。

年齢は、1と2に基づいて選択されたものに従って計算されます。この柔軟性は、ユーザーが年齢層をどのように計算したいかを選択できるようにし、これがどのように決定されるかの曖昧さを解消するためにエンコードされています。
