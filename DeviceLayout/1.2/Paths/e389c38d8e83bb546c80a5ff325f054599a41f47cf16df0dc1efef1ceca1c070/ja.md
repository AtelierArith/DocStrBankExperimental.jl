```
attach!(p::Path, c::GeometryReference, t::Coordinate;
    i::Integer=length(p), location::Integer=0)
attach!(p::Path, c::GeometryReference, t;
    i::Integer=length(p), location=zeros(Int, length(t)))
```

`c`をパスに沿って取り付けます。2番目のメソッドでは、`t`と`location`の範囲または配列を指定することができます（長さが一致しない場合、`location`は循環します）。

デフォルトでは、取り付けは`t ∈ [zero(pathlength(s)),pathlength(s)]`で最も最近のパスセグメント`s`に沿って行われますが、異なるパスセグメントインデックスを`i`を使用して指定することができます。参照は、パスが0°を指している場合はゼロ回転で向けられ、それ以外の場合はパスに沿って回転します。

セル参照の原点は、パスと共に回転する座標系に対してセルをどこに配置するかをメソッドに指示します。パスが角度0°の直線であると仮定します。すると、`Point(0.,10.)`の原点は、パスの上に10の位置にセルを配置します。また、パスが90°左に曲がると、パスの左側に10の位置にセルを配置します。

`location`オプションは便利のためのものです。`location == 0`の場合、特別なことは起こりません。`location == -1`の場合、参照の取り付けポイントは波導の最も左の端（描画されたポリゴン; パス自体には幅がありません）にあります。同様に、`location == 1`の場合、取り付けポイントは最も右の端にあります。このオプションは、最初の段落で説明されていることを除いて、セル参照を自動的に回転させることはありません。このオプションは、パスと共に回転する座標系の特別な原点を設定するものと考えることができます。たとえば、`Point(0.,10.)`のセル参照の原点と`location == -1`を組み合わせると、角度0°の描画された（有限幅の）パスの端の上に10の位置にセルが配置されます。
