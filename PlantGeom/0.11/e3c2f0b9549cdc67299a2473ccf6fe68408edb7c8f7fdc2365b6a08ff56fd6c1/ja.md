```
merge_children_geometry!(mtg; from, into, delete=:nodes, verbose=true, child_link_fun=x -> new_child_link(x, verbose))
```

MultiScaleTreeGraph (MTG) のジオメトリを簡略化し、低スケールのジオメトリを高スケールのジオメトリに統合します。

# 引数

  * `mtg`: 処理する MultiScaleTreeGraph。
  * `from`: 簡略化するノードのタイプの文字列。これは統合する必要がある低スケールのメッシュです。文字列または文字列のベクターである必要があります。*例* ["Petiole", "Rachis"]。
  * `into`: 統合するノードのタイプの文字列。単一の文字列である必要があります。*例* "Leaf"。
  * `delete`: 統合後にノードまたはジオメトリを削除するかどうかを示すシンボル：

      * `:none`: 削除は行われず、ジオメトリは `into` ノードに統合され、`from` ノードにも以前のまま保持されます。
      * `:nodes`: `from` タイプのノードは統合後に削除されます。
      * `:geometry`: ジオメトリのみが削除されますが、`from` ノードは MTG に残ります。
  * `verbose`: ノードまたはジオメトリが期待されるノードで見つからなかった場合に情報を返すかどうかを示すブール値。
  * `child_link_fun`: 削除対象の親ノードを受け取り、その子ノードの新しいリンクを返す関数。`delete` が `true` の場合は必須です。デフォルトの関数は `new_child_link` で、親と子のリンクを考慮して賢く動作しようとします。詳細についてはヘルプページを参照してください。リンクを変更する必要がない場合は、代わりに `link` 関数を使用してください。

# 戻り値

  * 何も返しません。この関数は `mtg` をその場で変更します。

# 注意事項

`from` タイプの子ノードにジオメトリが見つからない場合、情報メッセージがログに記録されます。
