```
UniV3(current_price, lower_ticks, liquidity, γ, Ai)
```

二つのコインのUniswap v3 CFMMを作成します。このCFMMは、価格の区間が互いに重ならない[BoundedProduct](@ref)プールのコレクションです。価格は、資産1の単位あたりの資産2の量を指します。`lower_ticks`ベクトルは、価格を降順に格納します（すなわち、インデックスが増加するにつれて資産2が高くなります）。`k`がプールの数であるとき、`k+1`番目の価格は`Inf`であると仮定されます。`liquidity`ベクトルの`i`番目のエントリは、価格`p[i]`と`p[i+1]`の間の各プールの不変量を格納します。論文では、不変量の二乗を使用し、次のように定義されます。

$$
\varphi(R) = (R_1 + \alpha)(R_2 + \beta).
$$

前述のように、`γ`は手数料率であり、`Ai`ベクトルはローカルインデックスをグローバルインデックスにマッピングします。

詳細については、「定常関数市場メーカーを通じた最適ルーティングのための効率的なアルゴリズム」を参照してください。
