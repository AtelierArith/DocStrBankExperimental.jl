ソース: FMISpec3.0, バージョン D5ef1c1: 2.4.7.4. 変数属性列挙型は変数の因果関係を定義します。この列挙型の許可される値:

parameter - シミュレーション中に一定のデータ値（調整可能なパラメータは除く）であり、環境によって提供され、接続に使用することはできません。ただし、セクション 2.4.9.2.6 に記載されている端末でのパラメータ伝播には使用できます。変動性は固定または調整可能でなければなりません。これらのパラメータは独立して変更できますが、計算されたパラメータとは異なります。初期値は正確でなければならず、存在しない場合（つまり正確であることを意味します）。

calculatedParameter - シミュレーション中に一定のデータ値であり、初期化中または調整可能なパラメータが変更されたときに計算されます。変動性は固定または調整可能でなければなりません。初期値はおおよそ、計算された、または存在しない（つまり計算されたことを意味します）。

input - 変数の値はインポータによって提供されることがあります。[たとえば、インポータは別の FMU の出力をこの入力に転送することができます。]

output - 変数の値はインポータによって使用されることがあります。[たとえば、この値は別の FMU の入力に転送されることがあります。] 入力に対する代数的関係は、<fmiModelDescription><ModelStructure><Output> の dependencies 属性を介して定義できます。

local - ローカル変数は次のとおりです:

  * 連続状態とその ContinuousStateDerivatives、ClockedStates、EventIndicators または InitialUnknowns。これらの変数は <fmiModelDescription><ModelStructure> にリストされています。
  * デバッグ目的で読み取ることができ、<fmiModelDescription><ModelStructure> にリストされていない内部、中間変数またはローカルクロック。

ローカル変数の設定:

  * 初期化モードおよびそれ以前では、ローカル変数は開始値がある場合や <InitialUnknown> としてリストされている場合に設定する必要があります。
  * スーパー状態 Initialized では、ローカル変数に対して fmi3Set{VariableType} を呼び出してはいけません。モデル交換では、連続状態は fmi3SetContinuousStates で設定できます。ローカル変数の値は、別のモデルまたは FMU の入力として使用してはいけません。

independent - 独立変数（通常は時間 [ただし、角度などもあり得ます]）。すべての変数はこの独立変数の関数です。変動性は連続でなければなりません。FMU の正確に 1 つの変数は独立として定義されなければなりません。モデル交換の場合、値は fmi3SetTime によって設定された最後の値です。コシミュレーションの場合、独立変数の値は、fmi3DoStep への最後の呼び出しによって返される lastSuccessfulTime または fmi3CallbackIntermediateUpdate の引数 intermediateUpdateTime の値です。スケジュール実行の場合、独立変数の値は定義されていません。[スケジュール実行におけるこの変数の主な目的は、独立変数の量と単位を定義することです。] 独立変数の初期値は、コシミュレーションとモデル交換の両方に対して fmi3EnterInitializationMode の引数 startTime の値です。独立変数の単位が定義されていない場合、暗黙的に s（秒）となります。1 つの変数が独立として定義されている場合、開始属性なしで浮動小数点型で定義されなければなりません。独立変数に対して fmi3Set{VariableType} 関数を呼び出すことは許可されていません。代わりに、その値は fmi3EnterInitializationMode で初期化され、初期化後にモデル交換のために fmi3SetTime で設定され、コシミュレーション FMU の fmi3DoStep の引数 currentCommunicationPoint と communicationStepSize で設定されます。[実際の値は fmi3Get{VariableType} で照会できます。]

structuralParameter - 変数の値は構成モードまたは再構成モードでのみ変更できます。変動性属性は固定または調整可能でなければなりません。初期属性は正確でなければならず、存在しない場合（つまり正確であることを意味します）。開始属性は必須です。構造パラメータには <Dimension> 要素を持ってはいけません。構造パラメータは <Dimension> 要素で参照される場合があります。構造パラメータが <Dimension> 要素で参照される場合、それは <UInt64> 型でなければならず、その開始属性は 0 より大きくなければなりません。min 属性はまだ 0 である可能性があります。

因果関係のデフォルトはローカルです。連続時間状態またはイベントインジケータは因果関係 = ローカルまたは出力でなければなりません。セクション 2.4.8 も参照してください。

[因果関係 = calculatedParameter および因果関係 = local で変動性 = fixed または tunable は似ています。違いは、calculatedParameter は別のモデルまたは FMU で使用できるのに対し、ローカル変数は使用できないことです。たとえば、Modelica 環境で FMU をインポートする場合、calculatedParameter は最終パラメータとして公開セクションにインポートされるべきですが、ローカル変数はモデルの保護されたセクションにインポートされるべきです。]

Clock 型の変数の因果関係は、input または output でなければなりません。

列挙型の定数の再定義を助けるために、接頭辞 "fmi3" を追加しました。
