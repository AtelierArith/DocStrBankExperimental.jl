# 拡張ヘルプ

```
norm(H::AbstractHyperrectangle, [p]::Real=Inf)
```

### アルゴリズム

ノルムは次のように定義されます。

$$
‖ X ‖ = \max_{x ∈ X} ‖ x ‖_p = max_{x ∈ \text{vertices}(X)} ‖ x ‖_p.
$$

最後の等式は、凸関数の最適解が多面体の頂点のいずれかで達成されるため成り立ちます。

この実装は、最大値が任意の$p$-ノルムに対して頂点$c + \text{diag}(\text{sign}(c)) r$で達成されるという事実を利用しています。したがって、この特定の頂点の$p$-ノルムを取るだけで十分です。この主張は以下で証明されます。特に、*各*頂点の$p$-ノルムを計算する必要はなく、これは非常に高価になる可能性があります。

$$
X
$$

がハイパーレクタングルであり、ハイパーレクタングルの$n$次元ベクトルの中心と半径がそれぞれ$c$と$r$であるとすると、$2^n$の頂点に関する推論から次のようになります。

$$
\max_{x ∈ \text{vertices}(X)} ‖ x ‖_p = \max_{α_1, …, α_n ∈ \{-1, 1\}} (|c_1 + α_1 r_1|^p + ... + |c_n + α_n r_n|^p)^{1/p}.
$$

関数$x ↦ x^p$、$p > 0$は単調増加であり、したがって各項$|c_i + α_i r_i|^p$の最大値は、各$i$に対して$|c_i + \text{sign}(c_i) r_i|^p$によって与えられます。したがって、$x^* := \text{argmax}_{x ∈ X} ‖ x ‖_p$は頂点$c + \text{diag}(\text{sign}(c)) r$です。
