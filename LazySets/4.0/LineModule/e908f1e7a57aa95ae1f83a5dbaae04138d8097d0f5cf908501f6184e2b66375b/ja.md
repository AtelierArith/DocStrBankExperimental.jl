```
Line{N, VN<:AbstractVector{N}} <: AbstractPolyhedron{N}
```

形式の線を表す型

$$
    \{y ∈ ℝ^n: y = p + λd, λ ∈ ℝ\}
$$

ここで、$p$ は線上の点で、$d$ はその方向ベクトル（必ずしも正規化されているわけではない）です。

### フィールド

  * `p` – 線上の点
  * `d` – 方向

### 例

3つのコンストラクタがあります。オプションのキーワード引数 `normalize`（デフォルト: `false`）を使用して、結果の線の方向をノルム1に正規化することができます（ユークリッドノルムに関して）。

1. デフォルトのコンストラクタはフィールド `p` と `d` を取ります：

点 $[-1, 2, 3]$ を通り、ベクトル $[3, 0, -1]$ に平行な線：

```jldoctest
julia> Line([-1.0, 2, 3], [3.0, 0, -1])
Line{Float64, Vector{Float64}}([-1.0, 2.0, 3.0], [3.0, 0.0, -1.0])

julia> Line([-1.0, 2, 3], [3.0, 0, -1]; normalize=true)
Line{Float64, Vector{Float64}}([-1.0, 2.0, 3.0], [0.9486832980505138, 0.0, -0.31622776601683794])
```

2. 2番目のコンストラクタは、キーワード引数として2つの点 `from` と `to` を取り、それらを通る線を返します。詳細はアルゴリズムセクションを参照してください。

```jldoctest
julia> Line(from=[-1.0, 2, 3], to=[2.0, 2, 2])
Line{Float64, Vector{Float64}}([-1.0, 2.0, 3.0], [3.0, 0.0, -1.0])
```

3. 3番目のコンストラクタは `Line2D` に似ており、2次元の線のみに機能します。2つの入力 `a` と `b` を取り、$a ⋅ x = b$ となる線を構築します。

```jldoctest
julia> Line([2.0, 0], 1.)
Line{Float64, Vector{Float64}}([0.5, 0.0], [0.0, 1.0])
```

### アルゴリズム

2つの点 $p ∈ ℝ^n$ と $q ∈ ℝ^n$ が与えられたとき、これら2つの点を通る線は `L:`{y ∈ ℝ^n: y = p + λ(q - p), λ ∈ ℝ}`` です。
