```
root_optim!(tree::FelNode, models; <keyword arguments>)
```

ツリーのルート位置とルート状態を最適化します。新しい最適なルートノードを返します。modelsは、ツリーのメッセージが1つのパーティションのみを含む場合は単一のモデル、またはメッセージが>1パーティションを持つ場合はモデルの配列、またはノードを受け取り、異なるブランチごとにモデルを変える必要がある場合はVector{<:BranchModel}を返す関数であることができます。

# キーワード引数

  * `partition_list=1:length(tree.message)`: (例: 1:3 または [1,3,5]) 実行するパーティションを選択できます（ただし、すべてのモデルでルート位置とルート状態を最適化したいと思うでしょう。デフォルトオプションです）。
  * `root_LL!=default_root_LL_wrapper(tree.parent_message[partition_list])`: メッセージを受け取り、(最適な)親メッセージとLL（対数尤度）を返す関数。デフォルトオプションは、すべてのルート候補の親メッセージとして定数 `tree.parent_message[partition_list]` を使用します。
  * `K=10`: ブランチに沿った等間隔のルート候補ポイントの数。（頻度主義のフレームワークでのみ使用されるべきですか？）
