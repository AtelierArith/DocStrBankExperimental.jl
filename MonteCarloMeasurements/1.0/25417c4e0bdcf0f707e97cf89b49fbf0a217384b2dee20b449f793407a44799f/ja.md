場合によっては、粒子が通過する原始関数を定義することが不可能であり、安全でない比較を許可することが受け入れられないことがあります。そのようなケースの一つは、不確定行列の固有値を内部的に計算する関数です。固有値の計算は比較演算子を利用します。不確実性が大きい場合、固有値は返された固有値のソートされたリストの中で位置を変える可能性があり、下流の計算を完全に台無しにしてしまいます。このため、以下の順序で推奨します。

1. ドキュメントで詳述されている `@bymap` を使用する [こちら](https://github.com/baggepinnen/MonteCarloMeasurements.jl#monte-carlo-simulation-by-mappmap)。すべての不確定値がエントリ関数への引数として現れる場合に適用可能です。
2. `Workspace` オブジェクトを作成し、それをエントリ関数を使用して呼び出します。不確定パラメータがエントリ関数への引数であるオブジェクト内にネストされている場合に適用可能です：

```julia
# 望ましい計算: y = f(obj), obj は内部に不確定パラメータを含む
y = with_workspace(f, obj)
# または同等に
w = Workspace(f, obj)
use_invokelatest = true # w が同じ関数内で作成され使用される場合、世界年齢の問題の代償としてこれを false に設定すると 0.1-1 ms の時間を短縮できます。
w(obj, use_invokelatest)
```
