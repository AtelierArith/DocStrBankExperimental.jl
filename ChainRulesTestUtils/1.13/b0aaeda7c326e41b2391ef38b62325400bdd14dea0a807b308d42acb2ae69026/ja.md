```
test_rrule([config::RuleConfig,] f, args...; kwargs...)
```

# 引数

  * `config`: デフォルトは `ChainRulesTestUtils.TestConfig`。
  * `f`: `rrule` をテストする関数。接線は `f ⊢ f̄` を使用して提供できます。（Julia REPLおよびサポートされているエディタで `⊢` を入力するには `\vdash` + tab を使用します。）
  * `args...`: プライマル引数 `x`、またはプライマルとその接線: `x ⊢ x̄`

      * `x`: `f` を評価するための入力（一般的にはドメイン内の任意の点に設定する必要があります）。
      * `x̄`: 現在蓄積されたコタンジェント; 提供されていない場合は自動的に生成されます。

    インデックスなどの非微分可能な引数は、`x̄` を `NoTangent()` に設定する必要があります。

# キーワード引数

  * `output_tangent`: テストのために逆方向に伝播させるシード（技術的にはコタンジェント）。`f` の出力の微分である必要があります。提供されていない場合は自動的に設定されます。
  * `check_thunked_output_tangent=true`: 出力接線のサンクされたバージョンをプルバックに渡すと同じ結果が返されることも確認します。
  * `fdm::FiniteDifferenceMethod`: 使用する有限差分法。
  * `rrule_f=rrule`: テストされる `rrule` に似た API を持つ関数（デフォルトは `rrule`）。ADシステムからの勾配をテストするために使用されます。
  * `check_inferred=true` の場合、`rrule` の推論可能性（型安定性）がチェックされます — `f` 自体が推論可能である場合 — そしてそれが返すプルバックの推論可能性もチェックされます。
  * `fkwargs` はキーワード引数として `f` に渡されます。
  * `testset_name`: 提供されている場合、テストをラップするために使用されるテストセットの名前。

それ以外の場合は、関数と引数の型から決定されます。

  * 残りのすべてのキーワード引数は `isapprox` に渡されます。
