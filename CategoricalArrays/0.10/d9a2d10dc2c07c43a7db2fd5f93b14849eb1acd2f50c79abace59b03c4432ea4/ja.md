```
categorical(A::AbstractArray; levels=nothing, ordered=false, compress=false)
```

`A`から値を持つカテゴリカル配列を構築します。

`levels`キーワード引数は、データの可能な値を指定するベクトルであることができます（これは、結果の配列に対して[`levels!`](@ref)を呼び出すよりも効率的です）。`levels`が省略され、要素型がそれをサポートしている場合、レベルは昇順にソートされます。そうでない場合、レベルは`A`に現れた順序のまま保持されます。`ordered`キーワード引数は、配列の値がレベルの順序に従って比較できるかどうかを決定します（[`isordered`](@ref）を参照）。

`compress`が`true`の場合、`A`内のユニークな値の数を保持できる最小の参照型が使用されます。これによりメモリ使用量は減少しますが、このパラメータを渡すことで、呼び出しが行われる関数内のパフォーマンスに影響を与える型の不安定性も導入されます。したがって、このオプションは注意して使用してください（1引数バージョンはこの問題に悩まされません）。

```
categorical(A::CategoricalArray; compress=false, levels=nothing, ordered=false)
```

`A`がすでに`CategoricalArray`である場合、そのレベル、順序性、および参照型は明示的に上書きされない限り保持されます。 ```
