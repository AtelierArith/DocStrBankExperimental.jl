```
抽象型 AbstractLocalIndices
```

抽象型は、範囲 `1:n` の一部のパーティションにおける *ローカル*、*所有*、および *ゴースト* インデックスを表します。範囲の長さは `n` です。

# 表記法

`1:n` を長さ `n` の整数範囲とします。`1:n` のインデックスを *グローバル* インデックスと呼びます。`1:n` のパーティションを考えます。パーティション内の一部のインデックスは、この部分の *所有* インデックスと呼ばれます。つまり、各部分は `1:n` のサブセットを *所有* します。これらのサブセットはすべて互いに排他的です。各部分には *ゴースト* インデックスと呼ばれる第二のインデックスセットが装備されていると仮定します。特定の部分におけるゴーストインデックスのセットは、他の部分が所有するグローバルインデックス `1:n` の任意のサブセットです。所有インデックスとゴーストインデックスの和集合は、この部分の *ローカル* インデックスと呼ばれます。ローカルインデックスのセットは、異なる部分間で重複する可能性があります。

所有、ゴースト、およびローカルインデックスのセットは、`AbstractLocalIndices` の具体的な実装においてベクターのようなコンテナを使用して格納されます。これにより、特定の順序が与えられます。部分内の `i` 番目の所有インデックスは、この部分の所有インデックスを含む配列のインデックス `i` に格納されているものとして定義されます（ゴーストインデックスおよびローカルインデックスについても同様です）。これらの順序付けられたインデックスセット内のインデックス間のマッピングは、[`local_to_global`](@ref)、[`own_to_local`](@ref) などの関数によって与えられます。
