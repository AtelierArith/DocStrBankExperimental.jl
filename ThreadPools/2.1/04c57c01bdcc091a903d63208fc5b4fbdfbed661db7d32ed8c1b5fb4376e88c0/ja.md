```
logbforeach(fn, itrs...)
```

`Base.foreach`を模倣しますが、プライマリ以外のすべての利用可能なスレッドに関数評価を開始し、均一なタスク期間に適した事前割り当てスケジューリング戦略を使用します。 ログが記録されたプールを返し、ログ関数や`plot`で分析できます。    

# 例

```julia
julia> pool = logbforeach(x -> println((x,Threads.threadid())), 1:8);
(1, 2)
(3, 3)
(6, 4)
(4, 3)
(2, 2)
(7, 4)
(5, 3)
(8, 4)

julia> plot(pool)
```

実行順序は保証されておらず、プライマリスレッドは使用されないことに注意してください。
