```
logtforeach(fn, itrs...) -> pool
```

`Base.foreach`を模倣しますが、関数の評価をすべての利用可能なスレッドに起動し、均一なタスクの所要時間に適した事前に割り当てられたスケジューリング戦略を使用します。 ログが記録されたプールを返し、ログ関数や`plot`で分析できます。

# 例

```julia
julia> pool = logtforeach(x -> println((x,Threads.threadid())), 1:8);
(1, 1)
(3, 2)
(7, 4)
(2, 1)
(4, 2)
(5, 3)
(8, 4)
(6, 3)

julia> plot(pool)
```

実行順序は保証されておらず、主スレッドが使用されることに注意してください。
