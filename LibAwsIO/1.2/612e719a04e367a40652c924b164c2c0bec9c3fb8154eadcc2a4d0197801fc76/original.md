Invoked upon the acquisition, or failure to acquire a retry token. This function will always be invoked if and only if [`aws_retry_strategy_acquire_retry_token`](@ref)() returns AWS_OP_SUCCESS. It will never be invoked synchronously from [`aws_retry_strategy_acquire_retry_token`](@ref)(). Token will always be NULL if error_code is non-zero, and vice-versa. If token is non-null, it will have a reference count of 1, and you must call [`aws_retry_token_release`](@ref)() on it later. See the comments for [`aws_retry_strategy_on_retry_ready_fn`](@ref) for more info.
