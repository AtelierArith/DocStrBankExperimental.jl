```
aws_channel_schedule_task_now(channel, task)
```

タスクをイベントループ上でできるだけ早く実行するようにスケジュールします。これは、タスクを正しいスレッドに移動させる理想的な方法です。また、コンテキストスイッチにも便利です。この関数は、任意のスレッドから呼び出すことが安全です。

チャネルのイベントループから呼び出された場合、タスクは直接実行リストに追加されます。チャネルのイベントループの外から呼び出された場合、タスクはクロススレッドタスクキューに入ります。

タスクが特定のソース同期に対して直列化される必要がある場合、このAPIを使用しない方が良いかもしれません。なぜなら、イベントループスレッドから提出されたタスクは、この最適化のために外部スレッドから提出されたタスクよりも「先に進む」ことができるからです。これが問題である場合は、提出ロジックをリファクタリングするか、この最適化を行わない[`aws_channel_schedule_task_now_serialized`](@ref)バリアントを使用することができます。

タスクは、その関数が実行されるまでクリーンアップまたは変更されるべきではありません。

### プロトタイプ

```c
void aws_channel_schedule_task_now(struct aws_channel *channel, struct aws_channel_task *task);
```
