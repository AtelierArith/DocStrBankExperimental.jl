```
aws_tls_ctx_pkcs11_options
```

この構造体は、[`aws_tls_ctx_options`](@ref) の init-with-pkcs11 関数を呼び出す際に多くの引数を優雅に渡す方法として存在します（これにより、将来的にオプションの引数を導入するのも簡単になります）。この構造体のインスタンスは、スタック上に短時間だけ存在するべきです。

これを高水準言語にバインディングするための指示: - Python: この構造体のメンバーは、init-with-pkcs11 関数へのキーワード引数であるべきです。 - JavaScript: これは、init-with-pkcs11 関数に渡されるオプションマップであるべきです。 - Java: これは、init-with-pkcs11 関数に渡されるオプションクラスであるべきです。 - C++: Java と同様です。

整数型に関する注意: PKCS#11 は ID、ハンドルなどに `unsigned long` を使用しますが、公開 API では `uint64_t` として公開しています。これは、sizeof(long) がプラットフォーム/アーキテクチャ/言語によって一貫性がないためです（例: Java では常に 64bit、Windows の C では常に 32bit、Linux と Apple の C では CPU に一致します）。公開 API で uint64_t を使用することで、すべての注意深い境界チェックを一箇所にまとめ、高水準言語のバインディングが正しく行われることを期待する必要がなくなります。
