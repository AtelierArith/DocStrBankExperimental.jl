```
provenance(object) -> NamedTuple
```

`object`の*出所*を`NamedTuple`として説明します。ここでの出所は以下を指します：

> 出所とは、データや物の生成に関与するエンティティ、活動、人物に関する情報であり、これを用いてその品質、信頼性、または信頼性についての評価を形成することができます。
>
> – https://www.w3.org/TR/prov-overview/


`object`から出所がキャプチャされていない場合、`nothing`を返します。

出所キャプチャに参加したい型を持つパッケージの著者は、自分の型に対して`provenance`を拡張する必要があります。定義は*必ず*、`object`に関連する出所データを含む`NamedTuple`を返さなければなりません。

# 例

```julia
struct MyType
    # ...
end

function ProvenanceBase.provenance(t::MyType)
    # `t`に関連する出所データを計算します。
    return (
        # 出所データを名前付きフィールドに割り当てて返します。
    )
end
```

# データ

`ProvenanceBase`は、これらのメソッドによって返されるべき出所データの必須セットを規定していません。関連するデータを決定するのは各パッケージの著者自身です。

通常キャプチャする必要がないデータは、関連する`object`に直接関係しない限り、グローバルに利用可能なシステムレベルのデータです。`provenance`の返り値に通常含まれるデータは、均一なアクセスパターンが必要な`object`の内部（または公開）データです。
