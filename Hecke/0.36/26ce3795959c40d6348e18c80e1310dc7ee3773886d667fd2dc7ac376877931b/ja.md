複素体は常にこのように保存されます

```
      maps[1]   maps[2]
    * ------> * ------> * ....
```

論理的には2種類の複素体があります

  * チェーン
  * コチェーン

（`typ`に保存されます（`type`はキーワードのため使用できません））

タイプは、`seed`と共同で、写像とオブジェクトのホモロジー、外部、番号付けを決定します。`seed`は常に論理的に最小のオブジェクトインデックスを保存します（ホモロジー次数の意味で）

# チェーン複素体

`typ == :chain`

定義は `d_i(d_i+1(x)) = 0` であり、ここで `d_i` は（論理的に）番号付けされた写像（微分）です。対応するオブジェクト `M_i` は次のように定義されます

`d_i : M_i -> M_i-1`

したがって、最小オブジェクト（インデックス）は最後の写像の像です =>   `maps[end] = d_seed+1`   `codomain(maps[end]) = M_seed`   `domain(maps[1]) = M_(seed+length(maps))`

# コチェーン複素体

`typ == :cochain`

定義は `d_i+1(d_i(x)) = 0` であり、

`d_i : M_i -> M_i+1`

したがって、最小オブジェクトは `domain(maps[1]) = M_seed` であり、最大のものは `codomain(maps[end]) = M_(seed + length(maps))` です。

# アクセス

## 範囲

オブジェクト - インデックス - 範囲は次のいずれかです

  * `(seed + length):-1:seed`     （チェーン複素体）
  * `seed:(seed + length(maps))`  （コチェーン複素体）

## マップ

map(C, i) =

  * `maps[length - (i - s)]`      （チェーン）
  * `maps[i-s]`                   （コチェーン）

## オブジェクト

`obj(C, i) = domain(map(C, i))` の両方の場合   しかし、最後には、最後のオブジェクトは最後の写像のコドメインです...
