```
unique_name(prefix::AbstractString, separator::AbstractString = "#")::AbstractString
```

短く、人間が読みやすいユニークな名前を使うことは、デバッグ時に大いに役立ちます。通常、人間が提供する短い非ユニークな名前と、不透明なオブジェクト識別子、またはその組み合わせのいずれかを選択する必要があります。この関数は、不透明なオブジェクト識別子を短いカウンターに置き換え、ユニークで短い名前を提供します。

つまり、これは `prefix` で始まり、デフォルトで `#` の `separator` に続き、プロセスインデックス（複数のプロセスを使用している場合）とインデックス（この名前がプロセス内で何回使用されたか）を返します。例えば、`unique_name("foo")` は最初の使用時に `foo` を返し、2回目には `foo#2` を返します。複数のプロセスを使用している場合は、`foo`、`foo#1.2` などを返します。

つまり、名前がユニークなコード（例えば、シンプルなスクリプトやJupyterノートブック）では、名前が混乱することはありません。同じ名前の複数の使用を区別する必要がある場合にのみ、名前にサフィックスを追加します。

テストを助けるために、`prefix` に `!` が含まれている場合は、それをそのまま返し、ユニークでない可能性があることを受け入れます。
