[`FormatReader`](@ref) および [`FormatWriter`](@ref) インターフェースは、`Daf` データを保存するための低レベル API を指定します。追加のフォーマットをサポートするために `Daf` を拡張するには、この API の新しい実装を作成します。

ストレージフォーマットオブジェクトは、いくつかの名前付きスカラー データ、軸のセット (各エントリに一意の名前が付けられています)、およびこれらの軸に基づく名前付きベクトルおよび行列データを含みます。

データプロパティは、それに基づく軸に与えられた一意の名前によって識別されます。つまり、スカラー プロパティ、特定の軸ごとのベクトル プロパティ、および各 (順序付き) 軸のペアごとの行列プロパティには、別々の名前空間があります。

行列については、[`MatrixLayouts`](@ref) を注意深く追跡します。具体的には、ストレージフォーマットは列優先の行列のみを扱い、最初に行軸、次に列軸の下にリストされます。ストレージフォーマットオブジェクトは、同じ行列の2つのコピーを保持する場合があり、両方のメモリレイアウトで、両方の軸の順序の下に2回リストされます。

一般に、ストレージフォーマットオブジェクトはできるだけ「単純」であり、新しいストレージフォーマットをサポートしやすくします。必要な関数は、上記の別々のプロパティ名前空間を扱うために絶対に最小限の必要なロジックを持つ、栄光あるキー-バリューリポジトリを実装します。

ドキュメントの明確さのために、型階層を [`DafWriter`](@ref) `<:` [`FormatWriter`](@ref) `<:` [`DafReader`](@ref) `<:` [`FormatReader`](@ref) に分割します。

ここにリストされている関数は、読み取り専用操作のために [`FormatReader`](@ref) を使用し、`Daf` ストレージへの書き込み操作のために [`FormatWriter`](@ref) を使用します。これは低レベルの API であり、パッケージの外部から使用することを意図しておらず、したがってトップレベルの `DataAxesFormats` 名前空間から再エクスポートされていません。

対照的に、[`DafReader`](@ref) および [`DafWriter`](@ref) を使用する関数は、パッケージの外部から使用することを意図した高レベルの API を説明し、再エクスポートされます。これらの関数は `DataAxesFormats.Readers` および `DataAxesFormats.Writers` モジュールにリストされています。これにより、任意のストレージフォーマットに共通するすべてのロジックを提供し、フォーマット固有の関数をできるだけシンプルに保つことができます。

つまり、新しい `Daf` ストレージフォーマットを実装する際には、`struct MyFormat <: DafWriter` と記述し、ここにリストされている関数を [`FormatReader`](@ref) および [`FormatWriter`](@ref) の両方に対して実装する必要があります。
