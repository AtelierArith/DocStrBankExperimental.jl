特定の軸に対して[`get_frame`](@ref)の列を指定します。最も一般的な形式は、ペア`"column_name" => query`です。2つの省略形が適用されます：ペア`"column_name" => "="`はペア`"column_name" => ": column_name"`の省略形であり、また省略形`"column_name"`（単純な文字列）も同様です。

また、他の言語（Pythonなど）からAPIを呼び出しやすくするために、ペアの代わりにタプルを指定することも許可されています。タプルは`Pair`の概念を持たない言語に便利です。

クエリは、次のように軸クエリと組み合わされます（[`full_vector_query`](@ref)を使用）：

  * クエリに[`GroupBy`](@ref)が含まれている場合、クエリは軸クエリに指定されたマスクを繰り返す必要があります。つまり、軸クエリが`metacell & type = B`の場合、列クエリは`/ cell & metacell => type = B @ metacell : age %> Mean`でなければなりません。ご不便をおかけして申し訳ありません。TODO: [`GroupBy`](@ref)列クエリにマスクを自動的に注入します。
  * そうでない場合、クエリが（単一の）軸で始まる場合、それは削減のみを含むべきです。軸クエリは自動的にその後に注入されます。つまり、軸クエリが`gene & is_marker`の場合、列クエリ`/ metacell : fraction %> Mean`の完全なクエリは`/ metacell / gene : fraction %> Mean`になります（すべてのメタセルにおける平均遺伝子発現）。ここで軸クエリと列クエリを単純に連結できないのは、Juliaが無限の知恵を持って、Rやmatlabのように列優先の行列を使用するためです。したがって、削減は行列の列ではなく行を排除します。
  * それ以外の場合（典型的なケース）、単純に軸クエリと列クエリを連結します。つまり、軸クエリが`cell & batch = B1`で、列クエリが`: age`の場合、完全なクエリは`cell & batch = B1 : age`になります。これが最も単純で一般的なケースです。

すべての場合において、（完全な）クエリは軸の各エントリに対して値を返さなければなりません。
