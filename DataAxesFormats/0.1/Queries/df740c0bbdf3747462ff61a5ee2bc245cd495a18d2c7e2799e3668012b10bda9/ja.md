```
CountBy(property::AbstractString) <: QueryOperation
```

同じ軸のエントリのペアの値の組み合わせのカウントの行列を生成するクエリ操作です。つまり、いくつかのベクトルプロパティを取得した後、同じ軸の2番目のベクトルプロパティを取得することに従います。結果は、1番目のプロパティの値が行、2番目のプロパティの値が列であり、値は値の組み合わせが出現する回数です。文字列[`Query`](@ref)では、`*`演算子を使用し、ルックアップするプロパティ名を続けて指定します（例：`/ cell : type * batch`は、行がセルタイプに対応し、列がセルバッチに対応し、値が各バッチとタイプの組み合わせのセルの数になる行列を生成します）。

デフォルトでは、行および/または列には実際に見られた値のみが含まれ、アルファベット順に並べられます。ただし、1つまたは両方のプロパティが軸に対応することが一般的です。この場合、[`AsAxis`](@ref)サフィックスを使用して、行列の行および/または列を特定の軸のエントリに正確にすることができます（例：`/ cell : type ! * batch`は、`type`軸のエントリに正確に対応する行を持つ行列を生成します。たとえセルがないタイプがあっても）。両方のプロパティが軸である場合、これは特に便利で、結果を行列プロパティとして保存できます（例：`/ cell : type ! * batch !`は、行がタイプ軸のエントリに対応し、列がバッチ軸のエントリに対応する行列を生成します。これにより、`set_matrix!(daf, "type", "batch", ...)`に渡すことができます）。

生のカウント行列は、他の行列と同様に後処理できます（[`ReductionOperation`](@ref)または[`EltwiseOperation`](@ref)を使用）。これにより、便利な集約プロパティを計算できます（例：`/ cell : type * batch % Fractions`は、列がバッチに対応し、行が各バッチ内の各タイプからのセルの割合になる行列を生成します）。
