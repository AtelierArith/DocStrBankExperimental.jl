別の軸からプロパティの値を取得するためのクエリ操作で、値が軸のエントリ名であるベクタープロパティに基づいています。文字列 [`Query`](@ref) では、`=>` 演算子を使用して、検索する名前を指定します。

つまり、ベクタープロパティ（例えば、各 `cell` の `batch`）の値をクエリする場合、このプロパティの名前がいくつかの軸名と同じであると仮定します。各値はこの軸のエントリの名前であると見なします。これを使用して、その軸の他のプロパティ（例えば、`age`）の値を取得します（例：`/ cell : batch => age`）。

同じ軸にマッピングされる複数のベクタープロパティを保存できることは便利です。これをサポートするために、プロパティ名が軸名で始まり、その後に `.suffix` が続く命名規則をサポートしています（例：`/ cell : type => color` と `/ cell : type.manual => color` の両方は、`cell` 軸のプロパティの `type` の `color` を検索します - 各 `cell` の「the」`type` か、各 `cell` の代替 `type.manual` のいずれかです）。

取得はチェーンすることができます（例：`/ cell : batch => donor => age` は、各 `cell` の `batch` の `donor` の `age` を取得します）。

プロパティが存在しない場合、これはエラーです。ただし、これが [`IfMissing`](@ref) に続く場合は除きます（例：`/ cell : type => color || red`）。プロパティが空の値を含む場合、これもエラーですが、[`IfNot`](@ref) に続く場合は除きます（例：`/ cell : type ? => color` は、空でないタイプを持つセルのタイプの色のベクターを計算し、`/ cell : batch ? 0 => donor => age` は、空のバッチを持つセルにゼロの年齢を割り当てます）。
