```
AsAxis([axis::AbstractString = nothing]) <: QueryOperation
```

ベクトルプロパティを取り、各値をある軸のエントリの名前と見なす必要がある3つのケースがあります：[`Fetch`](@ref)、[`CountBy`](@ref)、および[`GroupBy`](@ref)。文字列[`Query`](@ref)では、これは`!`演算子によって示され、オプションで使用する軸の名前が続きます。

[`Fetch`](@ref)を使用する場合、常にある軸でルックアップを行うため、[`AsAxis`](@ref)が暗黙的に含まれます（例：`/ cell : type => color`は`/ cell : type ! => color`と同じです）。対照的に、[`CountBy`](@ref)および[`GroupBy`](@ref)を使用する場合、カウントまたはグループ化のために軸のすべてのエントリを使用するように`AsAxis`を明示的に指定する必要があります（例：`/ cell : age @ type %> Mean`は、関連するセルを持つ各タイプの平均年齢のベクトルを返しますが、`/ cell : age @ type ! %> Mean`は、タイプ軸の各値の平均年齢のベクトルを返します。同様に、`/ cell : type * age`は、関連するセルを持つタイプの行を持つカウント行列を生成しますが、`/ cell : type ! * age`は、タイプ軸のエントリそのものの行を持つカウント行列を生成します）。

値のセットはベクトルプロパティに一致する軸によって固定されているため、[`GroupBy`](@ref)でこれを使用する場合、一部のグループに値がないとエラーが発生する可能性があります。これは、削減に[`IfMissing`](@ref)サフィックスを提供することで回避できます（例：`/ cell : age @ type ! %> Mean`は、一部のタイプに関連するセルがない場合に失敗しますが、`/ cell : age @ type ! %> Mean || 0`は、そのようなタイプにゼロの平均年齢を与えます）。

通常、基本プロパティの名前は軸の名前と同じです。この場合、軸の名前を指定する必要はありません（上記の例のように）。時には、同じ軸にマッピングされる複数のベクトルプロパティを保存できることが便利です。これをサポートするために、プロパティ名が軸名で始まり、その後に`.suffix`が続く命名規則をサポートしています（例：`/ cell : type => color`と`/ cell : type.manual => color`の両方は、`cell`軸のあるプロパティの`type`の`color`をルックアップします - 各`cell`の「the」`type`、または各セルの代替`type.manual`）。

プロパティ名が上記の規則に従わない場合、軸の名前を明示的に指定することが可能です（例：`/ cell : manual ! type => color`は、`manual`プロパティの各値を`type`軸のエントリの名前と見なし、この軸の一致する`color`プロパティ値をルックアップします）。
