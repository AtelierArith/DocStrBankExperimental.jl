```
GroupBy(property::AbstractString) <: QueryOperation
```

値の各グループを集約するために（以下の）[`ReductionOperation`](@ref)を使用するクエリ操作です。指定された`property_name`（追加の[`Fetch`](@ref)操作が続く可能性があります）を取得し、各値のグループの名前に結果のベクトルを使用します。

ベクトルに適用した場合、結果はグループごとに1つのエントリを持つベクトルになります（例：`/ cell : age @ type %> Mean`は、各細胞タイプの平均年齢を持つエントリを持つベクトルを生成します）。行列に適用した場合、結果はグループごとに1行を持つ行列になります（例：`/ cell / gene : UMIs @ type %> Max`は、各タイプごとに1行、遺伝子ごとに1列を持つ行列を生成し、その値は各タイプの細胞内の遺伝子の最大UMI数です）。

デフォルトでは、結果は実際に観測されるグループ値のみを使用し、ソートされた順序になります。ただし、操作が[`AsAxis`](@ref)サフィックスで続く場合、取得されたプロパティは既存の軸に対応する必要があります（[`Fetch`](@ref)を使用する場合と似ています）。この場合、結果は軸のエントリを使用し、データで観測されない場合でも（空の値を持つベクトルエントリは無視されます）。この場合、いくつかのグループに値がない場合、削減操作は失敗しますが、[`IfMissing`](@ref)サフィックスが続く場合を除きます（例：`/ cell : age @ type ! %> Mean`は、`type`軸のすべてのエントリを持つベクトルを生成し、空のタイプの細胞は無視します。これは、どの細胞とも関連付けられていないタイプがある場合に失敗します。対照的に、`/ cell : age @ type ! %> Mean || 0`は成功し、関連付けられていないタイプにはゼロの値を割り当てます）。
