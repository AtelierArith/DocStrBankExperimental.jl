```
dlegendre!(dP::AbstractMatrix, ϕ::Number, P::AbstractMatrix, n_max::Integer = -1, m_max::Integer = -1; kwargs...) -> Nothing
```

関連するレジェンドル関数 `P_n,m[cos(ϕ)]` の一階導関数を `ϕ` [rad] に関して計算します：

```
∂P_n,m[cos(ϕ)]
──────────────
     ∂ϕ
```

計算される最大次数と次数は、パラメータ `n_max` と `m_max` によって与えられます。これらが負の場合（デフォルト）、行列 `dP` の次元が使用されます：

```
最大次数 -> 行数 - 1
最大次数  -> 列数 - 1
```

導関数は行列 `dP` に格納されます。

パラメータ `N` は正規化を選択します。以下の値が有効です：

  * `Val(:full)`: 完全に正規化された関連レジェンドル関数を計算します。
  * `Val(:schmidt)`: シュミット準正規化された関連レジェンドル関数を計算します。
  * `Val(:unnormalized)`: 非正規化された関連レジェンドル関数を計算します。

このアルゴリズムは、同じ正規化 `N` を使用して関連レジェンドル関数の値を持つ行列 `P` を必要とします。これは関数 [`legendre`](@ref) を使用して計算できます。

!!! warning
    ユーザーは正しい値を持つ行列 `P` を渡す責任があります。たとえば、`ph_term` が `true` の場合、`P` も `ph_term` を `true` に設定して計算する必要があります。


# キーワード

  * `ph_term::Bool`: `true` の場合、コンドン-ショートリー位相項 `(-1)^m` が含まれます。   (**デフォルト** = `false`)
