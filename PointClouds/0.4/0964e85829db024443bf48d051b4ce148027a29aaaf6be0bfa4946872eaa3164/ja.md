```
PointCloud(input; kws...)
```

処理のためにポイントデータの（サブセット）をメモリにロードします。データは「カラム形式」で保存されており、各ポイント属性（座標、強度など）は、各ポイントごとに1つのエントリ（行）を持つカラムを形成します。属性は、さらなる処理ステップで追加/更新/削除できます。

`input` は、1つまたは複数の（タプル/配列として渡される）[`LAS`](@ref) 値、または [`read(input, LAS)`](@ref Base.read(::IO, ::LAS)) に渡すことができる値（LAS/LAZファイルへのパスなど）です。あるいは、`Vector` の `NamedTuple` を渡すこともでき、`x`、`y`、`z` の名前は座標として解釈され、他のすべての名前は追加属性として解釈されます。

# キーワード

  * `attributes`: 各ポイントに含まれる追加属性。属性は、各ポイントに適用されて属性を計算する関数のタプル/配列として指定されます。デフォルトでは、関数の名前が属性名として使用されますが、`Pair{Symbol,Function}` を渡すことで手動で名前を指定できます。属性は、キーが属性名で値が関数である `NamedTuple` でも定義できます。デフォルト: `()`.
  * `coordinates`: ロードする座標のサブセット、例えば、垂直座標が必要ない場合は `(:x, :y)`。デフォルト: `(:x, :y, :z)`.
  * `crs`: x/y/z座標が変換されるべき座標参照系（CRS）。[PROJライブラリ](https://proj.org/en/9.4/usage/quickstart.html) が理解できる任意の文字列として指定します。デフォルト: 最初の入力のCRS。
  * `extent`: 限定された座標範囲内にあるポイントのみを含める。下限と上限を持つタプルとして指定します。各境界はx値とy値のタプルです。例えば、$0 ≤ x ≤ 1$ および $0 ≤ y ≤ 2$ のために `extent = ((0, 0), (1, 2))`。
  * `filter`: 各ポイントに対して呼び出され、`filter` 関数が `false` を返すポイントを除外するための述語関数。

# 例

```
PointCloud(las; attributes = (intensity, :t => gps_time))
PointCloud(las; attributes = (i = intensity, t = gps_time))
PointCloud(las; crs = "EPSG:4326", extent = ((-73.97, 40.80), (-73.95, 40.82)))
```
