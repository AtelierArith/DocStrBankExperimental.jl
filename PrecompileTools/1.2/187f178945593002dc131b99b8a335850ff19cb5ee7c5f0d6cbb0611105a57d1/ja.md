```
@compile_workload f(args...)
```

`precompile`（および compile*workload ファイルに保存）するのは、式内で発生するすべてのメソッド呼び出しです。`@compile*workload` ブロック内のすべての呼び出し（直接または間接）はキャッシュされます。

`@compile_workload` には3つの重要な機能があります：

1. コードはパッケージがプリコンパイルされているときのみ実行されます（つまり、`*.ji` プリコンパイル compile_workload ファイルが書き込まれているとき）。
2. インタープリターが無効になり、呼び出しがコンパイルされることが保証されます。
3. 直接および間接の呼び出し先は、他のパッケージで定義されたメソッドやランタイムディスパッチされた呼び出し先に対してもプリコンパイルされます（Julia 1.8 以上が必要です）。

!!! note
    包括的なプリコンパイルを行うには、特定のメソッド/引数タイプの組み合わせの最初の使用が `@compile_workload` 内で発生することを確認してください。

    詳細には：ランタイムディスパッチされた呼び出し先は、型推論が実行されるときにのみキャプチャされ、最初の使用時にのみ推論されます。あなたのパッケージで定義されたメソッドに遡ることができる推論可能な呼び出しと、その*推論可能な*呼び出し先は、呼び出し先の「所有権」に関係なくプリコンパイルされます（Julia 1.8 以上）。

    したがって、この推奨事項は次のことにのみ重要です：

    ```
    - Base または他のパッケージで定義されたメソッドへの直接呼び出し OR
    - そのようなメソッドへの間接的なランタイムディスパッチ呼び出し。
    ```
