ノイズグリッドは、ポイントの配列からノイズプロセスを構築します。たとえば、時間点 `t` を持つ配列 `W` を生成し、次のコンストラクタを使用して目的のノイズプロセスを構築できます。

```julia
NoiseGrid(t, W, Z = nothing; reset = true)
```

これにより、関連するノイズプロセスが構築されます。このプロセスは、与えられたポイントの線形補間を伴い、したがってグリッドは積分のグリッドと一致する必要はありません。したがって、これは適応的な解法にも使用できます。ただし、ノイズプロセスの忠実度は、ノイズグリッドがどれだけ細かく決定されるかに関連していることに注意する必要があります。ノイズグリッドが積分に比べてポイントがまばらである場合、線形補間によってその分布特性がわずかに変化する可能性があります。したがって、精度を確保するために、グリッドサイズは少なくとも積分の時間ステップ数を近似することが推奨されます。

1次元プロセスの場合、`W` は `Number` の `AbstractVector` である必要があります。多次元プロセスの場合、`W` は `noise_prototype` の `AbstractVector` である必要があります。

## NoiseGrid

この例では、事前に計算されたポイントの配列を使用して独自のブラウン運動を定義する方法を示します。通常の使用では、`WienerProcess` を代わりに使用するべきです。これは、ノイズグリッドが線形補間を使用するのに対し、分布的に正確な補間を持つためですが、これはワークフローの良い例です。

`NoiseGrid` を定義するには、一連の時間点とプロセスの値のセットが必要です。`dt=0.001` で `(0.0,1.0)` 上のブラウン運動を定義しましょう。これを行うには、

```julia
dt = 0.001
t = 0:dt:1
brownian_values = cumsum([0; [sqrt(dt) * randn() for i in 1:(length(t) - 1)]])
```

次に、これらの値を使用して `NoiseGrid` を構築します。

```julia
W = NoiseGrid(t, brownian_values)
```

その後、`SDEProblem` の `noise` 引数として `W` を渡して、SDE で使用できます。
