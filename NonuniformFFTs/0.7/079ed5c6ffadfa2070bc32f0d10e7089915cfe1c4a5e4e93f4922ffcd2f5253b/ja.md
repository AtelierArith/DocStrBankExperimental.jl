```
set_points!(p::PlanNUFFT, points)
```

NUFFTを実行する前に非一様な点を設定します。

1次元では、`points`は単に実数値のベクトル（非一様な位置）です。

複数次元では、`points`は次のように渡すことができます：

  * ベクトルのタプル `(xs::AbstractVector, ys::AbstractVector, …)`（**推奨される**）;
  * タプルまたは静的ベクトルのベクトル `[x⃗₁, x⃗₂, x⃗₃, x⃗₄, …]`（通常はStaticArrays.jlパッケージの`SVector`）;
  * サイズが `(d, Np)` の行列で、ここで `d` は空間次元、`Np` は非一様な点の数です。

余分なアロケーションを避けたい場合は、最初の形式を推奨します。

便利さのため（および後方互換性のため）、StructArrays.jlパッケージの`StructVector`を渡すこともでき、これは上記の最初のオプションと同等です。

点は周期セル $[0, 2π)^d$ の外にあっても構いません。この場合、それらはその領域に「折りたたまれ」ます。

!!! note "点配列の変更"
    予想されるように、このパッケージは入力点を変更しません。これを行うと驚くべきことになるためです。ただし、アロケーションとコピーを避けるために、変換を行う際に点データへの参照を保持します。これは、`set_points!` と [`exec_type1!`](@ref) または [`exec_type2!`](@ref) の呼び出しの間に渡された配列を**変更しない**べきであることを意味します。そうしないと、誤った結果やさらに悪い結果を招く可能性があります。
