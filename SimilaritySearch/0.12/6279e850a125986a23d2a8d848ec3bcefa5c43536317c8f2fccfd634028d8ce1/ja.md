```
abstract type AbstractDatabase end
```

データベースを表すための基本タイプ。データベースは、`AbstractVector`に似たインターフェースのようにアクセスできるオブジェクトのコレクションです。これは、`SimilaritySearch`メソッドがデータベースとオブジェクトを区別できるようにするために分離されています（ほとんどのオブジェクト表現はベクトルや行列のように見えるため）。

基本的な実装は次のとおりです：

  * `MatrixDatabase`: `Matrix`に格納されたオブジェクトベクトルのラッパーで、列がオブジェクトです。これは静的です。
  * `DynamicMatrixDatabase`: 新しいベクトルを追加できるベクトルの動的表現です。
  * `VectorDatabase`: ベクトルのような構造のラッパーです。あらゆる種類のオブジェクトを含むことができます。
  * `SubDatabase`: 特定のデータベースのサンプルです。

特に、ストレージの詳細は`VectorDatabase`と`MatrixDatabase`では使用されません。たとえば、`Matrix`、`SMatrix`、または`StrideArrays`のような行列を使用したり、`VectorDatabase`で生成されたオブジェクトを使用したり（ベクトルのようなインターフェースをサポート）することが可能です。

ストレージバックエンドがそれをサポートしている場合、次のようなベクトル操作を使用することができます：

  * `i`-番目の要素を取得 `obj = db[i]`、データベース内の要素は位置によって識別されます
  * インデックスのリスト`lst`における要素リストを取得 `db[lst]`（`view`を使用することも可能）
  * `i`-番目の要素に値を設定 `db[i] = obj`
  * ランダムサンプリング `rand(db)`, `rand(db, 3)`
  * データベース内のオブジェクトを反復して収集
  * データベース内の要素数を取得 `length(db)`
  * データベースの末尾に新しいオブジェクトを追加（すべての内部コンテナがこれをサポートするわけではありません）

      * `push_item!(db, u)` は単一の要素 `u` を追加します
      * `append_items!(db, lst)` はオブジェクトのリストをデータベースの末尾に追加します
