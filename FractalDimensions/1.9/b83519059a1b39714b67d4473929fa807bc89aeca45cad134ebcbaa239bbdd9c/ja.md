```
generalized_dim(X::StateSpaceSet [, sizes]; q = 1, base = 2) -> Δ_q
```

`X`の`q`階の一般化次元を返します。これは、各`ε ∈ sizes`に対してヒストグラムベースのレーニーエントロピーを計算することによって行われます。

`q = 0`の場合は「容量」または「ボックスカウント次元」と呼ばれ、`q = 1`は「情報次元」と呼ばれます。

## 説明

返される次元は、レーニーエントロピー$H_q$のスケーリングの（逆）べき法則指数によって近似されます。これはボックスサイズ`ε`に対して、`ε ∈ sizes`の関係です：

$$
H_q \approx -\Delta_q\log_{b}(\varepsilon)
$$

$$
H_q
$$

は`ComplexityMeasures: Renyi, ValueHistogram, entropy`を使用して計算されます。つまり、与えられたボックスサイズでデータのヒストグラムを作成することによって計算されます。

この関数を呼び出すと、多くの自動化されたステップが実行されます：

1. `sizes`が指定されていない場合、`sizes = estimate_boxsizes(dataset)`を呼び出すことによってボックスサイズのベクトルが決定されます。
2. `sizes`の各要素に対して、適切なエントロピーが次のように計算されます。

    `julia  H = [entropy(Renyi(; q, base), ValueHistogram(ε), data) for ε ∈ sizes]`

    `x = -log.(sizes)`とします。
3. 曲線`H(x)`は、[`slopefit`](@ref)`(x, h)[1]`を使用して線形領域に分解されます。
4. 最も大きな線形領域が選択され、その領域の傾きを[`linear_region`](@ref)を使用してフィットします。この関数は[`linreg`](@ref)を使用して単純な線形回帰フィットを行います。この傾きが`generalized_dim`の返り値です。

これらのステップを一つずつ自分で行うことで、これらの関数呼び出しに与えられるキーワード引数を調整し、結果の精度を向上させることができます。この関数のソースコードはわずか3行のコードです。

このフラクタル次元を推定するアプローチは、私たちの知る限り、[Russell1980](@cite)で初めて使用されました。
