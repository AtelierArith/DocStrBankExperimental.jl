```
PlainMeshBlockHeader{T,N}
```

メッシュは、変数が定義される位置を定義します。問題の幾何学が固定されており、ほとんどの変数が固定グリッドに対して相対的な位置で定義されるため、この位置データを独自のブロックに一度書き込むことは理にかなっています。各変数は、位置データを提供するためにこれらのメッシュブロックのいずれかを参照します。

`PlainMeshBlockHeader`は、スカラー場の離散化が定義される位置を表すために使用されます。ブロックヘッダーには、`base_header`（`BlockHeader`）と以下のメタデータが含まれています。

  * `mults`: グリッドデータに適用される正規化係数

各方向において。

  * `labels`: 各方向におけるこのグリッドの軸ラベル。
  * `units`: 正規化係数が適用された後の各方向におけるこのグリッドの単位。
  * `geometry`: ブロックの幾何学。
  * `minval`: 各方向における最小座標値。
  * `maxval`: 各方向における最大座標値。

ブロックの幾何学は以下の値を取ることができます。

  * `Null`: 未指定の幾何学。これはエラーです。
  * `Cartesian`: デカルト幾何学。
  * `Cylindrical`: 円筒幾何学。
  * `Spherical`: 球面幾何学。

ヘッダーの最後の項目は`dims`で、各次元のグリッドポイントの数です。

書き込まれるデータは、シミュレーションの各次元におけるメッシュのノードポイントの位置です。したがって、解像度が$(nx; ny; nz)$の3dシミュレーションの場合、データは$(nx + 1)$要素を持つX位置の1d配列、次に$(ny + 1)$要素を持つY位置の1d配列、最後に$(nz + 1)$要素を持つZ位置の1d配列で構成されます。ここで、解像度はシミュレーションセルの数を指定し、したがってノード値には1つの追加要素があります。1dまたは2dシミュレーションでは、それぞれXまたはXとYの配列のみを書き込みます。
