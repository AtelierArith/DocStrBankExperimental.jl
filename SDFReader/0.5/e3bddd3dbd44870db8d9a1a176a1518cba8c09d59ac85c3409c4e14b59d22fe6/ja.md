```
PlainVariableBlockHeader{T,N}
```

`PlainVariableBlockHeader`は、メッシュブロック内の指定されたポイントに対して相対的に位置する変数を記述するために使用されます。

ブロックヘッダーには`base_header`（`BlockHeader`）と以下のメタデータが含まれています。

  * `mult`: 変数データに適用される正規化係数。
  * `units`: 正規化係数が適用された後のこの変数の単位。
  * `mesh_id`: このブロックのデータが定義されているメッシュに対する名前（`id`）。
  * `dims`: 各次元のグリッドポイントの数。
  * `stagger`: 関連するメッシュに対する変数の位置。

変数に関連付けられたメッシュは常にノード中心であり、すなわち、メッシュデータとして書き込まれる値はグリッドのノード値を指定します。変数は、コード内のグリッドのスタッガリングにより、このグリッドからオフセットされたポイントで定義される場合があります。`stagger`エントリは、メッシュに対して変数がどこで定義されているかを指定します。関連するメッシュが含むポイントの数はすでに定義されているため、これは変数を表示するために必要なポイントの数を決定します。

`stagger`エントリは以下のいずれかの値を取ることができます。

  * `CellCentre`: セル中心。ノード間の中点に位置します。$(nx; ny; nz)$グリッドを意味します。
  * `FaceX`: X方向の面中心。Y-Z平面上のノード間の中点に位置します。$(nx + 1; ny; nz)$グリッドを意味します。
  * `FaceY`: Y方向の面中心。X-Z平面上のノード間の中点に位置します。$(nx; ny + 1; nz)$グリッドを意味します。
  * `FaceZ`: Z方向の面中心。X-Y平面上のノード間の中点に位置します。$(nx; ny; nz + 1)$グリッドを意味します。
  * `EdgeX`: X方向のエッジ中心。X軸に沿ったノード間の中点に位置します。$(nx; ny + 1; nz + 1)$グリッドを意味します。
  * `EdgeY`: Y方向のエッジ中心。Y軸に沿ったノード間の中点に位置します。$(nx + 1; ny; nz + 1)$グリッドを意味します。
  * `EdgeZ`: Z方向のエッジ中心。Z軸に沿ったノード間の中点に位置します。$(nx + 1; ny + 1; nz)$グリッドを意味します。
  * `Vertex`: ノード中心。メッシュと同じ場所に位置します。$(nx + 1; ny + 1; nz + 1)$グリッドを意味します。

グリッドベースの変数の場合、書き込まれるデータはメッシュ上の各ポイントでの指定された変数の値を含みます。これは、シミュレーションの次元に応じて1次元、2次元、または3次元の配列の形式です。配列のサイズは、関連するメッシュのサイズと上記で示されたグリッドのスタッガリングに依存します。これは、このブロックのために書き込まれた`dims`配列に書き込まれた値に対応します。
