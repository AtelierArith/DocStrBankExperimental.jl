```
Mesh!(turtle, m::Mesh; scale = Vec(1.0, 1.0, 1.0), move = false, transform = true,
      deepcopy = true, kwargs...)
```

既存のメッシュをタートルに供給します（オプションの変換付き）。

## 引数

  * `turtle`: メッシュを供給するタートル。
  * `m`: 標準の位置と向きにある既存のスケールされていないメッシュ。
  * `scale`: x、y、z軸のスケーリングファクターを持つベクトル。
  * `move`: タートルを前に移動させるかどうか（`true` または `false`）。
  * `transform`: ユーザーが提供したスケーリングベクトルに従って、タートルの位置と向きにメッシュを変換するかどうか。
  * `deepcopy`: タートルに供給する前にメッシュのディープコピーを作成するかどうか。
  * `kwargs`: メッシュ内の三角形ごとに設定されるプロパティ。

## 詳細

既存のメッシュは（`scale`に従って）スケーリングされ、タートルと同じ方向に向くように回転し、タートルの前にメッシュが生成されるように平行移動されます。変換の前に元のメッシュのディープコピーが作成されます。

`move = true` の場合、タートルは `height` に等しい距離だけ前に移動します。

この関数には2つの想定される使用ケースがあります：

1. ユーザーが異なるスケールでの変換を伴う同じメッシュの複数のインスタンスを生成するために使用される*参照*メッシュを定義する場合。例としては、葉の形を表すアドホックメッシュがあります。この場合、ユーザーは `transform = true` と `deepcopy = true` を設定したいと考えています。
2. ユーザーがタートルを引数として受け取り、そのメッシュに対していくつかの操作（例：面積の計算、中心の座標など）を行い、そのメッシュをタートルに供給する場合。この場合、ユーザーは `transform = false`（変換はすでに適用されているため）を設定したいと考え、ディープコピーは不要なコストであるため `deepcopy = false` にする可能性があります。

## 戻り値

`nothing` を返しますが、サイドエフェクトとして `turtle` を変更します。

## 例

```jldoctest
julia> import PlantGeomPrimitives as PG

julia> e = PG.Ellipse();

julia> turtle = Turtle();

julia> Mesh!(turtle, e, scale = PG.Vec(2.0, 2.0, 2.0));
```
