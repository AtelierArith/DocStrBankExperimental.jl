```
acceptshare(client::PCloudClient; kwargs...)
```

共有リクエストを承認します。

リクエストは、diffによって報告された`sharerequestid`で識別されるか、メールから送信された`code`によって識別されます。

フォルダ名のためにオプションの`name`を指定できます。指定しない場合は、共有名が使用されます。

実装者は、`name`が事前に入力された状態でローカル名を尋ねることを推奨します。

オプションで、共有をマウントするターゲットフォルダは`folderid`または`path`で識別できます。

フォルダが指定されていない場合、共有を受け入れるためのユーザーのデフォルトフォルダが使用されます。もしそのようなフォルダが存在しない場合は、ユーザーのルートディレクトリに適切な名前のフォルダが作成されます。

オプションのパラメータ`always`が設定されている場合、以降、承認するユーザーはデフォルトの共有フォルダに対して共有ユーザーからのリクエストを自動的に承認します。

出典: https://docs.pcloud.com/methods/sharing/acceptshare.html

# 引数

  * `sharerequestid::Int`: 共有リクエストのID
  * `code::String`: ユーザーのメールに送信されたコード

`sharerequestid`または`code`を使用します。

# オプション引数

  * `name::String`: フォルダ名を指定します。指定しない場合は、共有名を使用します。
  * `folderid::Int`: 共有をマウントするフォルダのID
  * `path::String`: 共有をマウントするポイントのファイルパス
  * `always::Int`: 設定されている場合、以降、承認するユーザーはデフォルトの共有フォルダに対して共有ユーザーからのリクエストを自動的に承認します。

`folderid`または`path`を使用します。

# 出力例

```
{
    "result": 0
}
```
