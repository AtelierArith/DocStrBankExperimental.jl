```
getzip(client::PCloudClient; kwargs...)
```

ユーザーのファイルシステムからzipファイルを受信します。

定義されたツリーをパラメータとして期待します。

ソース: https://docs.pcloud.com/methods/archiving/getzip.html

# オプション引数

  * `forcedownload::Int`: 設定されている場合、content-typeは'application/octet-stream'になります。設定されていない場合は'application/zip'になります。
  * `filename::String`: 提供されている場合、これは'Content-Disposition'ヘッダーとして返され、ブラウザがファイルをダウンロードする際にこのファイル名を採用するよう強制します。ファイル名は変更されずに渡されるため、.zip拡張子を含む必要があります。
  * `timeoffset::String`: 希望する時間オフセット

# 出力

成功すると、要求されたツリー内のすべてのファイルとディレクトリを含むzipアーカイブが現在のAPI接続を介して返されます。

生成されるファイルのサイズが4Gbを超える場合、または65535エントリを超える場合、zip64形式が使用されます。それ以外の場合、ファイルは通常のzipです。これは、APIサーバーがアーカイブをオンザフライで構築するため、zipファイルを生成する最も高速な方法です。したがって、マルチギガバイトのファイルであっても、ダウンロードは即座に開始されます。

zipファイルはファイルの変更時間に関するタイムゾーン情報をサポートしていないため、デフォルトでは生成されるzipファイル内のすべての日時値はUTCになります。代わりに、通常の+xxxxまたは-xxxx形式で希望する時間オフセットを指定する`timeoffset`パラメータを提供できます。また、+/-xx:xxやいくつかの名前付きタイムゾーン（EET、EEST、PST、CSTなど）もサポートされています。
