```
sharefolder(client::PCloudClient; kwargs...)
```

他のユーザーとフォルダーを共有します。

共有は、他のユーザーの確認が必要な場合があります。共有されるフォルダーは `folderid` または `path` で識別されます。

オプションの共有 `name` を提供できます。提供しない場合は、フォルダー名が共有名として使用されます。実装者は、共有ユーザーが共有 `name` を選択できる機会を与えることが推奨されており、これはフォルダー名で事前に入力されるべきです。

必須パラメーター `mail` には、フォルダーを共有するユーザーのメールアドレスが含まれます。

必須パラメーター `permissions` は、フォルダーの権限を設定します。

読み取り専用の場合はゼロ、または次のいずれかの組み合わせ（合計/または）：

  * `1`: 作成権限
  * `2`: 修正権限
  * `4`: 削除権限

オプションのパラメーター `message` は、受信ユーザーに渡すメッセージを追加することを許可します。

フォルダーの共有は複雑な操作であり、次のエラーが返される可能性があります：- `2014`: ユーザーのアドレスが確認されていません。実装者は、ユーザーの確認後に sendverificationemail を呼び出し、ユーザーにメールを確認するように依頼することが推奨されます。

  * `2015`: ルートフォルダーは共有できません。このチェックはクライアントでも実行されるべきです。
  * `2016::ismine`: 自分のものであるフォルダーのみを共有できます。実装者はこれをローカルでも確認するべきです。
  * `2017`: ユーザーはあなたや他の誰からのリクエストを受け入れません。実装者はこれが発生するかどうかを知ることはできませんが、このエラーに対して適切に行動することが期待されています。

出典: https://docs.pcloud.com/methods/sharing/sharefolder.html

# 引数

  * `folderid::Int`: 共有フォルダーのフォルダーID
  * `path::String`: 共有フォルダーへのパス
  * `mail::String`: フォルダーを共有するユーザーのメール
  * `permissions::Int`: 権限フラグのビット単位の組み合わせ

`path` または `folderid` を使用

# オプションの引数

  * `name::String`: 共有の名前。デフォルト - フォルダー名。
  * `message::String`: 受信ユーザーに渡すメッセージ。
