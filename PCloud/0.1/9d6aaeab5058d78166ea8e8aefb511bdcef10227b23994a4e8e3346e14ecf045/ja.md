```
getthumblink(client::PCloudClient; kwargs...)
```

ファイルのサムネイルへのリンクを取得します

`fileid`（または`path`）をパラメータとして受け取り、ファイルのサムネイルをダウンロードできるリンクを提供します。

サムネイルは、メタデータの`thumb`値が`true`に設定されているファイルからのみ作成できます。

パラメータ`size`は、WIDTHxHEIGHTの形式で提供する必要があります。

幅は16から2048の間で、4または5で割り切れる必要があります。

高さは16から1024の間で、4または5で割り切れる必要があります。

デフォルトでは、サムネイルは元の画像と同じアスペクト比を持つため、結果として得られるサムネイルの幅または高さ（ただし両方ではない）が要求されたサイズよりも小さくなる場合があります。

指定されたサイズのサムネイルを正確に作成したい場合は、`crop`パラメータを設定できます。`crop`を使用すると、サムネイルは依然として正しいアスペクト比を持ちますが、必要に応じて両側からいくつかの行または列（ただし両方ではない）が切り取られます。したがって、1024x768の画像があり、128x128のサムネイルを作成しようとすると、最初に画像は128列を両側から切り取って768x768に変換され、その後128x128にリサイズされます。4:3の画像から長方形のサムネイルを作成するには、各側から正確に1/8が切り取られます。デフォルトでは、サムネイルはjpeg形式です。

`type`パラメータがpngに設定されている場合、png画像が生成されます。

サムネイルは最初のリクエスト時に作成され、指定されていない期間（またはファイルが変更されるまで）キャッシュされます。

クライアントは、スペースが許す限りサムネイルをキャッシュすることを試みるべきです。

元のファイルの`hash`が変更されたかどうかを監視することも推奨されます。変更されている場合は、新しいサムネイルをリクエストする必要があります。

出典: https://docs.pcloud.com/methods/thumbnails/getthumblink.html

# 引数

  * `fileid::Int`: サムネイル用のファイルのID
  * `path::String`: サムネイル用のファイルのファイルパス
  * `size::String`: WIDTHxHEIGHT

`fileid`または`path`を使用します

# オプション引数

  * `crop::Int`: 設定されている場合、サムネイルが切り取られます
  * `type::String`: pngに設定されている場合、サムネイルはpng形式になります

# 出力

成功した場合、`getfilelink`と同じデータが返されます。

さらに、生成された実際の画像の`size`が返されます。

`crop`が指定されている場合は要求されたサイズと一致し、それ以外の場合は異なる場合があります。

# 出力例

```
{
    result: 0,
    size: "32x32",
    path: "/hash/My%20thumb.jpg",
    expires: "Thu, 03 Oct 2013 01:06:49 +0000",
    hosts: [
        "c63.pcloud.com",
        "c1.pcloud.com"
    ]
}
```
