```
bsd_checksum(UInt16, data, init::UInt16 = zero(UInt16))
```

データのハッシュをBSDのsumユーティリティで定義された方法（およびGNU sumプログラムで実装された方法）を使用して計算します。

BSDチェックサムは、前のステップからのチェックサム値を右に1ビットビットローテーションし、次のデータの値を加算して繰り返すことによって16ビットのチェックサム値を計算します。合計はオーバーフローすることが許可されており、合計はゼロにリセットされます。

この関数は高速ですが、多くの衝突に悩まされます。考慮すべき欠陥は次のとおりです：

1. データの最初にあるゼロ値は、初期値がゼロ以外に設定されていない限り、チェックサム値に全く影響を与えません。
2. データ内のゼロの連続が16の倍数の長さである場合、チェックサム値には全く影響を与えません。

## 引数

  * `data`: 単一の`Unsigned`値または値のイテレータ。
  * `init::UInt16 = zero(UInt16)`: オプションの開始値。

事前定義された便利な関数は`bsd16`です。
