```
vcreate([P,] A, x, scratch=false) -> y
```

は、引数 `x` にマッピング `A` を適用した結果を格納するのに適した新しいインスタンス `y` を生成します。オプションのパラメータ `P ∈ Operations` は、`Direct`（デフォルト）、`Adjoint`、`Inverse` および/または `InverseAdjoint` のいずれかであり、`A` をどのように適用するかを指定するために使用できます。これは [`apply`](@ref) メソッドのドキュメントで説明されています。

オプションの引数 `scratch` は、入力引数 `x` が操作によって上書きされる可能性があるかどうかを示し、その結果を格納するために使用されることができます。これは、一部のマッピング（*インプレース*で操作できるもの）が新しいオブジェクトを結果 `y` のために割り当てるのを避けるために利用されることがあります。

呼び出し元は、`apply` を呼び出した後に `x` が必要ない場合は `scratch = true` に設定するべきです。`scratch = true` の場合、`y` が `x` と同じオブジェクトである可能性があります。そうでなければ、`y` は新しいオブジェクトであり、操作を適用した結果が `x` の内容と同じである場合を除きます（例えば、恒等性の場合は常に真です）。したがって、一般的には、返される `y` が入力 `x` と異なるとは限らないと仮定すべきではありません。

メソッド `vcreate(::Type{P}, A, x)` は、サポートされている任意の操作 `P` および引数タイプ `x` に対して線形マッピングによって実装されるべきです。`vcreate` によって返される結果は、*型の安定性*を確保するために予測可能な型であるべきです。`vcreate` で引数 `x` の有効性（*例* サイズ）をチェックすることは省略できます。なぜなら、この引数は最終的に `apply!` メソッドによってチェックされるからです。
