```
apply([P=Direct,] A, x, scratch=false) -> y
```

は、引数 `x` にマッピング `A` を適用した結果 `y` を返します。オプションのパラメータ `P` は、`A` をどのように適用するかを指定するために使用できます：

  * `Direct`（デフォルト）で `A` を適用し、`y = A⋅x` を得る；
  * `Adjoint` で `A` の随伴を適用し、`y = A'⋅x` を得る；
  * `Inverse` で `A` の逆を適用し、`y = A\x` を得る；
  * `InverseAdjoint` または `AdjointInverse` で `A'` の逆を適用し、`y = A'\x` を得る。

すべての操作が異なるタイプのマッピングによって実装されているわけではなく、`Adjoint` および `InverseAdjoint` は線形マッピングにのみ適用可能な場合があります。

オプションの引数 `scratch` は、入力引数 `x` が操作によって上書きされる可能性があるかどうかを示します。これは、一時的な作業領域の割り当てを避けるために利用できます。呼び出し元は、`apply` を呼び出した後に `x` が必要ない場合、`scratch = true` を設定するべきです。`scratch = true` の場合、`y` が `x` と同じオブジェクトである可能性があります；そうでなければ、`y` は新しいオブジェクトであり、操作を適用した結果 `x` の内容が `y` と同じである場合を除きます（例えば、恒等の場合は常に真です）。したがって、一般的には、マッピングを適用した結果が入力と異なるとは限らないと仮定すべきではありません。

Julia メソッドは、`apply(A', x)` が自動的に `apply(Adjoint, A, x)` を呼び出すように提供されているため、パフォーマンスに影響を与えることなく短い構文を使用できます。
