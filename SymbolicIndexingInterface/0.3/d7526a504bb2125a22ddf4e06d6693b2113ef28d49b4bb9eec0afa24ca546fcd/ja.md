```
remake_buffer(indp, oldbuffer, idxs, vals)
```

`oldbuffer`のコピーを返します。ここで、(オプションでシンボリックな) インデックス `idxs` が `vals` の対応する値で置き換えられます。`idxs` と `vals` は同じ長さのイテラブルでなければなりません。`idxs` には、バッファ内のインデックスが `indp` を使用して決定されるシンボリック変数が含まれている場合があります。`vals` の値の型は、バッファ内の対応するインデックスに格納されている値の型と一致しない場合があり、その場合、バッファの型はそれに応じて昇格されるべきです。一般的に、このメソッドは `vals` に格納されている値の型をできるだけ保持しようとします。型の昇格は、型の安定性を保ち、パフォーマンスを維持するために行われます。返されるバッファは、`oldbuffer` と同じ型（型パラメータを無視）であるべきです。

このメソッドはすでに `oldbuffer::AbstractArray` および `oldbuffer::Tuple` に対して実装されており、静的配列もサポートしています。

シンボルを値にマッピングする `Dict` を `idxs` と `vals` の代わりに取るこのメソッドの非推奨バージョンは、新しいメソッドにディスパッチされます。さらに、新しいシグネチャの `remake_buffer` メソッドが存在しない場合、`remake_buffer(sys, oldbuffer, Dict(idxs .=> vals))` が呼び出されます。

新しいメソッドシグネチャでは、`idxs` がシンボリック変数である必要がなく、インデックスであることを許可しています。したがって、新しいメソッドを実装する任意の型は、`idxs` にインデックスをサポートする必要があります。
