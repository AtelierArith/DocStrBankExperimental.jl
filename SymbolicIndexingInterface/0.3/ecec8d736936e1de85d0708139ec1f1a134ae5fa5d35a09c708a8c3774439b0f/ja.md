```
getp(indp, sym)
```

値プロバイダーを受け取り、パラメーター `sym` の値を返す関数を返します。値プロバイダーは、対応するインデックスプロバイダーにパラメーターの値を少なくとも格納している必要があります。`sym` はインデックス、記号変数、または前述の配列/タプルである可能性があります。

`sym` がパラメーターの配列/タプルである場合、返される関数はインプレースのゲッター関数としても使用できます。最初の引数はバッファ（`AbstractArray` である必要があります）で、パラメーターの値が書き込まれるべきものであり、2番目の引数は値プロバイダーです。

値プロバイダーが [`parameter_values`](@ref) を実装することが要求されます。この関数は常に実装する必要はなく、`getindex` を実装するコレクションにはデフォルトの実装があります。

返された関数がパラメーターのタイムシリーズを保存するタイムシリーズオブジェクトで使用されると、該当するタイムシリーズをインデックスするために使用できます。タイムシリーズオブジェクトは [`is_parameter_timeseries`](@ref) と [`get_parameter_timeseries_collection`](@ref) を実装する必要があります。さらに、パラメーターオブジェクトは [`with_updated_parameter_timeseries_values`](@ref) を実装する必要があります。

`sym` がタイムシリーズパラメーターである場合、関数は値プロバイダーがパラメータータイムシリーズオブジェクトである場合にパラメーターのタイムシリーズを返します。関数には、タイムシリーズ内の値にアクセスする特定のインデックスを示す追加の引数を提供できます。`sym` がパラメーターの配列である場合、次のケースが適用されます：

  * すべてのパラメーターが非タイムシリーズパラメーターである場合：関数は各パラメーターの値を返します。
  * すべてのパラメーターがタイムシリーズパラメーターである場合：すべてのパラメーターは同じタイムシリーズに属する必要があります（そうでない場合、`getp` はエラーになります）。関数はすべてのパラメーター値のタイムシリーズを返し、タイムシリーズ内の特定のインデックスでアクセスできます。
  * タイムシリーズパラメーターと非タイムシリーズパラメーターの混合：関数は*非タイムシリーズオブジェクト*でのみ使用でき、オブジェクト内の各パラメーターの値を返します。
