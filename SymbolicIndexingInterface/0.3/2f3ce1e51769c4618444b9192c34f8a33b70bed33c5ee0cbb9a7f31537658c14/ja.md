```
getsym(indp, sym)
```

値プロバイダーを受け取り、シンボリック変数 `sym` の値を返す関数を返します。`sym` が観測された量でない場合、返された関数は状態ベクトルを表す値の配列で直接呼び出すこともできます。`sym` は状態ベクトルへのインデックス、シンボリック変数、インデックスプロバイダー `indp` に含まれるシンボリック変数を含むシンボリック式、パラメータシンボル、独立変数シンボル、または前述のものの配列/タプルである可能性があります。返された関数がタイムシリーズオブジェクトで呼び出される場合、`sym` の値を返すインデックスを表す2番目の引数を与えることもできます。

最低限、これには値プロバイダーが [`state_values`](@ref) を実装する必要があります。シンボリック式をサポートするために、値プロバイダーは [`observed`](@ref)、[`parameter_values`](@ref)、および [`current_time`](@ref) を実装する必要があります。

この関数は通常実装する必要はなく、上記の関数に依存するデフォルト実装があります。

値プロバイダーがパラメータタイムシリーズオブジェクトである場合、[`getp`](@ref) と同じルールが適用されます。ここでの違いは、`sym` が非パラメータシンボルを含む可能性があり、値は常に状態タイムシリーズに対応して返されることです。
