```
function sympixels(; P, nrim, halfnint, patternvec, units, kwargs...)
```

# 説明:

対称的にピクセル化された正方形ユニットセルの三角形分割を含む `RWGSheet` 型の変数を作成します。金属ピクセルのパターンは C4 対称性を持ち、左右対称および上下対称の鏡像対称性を持ち、各象限は反対対角線の鏡像対称性を示します。返される値は、フィールド `s₁`、`s₂`、`β₁`、`β₂`、`ρ`、`e1`、`e2`、`fv`、`fe`、および `fr` が適切に初期化されています。三角形分割に含まれるピクセルは、以下に説明する `patternvec` 入力ベクトルによって決定されます。

# 引数:

すべての引数はキーワード引数であり、任意の順序で入力できます。

## 必須引数:

  * `P::Real > 0`: ユニットセルの正方形の辺の長さで、`units` キーワード引数で定義された単位で指定されます。
  * `nrim::Integer`: ユニットセルの境界内に配置される固体金属リムの幅で、ピクセルの単位で指定されます。`0` の値はリムがないことを意味します。
  * `halfnint`: ユニットセルの内部（リムの内側）正方形ピクセル化領域の辺の長さにおけるピクセルの半分の数です。
  * `patternvec::AbstractVector{<:Integer}`: 長さ `halfnint*(halfnint+1)÷2` のベクトルで、1 と 0 のみで構成されています。このベクトルの要素は、ユニットセルの不可約ゾーン内のピクセルにマッピングされ、![https://simonp0420.github.io/PSSFSS.jl/stable/assets/sympixel_with_irzone_numbering.png](https://simonp0420.github.io/PSSFSS.jl/stable/assets/sympixel_with_irzone_numbering.png) に示されている図のように配置されます。不可約ゾーン内では、値が `1`（または `true`）に対応するピクセルは金属化された領域と見なされ、`0` または `false` の値は金属のない（空洞）領域です。これは、`class` 値が `J` または `M` のいずれであっても当てはまります（重要な制限については以下の `class` 引数を参照）。
  * `units`: `P` の長さの単位（`mm`、`cm`、`inch`、または `mil` のいずれか）。

## オプション引数:

  * `pdiv::Int = 1`: 三角形分割を形成する際に各正方形ピクセルの辺に適用される「切り分け」または細分化の数です。`1`（デフォルト）の値は、三角形分割に含まれるピクセル（`class='J'` の `1` または `true` の値、`class='M'` の `0` または `false` の値）が、各正方形ピクセルに対して三角形を形成するための単一の対角線を除いて、さらに細分化されないことを意味します。`n>1` の値は、各正方形ピクセルが最初に `n×n` の正方形サブピクセルに分割され、その後、各サブピクセルに対して単一の対角線（`quad=false` の場合）が追加されて三角形が形成されることを意味します。
  * `quad::Bool=false`: `true` の場合、各サブピクセル（または `pdiv` が 1 の場合はピクセル）は、2 つの対角線を追加することによって 4 つの三角形に分割されます。`false`（デフォルト）の場合、各正方形には 2 つの三角形を生成するために単一の対角線のみが追加されます。
  * `sym::Bool = false`: `true` の場合、正方形に追加される対角線は、`true`（`false`）ピクセル位置の集合と同じ左右対称および上下対称の鏡像対称性を示します。`false` で `quad=false` の場合、ユニットセル全体に追加されるすべての対角線は同じ向きを持ちます。`quad=true` の場合、2 つの追加された対角線がすでに三角形分割の鏡像対称性を保証するため、`sym` は効果がありません（すなわち冗長です）。
  * `class::Char='M'`  クラスを指定します。`'J'` または `'M'` のいずれかです。`'J'` の場合、未知数はピクセルの `1` 値に対応する領域の電気表面電流です。`'M'` の場合、未知数はピクセルの `0` 値に対応する領域の磁気表面電流です。隣接する金属ピクセルが単一の点でのみ交差する幾何学に対して、`'J'` を使用すると非常に不正確な結果が得られることが知られています。したがって、ほとんどの `pixels` 要素およびすべての `sympixels` 要素に対しては、`'M'` のみの使用が強く推奨されます。
  * `dx::Real=0.0`, `dy::Real=0.0`: これらは、ユニットセル全体とその内容に適用される x および y 方向のオフセットを指定します。長さの単位は `units` キーワードで指定されたものです。
  * `rot::Real=0.0`: ユニットセル全体とその内容に適用される反時計回りの回転角度（度）です。この回転は、`dx` および `dy` で指定されたオフセットの前に適用されます。
  * `Zsheet::Complex=0.0`: FSS 導体の周波数に依存しない表面インピーダンスで、単位は [Ω] です。クラス `'J'` のシートに対してのみ指定できます。`Zsheet` が指定されている場合、`sigma`（または `σ`）は指定できません。
  * `sigma` または `σ`: DC、バルク導電率 [S/m]。クラス `'J'` のシートに対してのみ許可されます。`Zsheet` と同時に指定することはできません。これは、PSSFSS によって周波数ごとの有効なシート表面インピーダンスを計算するために、Gradient Model（Grujić 2022）を使用して `Rq` と共に使用されます。
  * `Rq=0.0`: RMS 表面粗さ [m]。クラス `'J'` の場合のみ合法です。`sigma`（または `σ`）が指定されている場合にのみ使用されます。その場合、`sigma` と共に使用して、Gradient Model を使用して周波数依存のシートインピーダンスを計算します。デフォルト値の 0 は滑らかな表面を示します。
  * `disttype::Symbol=:normal`: 表面粗さの確率分布タイプ。デフォルトは `:normal` です。他の合法的な値は `:rayleigh` です。
  * `fufp::Bool`: このキーワードは通常必要ありません。`fufp` は「ユニークな面ペアを見つける」の略です。`true` の場合、コードは三角形分割内でトポリッツの意味で同等の三角形ペアのクラスを検索します。すなわち、三角形ペア (A,B) と (C,D) が同じ同値クラスに属する場合、ペア (A,B) の 6 つの頂点は、単純な平行移動によってペア (C,D) の頂点と一致させることができます。このような同等のペアが多数存在する場合、同値性を利用することで行列のフィル時間が大幅に短縮されます。トレードオフは、それらを特定するのに必要な時間です。デフォルト値は、`strip`、`diagstrip`、`meander`、`manji`、`loadedcross`、`jerusalemcross`、`pixels`、4 辺の `polyring` スタイル（構造化メッシュを使用するもの）および `sympixels` に対して `true` であり、残りのスタイル（非構造化メッシュを使用するもの）には `false` です。
