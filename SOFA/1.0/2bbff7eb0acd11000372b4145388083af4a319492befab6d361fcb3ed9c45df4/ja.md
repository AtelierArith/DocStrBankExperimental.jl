地上の観測者のために、CIRSと観測座標間の変換のための星に依存しない天文学パラメータを準備します。呼び出し元はUTC、サイトの座標、周囲の空気条件、および観測波長を提供します。

この関数は、国際天文学連合のSOFA（基礎天文学の標準）ソフトウェアコレクションの一部です。

ステータス：サポート関数。

与えられたもの： utc1   double      UTCの2部... utc2   double      ...準ジュリア日（注1,2） dut1   double      UT1-UTC（秒） elong  double      経度（ラジアン、東+ve、注3） phi    double      測地緯度（ラジアン、注3） hm     double      楕円体上の高さ（m、測地的注4,6） xp,yp  double      極運動座標（ラジアン、注5） phpa   double      観測者の圧力（hPa = mB、注6） tc     double      観測者の周囲温度（℃） rh     double      観測者の相対湿度（範囲0-1） wl     double      波長（マイクロメートル、注7）

返されるもの： astrom iauASTROM*  星に依存しない天文学パラメータ： pmt    double       変更なし eb     double[3]    変更なし eh     double[3]    変更なし em     double       変更なし v      double[3]    変更なし bm1    double       変更なし bpn    double[3][3] 変更なし along  double       経度 + s'（ラジアン） xpl    double       極運動xpに対するローカル子午線（ラジアン） ypl    double       極運動ypに対するローカル子午線（ラジアン） sphi   double       測地緯度の正弦 cphi   double       測地緯度の余弦 diurab double       日周のアベレーションベクトルの大きさ eral   double       "ローカル" 地球回転角（ラジアン） refa   double       屈折定数A（ラジアン） refb   double       屈折定数B（ラジアン）

返されるもの（関数値）：         int         ステータス： +1 = 疑わしい年（注2）                                 0 = OK                         -1 = 受け入れられない日付

注：

1. utc1+utc2は準ジュリア日（注2）であり、2つの引数の間で任意の便利な方法で配分されます。たとえば、utc1がジュリア日数で、utc2が1日の分数である場合です。

ただし、JDは、特別な措置が講じられない限り、うるう秒中のUTCを明確に表すことはできません。現在の関数の慣例では、JDの日は、長さが86399、86400、または86401 SI秒であってもUTCの日を表します。

アプリケーションは、カレンダーの日付と時間を2部の準ジュリア日に変換するために、iauDtf2d関数を使用する必要があります。これは、前述のうるう秒の曖昧さの慣例を実装しています。

2. 警告ステータス「疑わしい年」は、時間尺度の導入前のUTCや、信頼できないほど未来のUTCをフラグします。詳細についてはiauDatを参照してください。
3. UT1-UTCはIERSの公報に表形式で示されています。これは、正のUTCうるう秒の終わりに正確に1秒増加します。これは、UT1-UTCを+/- 0.9秒以内に保つために導入されました。注意：この慣行は見直し中であり、将来的にはUT1-UTCが本質的に無制限に増加する可能性があります。
4. 地理座標はWGS84基準楕円体に関してです。経度の符号に注意してください：現在の関数で必要な経度は東が正（すなわち右手系）であり、地理的慣例に従っています。
5. 極運動xp,ypはIERSの公報から取得できます。値は、国際地球基準系に対する天体中間極の座標（ラジアン）であり、それぞれ子午線0度および90度西に沿って測定されます。多くのアプリケーションでは、xpとypをゼロに設定できます。

内部的には、極運動はローカル子午線に回転した形で保存されます。

6. 観測ステーションの楕円体上の高さhm（メートル）が不明で、圧力phpa（hPa = mB）が利用可能な場合、次の式からhmの適切な推定値を得ることができます。

```
    hm = -29.3 * tsl * log ( phpa / 1013.25 );
```

ここで、tslはKでの海面上の空気温度の近似値です（Astrophysical Quantities, C.W.Allen, 第3版、セクション52を参照）。同様に、圧力phpaが不明な場合、観測ステーションの高さhmから次のように推定できます。

```
    phpa = 1013.25 * exp ( -hm / ( 29.3 * tsl ) );
```

ただし、屈折は圧力にほぼ比例し、正確なphpa値が精密作業にとって重要であることに注意してください。

7. 引数wlは、マイクロメートル単位の観測波長を指定します。光学からラジオへの遷移は、100マイクロメートル（約3000 GHz）で発生すると仮定されています。
8. 入力パラメータの単位に注意を払うことが推奨されます。なぜなら、ありそうもない値の入力パラメータも受け入れられ、使用されるモデルに従って処理されるからです。
9. 呼び出し元が独自の地球回転情報と屈折定数を提供したい場合、iauApc関数を現在の関数の代わりに使用できます。
10. これは、ICRS <-> GCRS <-> CIRS <-> 観測の天文学的変換の連鎖に必要な星に依存しないパラメータをastrom構造に挿入するいくつかの関数の1つです。

さまざまな関数は、異なる観測者のクラスと変換チェーンの部分をサポートします：

```
    関数             観測者        変換
```

iauApcg iauApcg13    地心的      ICRS <-> GCRS iauApci iauApci13    地上の      ICRS <-> CIRS iauApco iauApco13    地上の      ICRS <-> 観測 iauApcs iauApcs13    宇宙         ICRS <-> GCRS iauAper iauAper13    地上の      地球回転の更新 iauApio iauApio13    地上の      CIRS <-> 観測

「13」で終わる名前のものは、さまざまな天体暦を計算するために現代のSOFAモデルを使用します。他のものは、呼び出し元が提供する天体暦を受け入れます。

ICRSからGCRSへの変換は、空間運動、視差、光の屈折、およびアベレーションをカバーします。GCRSからCIRSへの変換は、フレームバイアスと歳差- nutationを含みます。CIRSから観測への変換は、地球の回転、極運動、日周のアベレーション、および視差（ICRS <-> GCRS変換に組み込まれていない限り）、および大気の屈折を考慮します。

11. この関数によって生成されたコンテキスト構造astromは、iauAtioqおよびiauAtoiqによって使用されます。
