```
tiled_processing(tiled_view::TiledView, fct; verbose=true, dtype=eltype(tiled_view.parent), window_function=window_hanning)
```

タイルビューを使用して生データセットを処理し、各タイルを関数 `fct` に送信し、結果を `window_function` を介してマージします。

結果のタイルビューを返します。タイルされていない結果は `.parent` メンバーを介してアクセスできます。

複数の戻り値をタプルとして持つ関数も処理できるように、アクセサ関数を使用してタプルから関連する値を抽出できます。デフォルトは恒等関数で、単一の値を返す関数に使用されます。アクセサ関数のタプルが提供されると、各アクセサ関数が呼び出され、タプルの各要素に対して個別の配列が構築され、タプルとして返されます。

## 引数

  * `tiled_view`: TiledViewに分解する入力データ。TiledViewのためにコピーは作成されず、生データは `.parent` を介してアクセスできます。
  * `fct`: 各タイルに適用される関数。関数は単一の引数、すなわちタイルデータを受け入れる必要があります。
  * `verbose`: true の場合、ウィンドウレイアウトに関する診断情報が印刷されます。
  * `dtype`: 新しい配列の要素型。デフォルトでは、`tiled_view` の親配列の型と同じです。
  * `window_function`: TiledViewに適用するためにIndexFunArraysで定義されたウィンドウ関数。結果は現在、乗算をビューにラップする方法が不明なため、もはやビューではありません。この理由から、ウィンドウが適用されていないTiledViewも返され、代入に使用できます。デフォルトではハニングウィンドウ（window_hanning）が使用されます。
  * `accessor`: タイル内のデータにアクセスするための関数。デフォルトでは恒等関数です。
