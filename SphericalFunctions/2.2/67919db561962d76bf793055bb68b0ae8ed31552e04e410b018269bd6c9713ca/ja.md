```
D_iterator(D, ℓₘₐₓ, [ℓₘᵢₙ])
```

`D`の部分行列を返すイテレータを構築します。各部分行列は、$ℓ$が`ℓₘᵢₙ`から`ℓₘₐₓ`までの範囲で、要素$(ℓ,-ℓ,-ℓ)$から$(ℓ,ℓ,ℓ)$で構成されます。デフォルトでは、`ℓₘᵢₙ`は0です。

!!! danger "一貫性のない動作"
    このイテレータは、ドキュメントで示唆されている結果の転置を誤って返します。その結果、この関数が呼び出されるたびに警告が発生します。このマイナー/パッチバージョンでこのバグを実際に*修正*するのではなく（これは破壊的変更になります）、これはこのパッケージのメジャーバージョンの最終リリースであり、この関数（および`d_iterator`）に問題があることをユーザーに通知します。このパッケージの次のメジャーバージョンでは、実際の動作がこのドキュメントで示唆されているものに変更される可能性があります。これらの警告を抑えるには、一時的にキーワード引数`warn=false`を渡すことができますが、これは次のメジャーバージョンで削除される可能性があります。あるいは、`Dit = with_logger(NullLogger()) do D_iterator(...) end`を使用して、警告をキャッチすることもできます。


返されるオブジェクトは、元の`D`データへの*ビュー*であることに注意してください。つまり、これらの値を変更することができます。

結果は特定の$ℓ$値に制限された行列であるため、$(ℓ, m′, m)$要素を`[ℓ+m′+1, ℓ+m+1]`としてインデックスできます。たとえば、次のように使用することができます。

```
for (ℓ, Dˡ) in zip(ℓₘᵢₙ:ℓₘₐₓ, D_iterator(D, ℓₘₐₓ))
    for m′ in -ℓ:ℓ
        for m in -ℓ:ℓ
            Dˡ[ℓ+m′+1, ℓ+m+1]  # ... Dˡを使って何かをする
        end
    end
end
```

また、インスタンス化時やイテレーション中に境界チェックは行われないことに注意してください。`D`のサイズと`ℓₘₐₓ`および`ℓₘᵢₙ`の値が意味を持つことを確認するのはあなたの責任です。 ```
