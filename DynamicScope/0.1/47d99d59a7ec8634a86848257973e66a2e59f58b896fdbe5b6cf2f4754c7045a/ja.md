```
@dyn function name(...) ... end
```

マクロ `@name` を生成し、呼び出しスコープでデフォルト引数と宣言された `@requires` を解決します。

この関数が内部の動的関数に提供するローカル変数は、バブルアップを防ぐために `@provides` で宣言する必要があります。関数パラメータは自動的に提供されます。

`@requires` と `@provides` の順序は重要であり、関数が変数を提供し宣言できるようにします。例えば `a=a+2` の場合です。例外として、最初の `@requires` は関数引数によって提供された変数を上書きします。例えば `function foo(a=a+2); @requires a` の場合です。
