```
SandboxConfig(read_only_maps, read_write_maps, env)
```

サンドボックスエグゼキュータは、環境を適切に設定するための構成を必要とします。

  * `read_only_maps`: サンドボックスに読み取り専用マッピングとしてマッピングされるディレクトリ。

      * ペアとして指定されます。例: `sandbox_path => host_path`。すべてのパスは絶対パスでなければなりません。
      * ルートのマッピングを常に含める必要があります。例: `"/" => rootfs_path`。
  * `read_write_maps`: サンドボックスに読み書き可能なマッピングとしてマッピングされるディレクトリ。

      * ペアとして指定されます。例: `sandbox_path => host_path`。すべてのパスは絶対パスでなければなりません。
      * 一部のエグゼキュータは完璧なライブ更新を表示しない場合があります。整合性は実行が終了した後にのみ保証されます。
  * `env`: サンドボックス内で設定されるべき環境変数の辞書マッピング。
  * `entrypoint`: 実行される実際のコマンドが渡される実行可能ファイル。

      * これはサンドボックス内のパスであり、絶対パスでなければなりません。
      * デフォルトは `nothing` で、これによりコマンドが直接実行されます。
  * `pwd`: 実行されるコマンドの作業ディレクトリを設定します。

      * これはサンドボックス内のパスであり、絶対パスでなければなりません。
  * `persist`: エグゼキュータオブジェクトにrootfsに加えられた変更を永続化するように指示します。

      * これはブール値であり、エグゼキュータによって解釈されます。
      * 永続性は個々のエグゼキュータの特性であり、エグゼキュータオブジェクト自体が存在する限りのみライブで変更されます。
      * 永続的な変更を1つのエグゼキュータから別のエグゼキュータに転送することはできません。
  * `multiarch`: マルチアーキテクチャ実行可能ファイルのサポートをリクエストします。

      * これは `Platform` オブジェクトの配列です。
      * サンドボックスは、各プラットフォームに対してインタープリタ（`qemu-*-static` バイナリなど）が利用可能であることを保証します。
      * サポートされていないプラットフォームに対してマルチアーキテクチャサポートをリクエストすると、`ArgumentError` が発生します。
  * `uid` と `gid`: サンドボックス化されたプロセスを生成するための数値ユーザーおよびグループ識別子。

      * デフォルトでは、これらは両方とも `0` で、サンドボックス内の `root` を示します。
  * `stdin`, `stdout`, `stderr`: サンドボックス化されたプロセスの入力/出力ストリーム。

      * 任意の種類の `IO`、`TTY`、`devnull` などが使用できます。
  * `hostname`: サンドボックス内のホスト名を設定します。デフォルトは現在のホスト名です。
  * `verbose`: サンドボックス構築プロセスがより詳細または簡潔であるべきかを設定します。
