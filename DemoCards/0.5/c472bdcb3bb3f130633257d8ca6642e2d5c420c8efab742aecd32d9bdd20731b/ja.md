```
preview_demos(demo_path; kwargs...)
```

単一のデモカードのドキュメントプレビューを生成します。

# 戻り値

  * `index_file`: デモページのインデックスファイルへの絶対パス。このファイルをブラウザで開くと、生成されたデモを見ることができます。

# 引数

  * `demo_path`: デモファイルまたはフォルダへのパス。デモファイルへのパス、または複数のスクリプトのフォルダである可能性があります。標準のデモページフォルダ構造が検出された場合はそれを使用し、そうでない場合はそのプレビュー版を作成します。

# パラメータ

  * `theme`: プレビューで使用したいカードテーマ。デフォルトでは、ページ設定ファイルから推測されます。インデックスページを生成しないようにするには、`nothing`を渡すことができます。テーマの選択肢については、[`cardtheme`](@ref DemoCards.cardtheme)も参照してください。
  * `assets = String[]`: これは`Documenter.HTML`に渡されます。
  * `edit_branch = "master"`: `makedemos`と同様です。
  * `credit = true`: `makedemos`と同様です。
  * `throw_error = false`: `makedemos`と同様です。
  * `require_html = true`: Documenterワークフローをトリガーし、HTMLプレビューを生成する必要があるかどうか。`false`に設定すると、戻り値は生成された`index.md`ファイルへのパスになります。これは実験的なキーワードであり、安定したものとは見なされるべきではありません。
  * `clean = true`: 既存のサンドボックスビルディングディレクトリを最初にクリーンアップする必要があるかどうか。
