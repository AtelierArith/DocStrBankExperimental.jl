```
H, N = tfest(data, σ = 0.05, method = :corr)
```

コレログラムアプローチ（デフォルト）を使用して、信号モデル $y = H(iω)u + n$ を用いて伝達関数モデルを推定します。

`H` と `N` はどちらも `FRD`（周波数応答データ）型です。

  * `σ` は、FFTの前に推定された相関関数に適用されるガウスウィンドウの幅を決定します。大きな `σ` は、スムージングが少ないことを意味します。
  * `H` = Syu/Suu             プロセス伝達関数
  * `N` = Sy - |Syu|²/Suu     推定されたノイズPSD（$H$の分散の推定でもあります）。PSDは、システム同定文献で使用される「ノイズモデル」$N_m$に関連しており、$N_{psd} = N_m^* N_m$となります。ノイズモデルの大きさ曲線は `√(N)` をプロットすることで視覚化できます。
  * `method`: `:welch` または `:corr`。 `:welch` はパワースペクトル密度を推定するためにウェルチ法を使用し、`:corr`（デフォルト）はコレログラムアプローチを使用します。`method = :welch` の場合、追加のキーワード引数 `n`、`noverlap`、および `window` は、セグメントごとのサンプル数（デフォルトはデータの10%）、セグメント間で重複するサンプル数（デフォルトは50%）、および使用するウィンドウ関数（デフォルトは `hamming`）をそれぞれ決定します。

# 拡張ヘルプ

この推定方法は、入力 $u$ がノイズ $n$ と無相関である場合にバイアスがなくなりますが、そうでない場合（例えば、クローズドループでの同定の場合）はバイアスがかかります。
