```
BranchAndPruneSearch(S, process, bisect, initial_region)
```

分岐と剪定の探索。与えられた初期領域に対して、探索順序 `S` を使用します。これは、`bisect` を用いて探索領域を再帰的に二分し、すべての領域が `process` によって満足と見なされるまで続けます。探索が進むにつれて、領域同士の関係を示す二分木を構築します。

`process` と `bisect` はどちらも単一の領域を引数として取ります。

`process` は実行するアクションと、それを実行する領域を返さなければなりません。可能なアクションは次の通りです。

  * `:store`: 領域は最終的なものと見なされ、保存され、さらに処理されることはありません。
  * `:branch`: 要素は二分され、得られた2つの部分は独立して処理されます。
  * `:prune`: 要素は木から削除されます。もしそれが枝の最後の子孫であれば、全体の枝が木から剪定されます。単一の子孫を持つ中間ノードも木から削除されます。

初期領域は、この段階での洗練が不可能な場合、変更せずに返すことができます。

`bisect` は、領域が分岐することがマークされたときに使用されます。元の領域の2つのサブ領域を返さなければならず、同じタイプである必要があります。

`BranchAndPruneSearch` オブジェクトは反復可能であり、反復可能なものを使い果たすと完全な探索が行われます。

警告 デフォルトでは、探索はすべての残りの領域がプロセス関数によって `:store` の指示を受けるまで終了しません。異なる基準に基づいて停止するには、手動で反復ループを中断するか、カスタムコールバック関数を使用して `bpsearch(search ; callback)` を利用してください。
