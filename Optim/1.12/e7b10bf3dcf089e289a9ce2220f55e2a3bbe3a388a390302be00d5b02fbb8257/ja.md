# Adam

## 定数 `alpha` ケース（デフォルト）コンストラクタ：

```julia
    Adam(; alpha=0.0001, beta_mean=0.9, beta_var=0.999, epsilon=1e-8)
```

## スケジュールされた `alpha` ケースコンストラクタ：

`alpha` がすべてのイテレーションに対して固定された定数である上記（デフォルト）の使用法の代替として、以下のコンストラクタは `alpha` が現在のイテレーションカウントを現在の最適化イテレーションの更新ステップに使用される `alpha` の値にマッピングする呼び出し可能なオブジェクト（スケジューラ）である柔軟性を提供します。これにより、次の使用法を使用して、イテレーションにわたって `alpha` を希望通りにスケジュールすることができます。

```julia
    # alpha_scheduler をイテレーション -> alpha 値マッピングの呼び出し可能オブジェクトとする
    Adam(; alpha=alpha_scheduler, other_kwargs...)
```

## 説明

Adamは、勾配ベクトルの最初の2つのモーメントの推定を構築することによって探索方向を選択する勾配ベースの最適化手法です。これにより、確率的な目的関数およびそれに伴う勾配を持つ問題に適しています。この手法は[1]で紹介されており、関連するAdaMax手法も紹介されています。この手法に関する詳細は `?AdaMax` を参照してください。
