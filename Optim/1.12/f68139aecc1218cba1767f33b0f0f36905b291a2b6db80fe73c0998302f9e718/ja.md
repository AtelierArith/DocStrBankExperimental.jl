# パーティクルスウォーム

## コンストラクタ

```julia
ParticleSwarm(; lower = [],
                upper = [],
                n_particles = 0)
```

コンストラクタは3つのキーワードを受け取ります：

  * `lower = []`、下限のベクトル、空であれば下に制約がなく、`-Inf`の場合
  * `upper = []`、上限のベクトル、空であれば上に制約がなく、`Inf`の場合
  * `n_particles = 0`、スウォーム内の粒子の数、デフォルトは少なくとも3つ

## 説明

Optim.jlにおけるパーティクルスウォームの実装は、[1]で言及されているいわゆる適応型パーティクルスウォームアルゴリズムです。これは、探索、利用、収束、ジャンプアウトの4つの進化的状態の間で切り替えることによって、グローバルなカバレッジと収束を改善しようとします。ジャンプアウト状態では、意図的に最良の粒子を取り、その（潜在的におそらく）局所最適から遠ざけることを試み、グローバル最適を見つける能力を向上させます。もちろん、これは収束が遅くなる代償を伴いますが、その結果としてグローバル最適に収束することを期待しています。

収束はパーティクルスウォームに対しては決して評価されないことに注意してください。`Optim.Options(iterations=x)`で設定された最大反復回数に達するまで実行されます。
