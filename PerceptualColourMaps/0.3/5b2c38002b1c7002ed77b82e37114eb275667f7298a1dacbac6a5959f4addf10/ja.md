applycycliccolourmap/applycycliccolormap: 角度データの画像にサイクリックカラーマップを適用します

角度データが正しくレンダリングされるためには、データ値が尊重され、データ値がサイクリックカラーマップの特定のエントリに正しく割り当てられることが重要です。値の色への割り当ては、データがπまたは2πでサイクリックであるかどうかにも依存します。

対照的に、デフォルトの表示方法は通常、データ値を直接尊重せず、表示およびカラーマップでのレンダリングの前に角度データの不適切なオフセットや正規化を行うことがあります。

指定されたカラーマップを使用して角度データをレンダリングすることは、関連する画像の振幅の関数として調整できます。これにより、角度情報のカラーマップエンコーディングを調整して、角度データの振幅/信頼性/コヒーレンスを表現できます。

```
Usage: rgbimg = applycycliccolourmap(ang, cmap)
       rgbimg = applycycliccolourmap(ang, cmap, keyword args ...)

Arguments:
           ang - レンダリングされる角度データの画像
                 ::ImageMeta または ::Array{Float64,2}
          cmap - 角度データをレンダリングするためのサイクリックカラーマップ。

Keyword arguments:

           amp - 角度データのマッピングされた色を調整するために使用される振幅画像。
                 提供されない場合、色の調整は行われません。
                 ::ImageMeta または ::Array{Float64,2}
   modtoblack  - 振幅画像がカラーマップされた画像値を黒に向けて調整するか、
                 白に向けて調整するかを示すブールフラグ/1。デフォルトは黒に向けて調整。
   cyclelength - 角度データのサイクル長。データが方向を表す場合はπの値を、
                 データが位相値を表す場合は2πの値を使用します。入力データが度数の場合は、
                 サイクルを度数に設定するだけで、データが適切にレンダリングされます。デフォルトは2π。

Returns: rgbim - レンダリングされた画像。
                 ::ImageMeta{Float64,3} または ::Array{Float64,3}
```

生成可能なすべてのサイクリックカラーマップのリストを取得するには、cmap()を使用します:

```
> cmap("cyclic")
```

関連情報: cmap, scalogram, ridgeorient, applycolourmap, applydivergingcolourmap
