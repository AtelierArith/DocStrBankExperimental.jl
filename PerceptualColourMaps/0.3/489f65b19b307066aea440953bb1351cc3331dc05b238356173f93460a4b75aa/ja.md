sineramp  - ランプカラーマップテスト画像上にサイン波を生成します

テスト画像は、ランプ関数の上に重ねられたサイン波で構成されています。サイン波の振幅は、画像の上部での全値から下部での0に変調されます。

この画像は、異なるカラーマップの効果を評価するのに役立ちます。理想的には、サイン波パターンはカラーマップの全範囲で同様に識別可能であるべきです。さらに、画像の下部には、基になる信号が滑らかなランプであるため、特定の特徴が見えないはずです。実際には、多くのカラーマップはその範囲にわたって不均一な知覚コントラストを持ち、しばしば重要な特徴を隠すことができる「フラットスポット」を含んでいます。

```
Usage: img = sineramp(sze, amp, wavelen, p)
       img = sineramp()

Arguments:     sze - (rows, cols) テスト画像のサイズを指定します。
                     デフォルトは (256 512)  列の数は
                     名目上のもので、画像全体にサイン波のサイクル数が
                     整数になるように調整されます。
               amp - サイン波の振幅。デフォルトは 12.5
           wavelen - サイン波の波長（ピクセル単位）。デフォルトは 8。
                 p - 振幅の線形減衰のべき乗、
                     上から下への。減衰がない場合は
                     p = 0 を使用します。線形減衰の場合は
                     値 1 を使用します。コントラスト感度実験には
                     より大きな値の p を使用します。デフォルト値は 2 です。
```

サイン波が重ねられているランプ関数は、各行ごとにわずかに調整されており、画像の各行が0から255のデータ範囲全体をカバーします。したがって、大きなサイン波振幅を使用すると、テスト画像の上部のランプが画像の下部のランプの傾きに対して相対的に減少します。

まずは次のコマンドを試してください。

```
  > img = sineramp()
```

これは次のコマンドと同等です。

```
  > img = sineramp((256 512), 12.5, 8, 2)
```

'gray'で表示し、その後 'jet', 'hsv', 'hot' などのカラーマップを試してください。結果に不安を感じるかもしれません！

サイクリックカラーマップ、例えば hsv を評価したい場合は、circlesineramp() によって生成されたテスト画像を使用することをお勧めします。

デフォルトの波長と振幅に関する詳細はソースコードのコメントを参照してください。

関連: circlesineramp, chirplin, chirpexp, equalisecolourmap, cmap
