circlesineramp - サイクリックカラーマップを評価するためのテスト画像を生成します

```
Usage: (img, alpha) = circlesineramp(sze, amp, wavelen, p, hole)
       (img, alpha) = circlesineramp()

Arguments:     sze - テスト画像のサイズ。デフォルトは512x512。
               amp - サイン波の振幅。デフォルトはpi/10。
           wavelen - 円形テスト画像の半径でのサイン波の波長。デフォルトは8ピクセル。
                 p - 振幅の線形減衰を外側のエッジから中心に向かってどの程度上げるかを示す指数。減衰がない場合はp = 0を使用。線形減衰の場合は1の値を使用。デフォルト値は2、二次減衰。
              hole - テスト画像の中心に「穴」を持つべきかどうかを示すブールフラグ。デフォルトはtrueで、穴を持つことにより中心の方向の特異点の気を散らす要素を取り除きます。
Returns:
                im - テスト画像。
             alpha - 円形テスト画像の外側の領域に対応するアルファマスク。これらの領域を透明にした画像を書き込む場合に使用します。
```

テスト画像は、サイン波がスパイラルランプ関数に重ね合わされた円形パターンです。スパイラルランプは、右を指す値0から始まり、反時計回りに2*piの値に増加し、完全な円を完成させます。これにより、画像の右側に2*piの不連続性が生じます。重ね合わされたサイン波の振幅は、円形パターンの外側での全値から中心で0に変調されます。デフォルトのサイン波振幅pi/10は、ピークから谷までのサイン波の全体的なサイズが2*(pi/10)/(2*pi) = 10%のスパイラルランプの合計2*piを表すことを意味します。カラーマップをpiのサイクルでテストしている場合は、振幅をamp = pi/20に設定して、サイン波と円形ランプの同等の比率を得るべきです。

この画像は、サイクリックカラーマップの効果を評価するために設計されています。これはsineramp()のサイクリックな仲間です。理想的には、サイン波パターンはテスト画像の周囲のすべての角度で同じように識別可能であるべきです。実際には、多くのカラーマップはその範囲で不均一な知覚コントラストを持ち、しばしば重要な特徴を隠すことができる「フラットスポット」を含んでいます。HSVカラーマップを試してみてください。

理想的には、テスト画像はshowangularim()を使用してサイクリックカラーマップでレンダリングされるべきですが、この場合、SHOWまたはIMAGESCで画像をレンダリングすることも問題ありません。なぜなら、すべての画像値は0-2*piの範囲内にあり、完全な範囲を使用するからです。しかし、一般的に、デフォルトの表示方法は通常、データ値を直接尊重せず、表示前に角度データの不適切なオフセットや正規化を行うことがあります。

角度データが正しくレンダリングされるためには、データ値が尊重され、データ値がサイクリックカラーマップの特定のエントリに正しく割り当てられることが重要です。値を色に割り当てることは、データがpiまたは2*piでサイクリックであるかどうかにも依存します。SHOWANGULARIMはこれをサポートしています。
