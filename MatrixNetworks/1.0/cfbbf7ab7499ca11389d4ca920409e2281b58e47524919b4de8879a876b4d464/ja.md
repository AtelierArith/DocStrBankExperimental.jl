# `preferential_attachment_graph`

優先接続グラフのインスタンスを生成します。これは次のように生成される無向グラフです：

  * k0ノードのクリークから始めます。
  * n-k0の頂点を追加し、各頂点はその次数に基づいて選ばれたkノードにリンクします（重複は許可されます）。

## 関数

以下の関数は同義語です

  * `preferential_attachment_graph`
  * `pa_graph`

および

  * `preferential_attachment_edges!`
  * `pa_edges!`

計算関数は次のとおりです

  * `pa_graph(n,k,k0)` k0ノードのクリークとn個の総ノード、各ノードに追加されるkエッジを持つPAグラフを生成します。これはMatrixNetwork型を返します。

エッジ関数は次のとおりです

  * `pa_edges!(nnew,k,edges)` 既存のエッジセットに新しいエッジを追加します。ここで、各ノードは次数に基づいてkエッジを選択する新しいnnewノードをエッジセットに追加します。新しいノードIDはエッジ配列の最大エントリに基づいています。
  * `pa_edges!(nnew,k,edges,n0)` エッジセットにノードを追加し、n0+1が新しいノードセットの開始インデックスとなるエッジのセットを生成します。

## 入力

  * `n`: 最終グラフのノード数
  * `k`: 各ノードが追加されるときに選択するリンクの数。この数は、他のノードからのリンクや選択された重複によって、実際の次数がこの数より大きくなることも小さくなることもあります。
  * `k0`: 開始クリークのノード数。
  * `edges`: 新しいエッジを生成する過程で操作されるエッジのリスト。

## 出力

  * 優先接続グラフのためのマトリックスネットワーク型。
  * `edges` 更新されたエッジのリスト。

## 例

```
pa_graph(100,5,2)
```
