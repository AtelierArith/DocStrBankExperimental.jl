```
cubic_shell_embedding(s, γ, τ, B, k, bc=PeriodicBoundary()) → embedding
```

[`SpatioTemporalEmbedding`](@ref) インスタンスを作成し、ハイパーキュービック *シェル* 内の空間的隣接点を含めます。この埋め込みは `s` からのデータと共に使用されます。

## 説明

点は、現在の点の周りにハイパーキュービックシェルを形成することによって埋め込みに参加します。形成されるシェルの合計は `B` です。シェル上の点は、空間的距離 `k ≥ 1` を持ちます（インデックスの距離、都市ブロックメトリックのように）。`k = 1` は、シェルのすべての点が参加することを意味します。ハイパーキュービックグリッドの点は、`k ≥ 1` 点離れていることができます（つまり、点の間に `k-1` を落とすことができます）。要するに、システムの各空間次元において、カーテシアンオフセットインデックスは `-B*k : k : k*B` です。

`γ` は、埋め込みに含める過去の時間ステップの数であり、過去の各ステップには追加の遅延時間 `τ::Int` があります。`γ=0` は、現在のみを使用することに対応します。注意すべきは、**すべての** 埋め込まれた時間フレームが同じ空間構造を持っていることで、これは [`light_cone_embedding`](@ref) とは対照的です。

例として、2つの空間次元を持つシステムの `γ` 埋め込まれたフレームの1つ（すべて同じ）を考えてみましょう（`□` = 現在の点、埋め込みに *定義上* 含まれる、`n` = `n`-th シェルからの埋め込みに含まれる点、`.` = 埋め込みに含まれない点）
