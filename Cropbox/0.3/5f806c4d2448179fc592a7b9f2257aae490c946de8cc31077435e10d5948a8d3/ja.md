```
@config c.. -> Config | Vector{Config}
```

構成のセットまたは複数のセットを構築します。

システム `S` の基本的な構成単位は、`S => pv` の形式のペアで表されます。システム名 `S` はシンボルで表現されます。実際のシステムの型が使用される場合、その名前は自動的にシンボルに変換されます。

パラメータ名と対応する値は、`p => v` の形式の別のペアで表されます。複数のパラメータを指定する場合、ペアのタプル `(p1 => v1, p2 => v2)` や名前付きタプル `(p1 = v1, p2 = v2)` を使用できます。パラメータ名はシンボルでなければならず、`preserve` 状態変数でよく使用される `parameter` タグで宣言された変数を示す必要があります。例えば、`:S => (:a => 1, :b => 2)` は、同じスコープ内で `S => (a = 1, b = 2)` と同じ意味を持ちます。

複数のシステムの構成は、タプルで連結できます。カンマで区切られた `c` の複数の要素は、暗黙的にタプルを形成します。例えば、`:S => (:a => 1, :b => 2), :T => :x => 1` は、いくつかのパラメータを持つ二つのシステム `S` と `T` の構成のセットを表します。同じシステム名や変数名が連結中に再び現れると、タプル内に現れた順序で後のものによって上書きされます。例えば、`:S => :a => 1, :S => :a => 2` は `:S => :a => 2` になります。カンマの代わりに、`+` 演算子を使用して、`(:S => :a => 1) + (:S => :a => 2)` のように同様の方法で使用できます。演算子の優先順位のために置かれた括弧に注意してください。

複数の構成セットが必要な場合、[`simulate`](@ref) の `configs` のように、`Config` のベクトルが使用されます。このマクロは、より単純な構成を組み合わせることによってベクトルを構築する便利な方法をサポートします。接頭辞演算子 `!` は、構成値に置かれた任意の反復可能なものの *展開* を許可します。中置演算子 `*` は、構成のベクトルを別のベクトルまたは単一の構成と *乗算* して、複数の構成セットを構築することを許可します。例えば、`!(:S => :a => 1:2)` は、二つの別々の構成セット `[:S => :a => 1, :S => :a => 2]` に展開されます。`(:S => :a => 1:2) * (:S => :b => 0)` は `[:S => (a = 1, b = 0), :S => (a = 2, b = 0)]` に乗算されます。

# 例

```julia-repl
julia> @config :S => (:a => 1, :b => 2)
1つのシステムの構成:
  S
    a = 1
    b = 2
```

```julia-repl
julia> @config :S => :a => 1, :S => :a => 2
1つのシステムの構成:
  S
    a = 2
```

```julia-repl
julia> @config !(:S => :a => 1:2)
2要素のベクトル{Config}:
 <Config>
 <Config>
```

```julia-repl
julia> @config (:S => :a => 1:2) * (:S => :b => 0)
2要素のベクトル{Config}:
 <Config>
 <Config>
```
