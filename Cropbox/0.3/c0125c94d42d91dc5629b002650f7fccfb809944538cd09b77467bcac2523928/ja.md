```
@system name[{patches..}][(mixins..)] [<: type] [decl] -> Type{<:System}
```

新しいシステム `name` を宣言し、`decl` ブロックで新しい変数をカスタム構文を使用して宣言します。結果として得られるシステムは `System` またはカスタム `type` のサブタイプです。`mixins` は、既存のシステムの仕様を再利用して新しいシステムの宣言に貼り付けることを可能にします。`patches` は、高度な使用に必要な型の置換や定数の定義を提供することがあります。

# 変数

```
name[(args..; kwargs..)][: alias] [=> expr] [~ [state][::type|<:type][(tags..)]]
```

  * `name`: 変数名; 通常は短い略称です。
  * `args`: 自動的にバインドされる変数
  * `kwargs`: カスタムバインドされる変数; `call` と `integrate` で使用されます。
  * `alias`: 代替名; 長い説明。
  * `expr`: 状態特有のコードスニペット; 複数のステートメントには begin-end ブロックを使用します。
  * `type`: 内部データ型; 多くの変数のデフォルトは Float64 ですが、すべてではありません。
  * `tags`: 状態特有のオプション; `unit`, `min`/`max` など。

# 状態

  * `hold`: mixins 間で共有される変数のプレースホルダーをマークします。
  * `wrap`: 値をアンラップせずに他の関数に状態変数をそのまま渡します。
  * `advance`: 時間管理変数を管理します; `Clock` からの `time` と `tick`。
  * `preserve`: さらなる更新なしに初期に割り当てられた値を保持します; 定数、パラメータ。
  * `tabulate`: 名前付きキーを持つ二次元テーブルを作成します; *i.e.* パーティショニングテーブル。
  * `interpolate`: 離散値で補間された曲線関数を作成します; *i.e.* 土壌特性曲線。
  * `track`: 各更新のために変数式をそのまま評価します。
  * `remember`: 特定の条件が満たされるまで変数を追跡し続けます; 本質的には `track` が `preserve` に変わります。
  * `provide`: DataFrame における時系列のテーブルを管理します。
  * `drive`: 時系列から現在の値を取得します; `provide` によって供給されるかもしれません。
  * `call`: 一部の変数にバインドされた部分関数を定義します。
  * `integrate`: ガウス法を使用して積分を計算します; 時間領域には使用しません。
  * `accumulate`: 時間にわたる率変数の積分をエミュレートします; 本質的にはオイラー法です。
  * `capture`: 各時間ステップの積分間の差を計算します。
  * `flag`: ブールフラグを設定します; 本質的には `track::Bool` です。
  * `produce`: 動的に構築されたシステムの新しいインスタンスを添付します; *i.e.* 根構造の成長。
  * `bisect`: 二分法を使用して非線形方程式を解決します; *i.e.* ガス交換モデルの結合。
  * `solve`: 多項式方程式を象徴的に解決します; *i.e.* 光合成モデルにおける二次方程式。

# 例

```julia-repl
julia> @system S(Controller) begin
           a => 1 ~ preserve(parameter)
           b(a) ~ accumulate
       end
S
```
