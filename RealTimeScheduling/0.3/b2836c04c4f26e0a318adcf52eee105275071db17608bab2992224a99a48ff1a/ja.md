```
schedule_global(release!, T, m, endtime; kill=false, pass_schedule=false)
```

タスクシステム `T` の先取りグローバルスケジュールを `m` プロセッサで指定された `endtime` までシミュレートします。ジョブをリリースする際、関数 `release!(job)` が呼び出され、任意の変更を行うことができ、さまざまなグローバルスケジューラを実装することが可能になります。三つの例が提供されています: [`schedule_gfp`](@ref), [`schedule_gedf`](@ref), および [`schedule_gfl`](@ref)。

`release!(job)` に提供される `job` は、可能な限り早くリリースされることがデフォルトです（すなわち、時間 0 またはタスクの最後のジョブの後の1周期で）、相対的な締切とコストはタスクによって指定されます。優先度はデフォルトで `T` 内のタスクのインデックスに設定され、優先度値が低いほどスケジューラによって高い優先度として扱われます。

`kill` が `true` の場合、ジョブは締切時にまだ完了していなければ終了されます。

`pass_schedule` が `true` の場合、`release!` 関数にはジョブのリリースまでのスケジュールを含む第二引数が渡されます。
