```
@gen(X)(m, s, a)
@gen(X)(m, s, a, rng::AbstractRNG)
```

生成モデルを呼び出して (PO)MDP `m` のために; 動的意思決定ネットワークのいくつかのノードから値をサンプリングします。 Xは出力するノードを示す1つ以上のシンボルです。

ソルバーとシミュレーターは、`gen`関数ではなく、こちらを呼び出すべきです。問題作成者は、`@gen`を変更するのではなく、`transition`または`gen`関数のメソッドを実装するべきです。

# 引数

  * `m`: `MDP`または`POMDP`モデル
  * `s`: 現在の状態
  * `a`: アクション
  * `rng`（オプション）: ランダム数生成器（通常は`MersenneTwister`）

# 戻り値

`X`がシンボルの場合、対応するノードからサンプルされた値を返します。`X`が複数のシンボルの場合、指定されたノードからサンプルされた値の`Tuple`を返します。

# 例

`m`を`MDP`または`POMDP`、`s`を`m`の状態、`a`を`m`のアクション、`rng`を`AbstractRNG`とします。

  * `@gen(:sp, :r)(m, s, a)`は次の状態と報酬を含む`Tuple`を返します。
  * `@gen(:sp, :o, :r)(m, s, a, rng)`は次の状態、観測、および報酬を含む`Tuple`を返します。
  * `@gen(:sp)(m, s, a, rng)`は次の状態を返します。
