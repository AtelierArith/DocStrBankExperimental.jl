```
create_simulation(model::Model, [params = nothing, globals = nothing; name = model.name, filename = name, overwrite_file = true, logging = false, debug = false])
```

新しいシミュレーションオブジェクトを作成して返します。このオブジェクトはシミュレーションの完全な状態を保存します。

`model` は [`create_model`](@ref) によって作成された `Model` インスタンスです。

`params` はシミュレーションのすべてのパラメータを含む構造体（または `nothing`）でなければなりません。パラメータの値はシミュレーションの実行中は一定であり、[`param`](@ref) 関数を介して取得できます。

`globals` は可変構造体（または `nothing`）でなければなりません。これらのフィールドの値は、すべてのエージェントが [`get_global`](@ref) 関数を介してアクセスできます。値は [`set_global!`](@ref) または [`push_global!`](@ref) を呼び出すことで変更できます。

オプションのキーワード引数 `name` はシミュレーションに関するメタ情報として使用され、動的には影響を与えません。なぜなら、`name` は遷移関数内ではアクセスできないからです。`name` が指定されていない場合、モデルの名前が代わりに使用されます。

オプションの `filename` キーワードは、シミュレーションまたはその一部が e.g. [`write_snapshot`](@ref) を介して書き込まれるときに使用される文字列です。

ファイルは `write...` 関数が初めて呼び出されたときに作成され、`h5` サブフォルダ内に作成されます。デフォルトでは、同じ `filename` を持つ既存のファイルは上書きされますが、この動作は `overwrite_file` キーワードで無効にできます。この場合、ファイルには番号が付けられます。`filename` が提供されていない場合、`name` 引数が代わりに使用されます。

キーワード `logging` は自動ログファイルを有効にするブールフラグです。ログファイルには、さまざまな関数で費やされた時間などの情報が含まれます。また、`debug` が true に設定されている場合、ログファイルにはより詳細な情報が含まれ、各書き込み後にストリームがフラッシュされます。

hdf5 ファイルと同様に、overwrite_file キーワードはログファイルが上書きされるか番号付けされるかを決定します。番号付けは hdf5 ファイルの番号付けとは独立して設定されます。

シミュレーションは初期化されていない状態で開始されます。初期状態のためにエージェントとエッジを追加した後、最初に遷移関数を適用する前に [`finish_init!`](@ref) を呼び出す必要があります。

[`create_model`](@ref)、[`param`](@ref)、[`get_global`](@ref)、[`set_global!`](@ref)、[`push_global!`](@ref)、および [`finish_init!`](@ref) も参照してください。
