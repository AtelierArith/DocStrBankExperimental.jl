```julia
(1) function testCV(cv:CVresult, 
			expected::Union{Float64, Symbol} = :randomchance)

(2) function testCV(𝐂::Vector{Vector{Int}}; 
			expected::Union{Float64, Symbol} = :randomchance)

```

(1) [`CVres`](@ref) 構造体（`CVresult` 型）を与えられた場合、[`crval`](@ref) メソッドを呼び出して得られたもので、交差検証で得られた平均二値誤差損失の分布の差に対してベイリスのテスト（Bayles et *al.*, 2020[🎓](@ref)）を実行します。これは、交差検証で得られたパフォーマンスが指定されたチャンスレベルを上回っているかどうかをテストするために使用できます。

(2) メソッド (1) と同様ですが、最初の引数として `f` 混同行列のベクトルを与えます。混同行列の形式（`Vector{Vector{Int}}`）は、[`CVres`](@ref) 構造体に混同行列を格納するために使用されるものと同じです。

テストは常に方向性があります。$μ$ を平均損失、$E$ を期待される平均とすると、帰無仮説 $H_0: μ1 = E$ に対して代替仮説 $μ < E$ のもとでテストが行われます。テストが有意であれば、誤差損失は指定された期待レベルよりも統計的に劣っていることを意味し、観測された精度は期待される精度よりも統計的に**優れている**ことを意味します。

デフォルトでは、`expected` は $1-\frac{1}{z}$ に設定され、ここで $z$ はクラスの数です。別の値を渡すこともできますが、それは実数でなければなりません。

$$
z
$$

テスト統計量（標準偏差）、$p$ 値、漸近標準誤差を保持する 3 つのタプル (*z, p, ase*) を返します。

!!! warning "クラスの不均衡"
    誤差損失は精度を反映し、バランスの取れた精度ではありません。クラスが不均衡な場合、テストは多数派クラスによって駆動されます。[`crval`](@ref) に関するドキュメントも参照してください。


**例**:

```julia
using PosDefManifoldML

# ランダムデータのセットを生成
P, _dummyP, y, _dummyy = gen2ClassData(10, 60, 80, 30, 40, 0.1)

# 交差検証を実行
cv = crval(MDM(Fisher), P, y; scoring=:a)

# `z` と `p` の値は自動的に計算され、作成された構造体に保存されるため、印刷されます。
# 自分で計算するには:

z, p, ase = testCV(cv)

```
