```
@fread f [ rec=N ] [ spec ... ]
```

`FortranFile` `f` からデータを読み取るためのマクロインターフェースです。単一の `@fread` 呼び出しは、ファイルから単一のデータレコードを処理します。各 `spec` は次のいずれかです：

  * `var::T` 宣言。これは、ファイルから型 `T` のデータを読み取り、変数 `var` に割り当てます。`T` は通常の Fortran スカラーデータ型のいずれかです。
  * `var::(T,dims...)` 宣言。ここで `T` はスカラー型で、`dims...` は整数の系列です。これは、指定されたデータ型と次元の配列を読み取り、変数 `var` に割り当てます。
  * 配列を読み取るための明示的な形式として `var::Array{T}(undef,dims...)`
  * 事前に割り当てられた配列を参照する必要がある変数名。

`spec` は、以前の `spec` によって割り当てられた変数を参照することができます。`rec` キーワードは、`f` が直接アクセスファイルを参照する場合にのみ指定する必要があり、どのレコードを読み取るかを指定します。

例：

```
@fread f n::Int32 x::(Float64,n)
```

これは単一の `Int32` を読み取り、それを `n` に割り当て、その後 `n` 要素を持つ `Float64` 配列を読み取り、それを `x` に割り当てます。このようなレコードは、関数ベースの `read` インターフェースでは処理できません。同等の呼び出しは次のようになります。

```
n, x = read(f, Int32, (Float64,n))
```

しかし、これは `n` が `read` ステートメントが処理された後にのみ割り当てられるため、機能しません。このマクロインターフェースは、そのようなケースをカバーするために提供されています。 ```
