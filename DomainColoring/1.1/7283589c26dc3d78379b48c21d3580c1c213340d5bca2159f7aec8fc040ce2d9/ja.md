```
domaincolor(
    f :: "Complex -> Complex",
    limits = (-1, 1, -1, 1);
    pixels = (720, 720),
    abs = false,
    grid = false,
    color = true,
    all = false,
    box = nothing,
    kwargs...
)
```

複素関数を取り、そのドメインカラーリングプロットを生成します。

赤は位相 $0$ に対応し、黄色は $\frac{\pi}{3}$、緑は $\frac{2\pi}{3}$、シアンは $\pi$、青は $\frac{4\pi}{3}$、マゼンタは $\frac{5\pi}{3}$ に対応します。

# 引数

  * **`f`** はプロットする複素関数です。
  * **`limits`** はプロットする長方形の限界で、形式は `(minRe, maxRe, minIm, maxIm)` です。1つまたは2つの数値が提供された場合、それらは実軸と虚軸に沿って対称的に取られます。

# キーワード引数

  * **`pixels`** は、それぞれ実軸と虚軸で計算するピクセルの数で、1つの数値が提供された場合は両方に同じ数が取られます。
  * **`abs`** は、レベル曲線間の明るさの変化として大きさの自然対数のプロットを切り替えます。数値が設定されている場合、これが対数の底として使用され、`Inf` に設定されている場合、ゼロの大きさは黒で、極は白で色付けされます。さらに詳細な制御は、`base`、`transform`、または `sigma` のいずれかのパラメータを持つ名前付きタプルを渡すことで達成できます。`base` は対数の底を変更します。`transform` は大きさに適用される関数です（デフォルトでは `m -> log(base, m)`）、`sigma` はゼロと極が色付けされる速度を変更し、`base = Inf` を意味します。
  * **`grid`** は、整数の実部または虚部を持つ点を黒い点としてプロットします。より複雑な引数は、[`checkerplot`](@ref) と同様の方法で名前付きタプルとして渡すことができます。
  * **`color`** は位相角の色付けを切り替えます。名前または `ColorScheme`、または関数 `θ -> Color` に設定することもできます。`:print` に設定すると、デフォルトの色の脱飾バージョンが使用されます。
  * **`all`** は `abs = true`、`grid = true`、および `color = true` のショートカットです。
  * **`box`** が `(a, b, s)` に設定されている場合、出力がボックス `a` と `b` の範囲内にある領域を色 `s` でシェードします。`(f, s)` に設定されている場合、色付けされたドメインは `f(w) == true` によって定義されます。複数のボックスのリストでも可能です。

残りのキーワード引数はプロットバックエンドに渡されます。
