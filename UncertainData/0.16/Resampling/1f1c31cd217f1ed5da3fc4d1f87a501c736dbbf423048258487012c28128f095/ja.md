```
ConstrainedValueResampling{N_DATASETS}
```

制約付きの分布/母集団に対してリサンプリングを行う必要があることを示します。

## フィールド

  * `constraints`. データセットの制約。制約は長さ `N_DATASETS` のタプルとして表され、`i` 番目のタプル要素にはそのデータセットの制約が含まれます。各データセットの制約は、単一のサンプリング制約として、またはデータセットの長さに一致する長さのサンプリング制約のベクトルとして提供する必要があります (`Union{SamplingConstraint, Vector{<:SamplingConstraint}}}`)。例えば、`i` 番目のデータセットに352の観測値が含まれている場合、`constraints[i]` は単一のサンプリング制約（例: `TruncateStd(1.1)`）または352の異なるサンプリング制約のベクトル（例: `[TruncateStd(1.0 + rand()) for i = 1:352]`）でなければなりません。
  * `n::Int`. ドローの数。

## 例

不確実な値のコレクションが3つあり、それぞれの長さは `L = 50` であると仮定します。これらは `250` 回リサンプリングされるべきです。ただし、リサンプリングの前に、不確実な値を提供する分布/母集団は切り捨てられるべきです：

  * 最初のコレクションでは、各値をその平均周りの標準偏差の `1.5` 倍で切り捨てます。これは、安定した測定値を提供する機器からの測定誤差をシミュレートすることができ、誤差が正規分布しているが、分析のために外れ値や `1.5` 標準偏差を超える値には興味がない場合です。
  * 2番目のコレクションでは、各値を `80` パーセンタイル範囲で切り捨てます。これは、安定した測定値を提供する機器からの測定誤差をシミュレートすることができ、誤差が正規分布していないため、標準偏差よりも信頼区間を使用する方が良い場合です。この場合、外れ値には興味がなく、したがってデータの `10` パーセンタイルより小さい値と `90` パーセンタイルより大きい値を除外します。
  * 3番目のコレクションでは、`i` 番目の値をその平均周りの標準偏差の少し大きな分数で切り捨て、`i-1` 番目の値よりも大きくします。これにより、標準偏差は `0.5` から `0.5 + L/100` までの範囲になります。これは、例えば、時間とともに測定誤差が増加する機器をシミュレートすることができます。

```julia
L = 50
constraints_d1 = TruncateStd(1.5)
constraints_d2 = TruncateQuantiles(0.1, 0.9)
constraints_d3 = [TruncateStd(0.5 + i/100) for i = 1:L]
```
