一意のホモモルフィズムを、さまざまな制約に従った2つの属性付き$C$-セットの間に見つけます（存在する場合）。

ホモモルフィズムが存在しない場合は`nothing`を返します。多くのカテゴリ$C$において、$C$-セットホモモルフィズム問題はNP完全であり、この手続きは一般的に指数時間で実行されます。ドメインオブジェクトが小さい場合に最も効果的に機能します。

*モノモルフィズム*、すなわちすべての成分が単射関数であるホモモルフィズムに制限するには、キーワード引数`monic=true`を設定します。特定の成分のみを単射または全単射に制限するには、`monic=[...]`または`iso=[...]`を使用します。たとえば、グラフホモモルフィズムに対して`monic=[:V]`を設定すると、頂点マップが単射であることが保証されますが、辺マップには制約が課されません。

ホモモルフィズムを特定の部分割り当てに制限するには、キーワード引数`initial`を設定します。たとえば、グラフホモモルフィズムにおいて最初のソース頂点を3番目のターゲット頂点に固定するには、`initial=(V=Dict(1 => 3),)`を設定します。キーワード引数`type_components`を使用して、緩やかなホモモルフィズムの属性タイプに対する非自明な成分を指定します。これらの成分は呼び出し可能でなければなりません：適切なタイプのJulia関数またはFinFunction(Dict(...))です。

キーワード引数`alg`を使用して、ホモモルフィズム探索アルゴリズムを設定します。デフォルトでは、バックトラッキング探索アルゴリズムが使用されます（[`BacktrackingSearch`](@ref)）。キーワード引数`error_failures = true`を使用すると、指定された初期データの即時の不整合を説明するエラーが得られます。

キーワード`predicates`は、Dict{Ob, Dict{Int, Union{Nothing, AbstractVector{Int}}}}を受け入れます。ドメインの各部分に対して、現在の割り当てと仮の値を割り当てるためのブール関数として制約を与えるオプションがあります。たとえば、`predicates = (E = Dict(2 => [2,4,6]))`は、エッジ#2をコドメインのエッジ#2、#4、または#6に割り当てるマッチのみを見つけます。

キーワード`no_bind`は、すべてのAttrTypesに適用されるブール値であるか、特定の成分のイテラブルである可能性があります：これは、ドメイン内の属性変数がコドメイン内の具体的な値に送られるのを防ぎます。AttrType成分が`monic`の場合、属性変数は具体的な値に送られることができないため（したがって、そのような場合に`no_bind=true`を設定することは冗長です）、両方のケースで、"自由浮遊"属性変数（ドメイン内の任意のAttrによって参照されない）に対してホモモルフィズムを計算することが可能です。各そのような変数には、マッピングされる有限の可能性があります。

`any=true`を設定すると、返されるホモモルフィズムが一意であるという制約が緩和されます。

詳細については、[`homomorphisms`](@ref)、[`isomorphism`](@ref)を参照してください。
