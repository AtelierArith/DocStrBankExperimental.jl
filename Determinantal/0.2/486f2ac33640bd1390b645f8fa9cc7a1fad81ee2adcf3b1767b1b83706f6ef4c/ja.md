```
greedy_subset(L :: AbstractLEnsemble,k)
```

貪欲な部分集合選択を実行します：$\det{L}_{\mathcal{X}}$が高くなるように、サイズ$k$の部分集合$\mathcal{X}$を見つけます。これは、アイテムを1つずつ追加し、各ステップで行列式を最大化するアイテムを選択することで行います。これは、L-アンサンブルLを持つDPPの（近似的な）モードを見つけることと見なすことができます。

$$
k
$$

がLの（数値的な）ランクに対して大きすぎる場合、問題は明確ではなく、アルゴリズムは早期に停止します。

実装は$\O(nk^2)$で実行されますが、特に最適化されているわけではありません。最終結果が不安定に見える場合、数値的な不安定性が原因である可能性があります：$\bL$の条件を改善してみてください。

# 例

```@example
x = randn(2,100) #次元2の10ポイント
greedy_subset(EllEnsemble(gaussker(x)),12)
#同じことですが、より速い：
greedy_subset(gaussker(x),12)
```
