```
transform(::T, data)
```

型 `T` のための特徴エンジニアリングパイプラインを定義し、これは `data` に対して実行される [`Transform`](@ref) のコレクションやその他のステップで構成されます。

「変換インターフェース」の考え方は、特徴変換を合成可能にすることです。つまり、任意の `Transform` の出力は別の `Transform` の有効な入力であるべきです。

特徴エンジニアリングパイプラインも同じ原則に従うべきであり、パイプラインを構成する `Transform` ステップを追加/削除することが簡単で、壊れることはないはずです。

`transform` は、特徴エンジニアリングを必要とするカスタム型 `T` に対してオーバーロードされるべきです。唯一の要件は、`transform` の戻り値自体が「変換可能」であること、すなわち出力に対して [`is_transformable`](@ref) を呼び出すと true が返されることです。
