*Juliaのドキュメントシステムの拡張。*

# はじめに

このパッケージは、Juliaの組み込みドキュメントシステムのための便利な拡張のコレクションを提供します。これらは「実験的」と見なされている機能であり、`Base`に含めるにはまだ成熟していないか、デフォルトのJuliaインストールに含める価値が十分に見込まれないほどニッチな使用ケースを持っています。

現在、`DocStringExtensions.jl`は、ドキュメント文字列に便利な自動生成情報を追加するために使用できる「略語」と呼ばれるコレクションをエクスポートしています。これには以下のような情報が含まれます：

  * 簡略化されたメソッドシグネチャ；
  * 複合型の個々のフィールドに関するドキュメント；
  * モジュールのインポートおよびエクスポートリスト；
  * 特定のドキュメント文字列に適用可能なメソッドのソースリンク付きリスト。

ユーザーは追加の略語アイデアを提案したり、自分で実装して提出したりすることを大いに歓迎します。Juliaのプログラムイントロスペクションに対する強力なサポートにより、これは比較的簡単なプロセスです。

現在利用可能な略語の詳細は、以下の「エクスポート」セクションにリストされている個々のドキュメント文字列で確認できます。

# 例

簡単に言えば、略語は適切なドキュメント文字列に単純に補間することで使用できます。例えば：

```julia
using DocStringExtensions

"""
`func`の短い要約...

$(SIGNATURES)

ここで、`x`と`y`は両方とも正である必要があります。

# 詳細

`func`に関するいくつかの詳細...
"""
func(x, y) = x + y
```

`$(SIGNATURES)`は、上記のドキュメント文字列内で次のように置き換えられます。

````markdown
# シグネチャ

```julia
func(x, y)
```
````

生成されたコンテンツは、Juliaの`?`モードを介して表示するか、`Documenter.jl`が設定されている場合は生成された外部ドキュメントで確認できます。

[`SIGNATURES`](@ref)（および他の略語）を使用する利点は、ドキュメント文字列が周囲のコードと同期しなくなる可能性が低くなることです。ただし、ドキュメント文字列内に手動で埋め込まれた引数名`x`と`y`への参照は、自動的に更新されないことに注意してください。
