モジュール内の指定されたカテゴリまたは文書化されたバインディングのタプルに一致するすべてのドキュメント文字列に適用されるドキュメント文字列テンプレートを定義します。

実質的に、モジュール内のすべての一致するドキュメント文字列をテンプレートで置き換えます。`DOCSTRING` 略語は、テンプレートから生成された置き換えドキュメント文字列に元のドキュメント文字列を挿入するために使用できます。

# 例

```julia
@template DEFAULT =
    """
    $(SIGNATURES)
    $(DOCSTRING)
    """
```

`DEFAULT` は、他のテンプレート定義が文書化された式に一致しない場合にドキュメント文字列に適用されるデフォルトテンプレートです。`DOCSTRING` 略語は、テンプレート内で実際のドキュメント文字列の本文が各ドキュメント文字列に挿入される位置を示すために使用されます。

```julia
@template (FUNCTIONS, METHODS, MACROS) =
    """
    $(SIGNATURES)
    $(DOCSTRING)
    $(METHODLIST)
    """
```

ドキュメント文字列テンプレートを複数の異なるカテゴリに使用する必要がある場合、カテゴリのタプルを指定できます。

```julia
@template MODULES = ModName
```

上記のテンプレート定義は、モジュール `ModName` に見つかったモジュールのテンプレートに基づいてモジュールドキュメント文字列のテンプレートを定義します。

!!! 注     サポートされているカテゴリは `DEFAULT`、`FUNCTIONS`、`METHODS`、`MACROS`、`TYPES`、`MODULES`、および `CONSTANTS` です。
