```
Motility(motile_states::NTuple{N,MotileState}, rates...)
```

`Motility`を一連の`MotileState`とその遷移`rates`から構築します。

`motile_states`は`MotileState`インスタンスのタプルでなければなりません。`rates`は`(i => j, p)`の形式で指定する必要があります。ここで、`i`は`motile_states`タプル内の開始状態のインデックス、`j`は遷移が発生する状態のインデックス、`p`は`i`の期間が終了した後に遷移が`j`の状態に向かう確率です。不可能な遷移を示す必要はありません。明示的に指定されていない任意のペア`i => j`は遷移確率`p=0`を持つと仮定されます。

例えば、遅いが長い走行、瞬時の転倒（`angle_pdf`を持つ）、速いが短い走行から構成される3状態の運動性を定義したい場合、2つの走行は転倒にのみ遷移でき、転倒は30%の確率で遅い走行に、70%の確率で速い走行に遷移できるとします。この場合、次のように呼び出します：

```
Motility((Run(10.0, 2.0), Tumble(0.0, angle_pdf), Run(60.0, 0.5)),
    (1 => 2, 1.0),
    (2 => 1, 0.3), (2 => 3, 0.7),
    (3 => 2, 1.0)
)
```

一般的な運動性パターンのためのいくつかのデフォルトコンストラクタが提供されています：`RunTumble`、`RunReverse`、`RunReverseFlick`、`RunStop`
