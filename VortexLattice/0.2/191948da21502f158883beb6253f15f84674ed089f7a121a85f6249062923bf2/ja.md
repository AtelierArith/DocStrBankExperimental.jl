```
write_vtk(name, surfaces, [surface_properties]; kwargs...)
write_vtk(name, wakes; kwargs...)
write_vtk(name, surfaces, wakes, [surface_properties]; kwargs...)
```

サーフェスおよび/またはウェイクからParaviewファイルにジオメトリを書き出して視覚化します。

# 引数

  * `name`: 生成されるファイルの基本名
  * `surfaces`:

      * リフティングサーフェスを表す形状(3, nc+1, ns+1)のグリッドのベクトル

    または

      * サーフェスパネルを含む形状(nc, ns)の行列のベクトル（`nc`は弦方向のパネル数、`ns`はスパン方向のパネル数を参照）
  * `wakes`: （オプション）各サーフェスに対応するウェイクのベクトルで、形状(nw, ns)のウェイクパネルの行列（`nw`は弦方向のウェイクパネル数、`ns`はスパン方向のパネル数を参照）で表されます。
  * `surface_properties`: （オプション）各サーフェスのサーフェスパネルプロパティのベクトルで、形状(nc, ns)のパネルプロパティの行列（`nc`は弦方向のパネル数、`ns`はスパン方向のパネル数を参照）として保存されます。

# キーワード引数:

  * `symmetric`: （`surface_properties`が提供されている場合は必須）各サーフェスの誘導速度を計算する際にミラーイメージ（X-Z平面に対して）を使用したかどうかを示すフラグ。
  * `trailing_vortices`: モデルがトレーリングボルテックスを使用するかどうかを示すフラグ。ウェイクパネルがない場合はデフォルトで`true`、それ以外は`false`。
  * `xhat`: 使用される場合のトレーリングボルテックスが延びる方向。デフォルトは[1, 0, 0]。
  * `wake_length`: トレーリングボルテックスを延ばす距離。デフォルトは10。
  * `metadata`: 生成されるファイルに含めるメタデータの辞書。
