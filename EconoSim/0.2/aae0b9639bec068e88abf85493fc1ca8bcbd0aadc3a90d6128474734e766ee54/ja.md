```
復元可能
```

復元性のあるライフサイクルを示します。つまり、エンティティはダメージから回復できるということです。閾値は、閾値以下の健康に対する健康の乗数を決定します。

# フィールド

  * `damage_thresholds`: これらは、パーセンテージで順序付けられたタプルで、ダメージ乗数を保持します。適用される乗数は、エンティティの健康よりも高い最も低い閾値に対応します。
  * `restoration_thresholds`: これらは、パーセンテージで順序付けられたタプルで、復元乗数を保持します。適用される乗数は、エンティティの健康よりも高い最も低い閾値に対応します。
  * `wear`: 各使用から発生するダメージ。乗数の影響を受けます。

# 例

`Restorable     damage_thresholds = [(70.0%, 0.5), (100.0%, 0.2)]     restoration_thresholds = [(40.0%, 0.0), (70.0%, 0.2), (100.0%, 0.3)]` 1のダメージが与えられると、実際に適用されるダメージは0.2になります。健康が70%以下に下がると、0.5のダメージが適用されます。これは、さまざまなダメージレベルでの堅牢性を示しています。

1のダメージが復元されると、実際に復元されるダメージは0.3になります。健康が40%以下に下がると、ダメージは復元されません。これは復元性を示しています。最後のティアは、もはや復元が不可能なダメージのレベルを示しています。
