```
process(marginality::Marginality, units::Int64)
```

現在のユニット数に基づいて、限界ユニット数を決定します。補助ユニットのチェックが真を返す限り、さらに別のユニットのチェックが実行されます。

# 例:

限界性: [(2, 1.0), (5, 0.5)]

タプルはユニット数と、利用可能なユニット数が指定された数未満の場合の追加ユニットの確率を示します。上記の例では、ユニットが利用可能でない場合、最初のユニットの確率は100%です。2番目のユニットのチェックが実行され、これも100%の確率であるため、3番目のユニットのチェックが実行されます。3番目のユニットの確率は50%です（2ユニットが利用可能であるため、次のタプルが使用されます）。このチェックが3番目のユニットをもたらす場合、4番目のユニットのチェックが行われ、その後も続きます。5ユニットが利用可能になると、この例ではチェックは常に偽を返し、したがって最大ユニット数は5になります。

# 戻り値

限界ユニット数。
