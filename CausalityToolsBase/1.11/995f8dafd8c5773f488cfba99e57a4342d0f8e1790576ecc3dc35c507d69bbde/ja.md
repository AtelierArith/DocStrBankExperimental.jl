```
RectangularBinning(ϵ)
```

バイニングスキーム `ϵ` を使用して長方形のボックスパーティションを作成するための指示。

# フィールド

  * **`ϵ`**: 有効なバイニングスキーム。

# 有効なバイニングスキーム

以下の `ϵ` は有効です。

## データ範囲に基づく各軸のデータ範囲

1. `RectangularBinning(ϵ::Int)` は各軸を `ϵ` 等長の区間に分割し、すべての点がカバーされるように上限を1/100のビンサイズだけ拡張します。
2. `RectangularBinning(ϵ::Float64)` は各軸を固定サイズ `ϵ` の区間に分割します。
3. `RectangularBinning(ϵ::Vector{Int})` は i 番目の軸を `ϵᵢ` 等長の区間に分割し、すべての点がカバーされるように上限を1/100のビンサイズだけ拡張します。
4. `RectangularBinning(ϵ::Vector{Float64})` は i 番目の軸をサイズ `ϵ[i]` の区間に分割します。

これらの場合、長方形のパーティションは各座標軸に沿った最小値を特定し、データの最大値がカバーされるまで等長の区間を構築することによって作成されます。

## 各軸に沿ったカスタム範囲

長方形のバイニングは任意の最小-最大範囲でも指定できます。

5. `RectangularBinning(ϵ::Tuple{Vector{Tuple{Float64,Float64}},Int64})` は、(min, max) タプルのベクターによって示された範囲から各軸に沿った区間を作成し、各軸を同じ整数数の等長の区間に分割します。

以下のコンストラクタを使用する方が簡単かもしれません。

  * `RectangularBinning(RectangularBinning(minmaxes::Vararg{<:AbstractRange{T}, N}; n_intervals::Int = 10))` は、各軸に沿った (min, max) を示すタプルのベクターと、これらの範囲を分割する等長の区間の数を示す `n_intervals` を受け取ります。
  * `RectangularBinning(minmaxes::Vector{<:AbstractRange{T}}, n_intervals::Int)` は同様のことを行いますが、引数は範囲として提供されます。

# 例

データポイントによって決定されるグリッドの最小および最大位置：

  * `RectangularBinning(10)`: 各座標軸の点の最小値を見つけ、(拡張された) 範囲を `10` 等長の区間に分割します。
  * `RectangularBinning([10, 5])`: 各座標軸の点の最小値を見つけ、(拡張された) 範囲を最初の座標軸に沿って `10` 等長の区間に分割し、2 番目の座標軸に沿って `5` 等長の区間に分割します。
  * `RectangularBinning(0.5)`: 各座標軸の点の最小値を見つけ、軸の範囲をサイズ `0.5` の等長の区間に分割します。
  * `RectangularBinning([0.3, 0.1])`: 各座標軸の点の最小値を見つけ、最初の座標軸に沿った範囲をサイズ `0.3` の等長の区間に分割し、2 番目の軸に沿った範囲をサイズ `0.1` の等長の区間に分割します。

データ範囲を明示的に指定する（データポイントをカバーすることは保証されない）：

  * `RectangularBinning(-5:5, 2:2, n_intervals = 5)`: 範囲 `-5:5` と `2:2` を `n_intervals` 等長の区間に分割します。
