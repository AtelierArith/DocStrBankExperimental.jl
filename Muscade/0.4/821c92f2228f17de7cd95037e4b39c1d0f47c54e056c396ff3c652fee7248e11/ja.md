```
SweepX{ORDER}
```

非線形の時間領域ソルバーで、問題を時間ステップごとに解決します。モデルの`X`-dofのみが解決され、`U`-dofおよび`A`-dofは変更されません。

  * `SweepX{0}`はニュートン-ラフソン法で、非等式制約を扱うための実行可能性ラインサーチを行います。
  * `SweepX{1}`は暗黙のオイラー法で、実行可能性ラインサーチを行います。
  * `SweepX{2}`はニューマーク-β法で、ニュートン-ラフソン反復と実行可能性ラインサーチを行います。

重要な注意: Muscadeは要素が状態変数を持つことを許可しません。たとえば、塑性ひずみや乾燥摩擦のためのせん断自由位置などです。このような物理を実装する要素では、状態を要素の自由度として導入し、その進化を解決する必要があります。*静的問題*であっても、`ORDER≥1`の使用が必要です。

分析は次の構文で呼び出されます:

```
initialstate    = initialize!(model)
setdof!(initialstate,1.;class=:U,field=:λcsr)
states           = solve(SweepX{2};initialstate=initialstate,time=0:10)
```

# `solve`への名前付き引数:

  * `dbg=(;)`           呼び出しツリーをトレースするための名前付きタプル（デバッグ用）
  * `verbose=true`      出力を抑制するためにfalseに設定（テスト用）
  * `silenterror=false` エラーの出力を抑制するためにtrueに設定（テスト用）
  * `initialstate`      `initialize!`または`SweepX`から取得した`State`。
  * `time`              ニュートン-ラフソン反復の最大数
  * `β=1/4`,`γ=1/2`     ニューマーク-βアルゴリズムのパラメータ。`ORDER<2`の場合はダミー
  * `maxiter=50`        各ステップでの平衡反復の最大数。
  * `maxΔx=1e-5`        収束基準: `X`のノルム。D
  * `maxLλ=∞`           収束基準: 残差のノルム。
  * `saveiter=false`    出力`states`が最後に分析されたステップの反復時の状態を含むようにtrueに設定。収束しないステップを研究するのに便利です。
  * `maxLineIter=50`    実行可能性ラインサーチの最大反復数。ラインサーチをスキップするには0に設定（非推奨、非等式制約のあるモデルには）。
  * `sfac=0.5`          実行可能な点のためのラインサーチのパラメータ。仮の結果が実行可能でない場合、`sfac`の因子でバックトラックします。まだ実行可能でない場合は、残りを`sfac`の因子でバックトラックし、以下同様に`maxLineIter`回まで続けます。
  * `γfac=0.5`          実行可能性のためのパラメータ。非等式制約`g(X)`と反応力`λ`に対して、`g(X)*λ==γ`を要求し、各反復で`γ *= γfac`を掛けます。

# 出力

時間ステップでの状態を含む、名前付き入力引数`time`の長さと等しいベクトル。
