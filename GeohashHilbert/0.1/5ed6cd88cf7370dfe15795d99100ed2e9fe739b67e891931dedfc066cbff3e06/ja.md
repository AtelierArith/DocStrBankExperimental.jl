```
encode(lon, lat, precision, bits_per_char = 2)
```

経度-緯度のペアをジオハッシュ文字列としてエンコードします。結果のジオハッシュの長さは `precision` であり、より高い `precision` はより細かいエンコーディングを提供します。ジオハッシュ文字列を返します。

# 引数

  * `lon::Real`: エンコードする点の経度（度）。[-180,180] の範囲内でなければなりません。
  * `lat::Real`: エンコードする点の緯度（度）。[-90,90] の範囲内でなければなりません。
  * `precision::Integer`: 返されるジオハッシュの精度。正の値でなければなりません。
  * `bits_per_char ∈ [2,4,6]`: 各文字がエンコードする情報のビット数。

!!! note
    機械算術でのオーバーフローを避けるために、`precision * bits_per_char` は <= 62 でなければなりません。これにより、メートルレベルの粒度が可能です。もしあなたの使用ケースがサブメートルの粒度を必要とする場合は、経度-緯度座標の使用を再考することをお勧めします。
