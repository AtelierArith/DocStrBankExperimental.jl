```
Theiler(w::Int, nidxs = nothing)
```

サイズ `w ≥ 0` の Theiler ウィンドウを表すスキップ関数を生成する構造体です。これは、検索のクエリが検索構造を作成するために使用されるデータの一部である場合に便利です。これは、時系列分析で典型的です。この場合、クエリ自体を隣接点として見つけたくないため、クエリに非常に近いインデックスのポイントも避けることが一般的です。`w=0` を指定すると、クエリ自体が隣接点として除外されます。以下のブール式を参照してください。

`theiler = Theiler(w)` とします。すると、`theiler` 自体は [`bulksearch`](@ref) のスキップ関数として使用できます。なぜなら、`theiler(i, j) ≡ abs(i-j) ≤ w` だからです。さらに、引数 `nidxs` が *nothing* でない場合、`theiler(i, j) ≡ abs(i-nidxs[j]) ≤ w` となります。（元のデータ内のクエリのインデックスを `nidxs` として指定するのに便利です）

しかし、`theiler` は単一検索にも使用できます。`theiler(n)`（1つの引数を持つ）を使用すると、関数 `i -> abs(i-n) ≤ w` が生成されます。したがって、`theiler(n)` を [`search`](@ref) の `skip` 引数として渡すことができます。`theiler` は `Theiler` のインスタンスであり、要約すると `Theiler(w)(n)` を実行し、それを [`search`](@ref) に渡す必要があります。
