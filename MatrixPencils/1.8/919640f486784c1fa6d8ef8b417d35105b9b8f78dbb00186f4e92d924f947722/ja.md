```
rmkstruct(N, D; fast = false, atol::Real = 0, rtol::Real = atol>0 ? 0 : n*ϵ) -> (KRInfo, iz, nfp, ip)
```

有理行列 `R(λ) := N(λ)./D(λ)` のクロンカー構造と無限極限ゼロ構造情報を決定し、`KRInfo` オブジェクト、無限ゼロの重複度を `iz` に、有限極限の数を `nfp` に、無限極限の重複度を `ip` に返します。

分子 `N(λ)` は次の形式の多項式行列です：`N(λ) = N_1 + λ N_2 + ... + λ**k N_(k+1)`。係数行列 `N_i`（`i = 1, ..., k+1`）は3次元行列 `N` に格納されており、`N[:,:,i]` には `N_i`（`λ**(i-1)` を掛けたもの）が含まれています。

分母 `D(λ)` は次の形式の多項式行列です：`D(λ) = D_1 + λ D_2 + ... + λ**l D_(l+1)`。係数行列 `D_i`（`i = 1, ..., l+1`）は3次元行列 `D` に格納されており、`D[:,:,i]` には `D_i`（`λ**(i-1)` を掛けたもの）が含まれています。

また、`N(λ)` と `D(λ)` は、[Polynomials](https://github.com/JuliaMath/Polynomials.jl) パッケージによって提供される `Polynomial` 型の要素の行列として指定することもできます。この場合、`N(λ)` と `D(λ)` が同じ不確定要素を持つかどうかのチェックは行われません。

有理行列 `R(λ)` のクロンカー構造に関する情報は、右クロンカーインデックス `rki`、左クロンカーインデックス `lki`、有限ゼロの数 `nf`、および通常のランク `nrank` から構成され、`KRInfo` から `KRInfo.rki`、`KRInfo.lki`、`KRInfo.nf`、`KRInfo.nrank` として取得できます。詳細については、[`KRInfo`](@ref) を参照してください。さらに、`KRInfo.id` には、線形化に使用される基礎となるシステム行列ペンシル `S(λ)` の無限基本除数が含まれています。

有理行列 `R(λ)` のクロンカー構造情報は、[1] および [2] の結果を使用して、基礎となる線形化のクロンカー構造情報から取得されます。クロンカー構造情報の決定は、`R(λ) = C*inv(λE-A)*B+D` を満たす、順序 `n` の正則ペンシル `A-λE` を持つ不可約な記述子システム実現 `(A-λE,B,C,D)` を構築することによって行われ、構造化されたシステム行列ペンシル `S(λ) = [A-λE B; C D]` を適切なクロンカー様形式 (KLF) に縮約し、有限固有値の数（`R(λ)` の有限ゼロでもある）、無限固有値の重複度（`R(λ)` の無限ゼロの重複度に1を加えたもの）、左および右のクロンカーインデックス（`R(λ)` のものでもある）、および通常のランク（`R(λ)` の通常のランクに `n` を加えたもの）を示します。`R(λ)` に関するクロンカー構造情報と極限ゼロ構造情報は、それぞれ `S(λ)` と `A-λE` のクロンカー構造情報から回収されます。

右クロンカーインデックスは整数ベクトル `rki` に提供されます。`rki` の要素数は、有理行列 `R(λ)` の右零空間の次元であり、その合計は右多項式零空間基底の最小次数または右有理零空間基底の最小マクミラン次数です。

左クロンカーインデックスは整数ベクトル `lki` に提供されます。`lki` の要素数は、有理行列 `R(λ)` の左零空間の次元であり、その合計は左多項式零空間基底の最小次数または左有理零空間基底の最小マクミラン次数です。

`S(λ)` の無限固有値の重複度は整数ベクトル `id` に提供され、各 `i` 番目の要素 `id[i]` は無限基本除数の次数（すなわち、無限固有値の重複度）です。`id[i] > 1` に対応する無限ゼロは、重複度 `id[i]-1` の `R(λ)` の無限ゼロです。`R(λ)` の無限ゼロの重複度は `iz` に返されます。

`R(λ)` の極限は、正則ペンシル `A-λE` のゼロです。`A-λE` の有限固有値は `R(λ)` の有限極限であり、その数は `nfp` に返されます。`A-λE` の無限ゼロは `R(λ)` の無限極限であり、その重複度は `ip` に返されます。

不可約記述子システムに基づく線形化は、[3] に記載された方法と、[4] および [5] のペンシル操作アルゴリズムを組み合わせて、縮約順序線形化を計算するために構築されます。これらのアルゴリズムは、`fast = true` の場合は列ピボットを伴うランクを明らかにするQR分解、`fast = false` の場合はSVD分解に基づくランク決定を使用します。SVD分解に基づくランク決定は一般的により信頼性がありますが、関与する計算努力は高くなります。

`S(λ)` と `A-λE` の適切なKLFへの縮約は、[5] の直交類似変換を使用して行われ、`fast = true` の場合は列ピボットを伴うランクを明らかにするQR分解、`fast = false` の場合はより信頼性の高いSVD分解に基づくランク決定が含まれます。

キーワード引数 `atol` と `rtol` は、それぞれ `N(λ)` と `D(λ)` の非ゼロ係数に対する絶対および相対許容誤差を指定します。デフォルトの相対許容誤差は `n*ϵ` であり、ここで `n` は `N(λ)` の最小次元のサイズで、`ϵ` は `N(λ)` の係数の要素型のマシンエプシロンです。
