```
inner(A::MPS, B::MPS)
inner(A::MPO, B::MPO)
```

内積 `⟨A|B⟩` を計算します。`A` と `B` が MPO の場合、フロベニウス内積を計算します。

大きな MPS または MPO の重なりを取る際にアンダーフロー/オーバーフローを避けるために [`loginner`](@ref) を使用してください。

!!! compat "ITensors 0.3"



ITensors 0.3 より前は、`inner` にはデフォルトで `true` のキーワード引数 `make_inds_match` がありました。`true` の場合、関数は収束する前にサイトインデックスを一致させようとしました。したがって、入力は異なるサイトインデックスを持つことができますが、同じ次元または QN ブロックを持っている限りです。この動作は脆弱で、テンソルごとに単一のサイトインデックスを持つ MPS のみで機能しましたが、ITensors 0.3 以降は非推奨となりました。ITensors 0.3 以降は、収束させるために入力する MPS または MPO が互換性のあるサイトインデックスを持つことを確認する必要があります。たとえば、プライムレベルが適切に一致するようにすることです。

[`dot`](@ref) と同じです。

[`loginner`](@ref) や [`logdot`](@ref) も参照してください。
