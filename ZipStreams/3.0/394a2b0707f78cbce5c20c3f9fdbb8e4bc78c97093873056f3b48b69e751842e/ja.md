```
zipsource(io)
zipsource(f, io)
```

Zipアーカイブの読み取り専用の遅延ストリーミング表現を作成します。

最初の形式は、`io`の周りにラップされた`ZipArchiveSource`を返し、ユーザーが返されたオブジェクトを反復処理することでストリームから読み取られるファイルを抽出できるようにします。`io`は`Base.IO`から継承されたオブジェクト（技術的には`read`、`eof`、`isopen`、`close`、および`bytesavailable`が定義されていることを要求する）または`AbstractString`ファイル名であり、読み取り専用モードでファイルを開き、その`IOStream`をラップします。

2番目の形式は、最初の引数として単項関数を取ります。構築された`ZipArchiveSource`オブジェクトは関数に渡され、関数の結果がユーザーに返されます。これにより、`do`ブロックとの互換性が得られます。`io`が`AbstractString`ファイル名の場合、ブロックが終了するとファイルは自動的に閉じられます。`io`が上記のように`Base.IO`オブジェクトである場合、ブロックが終了しても*閉じられず*、呼び出し元が引数のライフタイムを制御できるようになります。

!!! warning "ファイルの終わりを知らずに読み取ることは危険です！"
    Zipアーカイブの中央ディレクトリは、ファイルの場所、圧縮および非圧縮サイズ、CRC-32チェックサムの*権威あるソース*です。ローカルファイルヘッダーはこの情報について虚偽の情報を提供する可能性があり、不適切なファイル抽出につながることがあります。私たちは、ユーザーが信頼できないソースから抽出されたファイルを信頼し始める前に、`is_valid!`メソッドを使用してファイルの内容を中央ディレクトリと照合することを**強く**推奨します。
