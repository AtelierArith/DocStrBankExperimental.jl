```
zipsink(fname; [キーワード引数]) -> ZipArchiveSink
zipsink(io; [キーワード引数]) -> ZipArchiveSink
zipsink(f, args...)
```

データを書き込むためのZipアーカイブの`IO`ストリームを開きます。

# 位置引数

  * `fname::AbstractString`: 書き込みのために開くZipアーカイブファイルの名前。ファイルが存在しない場合は作成されます。ファイルが存在する場合は、書き込む前に切り詰められます。
  * `io::IO`: 書き込み可能な`IO`オブジェクト。このオブジェクトは、返されたオブジェクトで`close`を呼び出すと閉じられます。
  * `f<:Function`: 開かれたストリームが渡される単項関数。このメソッドシグネチャは`do`ブロックの使用を許可します。シグネチャで呼び出された場合、`f`の戻り値がユーザーに返されます。

# キーワード引数

  * `utf8::Bool=true`: ファイル名とコメントをUTF-8エンコーディングでエンコードします。`false`の場合、テキストはIBM437エンコーディングでエンコードされていると見なされ、Zip標準に従います。
  * `comment::AbstractString=""`: Zipアーカイブに保存するコメント。この情報はアーカイブの最後にプレーンテキストで保存され、他の方法でZipアーカイブに影響を与えることはありません。コメントは常にIBM437エンコーディングを使用して保存されます。

!!! note "IO引数の使用"
    最初の引数として`IO`オブジェクトを渡すと、オブジェクトはそのまま使用され、ストリームの現在の位置から上書きされ、ストリームを閉じた後に中央ディレクトリが書き込まれ、残りは切り詰められません。この`zipsink`の使用は、書き込み専用ストリーム（例：ネットワークパイプ）にZipアーカイブを書き込む必要がある上級ユーザーのみに推奨されます。
