```
is_valid!([sink::IO,] zf::ZipFileSource) -> Bool
```

アーカイブファイルから読み取った内容がローカルファイルヘッダーに保存されている情報と一致することを検証し、オプションで残りのファイル情報をシンクに書き込みます。

ファイルの内容がローカルファイルヘッダーの情報と一致しない場合、メソッドは検出されたエラーを`@error`ロギングを使用して説明します。メソッドは、圧縮されたファイルサイズと解凍されたファイルサイズがヘッダーにあるものと一致すること、そして解凍されたデータのCRC-32がヘッダーに報告されているものと一致することを確認します。検証は、部分的に読み取られたファイルでも機能します。

関数名の感嘆符は、関数が`ZipFileSource`からバイトを破壊的に読み取ることをユーザーに警告するものです。`sink`が提供されている場合、`zf`からの残りの未読バイトは`sink`に抽出されます。

データを`ZipFileSource`に書き込むことができないため、`is_valid!`を繰り返し呼び出しても毎回同じ結果が返されますが、最初の呼び出し時にのみ`sink`にデータを抽出します。
