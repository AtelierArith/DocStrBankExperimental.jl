```
value_and_pullback!!(rule, ȳ, f, x...)
```

`f(x...)`の値とプルバックを計算します。`ȳ`は`f(x...)`によって返されるプライマルに対して有効な接線でなければなりません。

`rule`は`build_rrule`を使用して構築する必要があります。

*注意:* `value_and_pullback!!`の誤用には微妙な方法がたくさんあるため、可能な限り`value_and_gradient!!`の使用を推奨します。

*注意:* 異なる値の`x`に対して`value_and_pullback!!`を複数回呼び出す場合は、毎回同じ`rule`のインスタンスを使用する必要があります。

*注意:* `f`と`x`にエイリアスがないことを確認するのはあなたの責任です。例えば、

```julia
X = randn(5, 5)
rule = build_rrule(dot, X, X)
value_and_pullback!!(rule, 1.0, dot, X, X)
```

は間違った結果をもたらします。

*注意:* この`value_and_pullback!!`メソッドは、最初にすべての引数に対して`zero_codual`を呼び出す必要があります。これにより、追加のアロケーションが発生する可能性があります。これがあなたのユースケースで問題になる場合は、`CoDual`を事前にアロケートし、この関数の他のメソッドを呼び出すことを検討してください。`CoDual`はプライマル-接線ペアである必要があります（プライマル-データペアではなく）。これを間違える方法はたくさんあるため、一般的にはこれを行うことはお勧めしません。 ```
