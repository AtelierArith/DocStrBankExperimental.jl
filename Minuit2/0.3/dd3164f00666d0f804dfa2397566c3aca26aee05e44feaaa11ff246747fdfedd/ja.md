```
LeastSquares(x::AbstractArray, y::AbstractVector, yerror, model::Function; 
                 loss=:linear, verbose=0, model_grad=nothing, names=(), mask=nothing)
```

最小二乗コスト関数（別名カイ二乗関数）。

データが (x, y +/- yerror) の形式である場合に使用します。ここで、x は一次元または多次元であり、y は常に一次元です。

## 引数

  * `x::AbstractArray` : モデルが評価される位置。モデルが多変量の場合、x は形状 (D, N) を持ち、D は次元の数、N はデータポイントの数です。
  * `y::AbstractVector` : 観測値。x と同じ長さでなければなりません。
  * `yerror` : 観測値の推定不確実性。y と同じ形状を持つか、スカラーである必要があり、その場合は y と同じ形状にブロードキャストされます。
  * `model::Function` : f(x, par0, [par1, ...]) の形式の関数で、出力が観測値と比較されます。ここで、x は位置、par0, ... はモデルパラメータです。モデルが多変量の場合、x は形状 (D,) を持ち、D はデータポイントの数です。
  * `loss::Union{Symbol, Function}` : ロス関数は、外れ値に対してフィットをロバストにするために変更できます。現在実装されているのは ``:linear` と `:soft_l1` のみですが、ユーザーはこの引数として任意のロス関数を渡すことができます。それは単調で、2回微分可能な関数であり、二乗残差を受け取り、修正された二乗残差を返す必要があります。
  * `verbose::Int` : 冗長性レベル。0: 出力なし。
  * `model_grad::Union{Function, Nothing}` : モデルの勾配をオプションで渡します。モデルと同じ呼び出しシグネチャを持ちますが、形状 (K,) の配列を返す必要があります。ここで、K はパラメータの数です。勾配は Minuit によって収束を改善または加速するために使用できます。
  * `names` : モデルの各パラメータのオプションの名前（順番に）。モデルパラメータの数と同じ長さでなければなりません。
  * `mask::Union{Vector{Bool}, BitVector, Nothing}` : データのサブセットを選択するためのオプションのマスク。x と同じ長さでなければなりません。

## 注意事項

代替ロス関数は、外れ値の引力を弱めることによってフィットを外れ値に対してよりロバストにします。最小二乗フィットの機械的アナロジーは、引力を持つシステムです。ロス関数は、外れ値に対してフィットをロバストにするために変更できます。
