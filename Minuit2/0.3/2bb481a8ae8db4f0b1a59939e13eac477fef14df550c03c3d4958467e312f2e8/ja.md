```
ExtendedUnbinnedNLL(data::AbstractArray, scaled_pdf::Function; log=false, verbose=0, mask=nothing, grad=nothing, names=())
```

未ビン拡張負の対数尤度。

フィッティングされたPDFの形状と正規化に関心があり、元の未ビンデータが利用可能な場合に使用します。データは1次元または多次元である可能性があります。

## 引数

  * `data::AbstractArray` : 観測のサンプル。観測が多次元の場合、データは形状(D, N)を持つ必要があり、Dは次元の数、Nはデータポイントの数です。
  * `scaled_pdf::Function` : f(data, par0, [par1, ...])の形の確率密度関数で、dataはサンプルであり、par0, ...はモデルパラメータです。タプル(<データウィンドウ内のfの積分>, <データポイントで評価されたf>)を返す必要があります。最初の値は、フィットのために考慮するデータウィンドウ内で積分された密度です。たとえば、データが指数分布しているが、(0, 5)の区間のみをフィットする場合、最初の値は0から5まで積分された密度です。データが多変量の場合、fに渡されるデータは形状(D,N)を持ち、Dは次元の数、Nはデータポイントの数です。
  * `verbose::Int` : 冗長性レベル。0: 出力なし。
  * `log::Bool=false` : 指数族の分布（正規分布、指数分布、ポアソン分布など）は、pdfの対数を直接計算することを可能にし、$log(pdf)$を数値的に計算するよりも正確で効率的です。モデルがpdfの代わりにlogpdfを返す場合は、これを`true`に設定します。
  * `mask::Union{Vector{Bool}, BitVector, Nothing}` : データのサブセットを選択するためのオプションのマスク。データと同じ長さである必要があります。
  * `grad::Union{Function, Nothing}` : pdfの勾配をオプションで渡します。pdfと同じ呼び出しシグネチャを持ちますが、形状(K, N)の配列を返す必要があります。ここで、Nはデータポイントの数、Kはパラメータの数です。
  * `names` : モデルの各パラメータのオプションの名前（順番に）。モデルパラメータの数と同じ長さである必要があります。
