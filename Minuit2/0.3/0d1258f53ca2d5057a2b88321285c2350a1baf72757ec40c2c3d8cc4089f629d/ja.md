```
BinnedNLL(bincounts::AbstractArray, binedges::Union{AbstractArray, Tuple}, cdf::Function; use_pdf=:none, verbose=0,  grad=nothing, names=())
```

ビン化された負の対数尤度。

データがビン化されている場合に、フィットしたPDFの形状のみが関心のある場合に使用します。このコスト関数は、通常のヒストグラムおよび重み付きヒストグラムで機能します。ヒストグラムは1次元または多次元です。

## 引数

  * `bincounts::AbstractArray` : ヒストグラムのカウント。これが次元Dの配列である場合、Dはヒストグラムの軸の数です。
  * `xe::Union{AbstractArray, Tuple}` : ビンのエッジの位置、長さはlen(n) + 1でなければならず、nはビンの数です。ヒストグラムが複数の軸を持つ場合、xeは各軸に沿ったビンのエッジの位置のコレクションでなければなりません。
  * `cdf::Function` : 形式f(xe, par0, par1, ..., parN)の累積密度関数で、ここでxeはビンのエッジであり、par0, ...はモデルパラメータです。対応する密度は、ヒストグラムがカバーする空間で1に正規化されている必要があります。モデルが多変量の場合、xeは形状(D, N)の配列のようなものでなければならず、ここでDは次元、Nはモデルが評価されるポイントの数です。
  * `verbose::Int` : 冗長性レベル。0: 出力なし。
  * `grad::Union{Function, Nothing}` : オプションで`cdf`の勾配を渡します。cdfと同じ呼び出しシグネチャを持ちますが、形状(K,)の配列を返す必要があります。ここでKはパラメータの数です。勾配はMinuitによって収束を改善または加速するために使用できます。
  * `use_pdf::Symbol`: `:none`、`:numerical`、または`:approximate`のいずれか。モデルのcdfが利用できないが、モデルのpdfが利用できる場合、このオプションは「numerical」または「approximate」に設定して、ビン上のpdfの積分を計算できます。「numerical」オプションは数値積分を使用し、正確ですが計算コストが高く、1Dヒストグラムのみサポートされています。「approximate」オプションは、ビンの中心でpdfを評価するゼロ次近似を使用し、ビンの面積で掛け算します。これは高速で高次元でも機能しますが、ビン内のpdfの曲率が重要な場合、バイアスのかかった結果をもたらす可能性があります。
  * `names` : モデルの各パラメータのオプション名（順番に）。モデルパラメータの数と同じ長さでなければなりません。
