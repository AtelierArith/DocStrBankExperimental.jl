```
mnprofile(m::Minuit, var; size=30, bound=2, grid=nothing, subtract_min=false, 
               ncall=0, iterate=5, use_simplex=true)
```

指定された区間でMinosプロファイルを取得します。

1つのパラメータをスキャンし、各スキャンポイントに対して他のすべてのパラメータに関して関数を最小化します。

## 引数

  * `var` : スキャンするパラメータ。
  * `size=30` : スキャンポイントの数。グリッドが設定されている場合は無視されます。
  * `bound=2` : boundがタプルの場合、(左, 右) スキャン境界、または最小値の周りで対称的にスキャンするシグマの数。グリッドが設定されている場合は無視されます。
  * `grid` : プロファイルを計算するためのパラメータ値。`grid`が設定されている場合、`size`と`bound`は無視されます。
  * `subtract_min=false` : trueの場合、オフセットを引いて最小値をゼロにします。
  * `ncall=0` : 最小化が中止される前の最大呼び出し回数の近似値。0に設定すると、Minuit2ライブラリの適応ヒューリスティックを使用します。注意: 制限はわずかに違反される可能性があります。なぜなら、条件はアルゴリズムの完全な反復の後にのみチェックされるからです。通常、これは複数の関数呼び出しを行います。       iterate : int, optional           収束が達成されなかった場合、N回までMigradを自動的に呼び出します           (デフォルト: 5)。この単純なヒューリスティックは、           コスト関数の数値精度が低くてもMigradの収束をより頻繁にします。これを1に設定すると           この機能が無効になります。       use_simplex: bool, optional           反復する必要がある場合、各Migrad呼び出しの前に           シンプレックスアルゴリズムを呼び出すためにこれをTrueに設定します           (デフォルト: True)。これは、           病的なケースでの収束を改善する可能性があります（反復する必要がある場合）。

## 戻り値

  * タプル(`x`, `y`, `ok`) : パラメータ値、関数値、およびフィットが成功したかどうかのブール値を持つ1D配列のタプル。
