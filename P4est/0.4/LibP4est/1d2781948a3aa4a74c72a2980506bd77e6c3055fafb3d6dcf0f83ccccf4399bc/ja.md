```
sc_io_sink_complete(sink, bytes_in, bytes_out)
```

すべてのバッファされた出力データをシンクにフラッシュします。この関数は、別の書き込みが必要な場合に SC*IO*ERROR*AGAIN を返すことがあります。これは現在、BUFFER がバイトの整数倍を必要とする場合に発生する可能性があります。成功した場合、読み取られたバイト数と書き込まれたバイト数の更新された値が bytes*in/out に返され、シンクの状態はシンクが作成されたばかりのようにリセットされます。特に、バイトカウンタはゼロにリセットされます。シンクの内部状態はそれ以外は変更されません。ここから先もシンクへの書き込みを続けることは合法です。シンクで行われるアクションはそのタイプに依存します。BUFFER、FILEFILE: なし。FILENAME: sink->file に対して fclose を呼び出します。

### パラメータ

  * `sink`:[in,out] 書き込むシンクオブジェクト。
  * `bytes_in`:[in,out] 最後の新規または完了呼び出し以来受信したバイト。NULL である可能性があります。
  * `bytes_out`:[in,out] 最後の新規または完了呼び出し以来書き込まれたバイト。NULL である可能性があります。

### 戻り値

完了した場合は 0、エラーの場合は非ゼロ。

### プロトタイプ

```c
int sc_io_sink_complete (sc_io_sink_t * sink, size_t * bytes_in, size_t * bytes_out);
```
