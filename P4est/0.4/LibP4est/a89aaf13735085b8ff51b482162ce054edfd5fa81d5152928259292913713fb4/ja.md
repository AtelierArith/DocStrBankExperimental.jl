与えられた次数 > 0 のロバット点の並列番号付けを保存します。

各要素は、エッジごとに次数 + 1 のノードを持ち、ボリュームごとに vnodes = (次数 + 1)^3 のノードを持ちます。element*nodes は次元 vnodes * num*local*elements であり、各要素のノードを辞書式の yx 順序でリストします（x が最も早く変化します）。element*nodes は、次のように配置されたローカルノードのセットにインデックスを付けます: local nodes = [<––-owned*count––->|<––-nonlocal*nodes––->] = [<––––––––num*local*nodes––––––––->] nonlocal*nodes には、他のプロセスによって所有される独立ノードのグローバルに一意な番号が含まれています。ローカルノードの場合、グローバルに一意な番号は i + global*offset で与えられ、i はローカル番号です。ハンギングノードは常にローカルであり、グローバル番号を持ちません。これらは隣接する独立ノードの幾何学的に対応するノードをインデックスします。

ノードがハンギングかどうかは、要素の面とエッジに基づいて決定されます。この情報は、各要素ごとに 1 つの int16*t で face*code にエンコードされています。ハンギングしている面やエッジがない場合、値はゼロになります。それ以外の場合、face*code は p8est*lnodes_decode によって解釈されます。

独立ノードは複数の MPI ランクによって共有されることがあります。ノードの所有者ランクは、そのノードに *接触* している最も低い番号の要素を持つ最も低い番号のオクツリーからのものです。

*接触* とは何を意味するのでしょうか？四分木は、それに接触しているすべての面/エッジ/コーナー、およびそれらの面/エッジ/コーナーに含まれるすべてのノードに接触していると言われます。

X +–––––-+ x |\ \ x | \ \ . x | \ \ x X | +–––––-+ +––-+ . . | | | |\ \ X o + | | | +––-+ o . \ | p | + | q | o \ | | \| | o \| | +––-+ O +–––––-+

この例では、次数 = 3 です。q と p の間の面には 4 つのノードがあり、各エッジに 2 つ、面の各コーナーに 1 つあります。面は q に接触しているため、q はその面の '.' とマークされたノードを所有します（q がより低いツリーからのものであるか、p よりも低いインデックスを持っている場合）。下部および前面のエッジは q に接触しているため、q は 'o' とマークされたノードも所有します。前下コーナーは q に接触しているため、q はそのノード 'O' も所有します。他のエッジやコーナーは q に接触していないため、q はそれらのノードを所有できず、'x' および 'X' とマークされています。

global*owned*count には、各プロセスが所有する独立ノードの数が含まれています。

sharers 配列には、独立ローカルノードを所有または共有するランクを保持する [`p8est_lnodes_rank_t`](@ref) 型のアイテムが含まれています。このプロセッサに共有ノードがない場合、空です。それ以外の場合、ランクでソートされ、現在のプロセスが含まれます。

次数 < 0 は、lnodes データ構造が $C^0$ ロバットノードではなく、四分木境界オブジェクト（面、エッジ、コーナー）に番号を付けるために使用されていることを示します。

次数が -1 の場合、面ごとに 1 つのノードが割り当てられ、ボリューム、エッジ、またはコーナーごとにはノードが割り当てられません。この番号付けは、低次のラビアート・トーマス要素に使用できます。この場合、vnodes == 6 であり、ノードは面の順序でリストされます。

次数が -2 の場合、面ごとおよびエッジごとに 1 つのノードが割り当てられ、ボリュームまたはコーナーごとにはノードが割り当てられません。この場合、vnodes == 18 であり、ノードは面の順序でリストされ、その後エッジの順序でリストされます。

次数が -3 の場合、面ごと、エッジごと、コーナーごとに 1 つのノードが割り当てられ、ボリュームごとにはノードが割り当てられません。この場合、vnodes == 26 であり、ノードは面の順序でリストされ、その後エッジの順序でリストされ、その後コーナーの順序でリストされます。
