```
p8est_search_local(p4est_, call_post, quadrant_fn, point_fn, points)
```

森林のローカル部分を検索します。この検索は、以下で「ポイント」と呼ばれる複数のターゲットを同時に検索する場合に特に効率的です。

検索はすべてのローカル四分木を対象に行われ、再帰的に上から下へ進行します。各木に対して、木の根から開始するか、木のローカル四分木をすべて含む部分木の根からさらに下で開始することができます。同様に、再帰の中間レベルのいくつかは、プロセッサローカル部分が単一の深い部分木に含まれている場合はスキップされることがあります。したがって、外側のループは深さ優先のプロセッサローカル森林トラバーサルです。そのループ内の各四分木は、葉であるか、葉の（直接的または間接的な）厳密な祖先です。新しい四分木に入ると、ユーザー提供の四分木コールバックが実行されます。

便宜上、ユーザーは森林を下って追跡される匿名の「ポイント」を提供できます。この方法により、1回の検索呼び出しで複数のターゲットを使用できます。特定の四分木に潜在的に一致するポイントのセットは、根から葉に向かって減少します：各四分木に対して、潜在的に一致するポイントに対する内部ループが実行され、各候補に対してポイントコールバックが実行され、そのポイントが一致するかどうかを判断します。一致しない場合は、現在のブランチで破棄され、そうでない場合は次の深いレベルに渡されます。コールバックは、同じポイントに対して複数の四分木に対してtrueを返すことが許可されています。この場合、複数の一致する四分木が特定される可能性があります。また、コールバックは、以前にtrueを返した四分木のすべての子に対してfalseを返すことも許可されています。ポイントコールバックが四分木に関連するすべてのポイントに対してfalseを返すと、再帰は停止します。ポイントは実際には何でもよく、`p4est`は解釈を行わず、ポインタをコールバック関数に渡すだけです。

ポイントが存在し、最初の四分木コールバックがtrueを返した場合、現在のすべてのポイントに対してポイントコールバックを呼び出した後、2回目に実行します。これにより、ポイントに関する情報をより簡単に収集および後処理できます。falseを返すと、再帰は停止します。

ポイントがNULL配列の場合、それらは無視され、再帰は各四分木コールバックを照会することで進行します。ポイントがNULLでなく、空の配列である場合、再帰はすぐに停止します！

### パラメータ

  * `p4est`:[in] 検索対象の森林。
  * `call_post`:[in] trueの場合、四分木コールバックを事前および事後の両方で呼び出します。
  * `quadrant_fn`:[in] 四分木に入るときに1回、出るときに1回（2回目はポイントが存在し、最初の呼び出しがtrueを返した場合のみ）実行されます。この四分木は常にローカルであり、完全にローカルでない場合でも少なくとも1つの子孫があります。コールバックがfalseを返すと、この四分木とその子孫は検索再帰から除外されます。その**point**引数は常にNULLです。コールバックがNULLの場合は無視されます。
  * `point_fn`:[in] **points**がNULLでない場合、NULLであってはなりません。可能な一致ポイントに対してtrueを返す必要があります。**points**がNULLの場合、このコールバックは無視されます。
  * `points`:[in] ユーザー定義の「ポイント」の配列。NULLの場合、**quadrant_fn**コールバックのみが実行されます。それがNULLの場合、この関数は何もしません。NULLでない場合、検索中にそのメンバーに対して**point_fn**が呼び出されます。

### プロトタイプ

```c
void p8est_search_local (p8est_t * p4est, int call_post, p8est_search_local_t quadrant_fn, p8est_search_local_t point_fn, sc_array_t * points);
```
