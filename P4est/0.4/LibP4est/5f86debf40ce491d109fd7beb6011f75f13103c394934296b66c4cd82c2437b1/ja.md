```
p4est_iterate(p4est_, ghost_layer, user_data, iter_volume, iter_face, iter_corner)
```

ローカルフォレスト内のすべてのボリューム、フェイス、およびコーナーでユーザーが提供したコールバックを実行します。

[`p4est_iterate`](@ref) は、ローカルフォレスト内のすべてのボリューム、フェイス、およびコーナーでユーザーが提供したコールバック関数を実行します。ghost*layerはNULLである可能性があります。*user*data*ポインタは[`p4est_iterate`](@ref)によって変更されることはなく、各コールバックに渡されます。コールバックのいずれかはNULLである可能性があります。コールバック関数は互いに交互に実行されます。つまり、一部のフェイスコールバックはボリュームコールバックの間に発生し、一部のコーナーコールバックはフェイスコールバックの間に発生します。

1. ボリュームコールバックは、ソートされたモートンインデックス順に発生します。 2) フェイスコールバックは、それを共有するクアドラントのボリュームコールバックが実行された後にのみ実行されます。 3) コーナーコールバックは、コーナーに接触するすべてのフェイスのフェイスコールバックが実行された後にのみ実行されます。 4) 特定のクアドラントに対するすべてのフェイスコールバックが、コーナーコールバックの前に実行されるとは限りません。 5) コールバックは、ローカルフォレストのセクションで隣接していないゴーストクアドラントのみを含むフェイスやコーナーでは実行されません。

### パラメータ

  * `p4est`:[in] フォレスト
  * `ghost_layer`:[in] オプション: 指定されていない場合、ローカルパーティションの境界でのコールバックはゴーストクアドラントに関するデータを提供できません: 欠落している ([`p4est_quadrant_t`](@ref) *) ポインタはNULLに設定され、欠落しているインデックスは-1に設定されます。
  * `user_data`:[in,out] 各コールバックに供給するオプションのコンテキスト
  * `iter_volume`:[in] 各クアドラントの内部に対するコールバック関数
  * `iter_face`:[in] クアドラント間の各フェイスに対するコールバック関数
  * `iter_corner`:[in] クアドラント間の各コーナーに対するコールバック関数

### プロトタイプ

```c
void p4est_iterate (p4est_t * p4est, p4est_ghost_t * ghost_layer, void *user_data, p4est_iter_volume_t iter_volume, p4est_iter_face_t iter_face, p4est_iter_corner_t iter_corner);
```
