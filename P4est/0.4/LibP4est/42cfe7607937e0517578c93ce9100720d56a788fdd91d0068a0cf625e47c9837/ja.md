与えられた次数 > 0 のロバット点の並行番号を格納します。

各要素は、面ごとに次数 + 1 のノードを持ち、ボリュームごとに vnodes = (次数 + 1)^2 のノードを持ちます。num*local*elements は `p4est` 内のローカル四分木の数であり、element*nodes は次元 vnodes * num*local_elements で、各要素のノードを辞書式の yx 順序（x が最も早く変化）でリストします。したがって、次数が 2 の場合、ノードのレイアウトは次のようになります。

f*3 c*2 c*3 6–-7–-8 | | f*0 3 4 5 f*1 | | 0–-1–-2 c*0 c*1 f*2

element*nodes は、次のように配置されたローカルノードのセットにインデックスを付けます: local nodes = [<––-owned*count––->|<––-nonlocal*nodes––->] = [<––––––––num*local*nodes––––––––->] nonlocal*nodes には、他のプロセスが所有する独立ノードのグローバルに一意な番号が含まれています。ローカルノードの場合、グローバルに一意な番号は i + global_offset で与えられ、ここで i はローカル番号です。ハンギングノードは常にローカルであり、グローバル番号を持ちません。これらは隣接ノードの幾何学的に対応する独立ノードをインデックスします。

ノードがハンギングかどうかは、要素の面に基づいて決定されます。この情報は、各要素ごとに 1 つの int8*t でエンコードされた face*code に格納されます。ハンギング面がない場合、値はゼロであり、そうでない場合は face*code が p4est*lnodes_decode によって解釈されます。

独立ノードは複数の MPI ランクによって共有されることがあります。ノードの所有ランクは、そのノードに *接触* している最も低い番号の要素からのものです。

*接触* とは何を意味しますか？四分木は、それに接するすべての面/コーナー、そしてそれらの面/コーナーに含まれるすべてのノードに接触していると言われます。

X +–––––-+ o | | o | | +––-+ o | p | | q | o | | | | o | | +––-+ O +–––––-+

この例では、次数 = 6 です。q と p の間の面には 5 つのノードがあり、その面の各コーナーに 1 つずつあります。面は q に接しているため、q はその面のノードを所有します（q が低い木からのものであるか、p よりも低いインデックスを持っている場合）。下のコーナーは q に接しているため、q もそれを所有します。上のコーナーは q に接していないため、q はそれを所有できません。

global*owned*count には、各プロセスが所有する独立ノードの数が含まれています。

sharers 配列には、独立ローカルノードを所有または共有するランクを保持する [`p4est_lnodes_rank_t`](@ref) 型のアイテムが含まれています。このプロセッサ上に共有ノードがない場合、配列は空です。そうでない場合、ランク順にソートされ、現在のプロセスが含まれます。

次数 < 0 は、lnodes データ構造が $C^0$ ロバットノードではなく、四分木境界オブジェクト（面とコーナー）を番号付けするために使用されていることを示します。

次数が -1 の場合、面ごとに 1 つのノードが割り当てられ、ボリュームやコーナーごとにはノードが割り当てられません。この番号付けは、低次のラビアート-トーマス要素に使用できます。この場合、vnodes == 4 で、ノードは面の順序でリストされます。

f*3 c*2 c*3 +–-3–-+ | | f*0 0 1 f*1 | | +–-2–-+ c*0 c*1 f*2

次数が -2 の場合、面ごとおよびコーナーごとに 1 つのノードが割り当てられ、ボリュームごとにはノードが割り当てられません。この場合、vnodes == 8 で、ノードは面の順序でリストされ、その後コーナーの順序でリストされます。

f*3 c*2 c*3 6–-3–-7 | | f*0 0 1 f*1 | | 4–-2–-5 c*0 c*1 f*2
