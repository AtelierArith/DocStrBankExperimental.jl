```
PhysicalGrid(xlim::Tuple{Real,Real},ylim::Tuple{Real,Real},Δx::Float64;[opt_type=:none,optimize_threads=false,nthreads_max=1])
```

物理空間に接続されたグリッドを設定するためのコンストラクタです。グリッドによって離散化される領域は、制限 `xlim` と `ylim` によって定義され、セルの間隔（各方向で均一かつ同一）は `Δx` によって指定されます。このコンストラクタは、この情報を使用して各方向のセルの数を決定し、必要に応じて整数の数を収容するために与えられた範囲を拡張します。また、各側にゴーストセルをパディングします。物理的な原点に対応するセルの隅に対応するインデックスも決定します。各方向の最小制限に対応する隅のインデックスは (1,1) であることに注意してください。

グリッドサイズの最適化に専念するいくつかのオプション引数があります。`nthreads_max` は使用するFFT計算スレッドの数を設定し、アーキテクチャ上で利用可能な合計までの値に設定できます。デフォルトは1です。

キーワード `opt_type` は `:threads`、 `:prime`（デフォルト）、または `:none` に設定できます。`:none` の場合、グリッドは指定された範囲にできるだけ近く設定されます。`:prime` の場合、グリッドは各方向で素数の積となる数に拡張されます（したがってFFTにおいて効率的です）。`:threads` の場合、グリッドはさまざまなグリッドサイズで代表的な計算を使用してテストされ、最小のCPU時間を持つものが特定されます。`optimize_threads = true` の場合、スレッドの数も（1と `nthreads_max` の間で）変化します。
