```
options = ContinuationPar(dsmin = 1e-4,...)
```

`F(x, p) = 0`を解くために使用される`continuation`アルゴリズムに影響を与えるパラメータを含む変数を返します。

# 引数

  * `dsmin, dsmax`は許可される最小、最大アーク長の値です。計算された解の枝における点の密度を制御します。
  * `ds = 0.01`は初期アーク長です。
  * `p_min, p_max`は`p`の許可されたパラメータ範囲です。
  * `max_steps = 100`は最大の継続ステップ数です。
  * `newton_options::NewtonPar`: ニュートンアルゴリズムのオプションです。
  * `save_to_file = false`: ファイルに保存します。名前は自動的に生成されるか、[`continuation`](@ref)で定義できます。これには`using JLD2`が必要です。
  * `save_sol_every_step::Int64 = 0`現在の解を保存する継続ステップの数です。
  * `plot_every_step = 10`現在の解をプロットする継続ステップの数です。

## 固有要素の処理、計算は引数`detect_bifurcation`によってトリガーされます（以下参照）

  * `nev = 3`計算される固有値の数です。少なくとも`nev`の不安定な固有値を持つように自動的に増加します。適切な分岐検出のために設定する必要があります。詳細については[平衡点の分岐点の検出](@ref)を参照してください。
  * `save_eig_every_step = 1`指定されたステップごとに固有ベクトルを記録します。**重要**メモリ制限のあるリソース、*例* GPUにとって。
  * `save_eigenvectors = true` **重要**メモリ制限のあるリソース、*例* GPUにとって。

## 分岐検出の処理

  * `tol_stability = 1e-10`平衡点と周期軌道の安定性をテストするための固有値の実部の下限です。
  * `detect_fold = true`フォールド分岐を検出しますか？検出が安価であるため、便利なオプションです。実際、フォールドポイントが多数存在することがあり、これはメモリ制限のあるデバイス（例：GPU）でメモリを飽和させる可能性があります。
  * `detect_bifurcation::Int` ∈ {0, 1, 2, 3} 0に設定すると、何も行われません。1に設定すると、固有要素が計算されます。2に設定すると、継続実行中に分岐点が検出されますが、正確には特定されません。3に設定すると、分岐点を特定するために二分法アルゴリズムが使用されます（遅い）。検出をオフにする可能性は便利なオプションです。実際、分岐点が多数存在することがあり、これはメモリ制限のあるデバイス（例：GPU）のメモリを飽和させる可能性があります。
  * `dsmin_bisection = 1e-16`分岐点を特定するための二分法アルゴリズムのdsminです。
  * `n_inversion = 2`二分法アルゴリズムにおける符号反転の数です。
  * `max_bisection_steps = 15`最大の二分法ステップ数です。
  * `tol_bisection_eigenvalue = 1e-16`二分法ステップで分岐点を検出するための固有値の実部に対する許容誤差です。

## `ds`適応の処理（詳細については[`continuation`](@ref)を参照）

  * `a  = 0.5`攻撃性因子です。これは、継続ステップごとにニュートン反復の数をほぼ一定に保つために`ds`を適応させるために使用されます。`a`が大きいほど、各継続ステップで変更されるステップサイズ`ds`が大きくなります。

## イベント検出の処理

  * `detect_event::Int` ∈ {0, 1, 2} 0に設定すると、何も行われません。1に設定すると、継続実行中にイベントの位置が探されますが、正確には特定されません。2に設定すると、イベントを特定するために二分法アルゴリズムが使用されます（遅い）。
  * `tol_param_bisection_event = 1e-16`イベントを特定するためのパラメータに対する許容誤差です。

## その他

  * `η = 150.` パラメータp₀ + ds / ηで最初の点での接線を推定するためのパラメータです。
  * `detect_loop` [作業中] 枝のループを検出し、継続を停止します。

!!! tip "変異"
    パフォーマンスの理由から、パラメータを保持するために不変の構造を使用することに決めました。異なるフィールドの変異を大幅に簡素化するために、`Accessors.jl`パッケージを使用できます。詳細についてはチュートリアルを参照してください。
