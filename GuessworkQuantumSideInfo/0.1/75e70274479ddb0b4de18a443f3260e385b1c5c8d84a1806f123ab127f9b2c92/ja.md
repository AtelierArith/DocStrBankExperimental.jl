```
guesswork_MISDP(
    p::AbstractVector{T},
    ρBs::AbstractVector{<:AbstractMatrix},
    num_outcomes = size(ρBs[1], 1)^2;
    solver,
    c = T.(1:length(p)),
    verbose::Bool = true,
) where {T<:Number} -> NamedTuple
```

確率ベクトル `p` によって指定された c-q 状態のための `num_outcomes` 個の可能な推測順序に対する推測作業の近似を計算し、分布 `X` と関連する量子状態を与える `ρBs` を提供します。もし $num_outcomes ≥ d_B^2$ であれば、ここで `d_B` は量子状態が存在するヒルベルト空間の次元であり、正確に推測作業を計算します。この場合、パラメータ `K` を提供することはできません。混合整数 SDP の定式化では、推測者は常に `length(p)` 回の推測を行うことが許可されています。ただし、正しい答えを得るために必要な各推測数に対してどのようにペナルティを課すかを選択するために、カスタムコストベクトル `c` を提供することができます。

この関数によって解かれる混合整数 SDP のサイズは、`length(p)`、次元 `d_B`、および `num_outcomes` に対して多項式的に成長しますが、混合整数 SDP は一般に効率的に計算可能であることが知られていません。

キーワード引数 `solver` には、混合整数 SDP を解くことができるソルバーを指定する必要があります。現在、これは混合整数線形プログラムと SDP の交互のシーケンスを解くことによって混合整数 SDP を解決する [Pajarito.jl](https://github.com/JuliaOpt/Pajarito.jl) を意味します。例についてはドキュメントを参照してください。Pajarito のパフォーマンスは、与えられた基盤となる混合整数線形ソルバーと SDP ソルバーのパフォーマンスに依存し、商業用ソルバーはしばしば学術ライセンスを持ち、はるかに高い性能を発揮することがあります。

キーワード引数 `verbose` は、問題が最適に解決されていない場合に Convex.jl からの警告に加えて、ステータス、最適値、最適な推測順序、および POVM を印刷します。
