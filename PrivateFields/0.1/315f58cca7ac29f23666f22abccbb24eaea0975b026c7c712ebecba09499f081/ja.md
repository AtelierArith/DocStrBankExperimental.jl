```
@private_struct ...
```

特定の `@private` フィールドを持つ構造体に注釈を付けます。これらのフィールドは、通常の使用においてドット構文（`a.b`）で直接アクセスすることはできません。代わりに、構造体のフィールドには、フィールドへの特権アクセスを持つ `@private_method` メソッドを通じて間接的に対話する必要があります。

構造体に対して直接 `getfield` を呼び出すことは依然として機能しますが、デフォルトでは `getproperty` はフィールドのプライバシーを検証するプログラム生成された `PrivateFields.jl` メソッドを呼び出します。

フィールドのプライバシーを保持しながらカスタム `getproperty` メソッドで構造体をオーバーロードしたい場合は、代わりに `PrivateFields.jl` から [`getproperty_direct`](@ref) を実装する必要があります。

# 使用法

構造体のフィールド定義の前に `@private` を付けて、それらのフィールドをプライベートとしてタグ付けします。プライベート/パブリックフィールドは、任意の順序で宣言できます。

以下の例を参照してください。

# 例

```
@private_struct Foo{X,Y}
    @private x::X
    y::Y
end
```
