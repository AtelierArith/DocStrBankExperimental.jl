パターン：

  * `_` は任意のものにマッチします
  * `foo` は任意のものにマッチし、値を `foo` にバインドします
  * `Foo(x,y,z)` は `Foo` 型の構造体で、フィールドが `x,y,z` にマッチします
  * `Foo(y=1)` は `y` フィールドが `1` に等しい `Foo` 型の構造体にマッチします
  * `[x,y,z]` は 3 つのエントリを持つ `AbstractArray` にマッチし、`x,y,z` にマッチします
  * `(x,y,z)` は 3 つのエントリを持つ `Tuple` にマッチし、`x,y,z` にマッチします
  * `[x,y...,z]` は少なくとも 2 つのエントリを持つ `AbstractArray` にマッチし、`x` が最初のエントリにマッチし、`z` が最後のエントリにマッチし、`y` が残りのエントリにマッチします
  * `(x,y...,z)` は少なくとも 2 つのエントリを持つ `Tuple` にマッチし、`x` が最初のエントリにマッチし、`z` が最後のエントリにマッチし、`y` が残りのエントリにマッチします
  * `_::T` は T の任意のサブタイプ（`isa`）にマッチします
  * `x || y` は `x` または `y` のいずれかにマッチする値にマッチします（両方のブランチに存在する変数のみがバインドされます）
  * `x && y` は `x` と `y` の両方にマッチする値にマッチします
  * `x where condition` は `condition` が真である場合のみマッチします（`condition` はパターンの前に出現する任意の変数を使用できます。例：`(x, y, z where x + y > z)`）
  * その他のものは定数として扱われ、等価性がテストされます
  * 式は標準の補間構文 `$(x)` を介して定数として補間できます

パターンは任意にネストできます。

繰り返し変数は等しい場合（`==`）のみマッチします。例えば `(x,x)` は `(1,1)` にマッチしますが、`(1,2)` にはマッチしません。
