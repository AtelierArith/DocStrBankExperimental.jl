```
sql`SOME SQL ... $var`
sql`SOME SQL ... $(var...)`
sql`SOME SQL ... 'A $literal string'`
sql``
```

`@sql_cmd`マクロは、SQLクエリ文字列とそのパラメータを追跡するためのツールですが、パラメータをクエリ文字列に直接埋め込むことはありません。代わりに、`$x`のような埋め込みは、`x`の値がクエリパラメータとして渡されることになります。カンマ区切りのコンテキストに埋め込む値のコレクションがある場合、埋め込み内でスプラッティング構文を使用することもできます。例えば、`insert into foo values($(x...))`のようにです。

SQLインジェクション攻撃を防ぎ、`string()`を介さずにデータベース層を通じてJuliaタイプをそのSQL同等物に体系的に変換できるようにするために、直接の文字列埋め込みではなく、これを使用してください。

空のクエリフラグメントは、条件に基づいて動的にSQLコードを生成する必要がある場合に便利な```sql`` ```で生成できます。ただし、Juliaコード内ではなくSQL側に条件を埋め込むことも検討すべきです。

*埋め込みは*、単一引用符を持つ標準SQL文字列内では無視されるため、`'A $literal string'`を使用すると、変数`literal`の値ではなく`$literal`が含まれます。生の文字列を使用しているコードを変換する場合、埋め込みを引用する必要があったかもしれません。その場合、`SQLStrings.allow_dollars_in_strings[] = false`を設定することで変換を確認できます。

文字列の外でSQLコードにリテラル`$`を含める必要がある場合は、`\$`でエスケープできます。
