```
slicecube(I::GMTimage, layer::Union{Int, AbstractVector{<:Int}})
```

マルチレイヤーGMTimageのスライスを取得します。結果は依然としてGMTimageとして返されます。`layer`はzスライス番号です。

```
slicecube(G::GMTgrid, slice::Union{Int, AbstractVector{<:Int}}; axis="z")
```

GMTgridキューブからスライスを抽出します。

  * `slice`: Intの場合、そのレイヤーに対応するGMTgridが返されます。しかし、`slice`が浮動小数点数の場合、これは次のように解釈されます: その次元（以下の`axis`を参照）の座標を検索し、座標=`slice`に最も近いレイヤーを見つけます。`slice`の値が最も近いレイヤーの座標の10%以内でない場合、返されるレイヤーは隣接レイヤーの線形補間によって得られます。例えば、レイヤーが1単位間隔のキューブで`slice=2.5`の場合、レイヤー2と3の間で補間され、各レイヤーが最終結果で50%の重みを持ちます。注意: 戻り値の型は依然としてキューブですが、レイヤーは1つだけ（および対応する軸座標）です。`slice`は、抽出したいスライスを表す整数のベクトルでも構いません。出力は別のキューブです。
  * `axis`: スライスされる次元を示します。デフォルトの"z"は、スライスが垂直軸から取得されることを意味します。`axis="x"`は列に沿ってスライスし、`axis="y"`は行に沿ってスライスします。

```
slicecube(GI::GItype; slice::Int=0, angle=0.0, axis="x", cmap=GMTcpt())
```

GMTgridまたはGMTimageを斜め方向にスライスします。キューブのレイヤー`slice`を取り、`axis`の周りで`angle`度回転させます。これは`axis=:x`または`axis=:y`のいずれかでなければなりません。入力のデータ型に応じて異なる出力が生成されます。`GI`がGMTgridの場合、出力は2つのGMTgridになります: 1つは`z`レベルを持ち、もう1つはその平面に沿ったキューブのzレベルです。一方、GIがGMTimageの場合、最初の出力は前のケースに似ていますが、2番目はGMTimageになります。この場合、`cmap`オプションを使用して画像タイプにカラーテーブルを割り当てることができます。

スライスポイントP(x[i,j], y[i,j], z[i, j])での値は、同じ垂直方向の2つの最も近いボクセルの補間値です。

### 例

マルチレイヤーの'I' GMTimageオブジェクトの4番目のレイヤーを取得します。

```
I = slicecube(I, 4)
```
