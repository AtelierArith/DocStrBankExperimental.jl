```
orbits(xyz::Matrix{<:Real}; first=true, radius=6371.007, height=0, lon0=0, lat0=0, show=false, kw...)
```

地球や他の惑星の周りの軌道、または `xyz` に入力されたデータが表すものをプロットします。

### 引数

  * `xyz`: 軌道座標。デフォルトでは、地球中心地球固定（ECEF）システムの座標を期待しますが、球面座標（経度、緯度）でも構いません。この場合、`xyz` は Mx2 行列でなければならず、`height`（軌道の高さ）は > 0 でなければなりません。この引数が省略された場合、デモスパイラル「軌道」をプロットします。

### キーワード引数

  * `radius`: 惑星体（球状）の半径。この値はメートルまたはキロメートルで渡すことができ、球面座標で入力が渡されたときのみ使用されます。
  * `first`: $orbits$ が画像プロットの最初のレイヤーを作成するかどうかを示すブール値。デフォルト値の `true` を使用すると、$coast$ を呼び出して、正射影を使用して地球をプロットします。このデフォルトオプションは海岸線のみをプロットしますが、大陸を色付けするなどの他のオプションも `kw...` 引数を介して利用可能です。`first=false` に設定するか、より良い方法として $orbits!$ 形式を使用すると、$coast$ 呼び出しをスキップでき、このプロットを $grdimage$ によって生成された地球のDEMのような以前のプロットに追加できます。ただし、この場合、以前のプロットは同じ `lon0` と `lat0` を使用している必要があります。そうでないと、表示される軌道が間違ってしまいます。
  * `lon0`: 正射影の中央経度。
  * `lat0`: 正射影の中央緯度。
  * `height`: 入力 `xyz` が Mx2（経度、緯度）行列であり、半径 `radius` の球の上の *メートル* での高さを表すときに使用されます。（> 0 でなければならず、「あまり小さく」ないこと、さもないとこの関数のアルゴリズムが失敗します。）
  * `show`: 生成された画像を見たい場合は、これを `true` に設定します。`false` のままにすると、後でプロット呼び出しによってさらに要素を追加できます。
  * `kw`: $coast$ および $plot3$ 呼び出しで消費されるキーワード引数。たとえば、`land=:tomato, lw=1, lc=:blue` は大陸を `tomato` 色で塗り、軌道を青色の1ptの太さの線でプロットします。

### 例:

```
orbits(show=true)
```
