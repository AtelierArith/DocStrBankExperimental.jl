```
bv = isoutlier(x; critic=3.0, method::Symbol=:median, threshold=nothing, width=0.0) -> Union{BitVector, BitMatrix}
```

対応する `x` の要素で外れ値が検出されるときに、要素が true となる論理配列を返します。

もし `x` が行列であり、`width` が使用されていない場合、$isoutlier$ は `x` の各列に対して個別に動作します。デフォルトでは、外れ値は中央値から 3 倍の中央値絶対偏差 (MAD) よりも大きい値と定義されます。しかし、これは `critic` オプションを介して変更できます。

  * `critic`: 外れ値を検出するための閾値を設定する数値です。デフォルトは 3.0 で、これは中央値から 3 MAD よりも大きい値が外れ値と見なされることを意味します（`method` が `:median` の場合）、または平均から 3 標準偏差よりも大きい値（`method` が `:mean` の場合）です。
  * `method`: 外れ値の閾値を計算するために使用される方法です。次のいずれかの方法を使用できます：

      * `:median`: 中央値絶対偏差 (MAD) 方法を使用します。外れ値は中央値から `critic` MAD よりも大きい要素として定義されます。
      * `:mean`: 平均と標準偏差の方法を使用します。外れ値は平均から `critic` 標準偏差よりも大きい要素として定義されます。この方法は速いですが、「中央値」よりも堅牢性が低いです。
      * `:quartiles`: 四分位範囲 (IQR) 方法を使用します。外れ値は上四分位数 (75 パーセント) よりも 1.5 四分位範囲以上、または下四分位数 (25 パーセント) よりも 1.5 四分位範囲以下の要素として定義されます。この方法は、`x` のデータが正規分布していない場合に便利です。
  * `threshold`: `method` オプションの代替です。パーセンタイル閾値を指定し、[0, 100] の範囲にある要素を持つ 2 要素の配列として与えられます。最初の要素は下位パーセンタイル閾値を示し、2 番目の要素は上位パーセンタイル閾値を示します。また、(0, 100) の間の単一の数値でもあり、外れ値と見なされる端点の割合として解釈されます。たとえば、`threshold=1` は、下位および上位閾値が 1 パーセンタイルと 99 パーセンタイルであることを意味します。
  * `width`: このオプションが使用される場合（`x` が行列または GMTdataset の場合のみ）、ウィンドウ長 $width$ を持つ移動ウィンドウ法を使用してローカル外れ値を検出します。この長さは、`x` の最初の列に格納されている入力データと同じ単位で与えられます。

### 例:

```julia
x = [57, 59, 60, 100, 59, 58, 57, 58, 300, 61, 62, 60, 62, 58, 57];
findall(isoutlier(x))
2-element Vector{Int64}:
 4
 9
```

```julia
x = -50.0:50;	y = x / 50 .+ 3 .+ 0.25 * rand(length(x));
y[[30,50,60]] = [4,-3,6];	# 3 つの外れ値を追加
findall(isoutlier([x y], width=5))
```
