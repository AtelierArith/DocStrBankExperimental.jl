```
score, coeff, latent, explained, mu, ems = pca(X; DT::DataType=Float32, npc=0)
```

  * `X`: n-by-p のデータ行列 X。X の行は観測に対応し、列は変数に対応します。Float 型（32 または 64）でなければなりません。
  * `npc`: 解を構成するために使用される固有ベクトルの数。その値は [1, npc] の範囲内でなければなりません。デフォルトの `npc=0` は、完全な解を使用することを意味します。つまり npc = p です。このオプションは、`X` が *大きい* 場合に、非常に小さな説明分散を持つ成分を計算しないことでリソース（時間、メモリ）を節約したいときに使用します。
  * `DT`: データ型。内部的に、アルゴリズムは入力 `X` 行列のコピーを作成します。なぜなら、それは変更されるからです。`DT` はそのコピーがどのタイプを仮定するかを制御します。$Float32$ または $Float64$？デフォルトでは、`X` と同じデータ型を使用しますが、大きな行列の場合、メモリを節約するために $Float32$ を使用することが望ましい場合があります。注意：デフォルトは、以下で参照されるメソッドでは異なり、画像データはほぼ常に $UInt8$ または $UInt16$ であり、グリッドは $Float32$ であるため、デフォルトは $Float32$ です。

### 戻り値

  * `score`: 主成分。
  * `coeff`: 行列 X の主成分係数。X の行は観測に対応し、列は変数に対応します。係数行列は npc-by-n です。coeff の各列は1つの主成分の係数を含み、列は成分分散の降順に並んでいます。
  * `latent`: 主成分の分散。（cov(X) の固有値）
  * `explained`: 各主成分によって説明される全体の分散の割合。
  * `mu`: X の各変数の平均。
  * `ems`: 最大の固有値に対応する `npc` の固有ベクトルのみを使用することによって発生する平均二乗誤差。`ems` は npc = n の場合（デフォルト）に 0 です。

```
Ipca = pca(I::GMTimage; DT::DataType=Float32, npc=0) -> GMTimage{UInt8}
```

このメソッドは、通常は $RemoteS$ パッケージで作成された $UInt16$ 型の衛星データの $GMTimage$ キューブを受け取り、説明分散の降順で主成分の $UInt8$ の $GMTimage$ キューブを返します。$truecolor(Ipca)$（$RemoteS$ から）は、3つの最大成分で構成された偽色画像を表示します。

```
Gpca = pca(G::GMTgrid; DT::DataType=Float32, npc=0) -> GMTgrid{DT}
```

このメソッドは $GMTgrid$ キューブを受け取り、説明分散の降順で主成分を持つ別のグリッドを返します。タイプは `DT`（デフォルトは $Float32$）です。
