```
D = grid2tri(G, G2=nothing; bottom=false, downsample=0, isbase=false, ratio=0.01,
             thickness=0.0, wall_only=false, top_only=false, geog=false)
```

グリッド `G` によって定義された表面を三角形分割し、オプションで底面 `G2` を三角形分割します。

三角形分割の他に、この関数は `G` と `G2` の間、または `G` と一定のレベルまたは一定の厚さの間に垂直の壁を計算します。オプションとして、垂直の壁のみ、または完全に閉じた体（すなわち、底面を含む）を計算することができます。

この関数の出力は、$plot3d$ でボリュームレイヤーの3Dビューを作成するために使用できます。

注意: `G` グリッドは NaN の *アウトスカート* を持っている必要があります。そうでない場合は、$grdview$ を $N$ オプションで使用してください。

### 引数

  * `G`: 三角形分割される表面を表す GMTgrid オブジェクトまたはグリッドファイル名。
  * `G2`: レイヤーの底面を表すオプションの第二グリッド（またはファイル名）。このオプションを使用すると、`thickness` オプションは無視されます。

### キーワード引数

  * `bottom`: true の場合、底面で体を完全に閉じます。デフォルトでは、標高の視点角が正のときにその表面が見えないことが多いため、これを行いません。しかし、後でこのメッシュを STL 形式で保存して 3D ビューアソフトウェアにインポートしたい場合は、これが必要になるかもしれません。
  * `downsample`: グリッドの解像度が高すぎる場合、ここでファイルが大きくなり、操作が遅くなり、後の図があまり恩恵を受けないことがあります。そのような場合は、グリッドをダウンサンプルするのが良いアイデアです。`downsample` オプションは整数の削減係数を受け入れます。`downsample=2` はグリッドを各次元で2倍に縮小し、`downsample=3` は3倍に縮小します。
  * `isbase`: true の場合、`thickness` オプションを一定のレベル値を意味するものとして解釈します。つまり、垂直の壁は `G` の側面と `thickness` オプションを介して提供される一定のレベルから計算されます。
  * `ratio`: 計算された凹ハルが真の凹ハルにどれだけ近いかを決定するややトリッキーなパラメータです。0.005未満の値が適しているようですが、通常はそれほど近くしたくありません。なぜなら、これから得られる垂直の壁があまりにもギザギザになるからです。デフォルト値の0.01は、滑らかな凹ハルを得るのにうまく機能しますが、まだ良い垂直の壁を得るという目的に合っています。特定のケースに応じて調整が必要な場合があります。
  * `thickness`: 入力グリッドと同じ単位でレイヤーの厚さを表すスカラー。注意: 2つのグリッドが入力として渡された場合、このオプションは無視されます。
  * `wall_only`: true の場合、`G` と `G2` の間、または `G` + `thickness` の間の垂直の壁のみが計算され、返されます。
  * `top_only`: true の場合、`G` の三角形分割のみが返されます。
  * `geog`: `G` グリッドに参照情報がないが、地理座標にあることがわかっている場合は `geog=true` を設定します。この情報は三角形分割の出力に追加され、プロット目的に役立ちます。

### 戻り値

三角形分割された表面と垂直の壁、または壁のみ、または完全に閉じた体の GMTdataset のベクター。
