```
[I = ] lelandshade(G::GMTgrid; detail=1.0, contrast=2.0, intensity=false, zfactor=3, transparency=0.6,
                   show=false, color=false, opts=String[], cmap="", kw...)
```

グレースケールまたはカラーのシェーディング照明画像を、[Leland Brownの「テクスチャシェーディング」](http://www.textureshading.com/Home.html) によって開発された技術を使用して計算します。

  * `G`: Lelandテクスチャ照明画像を計算するための$GMTgrid$またはグリッドファイル名。
  * `detail`はテクスチャの詳細度です。詳細度の低い値は、より多くの標高情報を保持し、地形の全体的な大きな構造や標高の傾向をより感じさせますが、細かいテクスチャの詳細は犠牲になります。詳細度が高いとテクスチャが強調されますが、全体的に「平坦な」外観になり、標高の変化や大きな構造があまり目立たなくなります。
  * `contrast`は「垂直強調」と呼ばれるパラメータです。数値が高いほど中間調のコントラストが増加しますが、最も明るい特徴と最も暗い特徴の詳細が失われる可能性があります。数値が低いと、最も鋭い尾根や最も深い峡谷のみが強調されますが、全体的なコントラストは低下します。
  * `intensity | uint16`は、出力がUInt16またはUInt8画像（デフォルト）のいずれかを制御します。元のコードはUInt16画像のみを書き込みますが、$gdaldem$で計算されたヒルシェードと組み合わせたい場合、UInt8画像の方が便利です。
  * `zfactor`: 「ヒルシェード」を計算する際に$gdaldem$で使用される地形増幅係数。
  * `transparency`: $gdaldem$で計算されたヒルシェードとブレンドされたときにLelandアルゴリズムで計算されたテクスチャ画像の透明度。デフォルト値は0.5で、両方の画像に等しい重みを与えます。0.75の値はテクスチャ画像の重みを合計の3/4にします。
  * `color`: 出力がカラー画像かグレースケール画像かを選択するブール値（デフォルト）。カラー画像の場合、デフォルトの線形カラーマップを作成します（$makecpt$を呼び出すことによって）。ただし、`cmap`オプションで上書きできます。
  * `equalize`: カラー画像の場合、色をヒストグラム均等化することを選択できます（$grd2cpt$を呼び出すことによって）。このオプション単独（および`cmap`）は、`color=true`も設定します。
  * `opts`: $gdaldem$専用のオプションを持つ（オプションの）文字列ベクター（そのマニュアルを参照）。これを使用して、最終画像の「ヒルシェード」部分を微調整します。
  * `cmap`: カラー画像を作成する際にデフォルトのcmapを使用したくない場合、カラーマップ（cpt）名（ファイルまたはマスターcpt名）または$GMTcpt$を渡します。これにより、`color=true`も設定されます。
  * `colorbar`: ブール値で、`show=true`のときのみ使用され、画像の右側にカラーバーを追加します。
  * `show`: `true`に設定すると結果がすぐに表示されるブール値。`false`の場合、$GMTimage$オブジェクトが返されます。
  * `kw`: $makecpt$、$grd2cpt$、および$gdaldem$に引数を渡すために使用できるキーワード/値ペア。

### 例:

```
lelandshade(gmtread("@earth_relief_01s", region=(-114,-113,35,36)), color=true, colorbar=true, show=true)
```

### 戻り値

`show == false`の場合はGMTimageオブジェクト（`intensity`オプションに応じて8ビットまたは16ビット）、それ以外の場合は何も返されません。 ```
