```
create_model(
    case::Case,
    modeltype::EnergyModel,
    m::JuMP.Model;
    check_timeprofiles::Bool = true,
    check_any_data::Bool = true,
)
```

モデルを作成し、必要なすべての関数を呼び出します。

## 引数

  * `case::Case` - ケースタイプは、選択された時間構造、含まれる[`Resource`](@ref)s、および𝒳と𝒳間の潜在的な結合を表します。詳細はその*[docstring](@ref Case)*で説明されています。
  * `modeltype` - 使用されるモデルタイプで、`EnergyModel`のサブタイプです。
  * `m` - 空の`JuMP.Model`インスタンスです。提供されない場合、入力は標準の`JuMP.Model`であると見なされます。

## キーワード引数

  * `check_timeprofiles=true` - 各ノードの時間プロファイルをチェックするかどうかのブール指標です。新しいコンポーネントをテストしている場合を除き、チェックを無効にしないことをお勧めします。時間プロファイルがチェックされない場合、予期しない動作や入力データの潜在的な不整合が生じる可能性があります。
  * `check_any_data::Bool=true` - 入力データがチェックされるかどうかのブール指標です。新しい機能をテストしている場合を除き、チェックを無効にしないことをお勧めします。予期しない動作や、実現不可能なモデルが生じる可能性があります。

!!! note "古いものから新しいものへ"
    古い辞書を新しいケースタイプに変換するための追加のメソッドを提供します。
