```
struct Auto <: ContentSize
```

`GridLayout`の行/列サイズとして使用され、`trydetermine == true`の場合、`GridLayout`に対して行/列が内部の最大決定可能要素に合わせて縮小されるべきであることを示します。コンテンツ要素のサイズが決定できない場合、残りのスペースはすべての`Auto`行/列の比率に従って分割されます。

レイアウト可能な要素の幅/高さとして使用され、`trydetermine == true`の場合、要素の計算された幅/高さは、決定できる場合に自動幅/高さを報告します。これにより、親の`GridLayout`が要素の幅/高さに合わせて列/行サイズを調整できるようになります。`trydetermine == false`の場合、要素の計算された幅/高さは、たとえ自動幅/高さが決定できる場合でも`nothing`を報告し、親の`GridLayout`が要素の幅/高さに合わせて行/列を調整することを禁止します。これは、たとえば、タイトルの幅が自動的に決定できるにもかかわらず、`GridLayout`が列の幅をスーパータイトルの幅に縮小することを禁止するのに役立ちます。

`Auto`が要素サイズとして使用される場合、`ratio`は無視されます。
