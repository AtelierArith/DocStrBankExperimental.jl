```
parse(Colorant, desc)
```

色の説明を解析します。

これはHTML/CSSの色仕様のサブセットを解析します。特に、`currentColor`を除くすべてがサポートされています。

名前付き色（ただし、X11の名前付き色を使用しており、いくつかのケースではW3Cの名前付き色とはわずかに異なります）、`rgb()`、`hsl()`、`#RGB`、および`#RRGGBB`構文がサポートされています。

# 引数

  * `Colorant`: リテラルColorant
  * `desc`: 色の名前または説明

リテラルColorantは、`desc`文字列に従って解析されます（通常は`RGB`を返します）。より具体的な選択は、指定されたタイプの色を返します。

# 戻り値

  * `RGB{N0f8}`色、または
  * `hsl(h, s, l)`が使用された場合は`HSL`色
  * `rgba(r, g, b, a)`が使用された場合は`RGBA`色
  * `hsla(h, s, l, a)`が使用された場合は`HSLA`色
  * `0xAARRGGBB`/`0xARGB`が使用された場合は`ARGB{N0f8}`色
  * 最初の引数で指定された特定の`Colorant`タイプ

!!! note "X11の名前付き色に関する注意"
    スペースを含むX11の色名（例："sea green"）は、SVG/CSSで許可されていないため推奨されません。


!!! note "16進数表記に関する注意"
    CSSスタイルの16進数表記`#RRGGBB`/`#RGB`だけでなく、`0xRRGGBB`/`0xRGB`も解析できます。

    8桁または4桁の16進数表記をアルファ付きのRGB色に解析することもできます。ただし、結果はプレフィックス（つまり`#`または`0x`）に依存します。

    ```julia-repl
    julia> parse(Colorant, "#FF8800AA") # 透明なオレンジ
    RGBA{N0f8}(1.0,0.533,0.0,0.667)

    julia> parse(Colorant, "0xFF8800AA") # 不透明な紫
    ARGB{N0f8}(0.533,0.0,0.667,1.0)
    ```
