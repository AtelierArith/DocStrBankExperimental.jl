```
timeseries!([p::AbstractParticle,] bd::Billiard, t; dt, warning)
```

与えられた粒子 `p` をビリヤード `bd` 内で条件 `t` に従って進化させ、x, y, vx, vy の時系列と時間ベクトルを返します。もし `t` が `AbstractFloat` 型であれば、`t` だけの時間進化します。しかし、`t` が `Int` 型であれば、`t` 回の衝突のために進化します。それ以外の場合、`t` は任意の関数であり、入力として `t(n, τ, i, p)` を取り、進化を終了すべき時に `true` を返します。ここで `n` はこれまでに衝突した障害物の数、`τ` はこれまでに進化した時間、`i` は直前に衝突した障害物、`p` は粒子（例えば `p.pos` にアクセスできます）です。

この関数は粒子を変異させますので、そうでない場合は `timeseries` を使用してください。粒子が指定されていない場合、[`randominside`](@ref) を通じてランダムな粒子が選ばれます。

キーワード引数 `dt` は衝突の間の時系列を補間するために使用される時間ステップです。`dt` は衝突時間によって制限され、補間は*常に*衝突で停止します。直線的な伝播の場合は `dt = Inf`、磁気の場合は `dt = 0.01` です。

固定された磁気粒子の場合、`timeseries!` は警告を発し、粒子の軌道を返します。もし `t` が整数の場合、軌道は1周だけ進化します。

内部的には [`DynamicalBilliards.extrapolate`](@ref) を使用します。

### レイ分割ビリヤード

```
timeseries!(p, bd, t, raysplitters; ...)
```

レイ分割を実装するために、`timeseries!` 関数は4番目の引数 `raysplitters` を補足され、これは [`RaySplitter`](@ref) インスタンスのタプルです。レイ分割が使用される場合、`timeseries` は**常にビリヤードを変異させます**ので注意してください！レイ分割の使用に関する詳細情報と指示については、公式ドキュメントをご覧ください。
