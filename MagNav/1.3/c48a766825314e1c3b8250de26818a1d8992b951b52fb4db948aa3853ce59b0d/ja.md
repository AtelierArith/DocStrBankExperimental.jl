```
get_XYZ1(xyz_file::String,
         traj_field::Symbol = :traj,
         ins_field::Symbol  = :ins_data;
         info::String       = splitpath(xyz_file)[end],
         flight             = 1,
         line               = 1,
         year               = 2023,
         doy                = 154,
         dt                 = 0.1,
         tt_sort::Bool      = true,
         silent::Bool       = false)
```

保存されたCSV、HDF5、またはMATファイルからMagNavに必要な最小データセットを取得します。`XYZ1`フライトデータ構造内のすべてのフィールドは必要ではありません。CSV、HDF5、またはMATファイルに必要な最小データは以下を含みます：

  * `lat`, `lon`, `alt`（位置）
  * `mag_1_c`または`mag_1_uc`（スカラー磁気計測）

他のすべてのフィールドは計算/シミュレーションできますが、`flux_b`, `mag_2_c`, `mag_3_c`, `mag_2_uc`, `mag_3_uc`, `aux_1`, `aux_2`, および `aux_3`は除外されます。

CSVまたはHDF5ファイルが提供される場合、ファイル内の可能な列/フィールドは次のとおりです：

MATファイルが提供される場合、上記のフィールドも提供される可能性がありますが、非INSフィールドは指定された`traj_field` MAT構造内に、INSフィールドは指定された`ins_field` MAT構造内にあり、`ins_`プレフィックスなしである必要があります。これはMATLABコンパニオンがデータを出力する標準的な方法です。

**引数:**

  * `xyz_file`:   フライトデータCSV、HDF5、またはMATファイルのパス/名前（`.csv`、`.h5`、または`.mat`拡張子が必要）
  * `traj_field`: （オプション）MATファイル内で使用する軌道構造フィールド、CSVまたはHDF5ファイルには関連しない
  * `ins_field`:  （オプション）MATファイル内で使用するINS構造フィールド、利用できない場合は`:none`、CSVまたはHDF5ファイルには関連しない
  * `tt_sort`:    （オプション）trueの場合、時間でデータをソート（行ではなく）
  * `silent`:     （オプション）trueの場合、出力なし

`xyz_file`に提供されていない場合：

  * `info`:   （オプション）フライトデータ情報
  * `flight`: （オプション）フライト番号
  * `line`:   （オプション）行番号、すなわち`flight`内のセグメント
  * `year`:   （オプション）年
  * `doy`:    （オプション）年の日
  * `dt`:     （オプション）測定時間ステップ [s]

**戻り値:**

  * `xyz`: `XYZ1`フライトデータ構造
