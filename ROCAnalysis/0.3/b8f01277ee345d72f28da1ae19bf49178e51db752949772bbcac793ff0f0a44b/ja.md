```
ber()
```

ベイズ誤差率を計算します。これは、与えられたコスト関数によって指定された事前対数オッズに基づいて、ログ尤度比スコア `tar` と `non` のセットを考慮した場合の期待誤差率を計算します（`plo()` を参照）。

スコア `tar` と `non` が適切にキャリブレーションされているログ尤度比である場合、意思決定コスト関数 `dcf` を最小化する最適な決定閾値は驚くほどシンプルです。

```
θ = -plo(dcf).
```

この関数は、テストスコアに基づいてそのような決定の実際のコストを計算します。

引数：

  * `tar::Vector`, `non::Vector`, `plo`: ターゲットおよび非ターゲットの llr スコアと事前対数オッズ。ここで、`plo` はベクトルであり、複数のベイズ誤差率を生成する可能性があります。
  * `r::Roc`, `plo`: `roc(;collapse=false)` で計算された `Roc` 構造体。

Roc オブジェクトが圧縮されていないことが必須です。なぜなら、実際のコストは ROC 曲線の2つの「コーナー」ポイントの間の閾値で発生する可能性があるからです。`Roc` 構造体は、複数のコスト関数に対する最小ベイズ誤差率を計算するのに便利です。この場合、`Roc` は圧縮されることがあります。

  * `; thres`: 上記で説明した理論的最適値をオーバーライドするオプションの閾値。スコアが正しくキャリブレーションされていない場合に使用できます。
