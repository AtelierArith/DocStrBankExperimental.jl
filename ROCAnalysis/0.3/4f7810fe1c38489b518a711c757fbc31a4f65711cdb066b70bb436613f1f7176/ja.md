```
cllr(tar, non)
```

ログ尤度比のコストを計算します。これは、ログ尤度比（llr）スコア `tar` と `non`（ターゲットスコアと非ターゲットスコア）に対して行われます。ターゲットと非ターゲットのllrは、分子 `H1` と分母 `H2` の仮説が実際に真である場合のものです。

```
llr(x) = p(x | H1) / p(x | H2)
```

ここで `x` は、`H1` と `H2` の間で識別しようとするシステムのテストデータです。

完璧なシステムのCllrは、ターゲットスコアに `+Inf` を、非ターゲットスコアに `-Inf` を割り当てると0になります。無関心なシステムは、すべての入力 `x` に対して `llr=0` を生成し、`Cllr = 1` になります。悪くキャリブレーションされたシステムの場合、`Cllr > 1` であることに注意してください。Cllrの単位は *ビット* であり、cllrはデータから *抽出されていない* 情報の平均量として見ることができます。

Cllrは、システムの *キャリブレーション* と *識別* 能力を測定します。識別は、通常、非ターゲットスコアよりもはるかに高いターゲットスコアを生成する能力を含みます。キャリブレーションは、個々の `x` に対してコスト関数が知られている場合に最適なベイズ決定を行うことができることを含みます。
