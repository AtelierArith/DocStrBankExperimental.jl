```
freeze(cr::CommonRandomRecorder)::FrozenCommonRandomRecorder
```

[CommonRandomRecorder](@ref) は、クロックリクエストの乱数生成を見たときに毎回記録します。これは、複数のスレッドで比較のためにシミュレーションを実行する場合に問題になります。CRNを使用し、次のシミュレーション実行のために複数のスレッドを使用したい場合は、まず1つのスレッドでシミュレーションを何度も実行します。その後、シミュレーションをフリーズし、フリーズされたバージョンは新しいスレッドを記憶しなくなります。

フリーズされたレコーダーの一部はデバッグに便利なため可変になります。各スレッドのためにレコーダーをフリーズすると、各スレッドは自分のミスを追跡します。すべてのスレッドは、記録された乱数生成器の状態の同じコピーから動作します。
