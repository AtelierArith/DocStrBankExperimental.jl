```
mapsignal(fn,arguments...;padding,bychannel)
```

`fn`を引数のサンプルに適用し、`fn`の出力の信号を生成します。短い信号は最も長い有限長信号に合わせてパディングされます。関数`fn`は各引数を単一の数値として扱い、単一の数値を返す必要があります。この操作は入力のすべてのチャネルに対してブロードキャストされます。型安定な関数であることが期待されます。

## チャネル間関数

この関数は通常、チャネル間でブロードキャストされますが、各チャネルを個別に扱いたい場合は`bychannel=false`に設定できます。この場合、`fn`への入力は、特定のサンプルのすべてのチャネル値の`getindex`（タプルまたは配列）をサポートするオブジェクトとなり、`fn`は型安定なタプル値（マルチチャネルまたはシングルチャネルの結果用）または数値（シングルチャネルの結果のみ）を返す必要があります。例えば、以下のコードは左チャネルと右チャネルを入れ替えます。

```julia
x = rand(10,2)
swapped = mapsignal(x,bychannel=false) do (left,right)
    right,left
end
```

## パディング

パディングは、短い信号の終わりを超えたサンプルがどのように報告されるかを決定します。`padding`の値は[`pad`](@ref)に渡されます。そのデフォルト値は`fn`の値によって決まります。四つの基本的な算術演算子のデフォルト値はその恒等元（`*`の場合は`one`、`+`の場合は`zero`）です。これらのデフォルトは、`default_pad(fn)`を使用して`fn`に基づいて設定されます。`default_pad`のフォールバック実装は`zero`を返します。

特定の関数の新しいデフォルトを定義するには、`default_pad(fn)`の新しいメソッドを作成するだけです。

```julia

myfun(x) = 2x + 3
SignalOperators.default_pad(::typeof(myfun)) = one

```
