```
readtableCF!(データフレーム, 列; mergerows=false)
```

データフレームから四重奏CFを読み込みます。情報は列番号 `列` にあり、長さは **7 または 8** です：4つの分類群ラベルの後に3つのCF、その後にngenesが続く可能性があります。

同じ4分類群セット内に同じ種が複数回現れる場合（例：t1,t1,t2,t3）、データフレームは種間関係に関して情報を提供しない行（4分類群セット）を削除するように修正されます。この状況は、同じ種から複数の個体がサンプリングされた場合に発生する可能性があります。4分類群セットが情報を提供しない（その行が削除される）場合は、1つの分類群が3回または4回繰り返されるとき（t1,t1,t1,t1やt1,t2,t2,t2のように）です。2回現れる種のリストは、出力のDataCFオブジェクトに保存されます。これらの種については、外部エッジの長さが同系統単位で特定可能です。同じ4分類群セットに対応する複数の行がある場合、これらの行はマージされ、そのCF値（および遺伝子数）が平均化されます。どの種も4分類群セット内で繰り返されない場合、`mergerows`がtrueの場合にのみこの平均化が試みられます。

```
readtableCF!(DataCF, データフレーム, 列)
```

`DataCF`オブジェクト内の `.quartet.obsCF` 値を、列番号 `列` にあるデータフレームから読み取った値で修正します。`列` には、次の順序で3つのCFのための **3** 列番号が必要です：`12_34`、`13_24`、および `14_23`。

前提条件：

  * `DataCF` とデータフレーム内の同じ4分類群セットが同じ順序で存在しますが、この前提は *チェックされません*（速度のため、例えばブートストラップ中）。
  * 4分類群セットごとに1行のみ（同じ4分類群セットの複数の個体の代表はすでにマージされている必要があります）；基本的に：DataCFは `readtableCF!(df, 列)` によってデータフレームから作成されるべきです。
