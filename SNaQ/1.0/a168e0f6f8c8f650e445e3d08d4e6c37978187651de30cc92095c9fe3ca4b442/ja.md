```
topologymaxQpseudolik!(
    net::HybridNetwork,
    d::DataCF;
    verbose = true,
    ftolRel = 1e-6,
    ftolAbs = 1e-6,
    xtolRel = 1e-2,
    xtolAbs = 1e-3
)
```

与えられたネットワークトポロジーに対して、枝の長さと継承確率（γ）を推定します。ネットワークは*変更されず*、オブジェクト`d`のみが更新された期待コンコーダンス因子で変更されます。

出力: 最適化されたパラメータ（枝の長さとガンマ）を持つ新しいネットワーク。最大化されたクワルテット擬似逸脱は、加法定数までの負の対数擬似尤度であり、完全なフィットは0.0の逸脱に対応します。これはネットワークの属性でもあり、`loglik(net)`でアクセスできます。

オプションの引数:

  * `verbose`: `true`の場合、数値最適化に関する情報が画面に表示されます。
  * `ftolRel`, `ftolAbs`, `xtolRel`, `xtolAbs`: 擬似逸脱関数（`f`）とパラメータ（`x`）の絶対および相対許容値。

デフォルトの許容値はかなり緩やかで、実行時間を短縮します。これらは`snaq!`で使用されるものよりも緩やかであるため、同じネットワークトポロジーに対して`snaq!`がより良い（低い）スコアを返すことが期待できます。デフォルトよりも厳しい値、例えば`ftolRel`に対して1e-12、`ftolAbs`、`xtolRel`、`xtolAbs`に対して1e-10を使用することを*強く*推奨します。

枝の長さとγをさらに最適化するための別の戦略は、`topologymaxQpseudolik!`を複数回実行することです。なぜなら、ネットワーク内のエッジパラメータが改善されるたびに、良い場所から検索を再開することで、さらに良いエッジパラメータ値を見つけることができるからです。もし`snaq!`が与えられたネットワークトポロジーに対してより良いスコアを見つけた場合、この戦略（各試行でエッジパラメータを最適化する際に`snaq!`によって効果的に使用される）を使用し、厳しい許容値を使用することで、差異を排除できるはずです。
