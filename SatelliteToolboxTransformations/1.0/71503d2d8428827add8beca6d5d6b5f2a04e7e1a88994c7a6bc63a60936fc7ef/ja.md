```
r_itrf_to_gcrf_fk5([T, ]jd_ut1::Number, jd_tt::Number, x_p::Number, y_p::Number[, δΔϵ_1980::Number, δΔψ_1980::Number]) -> T
```

国際地球基準フレーム（ITRF）をユニバーサル天文時 `jd_ut1` [UT1] および地球時 `jd_tt` [Terrestrial Time] において地心天体基準フレーム（GCRF）に整列させる回転を計算し、IERS EOPデータ `x_p` [rad]、`y_p` [rad]、`δΔϵ_1980` [rad]、および `δΔψ_1980` [rad] を考慮します（[`fetch_iers_eop`](@ref)を参照）。このアルゴリズムはIAU-76/FK5理論を使用しています。

`x_p` はX軸周りの極運動の変位であり、これはIERS基準子午線の方向（0˚経度の子午線に沿って南方向に正）です。`y_p` はY軸周りの極運動の変位（90˚Wまたは270˚Eの子午線）です。`δΔϵ_1980` は傾斜の摂動です。`δΔψ_1980` は経度の摂動です。

UT1のユリウス日を使用してグリニッジ平均恒星時（GMST）を計算します（`jd_to_gmst`を参照）。一方、地球時のユリウス日は経度の摂動を計算するために使用されます。UT1のユリウス日と地球時のユリウス日は同じ瞬間に関連している必要があります。この関数は**これをチェックしません**。

回転のタイプはオプションの変数 `T` によって説明されます。もしそれが `DCM` であれば、DCMが返されます。そうでなければ、`Quaternion` であればクォータニオンが返されます。このパラメータが省略された場合、`DCM` にフォールバックします。

# 戻り値

  * `T`: ITRFフレームをGCRFフレームに整列させる回転。

# 備考

極運動に関連するEOPデータ（`x_p` と `y_p`）は必須です。これはITRF <=> PEFの変換を計算するための唯一の方法であり、モデルは運動がまだ十分に理解されていないため非常に不正確です **[1]**。ただし、傾斜の摂動（`δΔϵ_1980`）および経度の摂動（`δΔψ_1980`）に関連するEOPデータは省略可能です。この場合、GCRFフレームは通常J2000基準フレームと呼ばれます。
