```
Variables( 
    microspec           :: String,
    macrospec           :: String,
    dummyspec           :: String = "",
    nuisancedummyspec   :: String = "";
    market              :: String = "market",
    product             :: String = "product",
    outsidegood         :: String = "outsidegood",
    marketsize          :: String = "N",
    microinstruments    :: String = "",
    user                :: String = ""
    )
```

このメソッドは、[`Sources()`](@ref)で指定したソースから、回帰変数、インストゥルメント、ランダム係数、相互作用、変数ラベルなどを指定するために使用されます。これは*GrumpsVariables*型のオブジェクトを作成します。同じ名前の別のメソッドもあり、ほぼ同じことを達成しますが、ユーザーインターフェースが異なります。

ここで説明するメソッドは、2つの必須の文字列引数を含む引数を受け取りますが、他のメソッドはオプションの引数のみを受け取り、主にシンボル、シンボルのベクトル、およびキーワードを使用して任意の順序で渡すことができるシンボルの行列です。現在のメソッドは、他のメソッドを呼び出す前にユーザー入力を解析します。

オプション*market*は、市場記述子（名前）を含む列の列見出しを指定します。他のすべての引数についても同様ですが、*outsidegood*は、製品が外部財であることを示すスプレッドシートのエントリを説明します。外部財の同じラベルは、すべてのスプレッドシートおよびすべての市場で使用する必要があります。外部財のエントリは、消費者のマイクロデータ内でのみ使用されるべきであり、実際に外部財を選択する消費者がマイクロデータに存在する場合に限ります。すべての記述子は大文字と小文字を区別します。

個々の消費者の効用に入る変数と「平均効用」にのみ入る変数の間には区別があります。たとえば、個々の消費者の効用に何が入るかを指定するには、次のように指定できます。

"choice = loginc * msrp + famsize * logfootprint + famsize * van + urban * truck + rc * suv + rc * truck + rc * van"

これは*microspec*引数として、消費者の選択が「choice」という見出しの列にあるマイクロデータセットにあることを示し、4つの相互作用項と3つのランダム係数があることを示します。相互作用項は、消費者レベルの変数を最初の因子として、製品変数を2番目の引数として持ちます。この例では、3つのランダム係数はsuv、トラック、およびバン変数にあり、これらの変数名は製品レベルのデータセットの列見出しに対応する必要があります。*constant*を使用して定数が使用されていることを示すことができます：データに定数を含める必要はありません。

*macrospec*引数は次の形式を取ります。

"share = constant + logmpg + loghp + logfootprint + msrp | constant, logmpg, loghp, logfootprint, logcurbweight, lagplcon"

ここで*share*は製品レベルの市場シェアであり、=と|の間のすべては回帰変数を表し、|の後のすべてはインストゥルメントを表します。回帰変数とインストゥルメントは、製品レベルのモーメント部分のためのものです。再び*constant*を使用して定数が使用されていることを示すことができ、これはデータに含める必要はありません。

残りの引数については、他のメソッド[`Variables()`](@ref)の説明を参照してください。
