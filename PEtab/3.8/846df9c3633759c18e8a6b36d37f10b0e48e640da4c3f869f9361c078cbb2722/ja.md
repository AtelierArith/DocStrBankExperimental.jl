```
PEtabParameter(x; kwargs...)
```

パラメータ `x` のパラメータ推定情報。

`PEtabODEProblem` で推定されるすべてのパラメータは `PEtabParameter` として宣言されなければならず、`x` はモデル、観測式、またはノイズ式に現れるパラメータの名前でなければなりません。

## キーワード引数

  * `lb::Float64 = 1e-3`: パラメータ推定のための下限パラメータ境界。線形スケールで指定する必要があります。例えば、`scale = :log10` の場合、境界は `log10(1e-3)` ではなく `1e-3` として提供してください。
  * `ub::Float64 = 1e3`: パラメータ推定のための上限パラメータ境界。`lb` と同様に線形スケールで提供する必要があります。
  * `scale::Symbol = :log10`: パラメータを推定するスケール。許可されるオプションは `:log10`（デフォルト）、`:log2`、`:log`、および `:lin` です。`log10` スケールでの推定は通常パフォーマンスを向上させるため推奨されます。
  * `prior = nothing`: [Distributions.jl](https://github.com/JuliaStats/Distributions.jl) からのオプションの連続一変量パラメータ事前分布。事前分布は任意のパラメータ境界を上書きします。
  * `prior_on_linear_scale = true`: 事前分布が線形スケール（デフォルト）上にあるか、変換されたスケール上にあるか。例えば、`scale = :log10` で `prior_on_linear_scale = false` の場合、事前分布は変換された値 `log10(x)` に作用します。
  * `estimate = true`: パラメータを推定するか（デフォルト）、定数として扱うか。
  * `value = nothing`: `estimate = false` の場合に使用する値、および `get_x` 関数によって取得される値。デフォルトは `lb` と `ub` の間の中点です。

## 説明

もし事前分布 $\pi(x_i)$ が提供されると、パラメータ推定問題は最大尤度問題ではなく最大事後問題になります。実際には、負の対数尤度 $-\ell(x)$ を最小化する代わりに、負の事後を最小化します：

$$
\min_{\mathbf{x}} -\ell(\mathbf{x}) - \sum_{i} \pi(x_i)
$$

すべてのパラメータ $i$ に対して事前分布がある場合。
