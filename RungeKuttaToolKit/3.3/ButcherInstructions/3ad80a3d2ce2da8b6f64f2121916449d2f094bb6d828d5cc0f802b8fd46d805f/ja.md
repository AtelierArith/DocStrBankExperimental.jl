```
rooted_trees(
    n::Integer;
    tree_ordering::Symbol=:reverse_lexicographic,
) -> Vector{LevelSequence}
```

**正確に** $n$ 頂点を持つすべての根付き木のリストを計算して返します。異なる順序で根付き木を生成するいくつかのアルゴリズムが利用可能です：

  * `:reverse_lexicographic`（デフォルト）は、根付き木を木ごとに定数時間で生成する漸近的に最適なアルゴリズムです。これは、根付き木をその標準的なレベル列の表現の逆辞書順で生成します。これは知られている中で最も速いアルゴリズムですが、他のアルゴリズムとは異なり、最初に高い木を生成し、最後に茂った木を生成します。
  * `:lexicographic` は、根付き木をその標準的なレベル列の表現の辞書順で生成します。これは単に `:reverse_lexicographic` アルゴリズムを実行し、その出力を反転させます。このアルゴリズムは、最初に茂った木を生成し、最後に高い木を生成します。
  * `:attach` は、根付き木の順序 $n$ の各頂点に葉を付けることによって、順序 $n-1$ の各根付き木に葉を付けることによって根付き木を生成します。これは、根から始まり深さ優先で進みます。これは、基本的な微分の集合に対する積の法則の紙と鉛筆による適用によって生成される順序です。これは最も遅いアルゴリズムですが、最も直感的であると言えるでしょう。これは、最初に茂った木を生成し、最後に高い木を生成します。
  * `:butcher` は、[ジョン・ブッチャーのB系列に関するモノグラフ](https://link.springer.com/content/pdf/10.1007/978-3-030-70956-3.pdf)の[アルゴリズム3](https://link.springer.com/content/pdf/10.1007/978-3-030-70956-3_2)であり、ルンゲ–クッタ法に関する文献でよく使用される標準的な順序で根付き木を生成します。これは、最初に茂った木を生成し、最後に高い木を生成します。
