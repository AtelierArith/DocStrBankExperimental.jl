```
all_rooted_trees(
    n::Integer;
    tree_ordering::Symbol=:reverse_lexicographic,
) -> Vector{LevelSequence}
```

**最大** $n$ 頂点を持つすべての根付き木のリストを計算して返します。異なる順序で根付き木を生成するいくつかのアルゴリズムが利用可能です：

  * `:reverse_lexicographic`（デフォルト）は、根付き木を木ごとに定数時間で生成する漸近的に最適なアルゴリズムであり、[Terry Beyer と Sandra Mitchell Hedetniemi によって考案された](https://epubs.siam.org/doi/pdf/10.1137/0209055)ものです。これは、標準的なレベル列の表現の逆辞書順で根付き木を生成します。これは知られている中で最も速いアルゴリズムですが、他のアルゴリズムとは異なり、最初に高い木を生成し、最後に茂った木を生成します。
  * `:lexicographic` は、標準的なレベル列の表現の辞書順で根付き木を生成します。これは単に `:reverse_lexicographic` アルゴリズムを実行し、その出力を反転させます。このアルゴリズムは、最初に茂った木を生成し、最後に高い木を生成します。
  * `:attach` は、各根付き木の各頂点に葉を付けることによって順序 $n$ の根付き木を生成します。これは、根から始まり深さ優先で進みます。これは、基本的な微分の集合に対する積の法則の紙と鉛筆による適用によって生成される順序です。これは最も遅いアルゴリズムですが、最も直感的であると言えるでしょう。これは、最初に茂った木を生成し、最後に高い木を生成します。
  * `:butcher` は、[John Butcher の B-系列に関するモノグラフのアルゴリズム 3](https://link.springer.com/content/pdf/10.1007/978-3-030-70956-3_2)であり、[Runge–Kutta 法に関する文献でよく使用される標準的な順序で根付き木を生成します](https://link.springer.com/content/pdf/10.1007/978-3-030-70956-3.pdf)。これは、最初に茂った木を生成し、最後に高い木を生成します。
