```julia
function mcNemarTest(same args and kwargs as `cochranqTest`>)
```

単変量 [McNemar テスト](https://en.wikipedia.org/wiki/McNemar's_test) をデータの置換によって行います。 [`cochranqTest`](@ref) のエイリアスです。 $2 \cdot 2$ のコンティンジェンシーテーブルに使用できます。

[`cochranqTest`](@ref) はコンティンジェンシーテーブルの形式でデータ入力を受け付けないことに注意してください。データがコンティンジェンシーテーブルの形式である場合、次のように変換できます：

コンティンジェンシーテーブルが与えられた場合

| a | b |

| c | d |

次のようなベクトルを作成します：

  * $$
    b
    $$

    の頻度と同じ数のベクトル `[0, 1]`。
  * $$
    c
    $$

    の頻度と同じ数のベクトル `[1, 0]`。

例えば、コンティンジェンシーテーブル

| 1 | 2 |

| 3 | 4 |

は次のように入力されます：

```julia
table=[0 1; 0 1; 1 0; 1 0; 1 0]
```

!!! note "冗長な置換"
    上記のテーブルに任意の数のベクトル `[0 0]` または `[1 1]` を任意の組み合わせで追加しても、系統的な置換を使用して得られる p 値は全く同じです。漸近的 McNemar テストと同様に、これらのベクトルはコンティンジェンシーテーブルの要素 `a` と `d` に対応し、影響を与えません。


*方向性テスト*

  * 右方向のテストでは、$c$ が $b$ を超えることが期待されます。逆の場合、テストは 0.5 より高い p 値を返します。
  * 左方向のテストでは、$b$ が $c$ を超えることが期待されます。逆の場合、テストは 0.5 より高い p 値を返します。

*多重比較バージョン:* [`mcNemarMcTest`](@ref)

[UniTest](@ref) 構造体を返します。

*例*

```julia
using PermutationTests
table=[1 0; 1 0; 1 0; 1 0; 0 1]
t=mcNemarTest(table) # デフォルトではテストは双方向です

tR=mcNemarTest(table; direction=Right()) # 右方向のテスト
```
