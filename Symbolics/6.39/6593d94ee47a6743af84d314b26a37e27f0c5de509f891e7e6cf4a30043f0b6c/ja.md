```
transition(from, to, cond; immediate::Bool = true, reset::Bool = true, synchronize::Bool = false, priority::Int = 1)
```

状態 `from` から状態 `to` への遷移を作成します。この遷移は、遷移条件 `cond` が `true` に評価されると有効になります。

# 引数:

  * `from`: 遷移の出発状態。
  * `to`: 遷移の対象状態。
  * `cond`: Bool に評価される遷移条件。例えば、`ticksInState() >= 2` のようなものです。
  * `immediate`: `true` の場合、遷移が `true` になると同じティックで遷移が発生します。`false` の場合、状態のアクションが最初に評価され、遷移は次のティックで発生します。
  * `reset`: `true` の場合、遷移が発生すると、目的地の状態 `to` は初期条件にリセットされます。
  * `synchronize`: `true` の場合、遷移は出発状態のすべてのサブ状態機械が最終（端末）状態にある場合にのみ発生します。最終状態とは、外向きの遷移がない状態です。
  * `priority`: 状態に複数の外向き遷移がある場合、すべての外向き遷移は一意の優先度を持たなければなりません。遷移は優先度順に評価されます。つまり、優先度 1 の遷移が最初に評価されます。
