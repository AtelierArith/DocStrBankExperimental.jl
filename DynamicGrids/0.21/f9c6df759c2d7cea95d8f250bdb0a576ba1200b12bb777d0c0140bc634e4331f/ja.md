```
Delay <: AbstractDelay

Delay{K}(steps)
```

`Delay`は、`get`を使用してパラメータを取得する任意の`Rule`のフィールドとして、前のタイムステップからの[`Grid`](@ref)をパラメータソースとして使用することを可能にします。

すべてのフレームを保存する出力と組み合わせる必要があるため、`@assert DynamicGrids.isstored(output) == true`である必要があります。[`GraphicOutput`](@ref)を使用することで、出力オブジェクトを構築する際にキーワード引数`store=true`を使用することでこれを達成できます。

# 型パラメータ

  * `K::Symbol`: `init`内のグリッドの名前と一致します。

# 引数

  * `steps`: ユーザーが供給するパラメータとして、これは出力`tspan`のステップサイズの倍数です。これは各ステップの整数に自動的に置き換えられます。ルール内のコードで使用される場合、パフォーマンスのために`Int`のフレーム数である必要があります。

# 例

```julia
SomeRule(;
    someparam=Delay(:grid_a, Month(3))
    otherparam=1.075
)
```
