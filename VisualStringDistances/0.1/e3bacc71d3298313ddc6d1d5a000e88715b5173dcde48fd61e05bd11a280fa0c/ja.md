```
visual_distance(::Type{T}, s::Union{Char,AbstractString},
                     t::Union{Char,AbstractString}; D=KL(one(T)), ϵ=T(0.1),
                     normalize=nothing) where {T}
```

文字列 `s` と `t` の視覚的表現に基づいて距離の尺度を計算します。これは、GNU Unifont によってレンダリングされ、UnbalancedOptimalTransport.jl の非バランス Sinkhorn ダイバージェンスによって定量化されます。

  * キーワード引数 `D` は、"質量"（黒いピクセル）の生成または破壊に対してペナルティを課すために使用される `UnbalancedOptimalTransport.AbstractDivergence` を選択します。`D = VisualStringDistances.KL(ρ)` の場合、`ρ ≥ 0` の任意の数に対して、距離は非負であり、文字列の視覚的表現が同じである場合にのみゼロになります。これは一般的に望ましいです。
  * キーワード引数 `ϵ` は、Sinkhorn ダイバージェンスにおける "エントロピー正則化" を設定します。詳細については、[ドキュメント](https://ericphanson.github.io/UnbalancedOptimalTransport.jl/stable/optimal_transport/)を参照してください。簡単に言うと、`ϵ` が小さいほど、質量を移動するコストにより直接関連する量を計算しますが、計算に時間がかかります。
  * キーワード引数 `normalize` は、2 つの文字列の最大長に基づいて正規化定数を返す関数として選択できます。したがって、`normalize=identity` を選択すると、結果は 2 つの文字列の最大長で割られます。`normalize=sqrt` の選択は、いくつかの設定で良いバランスを提供することがわかっています。

[`printglyph`](@ref) を使用して、GNU Unifont によってレンダリングされた文字列の視覚的表現を見ることができます。

!!! note
    この執筆時点で、GNU Unifont は 57086 の異なる Unicode 文字をレンダリングすることができます。ただし、一部の Unicode 文字は同じグラフィカル表現でレンダリングされます。具体的には、689 の異なる Unicode 文字が重複した表現を持っています。例えば、以下の 6 つの重複のセットがあります：

      * 'Ꮋ': Unicode U+13BB (カテゴリ Lu: 大文字の文字)
      * 'Н': Unicode U+041D (カテゴリ Lu: 大文字の文字)
      * 'ꓧ': Unicode U+A4E7 (カテゴリ Lo: その他の文字)
      * 'Ⲏ': Unicode U+2C8E (カテゴリ Lu: 大文字の文字)
      * 'Η': Unicode U+0397 (カテゴリ Lu: 大文字の文字)
      * 'H': ASCII/Unicode U+0048 (カテゴリ Lu: 大文字の文字)

    したがって、これらの視覚的距離はゼロ（数値誤差の範囲内）として返されます。
