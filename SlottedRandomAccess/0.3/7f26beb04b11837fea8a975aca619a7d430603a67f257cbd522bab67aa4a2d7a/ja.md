```julia
struct PLR_SimulationParameters{RA, F}
```

PLRシミュレーションに使用されるパラメータを格納する型です。

# フィールド

  * `scheme::Any`: 特定のRAスキーム
  * `poisson::Bool`: シミュレーションがポアソンまたは定常トラフィックを仮定するかどうかを指定するフラグ
  * `coderate::Float64`: ユーザーによって送信されるパケットのコーディングレート
  * `M::Int64`: ユーザーによって送信されるパケットの変調のカーディナリティ
  * `power_dist::Any`: ランダムな電力値を生成するために使用される分布
  * `power_strategy::SlottedRandomAccess.ReplicaPowerStrategy`: 特定のパケットのレプリカに電力を割り当てるための戦略。[`ReplicaPowerStrategy`](@ref) 列挙型の有効な値でなければなりません。
  * `max_simulated_frames::Int64`: 各負荷ポイントのためにシミュレーションするRAフレームの数
  * `nslots::Int64`: 各RAフレーム内のスロットの数
  * `plr_func::Any`: 与えられたパケットのPLRをその等価なEb/N0の関数として計算するために使用される関数
  * `noise_variance::Float64`: シミュレーションでN0と仮定されるノイズの分散
  * `SIC_iterations::Int64`: デコーディングステップ中に実行する最大SICイテレーション数
  * `max_errored_frames::Int64`: シミュレーションするエラーのあるフレームの最大数。この数のエラーのあるフレームに達すると、シミュレーションは停止します。
  * `overhead::Float64`: シミュレーションにおけるフレーミングおよびガードバンド/タイムのために仮定されるオーバーヘッド。データ送信に完全にリソースが使用される理想的なケースと比較してオーバーヘッドを表す正の数でなければなりません（例：パイロットなし、ロールオフなし、ガードバンドなし、ガードタイムなし）。オーバーヘッド値が`1.0`であることは、スペクトル効率が理想的なケースの半分であることを意味します。
