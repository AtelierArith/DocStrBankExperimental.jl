```
context(x)
```

[`show(::IO, ::MIME"text/typst", ::Typst)`](@ref) のフォーマットデータを提供します。

カスタムタイプのためにこの関数を実装して、カスタム設定とパラメータを指定します。 [`Typst`](@ref) でラップされた値を渡すと、そのカスタムコンテキストがデフォルトと `merge!` され、デフォルトが上書きされる可能性があります。コンテキストのマージと `IOContext` の構築に互換性を持たせるために、メソッドは `AbstractDict{Symbol}` を返す必要があります。

| 設定              | デフォルト    | 型              | 説明                                                                               |
|:--------------- |:-------- |:-------------- |:-------------------------------------------------------------------------------- |
| `:backticks`    | `3`      | `Int`          | 生のテキストマークアップを囲むバックティックの数で、ネストされた生のテキストを区別するために増やすことができます。                        |
| `:block`        | `false`  | `Bool`         | `:mode => math` の場合、要素をインラインでレンダリングするために囲むドル記号にスペースをパディングするか、独自のブロックにするかを指定します。  |
| `:depth`        | `0`      | `Int`          | コンテナタイプ内の現在のネストレベルで、インデントの度合いを指定します。                                             |
| `:mode`         | `markup` | [`Mode`](@ref) | 現在のTypstの構文コンテキストで、`code` は番号記号の後に続き、`markup` は最上位にあり角括弧で囲まれ、`math` はドル記号で囲まれます。 |
| `:parenthesize` | `true`   | `Bool`         | 一部の数学的要素を括弧で囲んで演算子の優先順位を指定し、あいまいさを避けるかどうかを示します。                                  |
| `:tab_size`     | `2`      | `Int`          | マルチラインのTypstフォーマットを持つ一部の要素によって使用されるスペースの数で、各 `depth` レベルごとに繰り返されます。              |
