```julia
    struct Pad{N} <: AbstractBorder
        style::Symbol
        lo::Dims{N}    # 各次元の下端で拡張する数
        hi::Dims{N}    # 各次元の上端で拡張する数
    end
```

`Pad` は、画像の境界を超えてピクセルを外挿するために使用されるパディングの形式を指定する型です。インスタンスは、画像の境界条件を指定するシンボルである `style` を設定する必要があります。

# 出力

画像の境界がどのようにパディングされるべきかを指定する型 `Pad`。

# 拡張ヘルプ

空間的な二次元画像フィルタリング操作を画像と $D \times D$ フィルタとの間の離散畳み込みとして表現する場合、画像の境界から $D$ ピクセル以内のピクセルに対する結果は未定義です。境界近くおよび境界での操作を定義するには、エッジを超えてピクセルを外挿するためのスキームが必要です。`Pad` 型は、必要な外挿スキームを指定することを可能にします。

この型は、1次元、2次元、または多次元の画像のパディングを容易にします。

画像の各次元の下端と上端で異なる量のパディングを指定できます（2次元の場合、上、左、下、右）。

## オプション

いくつかの有効な `style` オプションが以下に説明されています。各オプションの指標例として、6つのピクセルからなる画像に対するパディングの結果が示されています。

```plain
        ┏━━━━━━┓ 
        ┃abcdef┃ 
        ┗━━━━━━┛ 
```

左端と右端のパディングの効果のみを示しますが、上端と下端にも類似の結果が適用されます。

### `:replicate` (デフォルト)

境界ピクセルは画像の境界を超えて拡張されます。

```plain
   ╭────┏━━━━━━┓────╮
   │aaaa┃abcdef┃ffff│
   ╰────┗━━━━━━┛────╯
```

関連情報: [`Fill`](@ref), [`padarray`](@ref), [`Inner`](@ref) および [`NoPad`](@ref)

### `:circular`

境界ピクセルは回転します。たとえば、左端を超えてインデックスを付けると、右端から始まる値が返されます。

```plain

   ╭────┏━━━━━━┓────╮
   │cdef┃abcdef┃abcd│
   ╰────┗━━━━━━┛────╯

```

関連情報: [`Fill`](@ref), [`padarray`](@ref), [`Inner`](@ref) および [`NoPad`](@ref)

### `:symmetric`

境界ピクセルはピクセル間の位置に対して反射します。つまり、ミラーリングする際に境界ピクセルは省略されます。

```plain

   ╭────┏━━━━━━┓────╮
   │edcb┃abcdef┃edcb│
   ╰────┗━━━━━━┛────╯

```

### `:reflect`

境界ピクセルはエッジ自体に対して反射します。

```plain

   ╭────┏━━━━━━┓────╮
   │dcba┃abcdef┃fedc│
   ╰────┗━━━━━━┛────╯

```

関連情報: [`Fill`](@ref),[`padarray`](@ref), [`Inner`](@ref) および [`NoPad`](@ref)。
