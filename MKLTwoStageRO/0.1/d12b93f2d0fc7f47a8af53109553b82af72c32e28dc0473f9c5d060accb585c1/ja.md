```
ECCG(Fields...)
```

拡張列と制約生成（ECCG）アルゴリズムのインスタンスを作成します（`Ref.1`）。

# フィールド

  * `MPsolver::Module = HiGHS`: 主問題（`MP`）のためのソルバー。`Sx` と `Sy` のタイプに基づいて選択する必要があります。一般的に、`Sx` または `Sy` が混合整数空間である場合、混合整数線形プログラム（MILP）をサポートするソルバーであるべきです。
  * `SPsolver::Module = HiGHS`: KKT条件に基づく再定式化形式のサブ問題（`SP`）のためのソルバー（実行可能性オラクルと最適性オラクルの両方）。サブ問題オラクルはビッグ-M法に基づく線形化された再定式化であるため、`U` が単に多面体不確実性集合である場合、少なくともMILPソルバーである必要があります。
  * `ϵ::Real = 1e-5`: 拡張CCG法の全体的な絶対停止基準。これは、オラクルの目的値が `ϵ` より小さくない場合、現在の `x` が実行可能でないと考えるための実行可能性オラクルの許容誤差としても使用されます。
  * `BigM::Real = 1e5`: ビッグ-M法に基づく線形化された再定式化における `SP` のビッグ-M値（実行可能性オラクルと最適性オラクルの両方）。ビッグ-Mに対して厳密な境界が解析的に得られる場合、アルゴリズムのパフォーマンスが向上する可能性があることに注意してください。
  * `verbose::Bool = true`: 解法プロセスの詳細出力を制御するスイッチ。

# 参考文献

1. Bertsimas, D., & Shtern, S. (2018). A scalable algorithm for two-stage adaptive linear optimization. arXiv preprint arXiv:1807.02812.
