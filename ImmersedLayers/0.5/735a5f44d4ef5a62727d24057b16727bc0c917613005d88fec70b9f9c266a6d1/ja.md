`@ilmproblem` マクロは、浸透層問題に特有の型を自動的に生成するために使用され、その型に基づいて問題をディスパッチすることができます。2つの引数を取ります：問題の名前（`Problem` が自動的に追加されます）と、問題がスカラー型かベクトル型かを示します。例えば、

```
@ilmproblem(StokesFlow,vector)
```

は、ベクトル値データを扱う型 `StokesFlowProblem` を生成します。生成された型は、グリッド情報とボディを渡すことができるコンストラクタを自動的に持ち、DDF関数とスケーリングタイプのオプション選択も可能です。例のコンストラクタは次のようになります。

```
StokesFlowProblem(grid,bodies[,ddftype=CartesianGrids.Yang3][,scaling=GridScaling])
```

サーフェスがない問題用の別のコンストラクタもあり、グリッド情報のみを渡すことが必要です。例えば、

```
StokesFlowProblem(grid)
```

問題のコンストラクタにはいくつかのキーワード引数があります。

  * `ddftype =` DDFタイプを設定します。デフォルトは `Yang3` です。
  * `scaling =` スケーリングタイプを設定します。`GridScaling`（デフォルト）または `IndexScaling` です。
  * `dtype =` 要素タイプを `Float64`（デフォルト）または `ComplexF64` に設定します。
  * `phys_params =` 物理パラメータを渡します。
  * `bc =` 境界条件データまたは関数を渡します。
  * `forcing =` 強制関数またはデータを渡します。
  * `motions =` 浸透サーフェスの速度を指定する関数を提供します。注意：このキーワードが使用される場合、サーフェスが動くと仮定されます。
  * `reference_body =` 問題が解かれる軸のボディのIDを提供します。
  * `timestep_func =` 時間依存問題のための関数を渡し、タイムステップサイズを提供します。この関数は2つの引数、`grid::PhysicalGrid` と `phys_params` を受け取り、タイムステップを返すことが期待されています。これを決定する方法はユーザーに委ねられています。引数に関係なく定数値を返すこともできます。
