```
OI_TARGET(lst=OITargetEntry[]; revn=0)
```

は `OI_TARGET` データブロックを生成します。オプション引数 `lst` はターゲットを指定する `OITargetEntry` のベクターで、デフォルトでは指定されていません。キーワード `revn` はリビジョン番号を指定します。

例えば `db` という `OI_TARGET` のインスタンスは、以下のプロパティを持っています：

```
db.extname # OI-FITS 拡張名
db.list    # ターゲットのリスト
db.revn    # リビジョン番号
```

そして、ターゲットに個別にインデックスまたは名前でアクセスするために、イテラブルまたは配列として使用できます：

```
length(db) # ターゲットの数
db[i]      # i 番目のターゲット
db[key]    # 名前が文字列 `key` と一致するターゲット

for tgt in db; ...; end # すべてのターゲットをループ
```

`OI_TARGET` インスタンスがデータセットにプッシュされると、ターゲット識別子（フィールド `target_id`）は自動的にデータセットのターゲットリストのインデックスと同じになるように書き換えられます。

標準メソッド `get` と `haskey` は、キーの型（整数または文字列）に応じて期待通りに動作します。`keys` メソッドについては、デフォルトではターゲット名のイテレータを返しますが、期待されるキーの型を指定することもできます：

```
keys(db)          # ターゲット名のイテレータ
keys(String, db)  # 同上
keys(Integer, db) # ターゲットインデックスのイテレータ
keys(Int, db)     # 同上
```

[`OIFITS.get_column`](@ref) を呼び出すことで、`OI_TARGET` データブロックのすべてのターゲットに対して、特定のターゲットフィールドをベクターの形式で取得できます。
