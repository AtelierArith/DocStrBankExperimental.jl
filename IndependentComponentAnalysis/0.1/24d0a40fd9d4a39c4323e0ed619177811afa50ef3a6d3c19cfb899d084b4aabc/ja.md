```
est, H = duet( x1, x2, n_sources, n = 1024;
    p            = 1, # ヒストグラムを重み付けするために使用される振幅パワー
    q            = 0, # ヒストグラムを重み付けするために使用される遅延パワー
    amax         = 0.7,
    dmax         = 3.6,
    abins        = 35, # ヒストグラムビンの数
    dbins        = 50, # ヒストグラムビンの数
    kernel_size  = 1, # ヒストグラムの平滑化を制御します。
    window       = hanning,
    bigdelay     = false,
    kernel_sizeδ = 0.1,
    kwargs..., # これらはstft関数に送信されます
)
```

DUETは盲目的なソース分離のためのアルゴリズムです。これはステレオミクスチャーで動作し、時間-周波数領域で重ならない限り、任意の数のソースを分離できます。

## `p` と `q`

論文は以下のガイドラインを示しています：

  * `p = 0, q = 0`: 元のDUETアルゴリズムで提案されたカウントヒストグラム
  * `p = 1, q = 0`: ML対称減衰推定器に基づく
  * `p = 1, q = 2`: ML遅延推定器に基づく
  * `p = 2, q = 0`: 遅延推定器のバイアスを減少させるため
  * `p = 2, q = 2`: 低信号対雑音比または音声ミクスチャー用

以下の論文から： "`p = 1, q = 0` は良いデフォルトの選択です。ソースが等しいパワーでない場合、`p = 0.5, q = 0` を提案します。これは、支配的なソースがヒストグラム内の小さなソースピークを隠すのを防ぎます。"

大きな遅延の場合、`dmax` を小さな値（20）に設定し、`nfft ≈ 8n, p=0.5, q=2` にする方が効率的なようです。

  * `bigdelay` は、2つのマイクが遠く離れているかどうかを示します。`true` の場合、遅延 `δ` は微分法を使用して推定され（論文のセクション8.4.1を参照）、遅延マップは `kernel_sizeδ` を使用して平滑化されます。

*Rickard, Scott. (2007). The DUET blind source separation algorithm. 10.1007/978-1-4020-6479-1_8.* に基づく実装。
