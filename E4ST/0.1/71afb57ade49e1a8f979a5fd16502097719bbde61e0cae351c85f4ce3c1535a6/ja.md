```
AggregationTemplate(;file, name) <: Modification
```

これは、集約されるべきもののテンプレートを表す `file` が与えられた場合に集約結果を出力するモジュールです。 `name` は単に修正の名前であり、集約された情報が保存されるファイル名のルートとして使用されます。

`file` は以下の列を持つcsvテーブルを表す必要があります：

  * `table_name` - 集約されるテーブルの名前。すなわち、`gen`、`bus` など。
  * `result_name` - 集約されるテーブル内の列の名前。列は [`get_table_col_unit`](@ref) を介してアクセス可能な単位を持っている必要があります。
  * `filter_` - テーブルの行に対するフィルタリング条件。すなわち、`filter1`。提供可能なフィルタの種類については [`parse_comparisons`](@ref) を参照してください。
  * `filter_years` - 集約される年に対するフィルタリング条件。年フィルタについての情報は [`parse_year_idxs`](@ref) を参照してください。
  * `filter_hours` - 集約される時間に対するフィルタリング条件。時間フィルタについての情報は [`parse_hour_idxs`](@ref) を参照してください。

`filter_` または `filter_hours` 列については、データテーブル（または時間テーブル）の列名が与えられた場合、その列の各ユニークな値に対して新しい行が作成されることに注意してください。すなわち、`gentype` の値が与えられた場合、`gentype=>coal`、`gentype=>ng` などの新しい行が作成されます。
