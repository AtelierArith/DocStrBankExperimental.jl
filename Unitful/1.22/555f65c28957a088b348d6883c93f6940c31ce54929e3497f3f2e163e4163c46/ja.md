```
@refunit(symb, abbr, name, dimension, tf, autodocs=false)
```

参照単位を定義します。通常はSI単位です。特定の次元の各単位間の変換係数を定義するのではなく、このマクロによって定義された参照単位との間の変換係数が与えられます。

このマクロは[`Unitful.abbr`](@ref)を拡張し、参照単位を省略形式で表示できるようにします。`tf == true`の場合、このマクロは標準SI接頭辞を使用して単位の10の各べきに対する記号を生成します。参照単位の次元を指定するために、`dimension`（[`Unitful.Dimensions`](@ref)オブジェクト）を指定する必要があります。`autodocs == true`の場合、SI接頭辞付き単位の自動生成されたドキュメント文字列が追加されます。このオプションは、`tf == false`の場合には影響を与えません。

!!! compat "Unitful 1.10"
    `@refunit`呼び出しの前にドキュメント文字列を追加して結果の単位を文書化するには、Unitful 1.10以降が必要です。`autodocs`引数もUnitful 1.10以降が必要です。


原則として、ユーザーはこのマクロを使用できますが、実際にはあまり意味がないかもしれません。[`@dimension`](@ref)を使用して新しい（おそらく物理的でない）次元を定義する場合、このマクロが必要になります。既存の次元を使用する場合、このマクロを使用するとほぼ確実に混乱を引き起こします。1つの潜在的な使用例は、SIに言及せずに単位系を定義することです。ただし、SIとこの仮想の単位系との間の変換を試みることを防ぐ明示的な障壁はなく、予期しない結果をもたらす可能性があります。

このマクロは、指定された次元に対して昇格のためのデフォルト単位として新しい単位（10のべき接頭辞なし）を選択します。

`symb`がバインドされている[`Unitful.FreeUnits`](@ref)オブジェクトを返します。

使用例: `@refunit m "m" Meter 𝐋 true`

この例は`src/pkgdefaults.jl`にあり、`km`、`m`、`cm`などを生成します。
