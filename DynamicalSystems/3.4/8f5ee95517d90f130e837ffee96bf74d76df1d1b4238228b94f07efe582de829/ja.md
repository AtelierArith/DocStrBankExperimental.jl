```
interactive_orbitdiagram(
    ds::DynamicalSystem, p_index, pmin, pmax, i::Int = 1;
    u0 = nothing, parname = "p", title = ""
)
```

離散時間力学系の軌道図（OD）を探索するためのインタラクティブなアプリケーションを開きます。`DynamicalSystems`が必要です。

本質的に、この関数は`ds`の`i`番目の変数の`orbitdiagram`の出力を提示し、それをインタラクティブにズームインすることを可能にします。

キーワードは、パラメータの名前、初期状態（*任意の*パラメータに使用される）または軌道図の上にタイトルを追加するかどうかを制御します。

## インタラクション

アプリケーションは「ODプロット」（左）と「コントロールパネル」（右）に分かれています。ODプロットでは、左のマウスボタンを使ってインタラクティブにクリック＆ドラッグしてOD内の領域を選択できます。この領域はその後、**再計算**されて高解像度で表示されます。

コントロールパネルのオプションは簡単で、

  * `n` 軌道図に記録されるステップの数（すべてがズームされた領域に含まれているわけではありません！）
  * `t` ステップの記録を開始する前の過渡ステップ
  * `d` x軸の密度（パラメータ軸）
  * `α` プロットされた点のアルファ値。

各更新時に`n*t*d`ステップが取られることに注意してください。これらのパラメータを変更した後は`update`を押す必要があります。`reset`を押すとODは元の状態（および変数）に戻ります。`back`を押すと探索の履歴を遡ります。履歴は「更新」ボタンが押されたときや領域がズームインされたときに保存されます。

ホイールから変数の1つを選択することで、どの変数のODを取得するかを決定することもできます！変数を変更する際にy軸の制限がわからないため、選択した変数のサイズにリセットされます。

## データへのアクセス

アプリケーションウィンドウにプロットされるのは*真の*軌道図であり、プロットの省略形ではありません。これは、すべてのデータが直接取得可能で使用可能であることを意味します。内部的には、常に軌道図を[0,1]²にスケーリングします（`Float32`ベースのプロットであっても`Float64`精度を許可するため）。しかし、これはデータを実際のスケールに変換する必要があることを意味します。これは、現在の関数から返される5つの引数を受け入れる関数[`scaleod`](@ref)を通じて行われます：

```julia
figure, oddata = interactive_orbitdiagram(...)
ps, us = scaleod(oddata)
```
