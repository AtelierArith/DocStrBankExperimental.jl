```
StreamInlet(info; max_buflen = 360, max_chunklen = 0, recover = True, procesing_flags = 0)
```

解決されたストリーム記述から新しいストリームインレットを確立します。

# 引数

  * `info::StreamInfo`: 解決されたストリーム記述オブジェクト（解決関数のいずれかから得られるもの）。注意: ストリーム*インレットは、望ましいチャネル形式とカウントが事前に既に知られている場合、完全に指定されたストリーム*情報で構築することもできますが、これは強く推奨されず、ストリームを事前に解決する時間がない場合（例: クライアントプログラムの制限による）にのみ行うべきです。

# キーワード引数

  * `max_buflen::Integer`: バッファリングするデータの最大量（名目上のサンプリングレートがある場合は秒単位、そうでない場合はサンプルのx100）。録音アプリケーションはここでかなり大きなバッファサイズを使用したいと考えますが、リアルタイムアプリケーションは次の計算を実行するために必要な分だけをバッファリングします。
  * `max_chunklen::Integer`: チャンクが送信される最大サイズ（サンプル単位）（デフォルトは送信者が使用するチャンクサイズに対応します）。録音プログラムはここで寛大なサイズを使用できます（ネットワークにパッキング方法を任せる）、一方でリアルタイムアプリケーションはより細かい（おそらく1サンプル）粒度を望むかもしれません。未指定のままにすると（=0）、送信者がチャンクの粒度を決定します。
  * `recover::Bool`: 回復可能な失われたストリーム（ソース*IDが設定されているもの）を静かに回復しようとします。他のすべてのケース（回復が偽、またはストリームが回復不可能な場合）では、ストリームのソースが失われた場合（例: アプリまたはコンピュータのクラッシュによる）に、関数が失われた*エラーをスローすることがあります。
