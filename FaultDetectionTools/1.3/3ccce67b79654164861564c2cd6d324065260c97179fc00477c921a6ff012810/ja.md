```
fdichkspec(sysf::FDIModel, SFDI::BitMatrix; sdeg, FDtol, FDGainTol, FDfreq, 
             atol, atol1, atol2, atol3, rtol, fast = true, minimal = false) -> (rdims, orders, leastorders)
```

与えられた合成モデル `sysf::FDIModel` に対して、一連の故障検出および隔離仕様 `SFDI` の実現可能性を確認します。 `SFDI` に `N` 行がある場合（すなわち、`N` の仕様が含まれている場合）、`N` 次元の整数ベクトル `rdims`、`orders`、`leastorders` が返され、FDIフィルタの合成に関連する情報が含まれます。 `SFDI[i,:]` に含まれる `i` 番目の仕様について、`rdims[i]` にはこの仕様を実現するために使用できる最小のヌル空間基底に基づく FDI フィルタの残差出力の数が含まれます。 `rdims[i] = 0` の場合、`i` 番目の仕様は実現不可能です。実現可能な `i` 番目の仕様については、`orders[i]` にはこの仕様を実現するために使用できる最小のヌル空間基底に基づく FDI フィルタの次数が含まれます。 `i` 番目の仕様が実現不可能な場合、`orders[i]` は `-1` に設定されます。 `minimal = true` の場合、`leastorders[i]` には `i` 番目の仕様を実現するために使用できるスカラー出力 FDI フィルタの達成可能な最小次数が含まれます。 `minimal = false` の場合、または `i` 番目の仕様が実現不可能な場合、`leastorders[i]` は `-1` に設定されます。

`FDFreq = freq` は、強い検出可能性チェックのための実数周波数値のベクトルまたはスカラー実数周波数値を指定します（デフォルト: `FDFreq = missing`）。

`FDtol = tol1` は、弱い仕様を評価するための閾値 `tol1` を指定します（関数 [`fditspec`](@ref) も参照） （デフォルト: `tol1 = 0.0001`）。

`FDGainTol = tol2` は、強い仕様を評価するための閾値 `tol2` を指定します。すなわち、`freq` に指定されたすべての周波数値に対する非ゼロ周波数応答ゲインの閾値です（関数 [`fdisspec`](@ref) も参照）（デフォルト: `tol2 = 0.01`）。

キーワード引数 `sdeg = β` は、内部生成された候補フィルタの極のための所定の安定度 `β` を指定します。すなわち、連続時間の場合はフィルタの極の実部が `β` 以下でなければならず、離散時間の場合はフィルタの極の大きさが `β` 以下でなければなりません。 `sdeg = missing` の場合、`FDFreq = missing` であれば安定化は行われません。 `sdeg = missing` かつ `FDFreq = freq` の場合、次のデフォルト値が使用されます: 連続時間の場合は `β = -0.05`、離散時間の場合は `β = 0.95`。

実行される削減におけるランクの決定は、`fast = true` の場合は列ピボットを用いたランクを明らかにする QR 分解に基づき、`fast = false` の場合はより信頼性の高い SVD 分解に基づきます。

キーワード引数 `atol1`、`atol2`、および `rtol` は、それぞれ `A`、`B`、`C`、`D` の非ゼロ要素の絶対許容誤差、`E` の非ゼロ要素の絶対許容誤差、`A`、`B`、`C`、`D` および `E` の非ゼロ要素の相対許容誤差を指定します。デフォルトの相対許容誤差は `n*ϵ` であり、ここで `ϵ` は作業用マシンエプシロン、`n` はシステム `sysf.sys` の次数です。キーワード引数 `atol3` は可観測性テストのための絶対許容誤差です（デフォルト: 内部で決定された値）。キーワード引数 `atol` は、同時に `atol1 = atol`、`atol2 = atol` および `atol3 = atol` を設定するために使用できます。

*メソッド:* [1] のヌル空間法が、適切に定式化された故障検出問題を解決する最小左ヌル空間基底として FDI フィルタを決定するために順次使用されます。
