```
query(document | node, selector; first = false, root = false) -> Node[]
query(f, document | node, selector; first = false, root = false) -> nothing
```

指定されたCSS `selector`に対して`document`または`node`をクエリします。`f`が提供されている場合は、見つかった各一致に対して`f`を呼び出し、`query`から`nothing`を返します。`f`が提供されていない場合は、すべての一致を含む`Vector{Node}`を返します。

`first::Bool`キーワードは、セレクターリストの最初の一致のみをマッチさせるかどうかを制御します。上流のドキュメントから引用します：

> リスト内のセレクターのいずれかで最初の一致が見つかった後、検索を停止します。
>
> デフォルトでは、コールバックは各セレクターリストに対してトリガーされます。つまり、ノードが異なるセレクターリストに一致する場合、コールバックで複数回返されます。
>
> 例えば：
>
> ```plaintext
> HTML: <div id="ok"><span>test</span></div>
> セレクター: div, div[id="ok"], div:has(:not(a))
> ```
>
> デフォルトの動作では、同じノード（div）で3回のコールバックが発生します。なぜなら、それはリスト内のすべてのセレクターによって見つかるからです。
>
> このオプションを使用すると、任意のセレクターで最初の一致の後に要素チェックを終了できます。つまり、上記の例ではコールバックが1回だけ呼び出されます。このようにして、検索中の重複を排除します。


`root::Bool`キーワードは、検索にルートノードを含めるかどうかを制御します。上流のドキュメントから引用します：

> 渡された（ルート）ノードを検索に含めます。
>
> デフォルトでは、ルートノードはセレクター検索に参加せず、その子のみが参加します。
>
> この動作は論理的です。ノードを見つけた後に、その中で他のノードを検索したい場合、再度チェックする必要はありません。
>
> しかし、ルートノードが検索に参加する必要がある場合もあります。これがこのオプションの目的です。
