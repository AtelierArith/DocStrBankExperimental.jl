```
lattice_plot(sol, sp, lrs::LatticeReactionSystem, filename::String; t = sol.tspan[2], kwargs...)
```

`LatticeReactionSystem`シミュレーションのプロットを作成します。プロットは`t`で指定された時点で作成されます（デフォルトではシミュレーションの最終時点になります）。

引数（すべての格子）:

  * `sol`: プロットしたいシミュレーション。
  * `sp`: プロットしたい種の値。シンボリック形式またはシンボルとして提供できます。
  * `lrs`: シミュレーションされた`LatticeReactionSystem`。
  * `t = sol.t[end]`: 解をプロットしたい時点

さらに、使用される格子のタイプに応じて、以下のオプション引数が関連する場合があります。

引数（1d格子）:

  * `markersize = 20`: 各コンパートメントの値を示すマーカーのサイズ。

引数（グラフおよび2d格子）:

  * `colormap = :BuGn_7`: アニメーションで種の量を表示するためのカラーマップ。
  * `plot_min = nothing`: カラー スケールの最小値（この値未満の値は、カラー スケールが計算されるときにこの値に設定されます）。`nothing`の場合、シミュレーションの最小値を使用します（プロットされた時点だけでなく、シミュレーション全体にわたって）。
  * `plot_max = nothing`: カラー スケールの最大値（この値を超える値は、カラー スケールが計算されるときにこの値に設定されます）。`nothing`の場合、シミュレーションの最大値を使用します（プロットされた時点だけでなく、シミュレーション全体にわたって）。

引数（グラフ格子）:

  * `node_size = 50`: プロット内のコンパートメントのサイズ。
  * `layout = Spring()`: プロット内のグラフノードのレイアウト。ベクトルとして提供でき、i番目の要素は2値のタプル（それぞれi番目のコンパートメントのyおよびx位置を決定）です。

注意:

  * マスクされた格子の場合、コンパートメントに対応しないグリッドポイントの値は表示されません。
  * 現在のプロットインターフェースは進行中の作業であり、修正が期待されます。フィードバックがある場合は、パッケージの著者に連絡してください。
  * 追加の引数は`lattice_plot`に渡すことができ、次にMakieの`lines`プロットコマンドに渡されます。
