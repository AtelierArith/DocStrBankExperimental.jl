```
lattice_animation(sol, sp, lrs::LatticeReactionSystem, filename::String; kwargs...)
```

`LatticeReactionSystem`シミュレーションのアニメーションを作成します。アニメーションは、`filename`引数で指定された名前のファイルに保存されます。

引数（すべての格子）:

  * `sol`: アニメーションしたいシミュレーション。
  * `sp`: アニメーションしたい種の値。記号形式またはシンボルとして提供できます。
  * `lrs`: シミュレーションされた`LatticeReactionSystem`。
  * `filename`: アニメーションを保存したいファイルの名前。
  * `nframes = 200`: アニメーションのフレーム数（これらはシミュレーション全体に均等にサンプリングされます）。
  * `framerate = 20`: アニメーションのフレームレート。
  * `ttitle = true`: アニメーション全体でシミュレーションの時間を示すタイトルを追加するかどうか。

さらに、使用される格子のタイプに応じて、以下のオプション引数が関連する場合があります。

引数（1d格子）:

  * `markersize = 20`: 各区画の値を示すマーカーのサイズ。
  * `plot_min = nothing`: yスケールの最小値。`nothing`の場合、シミュレーションの最小値を使用します。
  * `plot_max = nothing`: yスケールの最大値。`nothing`の場合、シミュレーションの最大値を使用します。

引数（グラフおよび2d格子）:

  * `colormap = :BuGn_7`: アニメーションで種の量を表示するためのカラーマップ。
  * `plot_min = nothing`: カラー スケールの最小値（この値未満の値は、カラー スケールが計算されるときにこの値に設定されます）。`nothing`の場合、シミュレーションの最小値を使用します。
  * `plot_max = nothing`: カラー スケールの最大値（この値を超える値は、カラー スケールが計算されるときにこの値に設定されます）。`nothing`の場合、シミュレーションの最小値を使用します。

引数（グラフ格子）:

  * `node_size = 50`: プロット内の区画のサイズ。
  * `layout = Spring()`: プロット内のグラフノードのレイアウト。ベクトルとして提供でき、i番目の要素は2値のタプル（それぞれi番目の区画のyおよびx位置を決定）です。

注意:

  * マスクされた格子の場合、区画に対応しないグリッドポイントの値は表示されません。
  * 現在のアニメーションインターフェースは進行中の作業であり、修正が期待されます。フィードバックがある場合は、パッケージの著者に連絡してください。
  * `lattice_animation`に追加の引数を渡すことができ、これらはMakieの`heatmap`プロットコマンドに渡されます。
