```
put(n::Int, locations => subblock) => PutBlock
```

`n`-量子ビットの [`PutBlock`](@ref) を作成します。第二引数は位置とサブブロックのペアであり、位置はタプル、整数、または範囲であり、サブブロックのサイズは位置の長さと一致する必要があります。$I$ を $2\times 2$ の単位行列、$G$ を $2\times 2$ の行列とすると、`put(n, i=>G)` の行列表現は次のように定義されます。

$$
I^{\otimes n-i} \otimes G \otimes I^{\otimes i-1}
$$

複数の位置の場合、式は複雑になることがあり、これは量子コンピューティングにおける `n`-量子ビット空間に適用される多量子ビットゲートの行列表現に対応します。

### 例

```jldoctest; setup=:(using Yao)
julia> put(4, 1=>X)
nqubits: 4
put on (1)
└─ X
```

特定の位置に多量子ビットゲートを置きたい場合は、すべての可能な位置を記述する必要があります。

```jldoctest; setup=:(using Yao)
julia> put(4, (1, 3)=>kron(X, Y))
nqubits: 4
put on (1, 3)
└─ kron
   ├─ 1=>X
   └─ 2=>Y
```

外側の位置はスコープを作成し、`PutBlock` 内のブロックに対して連続した2つの量子ビットのように見えます。

!!! tips
    大きなブロックには `put` の代わりに [`subroutine`](@ref) を使用する方が良いです。なぜなら、`put` はその内容の行列を直接使用するため、内部のものを利用することができません。`put` は小さなブロックに対してより効率的です。
