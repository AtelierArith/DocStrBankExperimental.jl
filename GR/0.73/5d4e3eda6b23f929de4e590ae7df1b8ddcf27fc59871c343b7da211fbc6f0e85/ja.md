```
path(x, y, codes)
```

与えられた頂点とパスコードを使用してパスを描画します。

**パラメータ:**

`x` :     X座標を含むリスト `y` :     Y座標を含むリスト `codes` :     パスコードを含むリスト

`x` と `y` の値はワールド座標で表されます。`codes` は複合パスを作成するために使用できるいくつかのパスプリミティブを説明します。

次のパスコードが認識されます：

+–––––+––––––––––––––––-+–––––––––-+–––––––––-+ | **コード** | **説明**                     | **x**             | **y**             | +–––––+––––––––––––––––-+–––––––––-+–––––––––-+ |     M, m | 移動                          | x                 | y                 | +–––––+––––––––––––––––-+–––––––––-+–––––––––-+ |     L, l | 線                            | x                 | y                 | +–––––+––––––––––––––––-+–––––––––-+–––––––––-+ |     Q, q | 二次ベジェ曲線              | x1, x2            | y1, y2            | +–––––+––––––––––––––––-+–––––––––-+–––––––––-+ |     C, c | 三次ベジェ曲線              | x1, x2, x3        | y1, y2, y3        | +–––––+––––––––––––––––-+–––––––––-+–––––––––-+ |     A, a | 弧                            | rx, a1, 予約      | ry, a2, 予約      | +–––––+––––––––––––––––-+–––––––––-+–––––––––-+ |        Z | パスを閉じる                |                   |                   | +–––––+––––––––––––––––-+–––––––––-+–––––––––-+ |        S | ストローク                  |                   |                   | +–––––+––––––––––––––––-+–––––––––-+–––––––––-+ |        s | パスを閉じてストローク      |                   |                   | +–––––+––––––––––––––––-+–––––––––-+–––––––––-+ |        f | パスを閉じて塗りつぶす      |                   |                   | +–––––+––––––––––––––––-+–––––––––-+–––––––––-+ |        F | パスを閉じて、塗りつぶしとストローク |                   |                   | +–––––+––––––––––––––––-+–––––––––-+–––––––––-+

  * 移動: `M`, `m`

    現在の位置を (`x`, `y`) に移動します。新しい位置は絶対的 (`M`) または現在の位置に対して相対的 (`m`) です。 :code:`path` の初期位置は (0, 0) です。

    例:

    > > > path([0.5, -0.1], [0.2, 0.1], "Mm")

```
この例の最初の移動コマンドは、現在の位置を絶対座標 (0.5, 0.2) に移動します。2番目の移動は、現在の位置に対して (-0.1, 0.1) の移動を行い、結果として点 (0.4, 0.3) になります。
```

  * 線: `L`, `l`

    現在の位置から指定された位置 (`x`, `y`) までの線を描画します。線の終点は絶対的 (`L`) または現在の位置に対して相対的 (`l`) です。現在の位置は線の終点に設定されます。

    例:

    > > > path([0.1, 0.5, 0.0], [0.1, 0.1, 0.2], "MLlS")

```
最初の線コマンドは、現在の位置 (0.1, 0.1) から絶対位置 (0.5, 0.1) までの直線を描画し、水平線を生成します。2番目の線コマンドは、現在の位置に対して相対的な垂直線を描画し、終点 (0.5, 0.3) になります。
```

  * 二次ベジェ曲線: `Q`, `q`

    現在の位置から終点 (`x2`, `y2`) までの二次ベジェ曲線を描画し、 (`x1`, `y1`) を制御点として使用します。両方の点は絶対的 (`Q`) または現在の位置に対して相対的 (`q`) です。現在の位置はベジェ曲線の終点に設定されます。

    例:

    > > > path([0.1, 0.3, 0.5, 0.2, 0.4], [0.1, 0.2, 0.1, 0.1, 0.0], "MQqS")

```
この例では、開始点と終了点がそれぞれ y=0.1 に位置する二つのベジェ曲線が生成されます。制御点はそれぞれのベジェ曲線の中間に水平に配置されており、y値が高いため、両方の曲線は対称的であり、中央でわずかに上に曲がります。現在の位置は終了時に (0.9, 0.1) に設定されます。
```

  * 三次ベジェ曲線: `C`, `c`

    現在の位置から終点 (`x3`, `y3`) までの三次ベジェ曲線を描画し、 (`x1`, `y1`) と (`x2`, `y2`) を制御点として使用します。すべての点は絶対的 (`C`) または現在の位置に対して相対的 (`c`) です。現在の位置はベジェ曲線の終点に設定されます。

    例:

    > > > path(

```
...     [0.1, 0.2, 0.3, 0.4, 0.1, 0.2, 0.3],  ...     [0.1, 0.2, 0.0, 0.1, 0.1, -0.1, 0.0],  ...     "MCcS"  ... )

この例では、開始点と終了点がそれぞれ y=0.1 に位置する二つのベジェ曲線が生成されます。制御点は x 軸に沿って均等に配置されており、最初の制御点は開始点と終了点の上に、2番目の制御点は下にあるため、両方のベジェ曲線は波のような形状を作ります。現在の位置は終了時に (0.8, 0.1) に設定されます。
```

  * 楕円弧: `A`, `a`

    現在の位置から始まる楕円弧を描画します。楕円の主軸は x 軸に沿って整列し、従軸はプロットの y 軸に沿って整列します。`rx` と `ry` は主軸と従軸に沿った楕円の半径です。`a1` と `a2` は弧の開始角度と終了角度をラジアンで定義します。現在の位置は弧の終点に設定されます。`a2` が `a1` より大きい場合、弧は反時計回りに描画され、それ以外の場合は時計回りに描画されます。`a` と `A` コマンドは同じ弧を描画します。`x` と `y` 配列の3番目の座標は無視され、将来の使用のために予約されています。

    例:

> > > path([0.1, 0.2, -3.14159 / 2, 0.0], [0.1, 0.4, 3.14159 / 2, 0.0], "MAS")


この例では、(0.1, 0.1) から始まる弧が描画されます。開始角度 -pi/2 が終了角度 pi/2 より小さいため、弧は反時計回りに描画されます。この場合、x 半径が 0.2 で y 半径が 0.4 の楕円の右半分が表示されます。したがって、現在の位置は終了時に (0.1, 0.9) に設定されます。

> > > path([0.1, 0.2, 3.14159 / 2, 0.0], [0.9, 0.4, -3.14159 / 2, 0.0], "MAS")


この例では、前の例と同じ弧が描画されます。唯一の違いは、開始点が (0.1, 0.9) にあり、開始角度 pi/2 が終了角度 -pi/2 より大きいため、楕円の弧が時計回りに描画されます。したがって、現在の位置は終了時に (0.1, 0.1) に設定されます。

  * パスを閉じる: `Z`

    現在の位置を最後の移動コマンド (`m` または `M`) のターゲット位置に直線で接続することによって、現在のパスを閉じます。このパスで移動が行われていない場合、現在の位置は (0, 0) に接続されます。パスがストロークされると、この線も描画されます。
  * パスをストローク: `S`, `s`

    現在の境界幅と境界色でパスをストロークします（:code:`gr.setborderwidth` と :code:`gr.setbordercolorind` で設定）。`s` の場合、パスは事前に閉じられ、これは `ZS` と同等です。
  * パスを塗りつぶす: `F`, `f`

    現在の塗りつぶし色を使用して、偶数-奇数ルールに従って現在のパスを塗りつぶします。パスを塗りつぶすことは暗黙的にパスを閉じます。塗りつぶし色は :code:`gr.setfillcolorind` を使用して設定できます。`F` の場合、パスはその後、現在の境界幅と色を使用してストロークされます。
