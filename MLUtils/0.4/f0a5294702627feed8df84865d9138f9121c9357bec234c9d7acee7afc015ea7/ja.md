```
oversample([rng], data, classes; fraction=1, shuffle=true)
oversample([rng], data::Tuple; fraction=1, shuffle=true)
```

`data`の再バランスされたバージョンを生成します。これは、すべてのクラスが`classes`の中で最も大きなクラスの観測数の少なくとも`fraction`倍を持つように、既存の観測を繰り返しサンプリングすることによって行われます。この方法により、すべてのクラスは、与えられた（元の）`data`における最も大きなクラスが持つ観測数に対して、結果のデータセット内での最小観測数を持つことになります。

例えば、デフォルト（すなわち`fraction = 1`の場合）では、結果のデータセットはほぼ完璧にバランスが取れます。一方、`fraction = 0.5`の場合、結果のデータ内のすべてのクラスは、最も大きなクラスの観測数の少なくとも50%を持つことになります。

`classes`の入力は、`numobs(data)`と同じ長さの配列です。

便利なパラメータ`shuffle`は、結果のデータが作成後にシャッフルされるかどうかを決定します。シャッフルされない場合、すべての繰り返しサンプルはクラスによってソートされ、最後に一緒に配置されます。デフォルトは`true`です。

乱数生成器`rng`は、最初の引数としてオプションで渡すことができます。

出力には、再サンプリングされたデータとクラスの両方が含まれます。

```julia
# 6つの観測とそれぞれ3つの特徴
X = rand(3, 6)
# 2つのクラス、非常に不均衡
Y = ["a", "b", "b", "b", "b", "a"]

# クラス"a"をクラス"b"に合わせてオーバーサンプリング
X_bal, Y_bal = oversample(X, Y)

# これにより、繰り返しデータを含む大きなデータセットが生成されます
@assert size(X_bal) == (3,8)
@assert length(Y_bal) == 8

# 現在、"a"と"b"の両方に4つの観測があります
@assert sum(Y_bal .== "a") == 4
@assert sum(Y_bal .== "b") == 4
```

この関数が機能するためには、`data`の型が[`numobs`](@ref)および[`getobs`](@ref)を実装している必要があります。

`data`がタプルであり、`classes`が指定されていない場合、タプルの最後の要素がクラスを含むと見なされます。
