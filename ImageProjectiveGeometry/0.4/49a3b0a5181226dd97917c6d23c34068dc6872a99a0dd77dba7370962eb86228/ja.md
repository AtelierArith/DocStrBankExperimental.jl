matchbycorrelation - 画像の特徴点を相関によってマッチングする

この関数は、2つの画像内で以前に検出された特徴点の間に仮のマッチを生成します。各点の周囲のウィンドウ内で、互いに最大相関を持つ点を探します。相互に最も強く相関する点のみが返されます。

これは、正規化された相互相関を最も基本的な形で実装しています。画像間のスケールや方向の違いに対処する試みはなく、したがって、画像があまりにも異ならない場合にのみ機能します。

これは単純なN^2比較です。

```
Usage: (m1, m2, p1ind, p2ind, cormat) =
                matchbycorrelation(img1, p1, img2, p2, w, dmax)

Arguments:
      img1, img2 - マッチさせたい点を含む画像。
        p1, p2   - img1およびimg2でコーナー検出器（例えばHarrisまたはphasecong3）を使用して検出された特徴点の座標。p1とp2は[2 x npts]配列ですが、p1とp2が同じ数の点を持つことは期待されていません。p1とp2の最初の行は各特徴点の行座標を示し、2行目は各点の列を示します。
        w        - 各特徴点の周囲で相関が行われるウィンドウサイズ（ピクセル単位）。これは奇数であるべきです。
        dmax     - （オプション）マッチングポイントの最大検索半径。画像間にあまり差異がない場合に速度を向上させるために使用されます。画像サイズの1/4という寛大な値に設定するだけでも有用な速度向上が得られます。このパラメータが省略された場合、デフォルトはInfになります。

Returns:
        m1, m2   - p1およびp2から選択された点の座標で、(仮に) m1[:,i]がm2[:,i]にマッチします。m1とm2は、各画像内の点を[行;列]の形式で定義する[2 x npts]配列です。
  p1ind, p2ind   - マッチを形成するp1およびp2内の点のインデックス。したがって、m1 = p1[:,p1ind]およびm2 = p2[:,p2ind]です。
        cormat   - 相関行列; 行はp1内の点に対応し、列はp2内の点に対応します。
```

この関数は非常に遅いです！注意が必要です。
