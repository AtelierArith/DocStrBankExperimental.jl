```
InfoNode{S, T}
InfoLeaf{T}
```

これらの型は、現在 `DecisionTree` 構造に存在しない追加情報、すなわち特徴名とクラスラベルを視覚化に使用できるように導入されています。この追加情報は、これらの型の属性 `info` に格納されます。これは `NamedTuple` です。したがって、前述の2つのポイントとは別に任意の情報を格納するために使用できます。

`DecisionTree` の型定義に類似して、汎用型 `S` はノード内で子ノード間の分割のしきい値として使用される特徴値の型であり、`T` は与えられたクラスの型です（これらはIDまたはラベルである可能性があります）。

```
!!! note
不足しているクラスラベルのみを追加できます。このメカニズムで既存のラベルを上書きすることはできません。クラスラベルを追加したい場合、汎用型 `T` は `Integer` のサブタイプでなければなりません。
```
