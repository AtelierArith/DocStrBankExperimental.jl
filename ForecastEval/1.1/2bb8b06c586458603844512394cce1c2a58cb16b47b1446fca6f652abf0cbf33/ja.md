```
mcs(l::Matrix{<:Number}, method ; kwargs...)
mcs(l::Matrix{<:Number} ; kwargs...)
```

この関数は、Hansen, Lunde, Nason (2011) の「The Model Confidence Set」、Econometrica, 79 (2), pp. 453-497 で提案された MCS テストを実装しています。

x_k (k = 1, ..., K) を K 個の予測（K の異なる予測モデルからの）とし、y を予測ターゲットとします。L(., .) を損失関数とします。mcs の最初の引数は、行列の k 番目の列が次の操作によって作成される行列です：

L(x_k, y)

Reality Check や SPA テストとは異なり、MCS には基準ケースがありません。

2 番目のメソッド引数は、使用する方法論を決定します。現在、MCSBoot のみが利用可能であり、この入力タイプが提供される場合、キーワード引数は必要ありません。あるいは、ユーザーは 2 番目の引数を省略でき、その場合は任意のキーワード引数が MCSBoot コンストラクタに渡されます。詳細については ?MCSBoot を参照してください。最も関連性の高いキーワード引数は次のとおりです：     alpha=0.05 <- テストで使用する信頼水準

```
basecaseindex=1 <- MCS には自然な基準ケースがありません。ただし、入力データは損失の行列であり、損失の差分ではなく、メソッド自体はすべての可能な損失差分の組み合わせを考慮します。これは、ブートストラップの最適ブロック長を決定するためにチェックするにはあまりにも多すぎるため、適切なブロック長を自動推定したいユーザーは、基準ケースに対するすべての損失差分の最適ブロック長を計算する目的で、仮想的な基準ケース変数インデックスを指定できます。
blocklength=0.0 <- ブートストラップで使用するブロック長。デフォルトの 0.0 は、データから最適にブロック長が推定されることを意味し、DependentBootstrap パッケージによって最も適切と見なされる方法（通常は Patton, Politis, and White (2009) の選択手順）を使用します。

numresample=1000 <- ブートストラップ時に使用する再サンプリングの数。

bootmethod=:stationary <- 使用するブートストラップ手法で、デフォルトは Politis と Romano (1993) の定常ブートストラップです。
```

ブートストラップオプションの詳細については、DependentBootstrap パッケージのドキュメントを参照してください。

MCS テストの出力は MCSTest 型です。詳細については ?MCSTest を参照してください。

RAM 制限に達している場合は、REPL で ?ForecastEval.MCSBootLowRAM と入力して、利用可能な代替アルゴリズムの詳細を確認してください。

開発者の方へ、主なアルゴリズム（MCSBoot に関連する）は、以下の潜在的なパフォーマンスの問題をまだ抱えています：

```
ISSUE 1: ループ内の一部の一時配列はおそらく削除できる

ISSUE 2: MCS メソッド A について、累積 p 値が method.alpha より大きくなった時点で K のループを終了できると思います。これを再確認する必要があります。

ISSUE 3: max(abs) メソッドまたは sum(sq) メソッド（または両方）のみを実行するオプションを追加する必要があります。
```

これらの問題に関するコメントやプルリクエストは、パッケージの GitHub ページで大歓迎です。
