```
rc(lD::Matrix{T}, method ; kwargs)
rc(lD::Matrix{T} ; kwargs)
```

この関数は、Hansen (2005) の方法論に従い、White (2000) の「データスヌーピングの現実チェック」で提案されたテストを実装します。

x*0 をベースケースの予測、x*k (k = 1, ..., K) を K の代替予測、y を予測ターゲットとします。L(., .) を損失関数とします。rc の最初の引数は、行列であり、行列の k 番目の列は次の操作によって作成されます：

L(x*k, y) - L(x*0, y)

予測損失が最初に来て、ベースケースの損失が二番目に来ることに注意してください。これは、White の論文で説明されていることとは逆です。

2 番目のメソッド引数は、使用する方法論を決定します。現在、RCBoot のみが利用可能であり、この入力タイプが提供される場合、キーワード引数は必要ありません。あるいは、ユーザーは 2 番目の引数を省略することができ、その場合は任意のキーワード引数が RCBoot コンストラクタに渡されます。詳細については ?RCBoot を参照してください。最も関連性の高いキーワード引数は次のとおりです：     alpha=0.05 <- テストで使用する信頼水準

```
blocklength=0.0 <- ブートストラップで使用するブロック長。0.0 のデフォルトは
    ブロック長がデータから最適に推定されることを意味し、DependentBootstrap パッケージによって
    最も適切と見なされる方法が使用されます（通常は Patton, Politis, and White (2009) の選択手順）。

numresample=1000 <- ブートストラップ時に使用するリサンプルの数。

bootmethod=:stationary <- 使用するブートストラップ方法論で、デフォルトは
    Politis and Romano (1993) の定常ブートストラップです。
```

現実チェックテストの出力は RCTest 型です。詳細については ?RCTest を参照してください。
