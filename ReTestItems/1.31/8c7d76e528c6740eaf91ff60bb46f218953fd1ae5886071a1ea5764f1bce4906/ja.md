```
@testsetup module MyTestSetup
    # @testitem間で共有できるコード
end
```

テスト専用のモジュールであり、`@testitem`が依存することができます。

複数のテスト項目で使用されるセットアップロジックに便利です。セットアップは、必要とする`@testitem`が実行される前に一度実行されます。複数のプロセスで実行する場合、各テストセットアップは各プロセスで一度実行されます。

各テストセットアップモジュールは、テストのライフタイムの間存続します。可変状態は避けるべきです。なぜなら、テスト項目が実行される順序は非決定的であり、テスト項目が同じプロセス内で同時に同じテストセットアップモジュールにアクセスする可能性があるからです。

`@testsetup`マクロで宣言される以外は、`@testitem`に認識されるようにするために、テストセットアップモジュールは他のモジュールと同様であり、依存関係をインポートしたり、名前をエクスポートしたりできます。

`@testitem`は、`setup`キーワードを介して`@testsetup`に依存します。例えば：

```
@testitem "MyTests1" setup=[MyTestSetup]
    # MyTestSetupを必要とするテスト
end
```
