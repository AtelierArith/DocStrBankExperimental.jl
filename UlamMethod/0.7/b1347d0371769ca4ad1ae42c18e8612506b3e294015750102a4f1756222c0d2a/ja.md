```
ulam_method(traj, binner; reinj_algo)
```

主なウラム法の計算を実行し、[`UlamResult`](@ref)を返します。

これは、すべてのオプションが利用可能な低レベルのメソッドです。

### 引数

  * `traj`: 短距離の軌道データを保持する[`Trajectories`](@ref)オブジェクト。
  * `binner`: 境界をビンに分割するために使用されるアルゴリズムを指定する[`BinningAlgorithm`](@ref)。

### オプション引数

  * `reinj_algo`: ニルヴァーナから内部に向かう軌道をどのように再注入するかを指定する[`ReinjectionAlgorithm`](@ref)。デフォルトは[`DataReinjection`](@ref)。

---

```
ulam_method(traj, nbins; nirvana = 0.10)
```

主なウラム法の計算を実行し、[`UlamResult`](@ref)を返します。

これは、ほとんどのオプションが自動的に選択される高レベルの「便利」メソッドです。

### ニルヴァーナオプション引数

境界は、データの割合`nirvana`がニルヴァーナにあるように配置されます。たとえば、デフォルト値の`0.10`では、すべてのデータポイントの約`10%`（`x0`と`xT`の間で分割される）が境界の外にあり、両側にほぼ等しい量があります。

境界の形状は、`nirvana`を長さ`Dim`のタプルのベクトルとして提供することでさらに制御できます。タプルの形式は`(min, max)`であり、各次元に沿って割合`min`（それぞれ`max`）が境界の「下」に（それぞれ「上」に）あることになります。
