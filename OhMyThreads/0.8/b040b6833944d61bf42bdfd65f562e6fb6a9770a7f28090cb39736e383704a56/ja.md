```
@set name = value
```

これは、ループの並列実行の設定を指定するために `@tasks for ... end` ブロック内で使用できます。

複数の設定がサポートされており、個別の `@set` ステートメントまたは `@set begin ... end` を介して指定できます。

## 設定

  * `reducer` (例: `reducer=+`): 提供された二項関数を使用して還元を行うことを示します。詳細については [`tmapreduce`](@ref) を参照してください。
  * `collect` (例: `collect=true`): 結果を収集する必要があることを示します（`map` に似ています）。

他のすべての設定は、基盤となる並列関数（例: [tmapreduce](@ref)）にキーワード引数として渡されます。したがって、これらの関数が受け入れるキーワード引数を提供できます。他にも、以下が含まれます。

  * `scheduler` (例: `scheduler=:static`): [`Scheduler`](@ref) または `Symbol`（例: `:dynamic`, `:static`, `:serial`, または `:greedy`）のいずれかです。
  * `init` (例: `init=0.0`): 還元に使用される初期値（`reducer=...` が必要です）。

`ntasks`、`chunksize`、`split` などの設定を使用して、スケジューリングポリシーを調整できます（選択したスケジューラがそれをサポートしている場合）。

代入はループ本体の上に持ち上げられるため、スコープはループのスコープではなく（見た目はそうですが）、ループ本体を*囲む*スコープです。(`@macroexpand` は `@tasks` ブロックの生成されたコードを検査するのに便利なツールです。)
