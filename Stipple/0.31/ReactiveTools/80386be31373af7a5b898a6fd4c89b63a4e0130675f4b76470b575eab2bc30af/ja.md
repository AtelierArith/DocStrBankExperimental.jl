```julia
@app(expr)
```

式 `expr` で提供されるリアクティブ変数とコードを設定し、有効にします。

**使用法**

@app に渡されたコードブロックは、アプリのロジックを実装し、UI コンポーネントの状態を処理し、これらの状態が変更されたときに実行されるコードを扱います。

```julia
@app begin
   # リアクティブ変数
   @in N = 0
   @out result = 0

   # N が変更されたときに実行されるリアクティブコード
   @onchange N begin
     result = 10 * N
   end
   
   # 明示的な優先度を持つリアクティブコード (デフォルト: 0)
   @onchange N begin
     # 感嘆符は、変数が静かに更新されることを示すために使用されます。
     # すなわち、無限ループを避けるために新しい変更イベントをトリガーせずに更新されます。
     N[!] = clamp(N, 0, 10)
   end priority = 1

   # `@app` ブロックの外で定義された関数を呼び出すリアクティブコード
   @onchange N on_N()
end

# `@app` ブロックの外で定義されたコールバック関数:
# この定義は `@app` ブロックの後に配置する必要があります。
# この構文は、長いコードを含めるのに便利で、別のファイルに移動することもできます。
# また、この関数はアプリを再読み込みすることなく更新できるため、デバッグやテストにも非常に便利です。
@handler function on_N()
  @info "on_N() が呼ばれました"
end
```

優先度を設定することは、値がクライアントに送信される前にコードを実行するのに役立ちます。たとえば、高価な計算をトリガーする前に範囲チェックを行うためです。上記の例では、結果が計算される前に `N` の値が範囲 [0, 10] に制限されます。

任意のハンドラー内で `Consume(true)` を返すことで、ハンドラーキューを中止できます。その場合、すべての保留中のハンドラーはスキップされます。優先度が > 0 に設定されている場合、クライアントは更新されません。
