```julia
marching_tetrahedra(
    coord,
    cellnodes,
    func,
    planes,
    flevels;
    tol,
    primepoints,
    primevalues,
    Tv,
    Tp,
    Tf
)

```

3D テトラヘドロンメッシュ上の関数の等値面と平面補間を抽出します。

基本的な観察は、テトラヘドロン上で局所的に、平面とのカットと P1 関数の等値面が同じように見えるということです。このメソッドは、複数の平面カットと複数の等値面のデータを一度に計算します。

入力パラメータ:

  * `coord`: 点の座標の 3 x n_points 行列
  * `cellnodes`: テトラヘドロンごとの点番号の 4 x n_cells 行列
  * `func`: 区分線形関数値の n_points ベクトル
  * `planes`: 平面方程式 `ax+by+cz+d=0` のベクトル、各々はベクトル [a,b,c,d] として格納
  * `flevels`: 関数の等値レベルのベクトル

キーワード引数:

  * `tol`: テトラヘドロン x 平面交差の許容誤差
  * `primepoints`: プロットされる領域の「コーナーポイント」の 3 x n_prime 行列。これらはメッシュには含まれていませんが、例えば Makie によって軸サイズを計算するために使用されます
  * `primevalues`: コーナーポイントにおける関数値の n_prime ベクトル。これらは例えば Makie によって関数の制限を計算するために使用できます
  * `Tv`: 戻り値の関数値の型
  * `Tp`: 戻り値の点の型
  * `Tf`: 戻り値のファセットの型

戻り値: (points, tris, values)

  * `points`: 点のベクトル (Tp)
  * `tris`: 三角形のベクトル (Tf)
  * `values`: 関数値のベクトル (Tv)

これらは、関数値が付加されたメッシュに容易に変換できます。

注意: 同様の座標を持つ点は識別されません。例えば、平面とエッジの交差は、そのエッジに隣接するテトラヘドロンの数だけエッジ交差点を生成します。その結果、視覚化のための法線計算は常にファセット法線に終わり、レンダリングされた等値面の視覚的印象はその区分線形系譜を示します。
