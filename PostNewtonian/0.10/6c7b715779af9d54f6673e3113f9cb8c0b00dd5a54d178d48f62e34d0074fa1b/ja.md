```
coorbital_waveform(inspiral; [ℓₘᵢₙ=2], [ℓₘₐₓ=8], [PNOrder])
coorbital_waveform(inspiral; [ell_min=2], [ell_max=8], [PNOrder])
```

与えられた `inspiral` 出力によって [`orbital_evolution`](@ref) で、共軌道フレームにおけるポストニュートン波形モード重みを評価します。

慣性フレームにおける波形については、[`inertial_waveform`](@ref) を参照してください。

返される量の形式と物理的解釈の詳細については、[マニュアルの「波形」セクション](https://moble.github.io/PostNewtonian.jl/stable/internals/waveforms.html) を参照してください。

`PNOrder` は `inspiral` を計算するために使用されたものがデフォルトですが、キーワード引数を渡すことで変更できます。

!!! tip
    `inspiral.t` に現在存在するのとは異なる時刻 `t′` でこの波形が必要な場合は、まず `inspiral` の組み込み補間機能を使用するべきです。例えば `inspiral′ = inspiral(t′)` のように。この関数の結果を補間するのではなく、または、より良い方法として、`orbital_evolution` を呼び出す際に `saves_per_orbit` または `saveat` キーワード引数を使用して時刻を選択することができます。これらのアプローチは、この関数の結果を補間するよりも正確で、速く、メモリを少なく消費します。`saves_per_orbit` を使用する場合は、*少なくとも* `2ℓₘₐₓ`、または好ましくは `4ℓₘₐₓ` に設定することをお勧めします。


モード重みは `ℓₘᵢₙ` から始まり（`0 ≤ ℓₘᵢₙ ≤ 2` を満たす必要があります）、`ℓₘₐₓ` まで拡張されます。波形は 2 次元の `Array` として返され、最初のインデックスはモードインデックス `(ℓ, m) = (ℓₘᵢₙ, -ℓₘᵢₙ)` から `(ℓ, m) = (ℓₘₐₓ, ℓₘₐₓ)` まで変化し、`m` が最も速く変化し、2 番目のインデックスは時間ステップにわたって変化します。

この関数では、波形は共軌道フレームで返されます — これは共回転フレームに似ています。特に、非前進系のモードは、インスパイラルの時間スケールでゆっくりと変化します。前進系のモードは、軌道の時間スケールで変化しますが、この変化も除外することができます。
