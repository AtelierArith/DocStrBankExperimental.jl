```
lock_mi_inference(::AbstractInterpreter, mi::MethodInstance)
```

`mi` が推論中であることを示唆しており、ブートストラップを加速するのに役立ちます。これは特に `NativeInterpreter` によって使用され、推論が最初に自分自身を推論しているときに無駄な作業の量を制限するのに役立ちます。これにより、推論がすでに進行中であることを検出し、それに対して二重に実行しないようにします。これによりデータ競合が発生しますが、このコードへの C からのエントリポイント（`jl_type_infer`）にはすでに再帰の検出と制限が含まれているため、これは主に無害な問題であることが期待されます。なぜなら、このフラグに遭遇するのはブートストラップの最初のフェーズ中だけであるべきだからです。
