```
struct LimitedAccuracy
```

`LimitedAccuracy` ラティス要素は、再帰のヒューリスティックな終了により真の推論結果が近似的であったことを示すために使用されます。例えば、`A` と `B` から始まる2つのコールスタックは次のようになります：

```
A -> C -> A -> D
B -> C -> A -> D
```

最初のケースでは、推論は `A->C->A` がサイクルを構成すると判断し、`C` に対して得られた結果を広げるかもしれません。たとえそれが他の方法で結果を得ることができたかもしれないとしてもです。この場合、`C` に対して推論された結果は、このラティスタイプで注釈され、得られた結果が制限のない推論の上限であることを示します。特に、`B` から始まるコールスタックは、`A` の推論中に得られた潜在的に不正確な結果に影響されることなく、再度推論を行います。

注意：私たちは逆のことを保証するために何の努力もしていません。例えば、`B` が最初に推論される場合、`C` に対して正確な結果をキャッシュし、この結果を `A` の推論中に再利用するかもしれません。たとえ `A` の推論が制限によりこの結果を得ることができなかったとしてもです。これは望ましくありません。なぜなら、いくつかの推論結果が順序に依存することになり、この状況をどのように回避できるかは不明だからです。

`LimitedAccuracy` 要素は別のラティス要素（これを `T` と呼びましょう）をラップし、制限が発生した原因である `causes` を追加で追跡します。ラティス要素として、`LimitedAccuracy(T)` は対応するラティス要素 `T` よりも ε 小さいと見なされますが、特に `⊑ T` であるすべてのラティス要素（ただし `T` と等しくないもの）も `⊑ LimitedAccuracy(T)` です。

`causes` リストは、特定の制限の原因が避けられないかどうかを判断するために使用され、もしそうであれば、`LimitedAccuracy(T)` を `T` に戻すために広げます。例えば、上記のコールスタックでは、`A` へのいかなる呼び出しも常に `A` に戻る場合、`A` から始めるか `B` を経由して到達するかは重要ではありません：`A` に到達する任意の推論は常に同じ制限に直面し、その結果はキャッシュされる可能性があります。
