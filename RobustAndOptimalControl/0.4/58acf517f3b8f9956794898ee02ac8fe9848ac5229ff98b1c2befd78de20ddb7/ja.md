```
K, γ, mats = hinfsynthesize(P::ExtendedStateSpace; gtol = 1e-4, interval = (0, 20), verbose = false, tolerance = 1.0e-10, γrel = 1.01, transform = true, ftype = Float64, check = true)
```

拡張植物 `P` に対して、$||F_l(P, K)||∞ < γ$(`lft(P, K)`) となるような H-infinity 最適コントローラ `K` を計算します。これは、`P` に対して最小の γ を求めるもので、γ-反復として知られています。この手法は、Glover と Doyle による論文「H∞-ノルム制約を満たすすべての安定化コントローラの状態空間公式とリスク感度との関係」に基づいています。

# 引数:

  * `gtol`: γ の許容誤差。
  * `interval`: 二分法の開始区間。
  * `verbose`: 進行状況を表示しますか？
  * `tolerance`: 虚数軸上の固有値を検出するための許容誤差。
  * `γrel`: `γrel > 1` の場合、最適な γ は γ 反復によって求められ、その後 `γ = γopt * γrel` のためのコントローラが設計されます。数値的な理由から、また最適コントローラが非常に速いダイナミクスを含む可能性があるため、わずかにサブオプティマルなコントローラを設計することはしばしば良い考えです。`γrel → ∞` の場合、計算されたコントローラは 𝑯₂ 最適コントローラに近づきます。𝑯∞ と 𝑯₂ の特性の混合を得ることも、`γrel > 1` を選択する理由の一つです。
  * `transform`: より広い範囲や問題仕様に耐えるために座標変換を適用します。
  * `ftype`: 数値的な堅牢性を高めるために、より高い精度で問題行列を構築します。計算されたコントローラが達成する場合
  * `check`: 計算されたコントローラによって達成された γ 値の設計後チェックを行います。達成された γ が設計中に計算された γ と異なる場合、警告が発行されます。この警告が発行された場合は、`ftype = BigFloat` のような高精度の数値型を使用することを検討してください。

使用例については、例のフォルダを参照してください。
