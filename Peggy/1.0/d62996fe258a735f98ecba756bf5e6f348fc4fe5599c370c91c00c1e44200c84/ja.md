```
peggy(expr...; whitespace=r"[[:space:]]*")
```

PEG式から`Parser`を作成します。

パーサーは各exprを順次マッチさせ、結合された結果を返します（詳細は以下）。

各exprは次のいずれかです。

  * `String` - 文字列リテラルをマッチさせて返します
  * `Regex` - `Regex`をマッチさせてマッチ値を返します（ただし、これを避けてください）
  * `Symbol` - [`grammar`](@ref)内のシンボルに関連付けられた式にマッチします。
  * `Symbol => expr` - `expr`にマッチし、それに名前を付けます。
  * `expr => callable` - `expr`にマッチし、その値に`callable`を適用した結果を返します。
  * `peg_epxr => k` - `expr` => _ -> k`の短縮形
  * `(expr, exprs...)` - `peggy(expr, exprs...)`と同じです
  * `[expr, exprs...]` - `many(expr, exprs; max=1)`と同じです
  * `Parser` - パーサーを生成する任意の式

# 名前とシーケンス結果

シーケンスの各要素には名前があります。`Symbol`および`Symbol => expr`はシンボルの名前を取ります。他のすべての式は":_"という名前が付けられます。シーケンスの値は次のように形成されます。

名前が"*"で始まる値は、名前が"*"で始まらない値がある場合は破棄されます。単一の値が残る場合、それがシーケンスの値です。そうでない場合、残りの値の`Vector`になります。

# ホワイトスペース

文字列リテラルはデフォルトで周囲のホワイトスペースを無視します。これを無効にするには、オプション`whitespace=r""`を使用します。
