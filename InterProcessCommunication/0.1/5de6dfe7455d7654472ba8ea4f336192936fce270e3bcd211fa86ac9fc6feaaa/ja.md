```
shm = SharedMemory(id, len; perms=0o600, volatile=true)
```

は、`id` で識別され、サイズが `len` バイトの新しい共有メモリオブジェクトを作成します。識別子 `id` は、POSIX 共有メモリオブジェクトを作成するために `'/'` で始まる文字列、または System V 共有メモリセグメントを作成するための System V IPC キーであることができます。この後者の場合、キーは `IPC.PRIVATE` に設定することで、存在しない共有メモリセグメントを自動的に作成できます。

キーワード `perms` は、付与されるアクセス権限を指定するためのものです。デフォルトでは、ユーザーによる読み取りと書き込みのみが許可されます。

キーワード `volatile` は、共有メモリが揮発性かどうかを指定します。非揮発性の場合、共有メモリは明示的な破棄またはシステム再起動までアクセス可能なままです。揮発性の場合（デフォルト）、共有メモリは、どのプロセスからも使用されなくなったときに自動的に破棄されます。

既存の共有メモリオブジェクトを取得するには、次のように呼び出します：

```
shm = SharedMemory(id; readonly=false)
```

ここで `id` は共有メモリ識別子（文字列、IPC キー、または [`ShmId`](@ref) によって返される共有メモリセグメントの System V IPC 識別子）です。キーワード `readonly` は、読み取り専用アクセスのみが必要な場合に true に設定できます。メソッド `shmid(obj)` を呼び出すことで、共有メモリオブジェクト `obj` の識別子を取得できることに注意してください。

共有メモリオブジェクト `shm` に対して利用可能なアクセサがいくつかあります：

```
pointer(shm)    # 共有メモリの基底アドレス
sizeof(shm)     # 共有メモリのバイト数
shmid(shm)      # 共有メモリの識別子
```

共有メモリオブジェクト `shm` が最終的に破棄されることを保証するには、次のように呼び出します：

```
rm(shm)
```

さらに [`ShmId`](@ref)、[`shmid`](@ref)、および [`shmrm`](@ref) を参照してください。
