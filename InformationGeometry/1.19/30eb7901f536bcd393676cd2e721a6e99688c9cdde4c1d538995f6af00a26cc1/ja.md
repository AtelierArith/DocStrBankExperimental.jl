`CompositeDataSet`型は、データを格納するためのより複雑（通常はパフォーマンスが低下する）なコンテナです。本質的には、複数の`y`コンポーネントを持つ観測データを、各`y`データのコンポーネントの1つに対応する別々のデータコンテナ（例えば`DataSet`型）に分割します。重要なことに、各小さなデータコンテナは、同じ「種類」の`x`データ、つまり同じ`xdim`、単位などを共有しますが、正確に同じ特定の`x`データを共有する必要はありません。

このアプローチの主な利点は、いくつかの観測に`missing`な`y`コンポーネントがある場合に適用できることです。`CompositeDataSet`の典型的な使用例は、複数の量が追跡されるが、すべての量が必ずしも各時間ステップで記録されるわけではない時系列です。例：

```julia
using DataFrames
t = [1,2,3,4]
y₁ = [2.5, 6, missing, 9];      y₂ = [missing, 5, 3.1, 1.4]
σ₁ = 0.3*ones(4);               σ₂ = [missing, 0.2, 0.1, 0.5]
df = DataFrame([t y₁ σ₁ y₂ σ₂], :auto)

xdim = 1;   ydim = 2
CompositeDataSet(df, xdim, ydim; xerrs=false, stripedYs=true)
```

ブール値のキーワード`stripedXs`と`stripedYs`は、コンストラクタに対して、値と対応する$1\sigma$不確実性が交互の順序で与えられているか、最初のブロックが`ydim`の多くの列が値であり、2番目の`ydim`の多くの列が対応する不確実性であるかを示すために使用できます。また、`xerrs=true`を使用して、`x`値も不確実性を持つことを示すことができます。基本的に、`DataSet`のような他のデータコンテナに対して呼び出すことができるすべての関数は、`CompositeDataSet`でも動作するように特化されています。
