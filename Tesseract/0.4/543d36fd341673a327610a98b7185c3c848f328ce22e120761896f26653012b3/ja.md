```
pix_write_jp2k(
    pix::Pix;
    quality::Integer = Int32(34),
    levels::Integer= Int32(5)
)::Union{Vector{UInt8}, Nothing}
```

JP2K画像形式でバイト配列に画像を書き込みます。エラーが発生した場合は`nothing`が返されます。

**引数:**

| T   | 名前      | デフォルト       | 説明                    |
|:--- |:------- |:----------- |:--------------------- |
| R   | pix     |             | バイト配列に書き込む画像。         |
| O   | quality | `Int32(34)` | 画像をエンコードする際の品質。       |
| O   | levels  | `Int32(5)`  | ファイルにエンコードする低解像度画像の数。 |

**制限:**

  * `quality` - `1`から`100`の範囲内でなければなりません。
  * `levels` - `1`から`10`の値を取ることができます。

**詳細:**

`quality`を`100`に設定すると、ロスレス画像が生成されます。

`levels`が`1`の場合、ファイルにはフル解像度の画像のみがエンコードされます。`2`の値はフル解像度画像と半解像度画像がエンコードされることを意味します。デフォルト値の`5`は、1、2、4、8、16の縮小係数を持つ画像がファイルにエンコードされることを引き起こします。

Leptonicaはレベルの数を10以下に制限しています。しかし、経験的なテストでは、8を超える値はOpenJpegライブラリで失敗することが示されています。
