```
tess_tsv(
    inst::TessInst,
    page::Integer = Int32(1)
)::Union{String, Nothing}
```

認識からタブ区切り値の文字列としてTSV結果を取得します。

**引数:**

| T   | 名前   | デフォルト      | 説明                   |
|:--- |:---- |:---------- |:-------------------- |
| R   | inst |            | 呼び出すTesseractインスタンス。 |
| O   | page | `Int32(1)` | データを取得するページ。         |

**詳細:**

このメソッドは、画像に対してまだ呼び出されていない場合、`tess_recognize()`を呼び出します。

データを読み取る場合は、使いやすい形式に解析する`tess_parsed_tsv()`を確認してください。ユーザーにデータを表示するか、ファイルに書き込むだけであれば、このメソッドで問題ないでしょう。

この呼び出しは基本的に認識プロセスの結果を返します。文字列内の各行は、Tesseractによって見つかった「オブジェクト」を識別します。各行の12の値は次のとおりです：

  * level - 行が何を説明しているかを識別します。
  * page - `tess_tsv()`メソッドに渡されたページ番号。
  * block - ページ上のブロックを識別します。
  * paragraph - ブロック内の段落番号。
  * line - 段落内の行。
  * word - 行内の単語。
  * left - アイテムの左端のピクセル数。
  * top - アイテムの上端のピクセル数。
  * width - アイテムの幅のピクセル数。
  * height - アイテムの高さのピクセル数。
  * conf - OCRエンジンが単語に対してどれだけ自信を持っているか（`0` - `100`）。レベルが5でない場合は`-1`。
  * text - 画像からデコードされた単語。

レベルは、行が提供している情報を識別します：

  * `1` - ページ情報、ページの開始時に追加されます。
  * `2` - ブロック情報、ブロックの開始時に追加されます。
  * `3` - 段落情報、段落の開始時に追加されます。
  * `4` - 行情報、行の開始時に追加されます。
  * `5` - 単語情報、ページから読み取られた単語を識別します。

左、上、幅、高さの値は、アイテムを囲むピクセル単位のボックスを定義します。したがって、レベルが1の場合、ボックスは全体の画像を説明します。レベルが`1`の場合、ボックスは抽出されたブロックを囲み、以下同様に抽出された単語まで続きます。

**例:**

```jldoctest; filter = r"(\s+)"
using Tesseract

download_languages()

instance = TessInst()
pix = sample_pix()

tess_image(instance, pix)
tess_resolution(instance, 72)

tsv = tess_tsv(instance)

for line in split(tsv, '\n'; keepempty = false)[2:6]
    println(strip(line))
end

# 出力

2	1	1	0	0	0	10	9	479	514	-1
3	1	1	1	0	0	11	9	406	14	-1
4	1	1	1	1	0	11	9	406	14	-1
5	1	1	1	1	1	11	9	14	11	95.791931	No
5	1	1	1	1	2	35	12	22	8	95.791931	one
```
