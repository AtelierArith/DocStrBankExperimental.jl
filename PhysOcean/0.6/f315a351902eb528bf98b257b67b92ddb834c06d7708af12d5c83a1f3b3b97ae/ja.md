```
velocity,ssh,fluxes=geostrophy(mask::BitArray,rhop,pmnin,xiin;dim::Integer=ndims(rhop),ssh=(),znomotion=0,fillin=true)
```

# 入力:

  * `mask` : 水中で真、陸上で偽のブール配列。
  * `rhop` : マスクと同じグリッド上の密度異常 (rho-1025) 配列。
  * `pmnin`: divand と同様のメトリックのタプル。ただし、m/s の速度を得るためにはメトリックもメートル単位である必要があります。
  * `xiin`: グリッドポイントの位置のタプル。
  * `dim` : 配列のどのインデックスが垂直方向に対応するかを示すオプションのパラメータ。デフォルトでは最後の次元が使用されます。
  * `ssh` : 垂直方向が除外されるマスクとしての配列。メートル単位の海面高さに対応します。デフォルトでは使用されませんが、診断されます。
  * `znomotion` : 動きのないレベルが仮定される垂直方向のインデックス。
  * `fillin` : 同じレベルの水点を使用して陸点を埋めるかどうかを示すブール値。デフォルトははいです。

# 出力:

  * `velocity` 各座標線の正規および左側の速度成分のタプル。
  * `eta` : 推定された海面高さ。入力に ssh が提供された場合、陸上で埋められた ssh を返します。
  * `fluxes`: 各水平方向のセクションを横断する統合速度。速度と同じ規則が適用されます。

# 注意:

地衡速度を計算します。1つまたは2つの水平方向の次元と追加の（時間）次元で動作します。次元は水平方向、垂直方向、その他の次元の順に並んでいる必要があります。動きのないレベルのインデックスまたは ssh eta を提供する必要があります。レベルはインデックス番号であることに注意してください。ssh の次元は、垂直次元が除外された rhop と同じでなければなりません。fillin=false を強制する場合、この呼び出しの外で欠損値のない密度配列を作成している必要があります。また、提供する場合は ssh も必要です。
