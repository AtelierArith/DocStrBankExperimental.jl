```
ForbiddenSequence <: AbstractGrammarConstraint
```

この [`AbstractGrammarConstraint`] は、与えられた `sequence` のルールノードの列を禁止します。列は厳密に垂直であり、ギャップを含むことがあります。木 `1{a, 2{b, 3{c, d}}}` を考えてみましょう：

  * `[2, 3, d]` は列です
  * `[1, 3, d]` は列です
  * `[3, c, d]` は列ではありません。なぜなら c と d は兄弟（水平）だからです。

例：

  * `ForbiddenSequence([3, 4])` は、ルール `4` が `c` または `d` で適用できないことを強制します。
  * `ForbiddenSequence([1, 2, 4])` は、ルール `4` が `b`、`c` または `d` で適用できないことを強制します。
  * `ForbiddenSequence([1, 4])` は、ルール `4` がどこでも適用できないことを強制します。

`ignore_if` に含まれるルールが列に現れる場合、制約は無視されます。禁止された `sequence = [1, 2, 3]` と `ignore_if = [99]` を考えてみましょう。ルートからの次のパスを考えます：

  * `[1, 2, 2, 3]` は禁止されています。なぜなら、列には `99` が含まれていないからです。
  * `[1, 99, 2, 3]` は禁止されていません。なぜなら、列には `99` が含まれているからです。
  * `[1, 99, 1, 2, 3]` は禁止されています。なぜなら、`99` を含まない部分列があるからです。
