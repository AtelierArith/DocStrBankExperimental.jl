```
new_child_link(node)
```

親ノードを削除する際の子ノードの新しいリンクを計算します。ルールは、削除された親ノードの子ノードリンクを子ノードに与えることですが、親が自身の親に従っている場合は除きます。

入力として与えられたノードはここでは子ノードです。

ルールは以下の表に要約されています：

| 削除されたノードリンク | 子ノードリンク | 新しい子ノードリンク |   警告    |
|:-----------:|:-------:|:----------:|:-------:|
|      /      |    /    |     /      |         |
|      /      |    +    |     +      | yes (1) |
|      /      |    <    |     /      |         |
|      +      |    /    |     /      | yes (2) |
|      +      |    +    |     +      |         |
|      +      |    <    |     +      |         |
|      <      |    /    |     /      |         |
|      <      |    +    |     +      |         |
|      <      |    <    |     <      |         |

警告は、新しいリンクを処理する適切な方法がない場合、*すなわち* 分岐とスケールの変更を混合している場合に発生します。

警告のケース (1) では、最初の子は "/" リンクのみを取り、他の子はそのリンクを保持します。
