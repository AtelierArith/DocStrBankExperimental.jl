```
FVMSystem(prob1, prob2, ..., probN)
```

PDEのシステムの表現を構築します。ここで、各 `probi` はシステムの `i` 番目のコンポーネントのための [`FVMProblem`](@ref) です。

これらの [`FVMProblem`](@ref) に対して、条件関数などの関数はすべて定義されている必要があり、`u` 引数は `u = (u₁, u₂, ..., uN)` の形を取ります（`Tuple` と `Vector` の両方が渡されます）。ここで `uᵢ` はシステムの `i` 番目のコンポーネントの解です。フラックス関数については、[`FVMProblem`](@ref) の形は次のようになります。

```
q(x, y, t, α, β, γ, p) ↦ (qx, qy),
```

同じ形が使用されますが、`α`、`β`、`γ` はすべて `Tuple` であり、`α[i]*x + β[i]*y + γ[i]` が `uᵢ` の近似となります。

!!! warning "デフォルトのフラックス関数を提供する"
    フラックス関数のこの違いと、フラックス関数に `α`、`β`、`γ` を提供する必要があるため、`FVMSystem` では拡散関数ではなくフラックス関数を提供する必要があります。拡散関数を提供した場合、問題を解こうとするとエラーが発生します。


この問題は [`FVMProblem`](@ref) と同じ方法で解決されますが、問題は解が各時間で行列を返すように定義されており、`(j, i)` 番目のコンポーネントは `j` 番目のコンポーネントの `i` 番目のノードでの解に対応します。

また、[`FVMProblem`](@ref) および [`SteadyFVMProblem`](@ref) も参照してください。

!!! note
    `FVMSystem` の定常状態問題を構築するには、システムに対して [`SteadyFVMProblem`](@ref) を適用する必要があり、システム内の各個別の [`FVMProblem`](@ref) に最初に適用するのではありません。
