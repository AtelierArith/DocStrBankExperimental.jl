```julia
triangulate(
    switches::String,
    tri_in::Triangulate.TriangulateIO
) -> Tuple{Triangulate.TriangulateIO, Triangulate.TriangulateIO}

```

三角形分割を作成します。出力三角形分割とオプションのボロノイ分割を含むタプル `(out::TriangulateIO, vor_out::TriangulateIO)` を返します。

`triangulate()` を呼び出した後、`out` と `vorout` の有効なフィールドは、使用されるスイッチの選択に明らかに依存します。 'p' スイッチが使用されると、ポインタ `holelist` と `regionlist` は `tri_in` から出力にコピーされますが、新しいスペースは割り当てられません。一方、Triangle は `pointlist` ポインタ（または他のポインタ）を決してコピーしません。 `out.pointlist` のために新しいスペースが割り当てられるか、'N' スイッチが使用される場合、`out.pointlist` は初期化されません。

これは triangle によって使用されるスイッチのリストです：

| スイッチ | 意味                                     |
| ----:|:-------------------------------------- |
|   -p | 平面直線グラフ (.poly ファイル) を三角形分割します。        |
|   -r | 以前に生成されたメッシュを洗練します。                    |
|   -q | 質の高いメッシュ生成。最小角度を指定できます。                |
|   -a | 最大三角形面積制約を適用します。                       |
|   -u | ユーザー定義の三角形制約を適用します。                    |
|   -A | 特定の領域内の三角形を識別するための属性を適用します。            |
|   -c | 凸包をセグメントで囲みます。                         |
|   -D | 準拠デローニー: すべての三角形は真のデローニーです。            |
|   -j | 出力 .node ファイルから未使用の頂点を削除します。           |
|   -e | エッジリストを生成します。                          |
|   -v | ボロノイ図を生成します。                           |
|   -n | 三角形の隣接リストを生成します。                       |
|   -g | Geomview 用の .off ファイルを生成します。           |
|   -B | 境界情報の出力を抑制します。                         |
|   -P | .poly ファイルの出力を抑制します。                   |
|   -N | .node ファイルの出力を抑制します。                   |
|   -E | .ele ファイルの出力を抑制します。                    |
|   -I | メッシュの反復番号の出力を抑制します。                    |
|   -O | .poly ファイルの穴を無視します。                    |
|   -X | 正確な算術の使用を抑制します。                        |
|   -z | すべてのアイテムをゼロから番号付けします（1 ではなく）。          |
|  -o2 | 二次サブパラメトリック要素を生成します。                   |
|   -Y | 境界セグメントの分割を抑制します。                      |
|   -S | 追加のシュタイナー点の最大数を指定します。                  |
|   -i | 分割統治法ではなく、増分法を使用します。                   |
|   -F | 分割統治法ではなく、フォーチュンのスイープラインアルゴリズムを使用します。  |
|   -l | 交互カットではなく、垂直カットのみを使用します。               |
|   -s | CDT を使用するのではなく、分割によってメッシュにセグメントを強制します。 |
|   -C | 最終メッシュの一貫性をチェックします。                    |
|   -Q | 静か: エラーを除いて端末出力なし。                     |
|   -V | 詳細: 私が何をしているかの詳細情報。                    |
