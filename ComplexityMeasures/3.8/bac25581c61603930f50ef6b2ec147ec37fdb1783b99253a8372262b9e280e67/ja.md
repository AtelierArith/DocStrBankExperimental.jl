```
RelativeMeanEncoding <: Encoding
RelativeMeanEncoding(minval::Real, maxval::Real; n = 2)
```

`RelativeMeanEncoding`は、ベクトルの平均の相対的な位置に基づいてベクトルをエンコードします。これは、事前に定義された最小値と最大値（それぞれ`minval`と`maxval`）に対して行われます。

## 説明

このエンコーディングは、[Azami2016](@citet)の振幅に配慮した順列エントロピーのアルゴリズムに触発されています。彼らは、確率を修正するために、振幅情報と状態ベクトルの一次差分情報の線形結合を使用します。しかし、ここでは、修正の振幅部分を整数シンボル`Λ ∈ [1, 2, …, n]`として明示的にエンコードします。エンコーディングの一次差分部分は、[`RelativeFirstDifferenceEncoding`](@ref)エンコーディングとして利用可能です。

## エンコーディング/デコーディング

[`encode`](@ref)と共に使用されると、$m$要素の状態ベクトル$\bf{x} = (x_1, x_2, \ldots, x_m)$は、$Λ = \dfrac{1}{N}\sum_{i=1}^m abs(x_i)$としてエンコードされます。$Λ$の値は、任意の単一要素$x_i$が取ることができる最小/最大値がそれぞれ`minval`/`maxval`であると仮定して、`[0, 1]`の区間に正規化されます。最後に、区間`[0, 1]`は`n`個の離散ビンに離散化され、正の整数`1, 2, …, n`で列挙され、正規化された$Λ$が落ちるビンの番号が返されます。

[`decode`](@ref)と共に使用されると、正規化された$Λ$が落ちたビンの左端が返されます。
