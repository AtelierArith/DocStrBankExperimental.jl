```
AddConstant <: ProbabilitiesEstimator
AddConstant(; c = 1.0)
```

カウントベースの [`OutcomeSpace`](@ref) 用の一般的な加算定数確率推定器であり、いくつかの文献の推定器は `c` を調整することで得られます。現在、$c$ はスカラーのみである必要があります。

  * `c = 1.0` はラプラス推定器、または「加算1」推定器です。

## 説明

$$
k
$$

番目の結果 $\omega_{k}$ の確率は次のように推定されます。

$$
p(\omega_k) = \dfrac{(n_k + c)}{n + mc},
$$

ここで、$m$ は結果空間の基数、$n$ は（エンコードされた）入力データポイントの数、$n_k$ は結果 $\omega_{k}$ が（エンコードされた）入力データポイントで観測される回数です。

`AddConstant` 推定器が [`probabilities_and_outcomes`](@ref) と共に使用される場合、$m$ は *観測された* 結果の数に設定されます。`[`allprobabilities*and*outcomes`](@ref)` と共に使用される場合、$m$ は *可能な* 結果の数に設定されます。

!!! note "未観測の結果には非ゼロの確率が割り当てられます！"
    上記の式を見てみると、$n_k = 0$ の場合、未観測の結果には非ゼロの確率 $\dfrac{c}{n + mc}$ が割り当てられます。これは、推定器が [`allprobabilities_and_outcomes`](@ref) と共に使用される場合、観測されていない結果でさえ非ゼロの確率が割り当てられることを意味します。これは、例えば [`missing_outcomes`](@ref) を使用する場合に結果に影響を与える可能性があります。
