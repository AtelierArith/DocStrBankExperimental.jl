```
counts_and_outcomes(o::OutcomeSpace, x) → (cts::Counts, Ω)
```

`x`（ソート可能でなければならない）を、提供された [`OutcomeSpace`](@ref) `o` によって指定された有限の結果の集合 `Ω` に離散化/エンコードし、各結果 `Ωᵢ ∈ Ω`（すなわち、各「離散化された値」または「エンコードされたシンボル」）がどれだけ頻繁に現れるかをカウントします。

最初の要素が [`Counts`](@ref) インスタンスであり、ベクトルのようにカウントを含むタプルを返し、2番目の要素 `Ω` はカウントに対応する結果であり、`cts[i]` が結果 `Ω[i]` のカウントであることを保証します。

結果は実際に `cts` に含まれており、`cts` の上で [`outcomes`](@ref) 関数を使用してそれらを取得できます。`counts_and_outcomes` は両方を返しますが、後方互換性のためです。

```
counts_and_outcomes(x) → cts::Counts
```

[`OutcomeSpace`](@ref) が指定されていない場合、[`UniqueElements`](@ref) が結果空間として使用されます。

## 説明

[`encode`](@ref) を使用して離散化する [`OutcomeSpace`](@ref) に対して、離散化された/エンコードされた入力データで観測される各結果 $\omega_i \in \Omega$（$\Omega$ は可能な結果の集合）がどれだけ頻繁に現れるかをカウントすることが可能です。したがって、各結果 $\omega_i$ にカウント $f(\omega_i)$ を割り当てることができ、$\sum_{i=1}^N f(\omega_i) = N$ となります。ここで、$N$ は（エンコードされた）入力データの観測数です。`counts` は、*観測された* 結果 $\omega_i^{obs}$（実際に入力データに現れる結果）のカウント $f(\omega_i)_{obs}$ と結果のみを返します。*未観測* 結果のカウントも必要な場合は、[`allcounts_and_outcomes`](@ref) を使用してください。
