```
orbit(sol::AbstractODESolution; vars=nothing, tspan=nothing, to_3d::Bool=false, interactive::Bool=true)
```

軌道や6つの位相空間座標と時間に関連する他の図をプロットするためのプロットレシピです。

# 引数

  * `sol::AbstractODESolution`: ODEソルバーによって返される解。

# キーワード

  * `vars`: プロットする変数を選択するために使用される引数。デフォルト値は[(1, 2, 3)]です。
  * `tspan::Tuple`: プロットする時間の範囲。例えば、tspan = (0, 1)。
  * `to_3d::Bool`: 点を3Dでプロットするかどうか。`true`の場合、プロットのために座標の順序が3Dポイントとして再配置されます。
  * `interactive::Bool`: インタラクティブモードで図を表示するかどうか。

## vars

`vars`は整数、関数、タプル、配列であることができます。`vars`の基本形はタプルです。例えば、時間の関数として変数`x`をプロットしたり、粒子の軌道をプロットしたりする場合、次のように使用できます。

```julia
orbit(sol, vars=(0, 1))
orbit(sol, vars=(1, 2, 3))
```

タプルの長さが3の場合、3Dに変換されます。時間、位置、または速度の関数をプロットするには、まず関数を定義する必要があります。関数の引数は7次元の配列でなければならず、その要素は位相空間座標と時間に対応し、戻り値は数値でなければなりません。例えば、

```julia
Eₖ(xu) = mₑ*(xu[4]^2 + xu[5]^2 + xu[6]^2)/2
orbit(sol, vars=(0, Eₖ))
```

このコマンドは、xの関数として運動エネルギーをプロットします。上記の形式は次のものと同等です。

```julia
orbit(sol, vars=Eₖ)
orbit(sol, vars=1)
```

複数の線を一度にプロットすることは、インデックスのタプルを提供することでサポートされています。例えば、

```julia
orbit(sol, vars=[1, (1, 2), Eₖ])
```

さらに高度な使用法もあります。タプルにベクトルが含まれている場合、それらは自動的に展開されます。例えば、

```julia
orbit(sol, vars=(0, [1, 2, 3]))
orbit(sol, vars=([1, 2, 3], [4, 5, 6]))
```

は次のものと同等です。

```julia
orbit(sol, vars=[(0, 1), (0, 2), (0, 3)])
orbit(sol, vars=[(1, 4), (2, 5), (3, 6)])
```
