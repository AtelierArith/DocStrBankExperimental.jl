```
lag(x::Node, n::Integer)
```

ノードを構築し、`x` から値を取得しますが、それらを `n` ノット遅延させます。

これは、`x` に存在しない新しいタイムスタンプを導入しないことを意味しますが、`x` を評価する際に最初の `n` 値に対してノットを出力しません。

!!! note
    もし `x` が定数ノードであり、`n > 0` の場合、`lag(x, n)` は `x` と同じ [`value_type`](@ref) の [`empty_node`](@ref) になります。

    概念的には、これは定数が時間の始まりに単一のノットで表されるという見方と一致しています。
