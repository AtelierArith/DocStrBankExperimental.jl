```
locate_modes!(evecs, evals=nothing; sort=true, modes=nothing)
```

各モード（`evecs`の行のペアごと）について、そのモードで最大の`norm`を持つ固有ベクトル（`evecs`の各列）を決定します。固有ベクトルは次のように次々と並んでいる必要があります：次元は2n、n ∈ ℤです。

`sort`がtrueの場合、固有ベクトルのペアはインプレースでソートされます。`evals`が提供されている場合、固有値もソートされます。`sort`がfalseの場合、`modes`ベクトルは現在の固有ベクトルの位置からそのモードへのマッピングで埋められます（`modes`のインデックスは現在の固有ベクトルの位置であり、そのインデックスの要素は対応するモードです）。`sort=true`の場合、この関数を呼び出した後、`modes == 1:n`になります。

モードの特定が成功した場合、`true`が返されます。モードの特定が失敗した場合、`false`が返され、`modes`にはゴミが含まれ、固有ベクトル/値は部分的にソートされている可能性があります。

固有ベクトル（および提供されている場合は固有値）は、最初の列から始まるペアで既に存在していると仮定します。

### 入力:

  * `evecs`  –  ペアで既に存在する固有ベクトルの2n x 2n行列
  * `evals`  –  `evecs`の固有ベクトルに対応する固有値を含む長さ2nのベクトル
  * `sort`   –  （オプション）固有ベクトルをソートするかどうかを指定するキーワード引数、デフォルトはtrue

### 出力:

  * `ret`    –  モードの特定が成功した場合はtrue、そうでない場合はfalseを返します
  * `modes` –  （オプション）固有ベクトル -> モードのマッピングのためのキーワード引数、`length(evecs) == Int(size(evecs,1)/2)`
