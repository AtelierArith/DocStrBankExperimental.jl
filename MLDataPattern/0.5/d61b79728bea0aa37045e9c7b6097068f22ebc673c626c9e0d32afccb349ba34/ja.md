```
kfolds(data, [k = 5], [obsdim]) -> FoldsView
```

`data` コンテナを `k` 回再分割し、`k` フォールド戦略を使用してフォールドのシーケンスを遅延 [`FoldsView`](@ref) として返します。結果として得られる `FoldsView` は、インデックスを付けたり、反復処理を行ったりできます。いずれにせよ、実際のデータのサブセットのみが作成されます。つまり、[`getobs`](@ref) が呼び出されるまで、実際のデータはコピーされません。

概念的には、k-フォールド再分割戦略は、与えられた `data` を `k` 個のほぼ同じサイズの部分に分割します。各部分は一度だけ検証セットとして使用され、残りの部分はトレーニングに使用されます。これにより、`data` の `k` 個の異なるパーティションが生成されます。

データセットのサイズが指定された `k` で割り切れない場合、残りの観測値は部分間で均等に分配されます。

```julia
for (x_train, x_val) in kfolds(X, k = 10)
    # コードは10回呼び出されます
    # nobs(x_val) は反復ごとに±1の範囲で異なる場合があります
end
```

複数の変数がサポートされています（例：ラベル付きデータの場合）

```julia
for ((x_train, y_train), val) in kfolds((X, Y), k = 10)
    # ...
end
```

デフォルトでは、フォールドは静的な分割を使用して作成されます。観測値をフォールドにランダムに割り当てるには、[`shuffleobs`](@ref) を使用します。

```julia
for (x_train, x_val) in kfolds(shuffleobs(X), k = 10)
    # ...
end
```

詳細については [`FoldsView`](@ref) を参照するか、関連する関数については [`leaveout`](@ref) を参照してください。
