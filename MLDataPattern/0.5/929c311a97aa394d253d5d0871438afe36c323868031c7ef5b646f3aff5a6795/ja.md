```
oversample([f], data, [fraction = 1], [shuffle = true], [obsdim])
```

既存の観測値を繰り返しサンプリングすることによって、`data`の再バランスされたバージョンを生成します。これにより、すべてのクラスは、最大のクラスの観測数の少なくとも`fraction`倍の観測数を持つことになります。このようにして、すべてのクラスは、与えられた（元の）`data`における最大のクラスが持つ観測数に対して、結果のデータセットに最小限の観測数を持つことになります。

例えば、デフォルト（すなわち`fraction = 1`）では、結果のデータセットはほぼ完璧にバランスが取れます。一方、`fraction = 0.5`の場合、結果のデータ内のすべてのクラスは、最大のクラスの観測数の少なくとも50%を持つことになります。

便利なパラメータ`shuffle`は、生成されたデータが作成後にシャッフルされるかどうかを決定します。シャッフルされない場合、すべての繰り返しサンプルはクラスによってソートされ、最後に一緒に配置されます。デフォルトは`true`です。

オプションのパラメータ`obsdim`は、観測を示す次元を指定するために使用できます。この概念が`data`のタイプに対して意味を持つ場合に限ります。詳細については`?ObsDim`を参照してください。

```julia
# 6つの観測値と3つの特徴
X = rand(3, 6)
# 2つのクラス、非常に不均衡
Y = ["a", "b", "b", "b", "b", "a"]

# クラス"a"をクラス"b"に合わせてオーバーサンプリング
X_bal, Y_bal = oversample((X,Y))

# これにより、繰り返しデータを含む大きなデータセットが生成されます
@assert size(X_bal) == (3,8)
@assert length(Y_bal) == 8

# 現在、"a"と"b"の両方に4つの観測値があります
@assert sum(Y_bal .== "a") == 4
@assert sum(Y_bal .== "b") == 4
```

この関数が機能するためには、`data`のタイプが[`nobs`](@ref)と[`getobs`](@ref)を実装している必要があります。例えば、以下のコードは`oversample`が`DataTable`で動作することを可能にします。

```julia
# DataTables.jlを動作させる
LearnBase.getobs(data::DataTable, i) = data[i,:]
StatsBase.nobs(data::DataTable) = nrow(data)
```

パラメータ`f`を使用して、与えられた`data`の各観測からターゲットを抽出または取得する方法を指定できます。`data`がタプルである場合、タプルの最後の要素にターゲットが含まれていると見なされ、その要素内の各観測に`f`が適用されます。
