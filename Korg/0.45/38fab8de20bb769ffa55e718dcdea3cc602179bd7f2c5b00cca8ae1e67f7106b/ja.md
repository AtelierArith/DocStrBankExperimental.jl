```
compute_LSF_matrix(synth_wls, obs_wls, R; kwargs...)
```

ガウシアン線幅関数（LSF）を適用し、波長 `obswls` に再サンプリングするために、波長 `synth_wls` に定義されたフラックスベクトルと掛け算されるスパース行列を構築します。

# 引数

  * `synth_wls`: `synthesize` に認識される任意の形式の合成波長、例えば、Å単位の下限と上限を含むペア、ペアのベクトル、またはJuliaの範囲オブジェクトのベクトル。
  * `obs_wls`: 観測スペクトルの波長
  * `R`: 解像度、$R = \lambda/\Delta\lambda$

# キーワード引数

  * `window_size` (デフォルト: 4): LSF幅（σ、HWHMではない）の単位で畳み込みカーネルをどれだけ外側に拡張するか
  * `verbose` (デフォルト: `true`): 警告や情報をstdout/stderrに出力するかどうか。
  * `step_tolerance`: `synth_wls`内の隣接波長間の最大差。これは、`synth_wls`が範囲や範囲のベクトルではなく波長のベクトルである場合にのみ使用されます。

データに最もよく一致させるために、あなたの波長範囲は比較する領域の外側に数$\Delta\lambda$延長するべきです。

[`Korg.apply_LSF`](@ref)は、単一のフラックスベクトルにLSFを効率的に適用できます。この関数は比較的遅いですが、LSF行列が構築されると、行列の掛け算を介して観測解像度にスペクトルを畳み込むのは速いです。
