```
prune_linelist(atm, linelist, A_X, wls...; threshold=1.0, sort=true, synthesis_kwargs...)
```

`linelist`の中で最も強い線を含むベクトルを返します（オプションで、近似等価幅でソートされます）。

# 引数

  * `atm`: 大気モデル
  * `linelist`: 線リスト（`Line`オブジェクトのベクトル）
  * `A_X`: 各元素の豊富さ（[`format_A_X`](@ref)を参照）
  * `wls...`: 合成する波長範囲。これは[`synthesize`](@ref)の`wls`と同じ方法で指定されます。

# キーワード引数

  * `threshold=0.1`: 戻り値リストに含めるための線中心吸収と光球で計算された連続吸収の比率の閾値。`0.1`は高解像度で高品質のスペクトルで測定可能なものを把握するための妥当なデフォルトですが、合成用の線リストを作成するためには使用すべきではありません。
  * `sort_by_EW=true`: `true`の場合、戻り値の線リストは近似された減少等価幅でソートされます。`false`の場合、線リストは波長順になります。波長順にリストを残す方がはるかに速いですが、強さでソートすることは最も強い線を視覚化するのに役立ちます。
  * `verbose=true`: `true`の場合、EWを測定している間にプログレスバーが表示されます。他のすべてのkwargsは[`synthesize`](@ref)への内部呼び出しに渡されます。
  * `max_distance=0.0`: 戻り値リストから除外される前に、`wls`から線がどれだけ離れていてもよいか（Å単位）。

!!! caution
    この関数は合成用に線リストをプルーニングするために使用できますが、デフォルトの動作はこの目的には攻撃的すぎます。合成を高速化するために線リストをプルーニングする場合は、はるかに低い閾値（例：`threshold=1e-4`）を設定し、`sort_by_EW=false`を使用してください。大きな線リストを使用しても、Korgは動的に含める線を選択します（[`synthesize`](@ref)への`line_cutoff_threshold`キーワード引数を参照）。


プロット用にこれを使用している場合は、[`merge_close_lines`](@ref)も参照してください。
