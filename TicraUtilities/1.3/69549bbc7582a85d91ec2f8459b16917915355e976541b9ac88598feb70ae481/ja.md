```
SPHQPartition
```

Ticra互換のQ型SPHファイルから1つの周波数のデータを読み込むための構造体です。`qsmns`フィールドにはQ係数が含まれています。これは`qsmns[s,m,n]`としてインデックス付けされ、`s ∈ {1,2}`、`m ∈ -mmax:mmax`、`n ∈ {1, ..., mmax}`です。しかし、すべてのエントリが使用されるわけではありません。非ゼロのエントリは、`n ∈ {max(1, abs(m)), ..., nmax}`の場所にのみ存在します。SPHファイルに保存されている係数が`Q'`であり、`SPHQPartition`インスタンスに保存されている係数が`Q`である場合、`Q = sqrt(8π) * conj(Q')`となります。これはTicra公式ドキュメントのファイルフォーマットセクションで説明されています。

### フィールド

  * `prgtag::String`: ファイルを作成したプログラムからのプログラムタグとタイムスタンプ。
  * `idstrg::String`: 識別テキスト。
  * `nthe::Int`: 360°にわたるθサンプルの数。偶数であり、4以上でなければなりません。
  * `nphi::Int`: 360°にわたるϕサンプルの数。3以上でなければなりません。
  * `nmax::Int`: SW展開における極座標インデックス`n`の最大値。`1 ≤ nmax ≤ nthe÷2`。
  * `mmax::Int`: SW展開における方位インデックス`|m|`の最大値。`0 ≤ mmax ≤ min((nphi-1)÷2, nmax)`。
  * `t4::String`: ダミー文字列。
  * `t5::String`: ダミー文字列。解析された場合、5つの実数を含みます。
  * `t6::String`: ダミー文字列。解析された場合、5つの実数を含みます。
  * `t7::String`: ダミー文字列。
  * `t8::String`: ダミー文字列。
  * `qsmns::OffsetArray{ComplexF64, 3}`: Q係数を含み、exp(jωt)の時間変動を仮定し、Ticraの正規化規約を使用します。配列は軸`(1:2, -mmax:mmax, 1:nmax)`を持ち、`n < abs(m)`のエントリは常にゼロです。
  * `powerms::OffsetArray{Float64, 1}`: ベクトルは軸`(0:mmax)`を持ち、`m`番目の要素は`±m`をmインデックスとするすべてのモードの合計パワー（大きさの二乗の合計の半分）を含みます。
