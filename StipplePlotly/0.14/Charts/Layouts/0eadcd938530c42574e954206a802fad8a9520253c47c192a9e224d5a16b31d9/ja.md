# プロパティ

---

  * `align::String` - ボックス内の `text` の水平方向の整列を設定します。`text` が2行以上にわたる場合（つまり、`text` に1つ以上の <br> HTML タグが含まれている場合）や、テキスト幅を上書きするために明示的な幅が設定されている場合にのみ効果があります。タイプ: 列挙型、いずれか（ `"left"` | `"center"` | `"right"` ）| デフォルト: `"center"`
  * `arrowcolor::String` - 注釈矢印の色を設定します。
  * `arrowhead::Int` - 終端注釈矢印のスタイルを設定します。タイプ: 0から8の間の整数 | デフォルト: `1`
  * `arrowside::String` - 注釈矢印の先端位置を設定します。タイプ: フラグリスト文字列。`"end"`、`"start"` の任意の組み合わせを `"+"` で結合するか、`"none"` | デフォルト: `"end"`
  * `arrowsize::Float64` - 矢印の先端のサイズを、`arrowwidth` に対して相対的に設定します。1（デフォルト）の値は、線の約3倍の幅の先端を提供します。タイプ: 0.3以上の数値 | デフォルト: `1`
  * `arrowwidth::Float64` - 注釈矢印線の幅（px単位）を設定します。タイプ: 0.1以上の数値
  * `ax::Union{String,Int,Float64}` - 矢印の先端に関する矢印の尾のx成分を設定します。`axref` が `pixel` の場合、正の（負の）成分は右から左（左から右）を指す矢印に対応します。`axref` が `pixel` でなく、`xref` と正確に同じである場合、これはその軸の絶対値であり、`xref` と同じ座標系で指定されます。タイプ: 数値またはカテゴリ座標文字列
  * `axref::String` - 注釈の尾（ax,ay）が指定される座標系を示します。ax 軸 ID（例: "ax" または "ax2"）に設定すると、`ax` の位置は ax 座標を指します。"paper" に設定すると、`ax` の位置は、正規化された座標でプロット領域の左からの距離を指し、"0"（"1"）は左（右）に対応します。ax 軸 ID の後に "domain"（スペースで区切る）を付けると、位置は "paper" の場合と同様に動作しますが、その軸のドメインの左からのドメイン長の分数での距離を指します。例えば、"ax2 domain" は2番目のax軸のドメインを指し、axの位置0.5は2番目のax軸のドメインの左と右の間の点を指します。矢印の絶対位置決めが機能するためには、"axref" は "xref" と正確に同じでなければなりません。そうでない場合、"axref" は "pixel" に戻ります（次に説明）。相対位置決めの場合、"axref" を "pixel" に設定できます。この場合、"ax" の値は "x" に対して相対的にピクセルで指定されます。絶対位置決めは、ズーム時に正しいトレンドを示し続けるべきトレンドライン注釈に便利です。相対位置決めは、注釈付きポイントのテキストオフセットを指定するのに便利です。タイプ: 列挙型、いずれか（ `"pixel"` | `"/^x([2-9]|[1-9][0-9]+)?( domain)?$/"` ）| デフォルト: `"pixel"`
  * `ay::Union{String,Int,Float64}` - 矢印の先端に関する矢印の尾のy成分を設定します。`ayref` が `pixel` の場合、正の（負の）成分は下から上（上から下）を指す矢印に対応します。`ayref` が `pixel` でなく、`yref` と正確に同じである場合、これはその軸の絶対値であり、`y` と同じ座標系で指定されます。タイプ: 数値またはカテゴリ座標文字列
  * `ayref::String` - 注釈の尾（ax,ay）が指定される座標系を示します。ay 軸 ID（例: "ay" または "ay2"）に設定すると、`ay` の位置は ay 座標を指します。"paper" に設定すると、`ay` の位置は、正規化された座標でプロット領域の下からの距離を指し、"0"（"1"）は下（上）に対応します。ay 軸 ID の後に "domain"（スペースで区切る）を付けると、位置は "paper" の場合と同様に動作しますが、その軸のドメインの下からのドメイン長の分数での距離を指します。例えば、"ay2 domain" は2番目のay軸のドメインを指し、ayの位置0.5は2番目のay軸のドメインの下と上の間の点を指します。矢印の絶対位置決めが機能するためには、"ayref" は "yref" と正確に同じでなければなりません。そうでない場合、"ayref" は "pixel" に戻ります（次に説明）。相対位置決めの場合、"ayref" を "pixel" に設定できます。この場合、"ay" の値は "y" に対して相対的にピクセルで指定されます。絶対位置決めは、ズーム時に正しいトレンドを示し続けるべきトレンドライン注釈に便利です。相対位置決めは、注釈付きポイントのテキストオフセットを指定するのに便利です。タイプ: 列挙型、いずれか（ `"pixel"` | `"/^x([2-9]|[1-9][0-9]+)?( domain)?$/"` ）| デフォルト: `"pixel"`
  * `bgcolor::String` - 注釈の背景色を設定します。デフォルト: `"rgba(0,0,0,0)"`
  * `bordercolor::String` - 注釈 `text` を囲むボーダーの色を設定します。デフォルト: `"rgba(0,0,0,0)"`
  * `borderpad::Int` - `text` と囲むボーダーの間のパディング（px単位）を設定します。デフォルト: `1`
  * `borderwidth::Int` - 注釈 `text` を囲むボーダーの幅（px単位）を設定します。タイプ: 0以上の数値 | デフォルト: `1`
  * `captureevents::Bool` - 注釈テキストボックスがマウスの移動およびクリックイベントをキャプチャするか、これらのイベントが注釈の背後にあるプロットのデータポイントに通過することを許可するかを決定します。デフォルトでは、`hovertext` が提供されない限り、`captureevents` は "false" です。`hovertext` なしでイベント `plotly_clickannotation` を使用する場合は、`captureevents` を明示的に有効にする必要があります。
  * `font::Font` - 注釈テキストのフォントを設定します。
  * `height::Int` - テキストボックスの明示的な高さを設定します。null（デフォルト）は、テキストがボックスの高さを設定することを許可します。背の高いテキストはクリップされます。タイプ: 1以上の数値
  * `hoverlabel::Dict` - 1つ以上のキーを含むオブジェクト: `bgcolor` `bordercolor` `font`
  * `name::String` - テンプレートで使用されるとき、名前付きアイテムは、この配列内の出力図に作成され、図がすでに持っているアイテムに加えて作成されます。`templateitemname` がこの `name` と一致するアイテムを作成することで、出力図内のこれらのアイテムを変更できます（`visible: false` または `enabled: false` を含めて非表示にすることができます）。テンプレートの外では効果がありません。
  * `opacity::Float64` - 注釈（テキスト + 矢印）の不透明度を設定します。タイプ: 0から1の間の数値 | デフォルト: `1`
  * `showarrow::Bool` - 注釈が矢印で描画されるかどうかを決定します。"true" の場合、`text` は矢印の尾の近くに配置されます。"false" の場合、`text` は提供された `x` と `y` に合わせて配置されます。デフォルト: `true`
  * `standoff::Int` - 矢印の先端を指している位置から離す距離（ピクセル単位）を設定します。たとえば、ズームに依存せずにマーカーの端を指すようにします。この値は、`ax` / `ay` ベクトルから矢印を短くします。`xshift` / `yshift` はすべてをこの量だけ移動させるのとは対照的です。タイプ: 0以上の数値 | デフォルト: `0`
  * `startarrowhead::Int` - 開始注釈矢印のスタイルを設定します。タイプ: 0から8の間の整数 | デフォルト: `1`
  * `startarrowsize::Int` - 開始注釈矢印のサイズを、`arrowwidth` に対して相対的に設定します。1（デフォルト）の値は、線の約3倍の幅の先端を提供します。タイプ: 0.3以上の数値 | デフォルト: `1`
  * `startstandoff::Int` - 開始矢印の先端を指している位置から離す距離（ピクセル単位）を設定します。たとえば、ズームに依存せずにマーカーの端を指すようにします。この値は、`ax` / `ay` ベクトルから矢印を短くします。`xshift` / `yshift` はすべてをこの量だけ移動させるのとは対照的です。タイプ: 0以上の数値 | デフォルト: `0`
  * `templateitemname::String` - テンプレート内のこの配列の名前付きアイテムを参照するために使用されます。テンプレートからの名前付きアイテムは、入力図に一致するアイテムがなくても作成されますが、`templateitemname` がその `name` と一致するアイテムを作成することで、1つを変更できます。テンプレートがない場合や一致するアイテムがない場合、このアイテムは明示的に `visible: true` で表示しない限り非表示になります。
  * `text::String` - 注釈に関連付けられたテキストを設定します。Plotly は、改行（<br>）、太字（<b></b>）、イタリック（<i></i>）、ハイパーリンク（<a href='...'></a>）などの操作を行うために、HTML タグのサブセットを使用します。タグ <em>、<sup>、<sub> <span> もサポートされています。タイプ: 文字列
  * `textangle::Union{Int,Float64}` - 水平方向に対して `text` が描画される角度を設定します。タイプ: `angle` | デフォルト: `0`
  * `valign::String` - ボックス内の `text` の垂直方向の整列を設定します。明示的な高さが設定されている場合にのみ効果があります。タイプ: 列挙型、いずれか（ `"top"` | `"middle"` | `"bottom"`） | デフォルト: `"middle"`
  * `visible::Bool` `visible` - この注釈が表示されるかどうかを決定します。タイプ: ブール値 | デフォルト: `true`
  * `width::Int` - テキストボックスの明示的な幅を設定します。null（デフォルト）は、テキストがボックスの幅を設定することを許可します。幅の広いテキストはクリップされます。タイプ: 1以上の数値
  * `x::Union{String,Int,Float64}` - 注釈のx位置を設定します。軸の `type` が "log" の場合、希望する範囲の対数を取る必要があります。軸の `type` が "date" の場合、日付データのように日付文字列である必要がありますが、DateオブジェクトやUnixミリ秒も受け入れられ、文字列に変換されます。軸の `type` が "category" の場合、各カテゴリにゼロから順番にシリアル番号が割り当てられるスケールを使用して数値である必要があります。
  * `xanchor::String` - テキストボックスの水平方向の位置アンカーを設定します。このアンカーは、`x` の位置を注釈の "left"、"center" または "right" に結びつけます。例えば、`x` が1に設定され、`xref` が "paper" に設定され、`xanchor` が "right" に設定されている場合、注釈の最も右側の部分はプロット領域の最も右側の端に合わせられます。"auto" の場合、アンカーはデータ参照の注釈に対して "center" と同等であるか、矢印がある場合は同様です。矢印がない場合の紙参照の場合、選択されたアンカーは最も近い側に対応します。タイプ: 列挙型、いずれか（ `"auto"` | `"left"` | `"center"` | `"right"`） | デフォルト: `"auto"`
  * `xref::Union{String,Int,Float64}` - 注釈のx座標軸を設定します。x 軸 ID（例: "x" または "x2"）に設定すると、`x` の位置は x 座標を指します。"paper" に設定すると、`x` の位置は、正規化された座標でプロット領域の左からの距離を指し、"0"（"1"）は左（右）に対応します。x 軸 ID の後に "domain"（スペースで区切る）を付けると、位置は "paper" の場合と同様に動作しますが、その軸のドメインの左からのドメイン長の分数での距離を指します。例えば、"x2 domain" は2番目のx軸のドメインを指し、xの位置0.5は2番目のx軸のドメインの左と右の間の点を指します。タイプ: 列挙型、いずれか（ `"paper"` | `"/^x([2-9]|[1-9][0-9]+)?( domain)?$/"` ）
  * `xshift::Union{Int,Float64}` - 注釈全体と矢印の位置を右（正）または左（負）にこの数だけピクセル移動させます。デフォルト: `0`
  * `y::Union{String,Int,Float64}` - 注釈のy位置を設定します。軸の `type` が "log" の場合、希望する範囲の対数を取る必要があります。軸の `type` が "date" の場合、日付データのように日付文字列である必要がありますが、DateオブジェクトやUnixミリ秒も受け入れられ、文字列に変換されます。軸の `type` が "category" の場合、各カテゴリにゼロから順番にシリアル番号が割り当てられるスケールを使用して数値である必要があります。
  * `yanchor::Union{String}` - テキストボックスの垂直方向の位置アンカーを設定します。このアンカーは、`y` の位置を注釈の "top"、"middle" または "bottom" に結びつけます。例えば、`y` が1に設定され、`yref` が "paper" に設定され、`yanchor` が "top" に設定されている場合、注釈の最も上側の部分はプロット領域の最も上側の端に合わせられます。"auto" の場合、アンカーはデータ参照の注釈に対して "middle" と同等であるか、矢印がある場合は同様です。矢印がない場合の紙参照の場合、選択されたアンカーは最も近い側に対応します。タイプ: 列挙型、いずれか（ `"auto"` | `"top"` | `"middle"` | `"bottom"`） | デフォルト: `"auto"`
  * `yref::Union{String,Int,Float64}` - 注釈のy座標軸を設定します。y 軸 ID（例: "y" または "y2"）に設定すると、`y` の位置は y 座標を指します。"paper" に設定すると、`y` の位置は、正規化された座標でプロット領域の下からの距離を指し、"0"（"1"）は下（上）に対応します。y 軸 ID の後に "domain"（スペースで区切る）を付けると、位置は "paper" の場合と同様に動作しますが、その軸のドメインの下からのドメイン長の分数での距離を指します。例えば、"y2 domain" は2番目のy軸のドメインを指し、yの位置0.5は2番目のy軸のドメインの下と上の間の点を指します。タイプ: 列挙型、いずれか（ `"paper"` | `"/^x([2-9]|[1-9][0-9]+)?( domain)?$/"` ）
  * `yshift::Union{Int,Float64}` - 注釈全体と矢印の位置を上（正）または下（負）にこの数だけピクセル移動させます。デフォルト: `0`
