生のトランザクション形式といくつかのピアツーピアネットワークメッセージは、次のデータのバイト数を示すために可変長整数の一種を使用します。Bitcoin Coreコードでは、これらの可変長整数をcompactSizeと呼びます。他の多くの文書では、これらをvar_intまたはvarIntと呼びますが、これは他の可変長整数エンコーディング（たとえば、データをディスクにシリアライズするためにBitcoin Coreで使用されるCVarIntクラス）と混同されるリスクがあります。トランザクション形式で使用されるため、compactSize符号なし整数の形式はコンセンサスルールの一部です。0から252までの数値に対して、compactSize符号なし整数は通常の符号なし整数のように見えます。0xffffffffffffffffまでの他の数値については、長さを示すためにバイトが数値の前に付加されますが、それ以外は数値はリトルエンディアン順の通常の符号なし整数のように見えます。

|                                       値 | 使用バイト数 |                    形式 |
| ---------------------------------------:| ------:| ---------------------:|
|                          >= 0 && <= 252 |      1 |               uint8_t |
|                     >= 253 && <= 0xffff |      3 | 0xfdの後にuint16_tとしての数値 |
|             >= 0x10000 && <= 0xffffffff |      5 | 0xfeの後にuint32_tとしての数値 |
| >= 0x100000000 && <= 0xffffffffffffffff |      9 | 0xffの後にuint64_tとしての数値 |

たとえば、数値515は0xfd0302としてエンコードされます。
