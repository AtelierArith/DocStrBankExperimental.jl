```
raff(model::Function, data::Array{Float64, 2}, n::Int; kwargs...)

raff(model::Function, gmodel!::Function, data::Array{Float64, 2},
    n::Int; MAXMS::Int=1, SEEDMS::Int=123456789,
    initguess::Vector{Float64}=zeros(Float64, n),
    ε::Float64=1.0e-4, noutliers::Int=-1, ftrusted::Union{Float64,
    Tuple{Float64, Float64}}=0.5)
```

ロバスト代数フィッティング関数（RAFF）アルゴリズム。この関数は、信頼できるデータポイントの数を自動的に見つけるために投票システムを使用します。

  * `model`: データをフィットさせる関数。そのシグネチャは次のように指定する必要があります。

    ```
    model(x, θ)
    ```

    ここで `x` は多次元引数で、`θ` は `n` 次元のパラメータベクトルです。
  * `gmodel!`: モデル関数の勾配。そのシグネチャは次のように指定する必要があります。

    ```
    gmodel!(g, x, θ)
    ```

    ここで `x` は多次元引数、`θ` は `n` 次元のパラメータベクトルで、勾配は `g` に書かれます。
  * `data`: フィットさせるデータ。この行列は次の形式である必要があります。

    ```
    x11 x12 ... x1N y1
    x21 x22 ... x2N y2
    :
    ```

    ここで `N` はモデルの引数の次元（すなわち `x` の次元）です。
  * `n`: モデル関数におけるパラメータベクトルの次元

オプションの引数は次のとおりです。

  * `MAXMS`: 使用するマルチスタートポイントの数
  * `SEEDMS`: ランダムマルチスタートポイントの整数シード
  * `initialguess`: 開始点の良い推測とマルチスタート戦略でのランダムポイント生成のための推測
  * `ε`: `lmlovo` に対する勾配停止基準
  * `noutliers`: 最大期待外れ値の数を示す整数。デフォルトは *半分*。*非推奨*。
  * `ftrusted`: 信頼できるポイントの最小期待割合を示す浮動小数点数。デフォルトは *半分* (0.5)。信頼できるポイントの割合の形式 `(fmin, fmax)` のタプルでも可能です。

見つかった最良のパラメータを持つ [`RAFFOutput`](@ref) オブジェクトを返します。
