```
get_prices(symbol::String; range::String="5d", interval::String="1d", startdt::Union{Date,DateTime,AbstractString}="", enddt::Union{Date,DateTime,AbstractString}="", prepost::Bool=false, autoadjust::Bool=true, timeout::Int=10, throw_error::Bool=false, exchange_local_time::Bool=false, divsplits::Bool=false, wait::Float64=0.0)
```

Yahoo Financeから価格を取得します。

## 引数

  * `symbol`: ティッカー（例：Apple Inc.のAAPL、またはS&P 500の^GSPC）

`range`または`startdt`と`enddt`の両方を提供できます。

  * `range`: 時間範囲を指定する文字列。事前定義された値（"ytd"、"max"）のいずれか、または次の接尾辞を使用したカスタム範囲である必要があります：

      * "m"は分を表します（例："30m"は30分）
      * "d"は日を表します（例："7d"は7日）
      * "mo"は月を表します（例："3mo"は3ヶ月）
      * "y"は年を表します（例："1y"は1年）
  * `startdt`と`enddt`: `Date`、`DateTime`、または"yyyy-mm-dd"形式の`String`のいずれかの型である必要があります。1つが指定された場合、両方を提供する必要があります。
  * `interval`: データの間隔。有効な値は"1m"、"2m"、"5m"、"15m"、"30m"、"60m"、"90m"、"1h"、"1d"、"5d"、"1wk"、"1mo"、"3mo"です。デフォルトは"1d"です。
  * `prepost`: 前後の市場データを含めるかどうかを示すブール値。デフォルトは`false`です。
  * `autoadjust`: デフォルトは`true`です。1d以上の間隔でのみ利用可能な調整後の終値と終値の比率で、始値、高値、安値、終値、およびボリュームを調整します。
  * `timeout`: HTTPリクエストのタイムアウト（秒単位）。デフォルトは10です。
  * `throw_error`: ブール値、デフォルトは`false`です。trueに設定すると、ティッカーが無効な場合や他の問題が発生した場合にエラーが発生します。falseの場合は警告が表示され、空の`OrderedDict`が返されます。
  * `exchange_local_time`: ブール値、デフォルトは`false`です。trueに設定すると、タイムスタンプは取引所の現地時間に対応します。そうでない場合はGMTです。
  * `divsplits`: ブール値、デフォルトは`false`です。trueに設定すると、配当金と株式分割データも返されます。分割データには分子、分母、および分割比率が含まれます。これが機能するには、間隔を"1d"に設定する必要があります。
  * `wait`: 浮動小数点数、デフォルトは0.0です。分単位のデータを長期間取得する際の連続API呼び出し間の待機時間（秒単位）を指定します。

## 注意事項

  * 7日を超える期間の分単位データリクエストの場合、関数は自動的にリクエストを複数の7日間のチャンクに分割し、結果を結合します。
  * `startdt`と`enddt`を使用する場合、両方を提供する必要があります。

## 戻り値

要求されたデータを含む`OrderedDict{String, Union{String,Vector{DateTime},Vector{Float64}}}`。
