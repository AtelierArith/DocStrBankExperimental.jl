```
generate_classification_plot_data(sequence_length,
                                  lower_x, lower_y, upper_x, upper_y,
                                  number_of_x_values, number_of_y_values;
                                  Hadamard_mode=false)
```

# 説明

ユーザーが指定した範囲内の各ペア (x,y) に対して、以下の操作を行います：

(1) トポロジー 12|34 を持つ四元木モデルから、長さ `sequence_length` bp の単一データポイントを生成します。このとき、葉 1 と 3 の枝の長さは y であり、他のすべての枝の長さは x です。

(2) そのデータポイントに対する最尤推定木を推定します。

(3) 得られた最尤推定と互換性のある二項四元木トポロジーを特定します（τ=1 は 12|34、τ=2 は 13|23、τ=3 は 14|23）。 

次に、推定された互換性のあるトポロジー分類と共にペア (x,y) を保存します。

# 引数

  * `sequence_length` : 各パラメータ領域 (x,y) に対して生成されるデータの長さ（bp）
  * `lower_x`, `lower_y`, `upper_x`, `upper_y` : x と y の値の下限と上限を指定します
  * `number_of_x_values, number_of_y_values` : 指定された上限と下限の間から選択される（均等に間隔を空けた）値の数。この値はステップサイズを決定し、したがってプロットの解像度を決定します。高い値は、ピクセル化が少ないプロットを生成しますが、生成に時間がかかります。
  * `Hadamard_mode` : 関数が実行されるモードのいずれかを決定するオプションのパラメータ。`Hadamard_mode = false` の場合、x と y はサイトあたりの期待される変異数（すなわち、進化的距離）で測定された枝の長さとして解釈されます。`Hadamard_mode = true` の場合、x と y はハダマードエッジパラメータとして解釈されます（すなわち、進化的距離に関数 d -> exp(-2d) を適用することによって得られます）。
