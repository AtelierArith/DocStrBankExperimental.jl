```
iht(filename, k, d, phenotypes=6, covariates="", summaryfile="iht.summary.txt",
    betafile="iht.beta.txt", kwargs...)
```

スパースレベル `k` で IHT を実行します。

# 引数

  * `filename`: VCF、バイナリ PLINK、または BGEN ファイルのための `String`。VCF ファイルは `.vcf` または `.vcf.gz` で終わる必要があります。バイナリ PLINK ファイルは `.bim/.bed/.fam` のトレーリングを除外する必要がありますが、トリオはすべて同じディレクトリに存在する必要があります。BGEN ファイルは `.bgen` で終わる必要があります。
  * `k`: 非ゼロ係数の数 = スパースパラメータのための `Int`
  * `d`: 表現型の分布。定量的形質には `Normal`、二項形質には `Bernoulli`、カウント形質には `Poisson` または `NegativeBinomial`、複数の定量的形質には `MvNormal` を指定します。

# オプション引数

  * `phenotypes`: 表現型ファイル名（`String`）、整数、または整数のベクター。整数は PLINK の `.fam` ファイルの表現型を格納する列に対応します（デフォルト `phenotypes=6`）。多変量解析のために複数の整数を入力します（例：`phenotypes=[6, 7]`）。欠損表現型は `NA` または `-9` として認識されます。定量的形質（単変量または多変量）の場合、欠損表現型は平均で補完されます。二項およびカウント表現型は補完できません。表現型ファイルは Julia ベースの `readdlm` 関数を使用して読み取られます。各被験者の表現型は異なる行に配置される必要があり、多変量解析の場合、各表現型はカンマで区切られます。表現型ファイルにはヘッダー行を含めないでください。各行は PLINK と同じ順序でリストされ、（多変量解析の場合）カンマで区切られる必要があります。
  * `covariates`: 共変量ファイル名。デフォルト `covariates=""`（この場合、切片項が自動的に含まれます）。`covariates` ファイルが指定された場合、Julia ベースの `readdlm` 関数を使用して読み取られます。共変量ファイルはカンマで区切られ、ヘッダー行を含めないでください。各行は PLINK と同じ順序でリストされる必要があります。最初の列はすべて 1 で、切片を示す必要があります。`exclude_std_idx` で指定されていない他のすべての列は、平均 0 分散 1 に標準化されます。
  * `summaryfile`: IHT の要約統計を保存するための出力ファイル名。デフォルト `summaryfile="iht.summary.txt"`。
  * `betafile`: IHT の推定遺伝子効果サイズを保存するための出力ファイル名。デフォルト `betafile="iht.beta.txt"`。
  * `covariancefile`: 多変量解析のための IHT の推定形質共分散行列を保存するための出力ファイル名。デフォルト `covariancefile="iht.cov.txt"`。
  * `exclude_std_idx`: 標準化から除外すべき非遺伝的共変量のインデックス。
  * `dosage`: 現在、VCF ファイルに対してのみ動作が保証されています。`true` の場合、遺伝子型の投与量を読み取ります（すなわち、標準化前の `X[i, j] ∈ [0, 2]`）。
  * [`fit_iht`](@ref) で利用可能なすべてのオプション引数。

# 出力

適合した `IHTResult`（単変量解析）または `mIHTResult`（多変量解析）。さらに、人間が読みやすいテキストファイルも `betafile` で指定されたファイル名で出力されます。
