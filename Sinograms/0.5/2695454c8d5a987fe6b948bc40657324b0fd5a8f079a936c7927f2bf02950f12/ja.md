```
SinoGeom{Td,To}
```

2Dシノグラムのレイジオメトリを表すための抽象型。これは、2D平行ビームまたはファンビームシステムの特定のシノグラムのサンプリング特性を説明します。

# 共通フィールド

  * `nb` "放射状"サンプルの数、別名 `nr` または `ns`
  * `d` 別名 `dr` または `ds`、"放射状"サンプル間隔
  * `offset` 単位のないサンプルオフセット（通常は0または0.25）
  * `na` "角度"サンプルの数、別名 `nϕ` または `nβ`  デフォルト: `2 * floor(Int, nb * π/2 / 2)`
  * `orbit` ソースの軌道（度またはUnitful）
  * `orbit_start` 開始角度（度またはUnitful） `orbit` と `orbit_start` はどちらも単位のない（度）か、同じ単位でなければなりません。

# `SinoFan`タイプの追加フィールド

  * `dsd` ソースから検出器までの距離
  * `dod` 原点から検出器までの距離
  * `source_offset` 通常は0

# 単位:

  * `d`、`source_offset`、`dsd`、`dod` はすべて同じ単位でなければなりません。

# 基本メソッド

  * `angles` (na) 単位は度
  * `dims (nb, na)`
  * `ones = ones(Float32, nb, na)`
  * `zeros = zeros(Float32, nb, na)`
  * `rays` サイズ `(nb, na)` の平行ビーム座標タプルのイテレータ `(r, ϕ)`
  * `downsample(st, down)` 整数因子でサンプリングを減少させる
  * `oversample(st, over)`
  * `sino_geom_plot!` システムジオメトリをプロットする

# 開発者向けの非公開ヘルパー関数:

  * `_ds|dr` 放射状サンプル間隔（`:moj`の場合は`NaN`）
  * `_s (nb) s` サンプル位置
  * `_w = (nb-1)/2 + offset` "中央"サンプル位置
  * `_ar (na)` ソース角 [ラジアン]
  * `_rfov` 放射状FOV
  * `_xds (nb)` 検出器要素の中心（ベータ=0）
  * `_yds (nb)` ""
  * `_tau(rg, x, y)` 各 `(x,y)` ペア `(length(x), na)` の投影s/ds
  * `_shape(rg, sino [,:])` `sino` を配列 `(nb,na[,:])` に再形成
  * `_unitv(rg [, (ib,ia)])` 単位 'ベクトル' で単一の非ゼロ要素を持つ

モジェットの場合:

  * `_d_ang (na)` 角度依存の放射状間隔

ファンビームの場合:

  * `_dso = dsd - dod` ソースから原点までの距離（平行ビームの場合はInf）
  * `_dfs` ソースから検出器焦点までの距離（第3世代CTの場合は0、平面検出器の場合は`Inf`）
  * `_gamma(rg [,s]) (nb)` ガンマサンプル値 `ラジアン`、オプションで`s`値を指定
  * `_gamma_max = max(|γ|)` ファン角の半分 `ラジアン`、`offset_s` が0の場合

# 注意事項

  * 3D軸方向またはヘリカルコーンビームCTには、代わりに `ct_geom()` を使用してください。
