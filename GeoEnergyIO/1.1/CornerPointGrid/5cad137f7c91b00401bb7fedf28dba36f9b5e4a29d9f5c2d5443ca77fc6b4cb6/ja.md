```
cpgrid_from_horizons(X, Y, depths)
cpgrid_from_horizons(X, Y, depths, (100, 100))
cpgrid_from_horizons(X, Y, depths, sz = missing;
    layer_width = 1,
    transforms = [(x, y, z, x_c, y_c, i, j, k) -> z],
    xy_transform = (x, y, i, j, z_t, z_b) -> (x, y, x, y)
)
```

コーナーポイントグリッドを一連の地層から作成します。地層は2D配列のセットとして与えられ、各配列はグリッド内の各点での地層の深さを表します。地層は同じサイズでなければならず、グリッド内の各セルの上部と下部を作成するために使用されます。少なくとも2つの地層を提供する必要があり、1つはグリッドの上部用、もう1つは下部用です。追加の地層を提供することもできます。地層が交差する場合、セルは最も低い地層が保持されるように圧縮されます。

グリッドは、深さ配列の行数と列数に等しい長さのベクトル/範囲である指定された`X`および`Y`座標を使用して作成されます。`sz`引数は、I/J方向でグリッドを異なるサイズに再サンプリングするために使用できます。`sz`が提供されない場合、グリッドは地層と同じサイズになります。

# キーワード引数

  * `layer_width`: 各層内のセルの数。単一の整数または、地層/深さの数から1を引いた長さの整数の配列であることができます。デフォルトは1で、すなわち各層が垂直方向に1つのセルを持つことを意味します。
  * `transforms`: 各セルの深さを変更するために使用できる関数または関数の配列。関数は次の引数を取る必要があります：`x`、`y`、`z`、`x_c`、`y_c`、`i`、`j`、`k`。ここで、`x`、`y`、`z`は変更される点の座標、`x_c`と`y_c`はその点が属するセルの中心の座標、`i`と`j`はI/J方向のセルのインデックス、`k`はK方向のセルのインデックスです。関数は点の新しい深さを返す必要があります。
  * `xy_transform`: 各柱のXおよびY座標を変更するために使用できる関数。関数は次の引数を取る必要があります：`x`、`y`、`i`、`j`、`z_t`、`z_b`。ここで、`x`と`y`は線の元のXおよびY座標、`i`と`j`はI/J方向の線のインデックス、`z_t`と`z_b`は線の上部と下部の深さです。関数は線の新しいXおよびY座標を返す必要があります。
