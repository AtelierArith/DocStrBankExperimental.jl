```
parse_data_file(filename; units = :si)
parse_data_file(filename; units = :field)
data = parse_data_file("MY_MODEL.DATA")
```

`filename`で指定されたパスの.DATファイル（業界標準の入力ファイル）をStringキーのDictに解析します。単位は、`units = :field`のような代替単位系を渡さない限り、厳密なSIに変換されます。`units = nothing`を設定すると、単位変換はスキップされます。JutulDarcy.jlのシミュレーターは、単位系が内部的に一貫していることを前提としています。JutulDarcy.jlでシミュレーションを行いたい場合は、SI単位に解析することを強くお勧めします。

このフォーマットに関する最良の公開ドキュメントは、Open Porous Media (OPM) プロジェクトのウェブページから入手できます: [OPM Flow manual ](https://opm-project.org/?page_id=955)。

# キーワード引数

  * `warn_parsing=true`: キーワードがパーサーによってサポートされていない（または部分的にサポートされている）場合に警告を生成します。
  * `warn_feature=true`: キーワードがサポートされているが、`JutulDarcy.jl`の数値ソルバーでのサポートが限られているか欠けている場合に警告を生成します。
  * `units=:si`: 出力に使用される単位系を示すシンボル。この値を`nothing`に設定すると、変換なしで値が返されます。つまり、入力ファイルに正確に含まれているものです。`:si`は厳密なSIを使用します。他の選択肢は`:field`と`:metric`です。`:lab`は現在サポートされていません。
  * `verbose=false`: 解析の進行状況に関する詳細な出力を生成します。大きなファイルの場合、多くの出力が生成されます。パーサーが失敗する場所や多くの時間を費やす場所を特定するのに役立ちます。

# 注意

この関数は、さまざまなシミュレーターに存在するキーワードのごく一部をカバーしています。サポートが限られている既知のキーワードに対して、パーサーと数値ソルバーの両方でのサポートレベルを示す警告が表示されます。新しいキーワードに関するプルリクエストは歓迎します！

SUMMARYセクションは、シミュレーションケースを定義するために必須ではない大量の利用可能なキーワードのためにスキップされます。
