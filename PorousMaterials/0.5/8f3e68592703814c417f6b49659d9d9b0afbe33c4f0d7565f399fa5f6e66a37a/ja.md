```
grid = energy_grid(crystal, molecule, ljforcefield; resolution=1.0, temperature=298.0, n_rotations=750)
```

結晶の単位セル上に、`crystal.box`の分数座標で定期的に配置された点の正規グリッドを重ねます。`n_gridpts`はa、b、c方向のグリッドポイントの数を決定します（0および1の分数座標を含む）。分数座標0と1はグリッドに含まれますが、冗長です。次に、各グリッドポイントで、分子の質量がそのポイントに中心を置いたときのアンサンブル平均ポテンシャルエネルギーを計算します。平均はボルツマン加重回転にわたって取られます。

アンサンブル平均は回転に対するボルツマン平均です： - R T log ⟨e⁻ᵇᵁ⟩

# 引数

  * `crystal::Crystal`: 分子のエネルギーグリッドを計算する結晶。`grid.box`は`framework.box`になります。
  * `molecule::Molecule`: エネルギーグリッドを求める分子
  * `ljforcefield::LJForceField`: 分子-結晶相互作用を計算するための分子モデル
  * `resolution::Union{Float64, Tuple{Int, Int, Int}}=1.0`: グリッドポイント間の最大距離（Å単位）または各次元のグリッドポイントの数を指定するタプル。
  * `n_rotations::Int`: 与えられたグリッドポイントに中心を置いた分子の自由エネルギーを見つけるためにモンテカルロシミュレーションで行うランダム回転の数。この値は、複数のレナード・ジョーンズ球から構成される分子にのみ関連します。
  * `temperature::Float64`: 回転が必要な分子の自由エネルギーを計算する温度。低温は、そのポイントでの最小ポテンシャルエネルギーの回転構成を過度に強調します。
  * `units::Symbol`: `:K`または`:kJ_mol`のいずれか、エネルギーが返される`Grid`に保存される単位。
  * `center::Bool`: グリッドの座標をシフトして、原点が単位セル`crystal.box`の中心になるようにします。
  * `verbose::Bool=true`: 一部の情報を印刷します。

# 戻り値

  * `grid::Grid`: システムのポテンシャルエネルギーを含むグリッドデータ構造
