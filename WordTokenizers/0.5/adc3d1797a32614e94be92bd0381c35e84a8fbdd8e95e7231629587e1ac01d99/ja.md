"... 任意の生テキストに対してペンツリーバンクのトークン化を生成するためのものです。ええ、もちろん" ロバート・マッキンタイアの引用

トークン化は、収縮形を分けるなど、いくつかのことを行います: "shouldn't" は ["should", "n't"] になります。他のほとんどの句読点は &'s になります。例外は、触れられないピリオドです。入力は単一の文であるべきですが、そうでなくても比較的問題ないでしょう。正確に何のために使いたいかによります。

これは元の sed スクリプトの直接的（自動的）な翻訳です。

正確に何をするかをいじりたい場合は、実際には非常に簡単です。このリポジトリから penn.sed ファイルをコピーし、心のままに修正してください。コメントアウトを解除できる行もいくつかあります。新しいトークナイザーを生成するには、次のようにします:

```
@generated function custom_tokenizer(input::AbstractString)
    generate_tokenizer_from_sed(joinpath(@__DIR__, "custom.sed"))
end
```
