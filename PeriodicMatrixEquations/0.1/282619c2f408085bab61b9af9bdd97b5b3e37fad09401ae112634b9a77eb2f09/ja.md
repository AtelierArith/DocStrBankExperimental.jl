```
pgclyap(A, C[, K = 1]; adj = false, solver, reltol, abstol) -> X
```

周期的リャプノフ微分方程式の周期的生成器を計算します。

```
.
X(t) = A(t)X(t) + X(t)A(t)' + C(t) , adj = false の場合、
```

または 

```
 .
-X(t) = A(t)'X(t) + X(t)A(t) + C(t) , adj = true の場合。
```

周期行列 `A` と `C` は同じ型、同じ次元、かつ整合性のある周期を持ち、さらに `C` は対称でなければなりません。 `K` は連続時間問題の離散化に使用されるグリッドポイントの数を指定します（デフォルト: `K = 1`）。 `A` と `C` が `PeriodicTimeSeriesMatrix` または `PeriodicSwitchingMatrix` の型である場合、`K` は2つの連続する切り替え時間値の間で使用されるグリッドポイントの数を指定します（デフォルト: `K = 1`）。  

`A` と `C` が `PeriodicFunctionMatrix`、`HarmonicArray`、`FourierFunctionMatrix` または `PeriodicTimeSeriesMatrix` の型である場合、結果の `X` は *周期的時系列行列* として決定される周期的生成行列のコレクションであり、`N` コンポーネントを持ちます。ここで、`A` と `C` が定数行列である場合は `N = 1`、それ以外の場合は `N = K` です。 `A` と `C` が `PeriodicSwitchingMatrix` の型である場合、`X` は *周期的切り替え行列* として決定される周期的生成行列のコレクションであり、その切り替え時間は `A` と `C` の切り替え時間の和集合に含まれるユニークな切り替え時間です。 `K > 1` の場合、和集合内の2つの連続する切り替え時間ごとに `K` の等間隔の値の洗練されたグリッドが使用されます。      

`X` の周期は `A` と `C` の最小公倍数の整合性のある周期に設定され、サブ周期の数はそれに応じて調整されます。 `X` の任意のコンポーネント行列は、適切な微分方程式を統合することによって完全な周期にわたって解を生成するために使用される有効な初期値です。 [1] の多重射撃法が最初に使用され、連続時間の周期的リャプノフ微分方程式をグリッドポイントで生成器解が満たす離散時間の周期的リャプノフ方程式に変換し、その後、[2] の周期的シュール法を使用して適切な離散時間の周期的リャプノフ方程式を解くことによって解を計算します。 

連続時間問題を離散時間問題に変換するために使用されるODEソルバーは、キーワード引数 `solver` を使用して指定でき、必要な相対精度 `reltol`（デフォルト: `reltol = 1.e-4`）および絶対精度 `abstol`（デフォルト: `abstol = 1.e-7`）とともに指定されます。 希望する相対精度 `reltol` に応じて、`reltol >= 1.e-4` の場合は低次のソルバーが使用され、一般的に非常に効率的ですが、精度は低くなります。 `reltol < 1.e-4` の場合は、高精度の要求に対応できる高次のソルバーが使用されます。 

[OrdinaryDiffEq.jl](https://github.com/SciML/OrdinaryDiffEq.jl) パッケージから以下のソルバーを選択できます：

`solver = "non-stiff"` - 非剛性問題用のソルバーを使用します（`Tsit5()` または `Vern9()`）;

`solver = "stiff"` - 剛性問題用のソルバーを使用します（`Rodas4()` または `KenCarp58()`）;

`solver = "auto"` - 剛性問題を自動的に検出するデフォルトのソルバーを使用します（`AutoTsit5(Rosenbrock23())` または `AutoVern9(Rodas5()`）。 

離散時間問題の行列の並列計算は、複数の実行スレッドでJuliaを起動することによって代わりに実行できます。 実行スレッドの数は、`-t/--threads` コマンドライン引数を使用するか、`JULIA_NUM_THREADS` 環境変数を使用して制御されます。
