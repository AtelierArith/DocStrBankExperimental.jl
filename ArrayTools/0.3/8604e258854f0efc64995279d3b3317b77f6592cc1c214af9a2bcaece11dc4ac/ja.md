`RubberIndex`は、任意の数のインデックスを表すシングルトン型です。定数`..`は`RubberIndex()`として定義されており、配列のインデックス付けにおいて他のインデックスを左揃えまたは右揃えするために使用できます。例えば、`A`が`3×4×5×6`の配列であると仮定すると、以下の等式がすべて成り立ちます：

```
A[..]           == A[:,:,:,:]
A[..,3]         == A[:,:,:,3]
A[2,..]         == A[2,:,:,:]
A[..,2:4,5]     == A[:,:,2:4,5]
A[2:3,..,1,2:4] == A[2:3,:,1,2:4]
```

ご覧の通り、ラバーインデックス`..`の利点は、必要なコロンの数に応じて自動的に拡張されることです。式もより読みやすくなります。

ラバーインデックスは値の設定にも使用できます。例えば：

```
A[..] .= 1         # Aを1で埋める
A[..,3] = A[..,2]  # A[:,:,:,2]をA[:,:,:,3]にコピーする
A[..,3] .= A[..,2] # 同様だがより高速
A[2,..] = A[3,..]  # A[3,:,:,:]をA[2,:,:,:]にコピーする
A[..,2:4,5] .= 7   # A[:,:,2:4,5]のすべての要素を7に設定する
```

先頭/末尾のインデックスは、`CartesianIndex`型のカーテシアンインデックスとして指定できます。

!!! warning
    既知の制限が2つあります：

    1. `end`という予約語は、ラバーインデックスの*前*に指定された区間でのみ使用できますが、*後*には使用できません。この制限は、Juliaパーサーによるもので回避できません。
    2. ラバーインデックスの前に指定できるインデックスは最大9つまでです。これはソースコードを編集することで拡張できます。
