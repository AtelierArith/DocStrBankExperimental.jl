t8_ctree

この構造体は、面の隣接情報を含むローカルツリーのデータを保持します。ツリーから面へのインデックスは次のように計算されます。Fをcmeshの次元の任意のeclassの最大面数とすると、ttf % Fは面番号で、ttf / Fは向きです。(t8*eclass*max*num*faces) 向きは次のように決定されます。my*faceとother*faceを接続するツリーの2つの面番号とします。次のように主面を選択します：両方のツリーが同じ要素クラスを持つ場合、面番号が低い方の面が主面となります。異なるクラスに属する場合は、順序三角形 < 四角 < 六角 < テトラ < プリズム < ピラミッドに従って、クラスが低いツリーに属する面が主面となります。次に、主面の面コーナー0が他の面の面コーナーkに接続します。面の向きはkとして定義されます。クラスが等しく、my*face == other*faceの場合、どちらの面を主面として扱っても同じ結果になります。詳細については、https://arxiv.org/pdf/1611.02929.pdfを参照してください。

| フィールド          | 注釈                                                 |
|:-------------- |:-------------------------------------------------- |
| treeid         | このツリーのローカル番号。                                      |
| eclass         | このツリーのeclass。                                      |
| neigh_offset   | このオフセットをツリーのアドレスに加えると、face_neighborエントリの配列が得られます。  |
| att_offset     | このオフセットをツリーのアドレスに加えると、attribute_infoエントリの配列が得られます。 |
| num_attributes | このツリーの属性の数。                                        |
