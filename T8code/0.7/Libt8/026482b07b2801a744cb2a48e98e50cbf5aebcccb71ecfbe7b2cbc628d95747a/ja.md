t8_ctree

この構造体は、面の隣接情報を含むローカルツリーのデータを保持します。ツリーから面へのインデックスは次のように計算されます。Fをcmeshの次元の任意のeclassの最大面数とすると、ttf % Fは面番号であり、ttf / Fは向きです。（t8*eclass*max*num*faces）向きは次のように決定されます。my*faceとother*faceを接続するツリーの2つの面番号とします。次のようにmain*faceを選択します：両方のツリーが同じ要素クラスを持つ場合、面番号が低い方の面がmain*faceとなります。異なるクラスに属する場合は、順序三角形<四角、六角<テトラ<プリズム<ピラミッドに従って、低いクラスのツリーに属する面がmain*faceとなります。次に、main*faceの面コーナー0は、他の面の面コーナーkに接続します。面の向きはkとして定義されます。クラスが等しく、my*face == other*faceの場合、どちらの面をmain_faceとして扱っても同じ結果になります。詳細については、https://arxiv.org/pdf/1611.02929.pdfを参照してください。

| フィールド          | 注釈                                                 |
|:-------------- |:-------------------------------------------------- |
| treeid         | このツリーのローカル番号。                                      |
| eclass         | このツリーのeclass。                                      |
| neigh_offset   | このオフセットをツリーのアドレスに加えると、face_neighborエントリの配列が得られます。  |
| att_offset     | このオフセットをツリーのアドレスに加えると、attribute_infoエントリの配列が得られます。 |
| num_attributes | このツリーの属性の数。                                        |
