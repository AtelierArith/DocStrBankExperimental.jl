```
@precompile_all_calls f(args...)
```

`precompile` は、式内で発生するすべてのメソッド呼び出しを行います。`@precompile_all_calls` ブロック内のすべての呼び出し（直接または間接）は、事前コンパイルされます。

`@precompile_all_calls` には3つの重要な機能があります：

1. コードは、パッケージが事前コンパイルされているときのみ実行されます（つまり、`*.ji` 事前コンパイルキャッシュファイルが書き込まれているとき）。
2. インタープリターは無効になり、呼び出しがコンパイルされることが保証されます。
3. 直接および間接の呼び出し先は、他のパッケージで定義されたメソッドや、ランタイムでディスパッチされた呼び出し先に対しても事前コンパイルされます（Julia 1.8 以上が必要です）。

!!! note
    包括的な事前コンパイルを行うには、特定のメソッド/引数タイプの組み合わせの最初の使用が `@precompile_all_calls` 内で行われることを確認してください。

    詳細には、ランタイムでディスパッチされた呼び出し先は、型推論が実行されるときにのみキャプチャされ、最初の使用時にのみ推論されます。あなたのパッケージで定義されたメソッドにトレースバックされる推論可能な呼び出しと、その *推論可能な* 呼び出し先は、呼び出し先の「所有権」に関係なく事前コンパイルされます（Julia 1.8 以上）。

    したがって、この推奨は次のような場合にのみ重要です：

    ```
    - Base または他のパッケージで定義されたメソッドへの直接呼び出し OR
    - そのようなメソッドへの間接的なランタイムディスパッチ呼び出し。
    ```
