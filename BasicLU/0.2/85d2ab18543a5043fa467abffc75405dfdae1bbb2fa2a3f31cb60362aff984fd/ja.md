```
update(F::LUFactor, pivot::Float64) -> Float64
```

列の変更後に因子分解を更新します。列の位置と新しい列は、以前の呼び出しで[`solve_for_update`](@ref)に設定されている必要があります。

`pivot`は更新操作に対応するピボット要素です。すなわち、列`j`の`B`がベクトル`v`に置き換えられるとき、`pivot = (B\v)[j]`となります。`pivot`と再計算されたバージョンとの絶対差は、`getinfo(F, :pivotError)`を使用して取得できます。これが返り値でもあります。1e-8より大きいピボットエラーは、数値的不安定性を示し、再因子分解を示唆します。

再計算されたピボット要素が絶対ピボット許容値を下回るとエラーが発生します。この場合、更新は行われず、古い因子分解は有効のままとなります。
