```
normalize(::Union{typeof(¬), typeof(∧), typeof(∨)}, p)
```

与えられた命題を、最初の引数が [`not`](@ref)、[`and`](@ref)、または [`or`](@ref) のいずれかに応じて、否定形、結合形、または選言形の標準形に変換します。

正規化された命題の構文木を考えると、各葉ノードはリテラルであり、原子またはその否定のいずれかです。否定標準形の命題は、構文木の枝が `and` と `or` の演算子のみを含むように展開されます。結合形および選言形の標準形は、他の演算子に対して再帰的に `and` または `or` 演算子を分配することによって平坦化された否定標準形です。言い換えれば、リテラルの集合は節であり、結合形または選言形の標準形の命題は、それぞれ選言節の結合または結合節の選言です。

結合形と選言形は、否定を除いて、*標準的*な形と呼ばれます。変換中に演算子を分配すると、構文木のサイズが指数関数的に増加する可能性があります。したがって、十分に大きな命題に対して標準形を計算することは実行不可能な場合があります。代わりに、[`tseytin`](@ref) 変換を使用して、与えられた命題に対して [`is_equisatisfiable`](@ref) な結合標準形の命題を見つけます。

!!! tip
    結合形と選言形の標準形の間で命題を変換することは、命題のサイズが指数関数的に増加するため、パフォーマンスが良くありません。すべての操作を命題に適用し、結果の命題を一度正規化するのが最もパフォーマンスが良いです。


# 例

```jldoctest
julia> @atomize normalize(∧, ¬(p ∨ q))
(¬p) ∧ (¬q)

julia> @atomize normalize(∨, p ↔ q)
(¬q ∧ ¬p) ∨ (q ∧ p)
```
