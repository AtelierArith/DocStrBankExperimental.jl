```
extract!(
    ndata :: AbstractArray{<:Real},
    odata :: AbstractArray{<:Real},
    ggrd  :: RectilinearGrid
) -> nothing
```

親 `GeoRegion` のデータを含む次元 N の配列 odata から、私たちが興味のあるサブ `GeoRegion` 内のデータのみを含む次元 N の別の配列 ndata に抽出します（N ∈ 2,3,4）。

これにより、次元が固定されている場合にメモリ空間を節約し、アロケーションを減らすための反復可能なインプレース修正が可能になります。

!!! warning
    続行する前に、1 次元が経度であり、2 次元が緯度であることを確認してください。ただし、3 次元および 4 次元（使用される場合）の順序は重要ではありません。


# 引数

  * `ndata` : アロケーションを最小限に抑えるために抽出されたデータのプレースホルダーとして意図された次元 N の配列。
  * `odata` : 抽出することに興味のある領域におけるグリッドデータを含む次元 N の配列。
  * `ggrd` : 抽出する内容に関する詳細情報を含む `RegionGrid`。
