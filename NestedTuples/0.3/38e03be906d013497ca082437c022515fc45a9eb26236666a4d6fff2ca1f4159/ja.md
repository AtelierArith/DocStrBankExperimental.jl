```
lazymerge(x::NamedTuple, y::NamedTuple)
```

`LazyMerge` 構造体を作成します。これは再帰的にマージされた NamedTuple のように振る舞いますが、構築がはるかに速くなります。

通常の `getproperty` に加えて、この構造体は静的キーによる `get` をサポートします：

```
(lm::LazyMerge).a == get(lm, static(:a))
```

特定の状況では、後者の方がはるかに速くなることがあります。

この構造体の元々の使用ケースは、特に確率的プログラミングの文脈で、名前付きタプルを名前空間として使用することをサポートすることです。そこで、観測データ用の（おそらくネストされた）名前付きタプルと、MCMC アルゴリズムの提案用の別の名前付きタプルを持つことが一般的です。したがって、マージは多くの回数実行されるループの本体内で行われます。
