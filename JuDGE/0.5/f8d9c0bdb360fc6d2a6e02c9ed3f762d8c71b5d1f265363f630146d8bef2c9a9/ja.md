```
state(model, variable, args...)
```

サブプロブレム `model` 内に状態変数 `variable` を定義します。すべてのサブプロブレムは同じ状態変数のセットを持つ必要があることに注意してください。これらの変数は、サブプロブレム間で持ち越される在庫をモデル化するために使用できます。

### 必須引数

`model` は、拡張変数を追加する JuDGE サブプロブレムです。

`variable` は、サブプロブレム内で作成される変数の名前であり、デフォルトでは連続的です。変数のセットを定義する場合は JuMP 構文に従います。サブプロブレム変数は、状態の変化に対応します。

### オプション引数

このマクロには、`@variable` マクロと同様に Con、Bin、または Int に設定できる第三の名前のない引数があります。

`state_name` は、マスタープロブレム内の状態変数の名前です。省略した場合、マスタープロブレム変数の名前はサブプロブレム変数と一致します（ただし、これは混乱を招く可能性があります。なぜなら、状態であるのはマスタープロブレム変数のみだからです）。これがどのように実装されるべきかは、`inventory.jl` の例を参照してください。

`initial` は、ルートノードにおけるマスタープロブレムの状態変数の初期値です。

`lb` は、マスタープロブレム内の変数の下限（通常は省略されます）です。

`ub` は、マスタープロブレム内の変数の上限（通常は省略されます）です。

### 例

```
@state(sp, -50<=Δstock<=50, state_name=stock, lb=0, ub=200, initial=0) #マスター内に stock という状態変数を定義します
                                                                       #(初期値は 0 で、0 から 200 の値を取ることができる)、
                                                                       #およびサブプロブレム内の Δstock（在庫レベルを ±50 変更できる）。
```
