```
shutdown(model, variable, args...)
```

サブプロブレム `model` 内にシャットダウン変数 `variable` を定義します。すべてのサブプロブレムは同じセットのシャットダウン変数を持つ必要があることに注意してください。

### 必須引数

`model` はシャットダウン変数を追加する JuDGE サブプロブレムです。

`variable` は作成される変数の名前で、デフォルトでは連続的です。変数のセットを定義する場合は JuMP 構文に従います。

### オプション引数

このマクロには、`@variable` マクロと同様に Con、Bin、または Int に設定できる第3の名前のない引数があります。

`lag` は、シャットダウンが決定されてから利用できなくなるまでのシナリオ内のノードの数です。

`duration` は、シャットダウンが持続するシナリオ内の連続したノードの数です。

`lb` はマスタープロブレムにおけるこの変数の下限（通常は省略されます）です。

`ub` はマスタープロブレムにおけるこの変数の上限（通常は省略されます）です。

### 例

```
@shutdown(model, shut[1:5], Bin) #ラグなしで無制限の寿命を持つ5つのバイナリ変数の配列を定義します
@shutdown(model, shut[1:5,1:2]>=0, lag=1) #ラグ1で無制限の持続時間を持つ10の連続変数の行列を定義します
@shutdown(model, 0<=shut<=10, Int, duration=2) #ラグ0で持続時間2の単一整数変数を定義します
```
