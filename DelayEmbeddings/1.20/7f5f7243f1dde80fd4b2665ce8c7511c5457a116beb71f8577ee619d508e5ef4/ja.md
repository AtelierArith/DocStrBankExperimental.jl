```
genembed(s, τs, js = ones(...); ws = nothing) → dataset
```

`s`の一般化埋め込みを作成します。`s`は時系列または任意の`Dataset`であり、結果を新しい`Dataset`として返します。

一般化埋め込みは次のように機能します：

  * `τs`は遅延ベクトルの各エントリに使用される遅延時間を示します。`τs[1] = 0`であることが推奨されます。`τs`には*負のエントリ*も許可されています。
  * `js`は、遅延ベクトルのエントリに使用される`s`に含まれる時系列のどれを示します。`js`には重複したインデックスを含めることができます。
  * `ws`はオプションの重みで、各埋め込まれたエントリに重みを付けます（遅延ベクトルのi番目のエントリは`ws[i]`で重み付けされます）。提供される場合、`ws[1] = 1`であることが推奨されます。

`τs, js, ws`は長さ`D`のタプル（またはベクトル）であり、埋め込み次元と一致します。たとえば、入力軌道$s = [x, y, z]$を想像してください。ここで$x, y, z$は時系列（`Dataset`の列）です。`js = (1, 3, 2)`および`τs = (0, 2, -7)`の場合、各ステップ$n$で作成される遅延ベクトルは次のようになります。

$$
(x(n), z(n+2), y(n-7))
$$

`ws = (1, 0.5, 0.25)`を使用すると、次のようになります。

$$
(x(n), \frac{1}{2} z(n+2), \frac{1}{4} y(n-7))
$$

`js`はスキップ可能で、すべての遅延エントリに対してインデックス1（最初の時系列）にデフォルト設定されます。また、`s`が`Dataset`ではなく時系列の場合は影響を与えません。

[`embed`](@ref)も参照してください。内部的には[`GeneralizedEmbedding`](@ref)を使用します。
