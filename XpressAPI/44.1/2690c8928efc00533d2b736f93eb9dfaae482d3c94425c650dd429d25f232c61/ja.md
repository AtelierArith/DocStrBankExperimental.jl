```
XPRS_MAXCHECKSONMAXTIME
```

この制御は、TIMELIMIT（または廃止されたMAXTIME）制御を使用して終了する最適化ランを正確に再現する必要がある場合に使用されることを意図しています。（整数）

この制御は、時間的基準を使用してアルゴリズムソフトウェアを一貫して終了させることの固有の難しさのために必要です。制御値は、最適化プログラムが終了がアクティブになったチェックを含めて`TIMELIMIT`基準をチェックする回数に関連しています。制御を使用するには、ユーザーは最初に実行が返された後にCHECKSONMAXTIME属性の値を取得する必要があります。この属性値は、最適化ルーチンXPRSmipoptimizeへの最後の呼び出し中に最適化プログラムが`TIMELIMIT`基準をチェックした回数です。この属性値は、最適化プログラムが`TIMELIMIT`基準で終了した場合は負の値になります。実行の再現が同じ方法で終了することを保証するために、ユーザーは最初に`TIMELIMIT`がデフォルト値または大きな値に設定されていることを確認し、実行が再び`TIMELIMIT`で終了しないようにし、その後単に制御`MAXCHECKSONMAXTIME`を`CHECKSONMAXTIME`値の絶対値に設定する必要があります。

デフォルト値: `0`

値: 0 : 非アクティブ。 n>0 : 終了をトリガーする前に最適化プログラムが`TIMELIMIT`（または`MAXTIME`）基準をチェックすべき回数。
