```
XPRS_PRESOLVE
```

この制御は、メインアルゴリズムを開始する前にプリソルブを実行するかどうかを決定します。（整数）

プリソルブは、冗長な制約を検出して削除したり、変数の境界を引き締めたりすることで、問題を簡素化しようとします。場合によっては、この段階で不適合性が判断されることもあり、最適解が見つかることもあります。

デフォルト値: `1`

値: -1 : プリソルブが適用されますが、プライマル不適合性が検出された場合に問題が不適合であると宣言されることはありません。問題はLP最適化アルゴリズムによって解決され、不適合な解が返されますが、これは時には役立つことがあります。 0 : プリソルブは適用されません。 1 : プリソルブが適用されます。 2 : プリソルブが適用されますが、冗長な境界は削除されません。これにより、バリアアルゴリズムの効率が向上することがあります。 3 : プリソルブが適用され、冗長であると判断された境界は常に削除されます。
