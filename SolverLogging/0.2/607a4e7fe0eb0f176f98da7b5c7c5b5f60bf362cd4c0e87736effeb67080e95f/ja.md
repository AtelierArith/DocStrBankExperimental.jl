```
setentry(logger, name::String, type; kwargs...)
```

`logger`に新しいエントリ/フィールドを追加するか、既存のエントリ/フィールドを修正するために使用されます。

# 新しいエントリの追加

ロガーに新しいエントリを追加するには、一意の`name`を指定します。`type`は合理的なフォーマットのデフォルトを提供するために使用され、必ず含める必要があります。キーワード引数はフィールドの動作/フォーマットを制御します：

  * `fmt` フィールドのフォーマットを制御するために使用されるCスタイルのフォーマット文字列
  * `index` 出力内のエントリの列。負の数は末尾から挿入します。
  * `level` エントリの冗長性レベル。レベルが高いほど、出力頻度が低くなります。レベル0は常に出力されますが、ロガーが無効になっている場合は除きます。最小レベル1の使用を推奨します。
  * `width` 列の幅。データはこの幅に左揃えされます。
  * `ccrayon` 条件付きフォーマット用の[`ConditionalCrayon`](@ref)。

# 既存のエントリの修正

このメソッドは、`name`がすでにロガー内のフィールドである場合、既存のエントリを修正することもできます。この場合、`type`は省略できます。新しい設定で任意のキーワード引数を指定するだけです。
