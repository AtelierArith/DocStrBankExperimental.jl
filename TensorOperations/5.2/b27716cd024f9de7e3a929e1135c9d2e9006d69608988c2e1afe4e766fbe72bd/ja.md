```
@ncon(tensorlist, indexlist; order = ..., output = ...)
```

`tensorlist`（`Vector`または`Tuple`型）のテンソルを、`indexlist`で指定されたネットワークに従って収束させます。ここで、`indexlist`は`tensorlist`と同じ長さのリスト（すなわち`Vector`または`Tuple`）であり、そのエントリはそれぞれリスト（できれば`Vector{Int}`）で、各整数エントリは`tensorlist`内の対応するテンソルの対応するインデックス/次元のラベルを提供します。正の整数は収束させる必要があるインデックスをラベル付けするために使用され、したがって`indexlist`内の2つの異なるエントリに現れます。一方、負の整数は出力テンソルのインデックスをラベル付けするために使用され、1回だけ現れるべきです。

デフォルトでは、収束は収束させるインデックスが増加する整数でラベル付けされる順序で実行されます。すなわち、最初にラベル`1`に対応する収束が実行されます。出力テンソルは、減少する（負のため絶対値で増加する）インデックスラベルに対応するインデックス順序を持ちます。キーワード引数`order`と`output`を使用すると、これらのデフォルトを変更できます。

マクロ`@ncon`の利点は、関数呼び出し`ncon`に対して、`tensorlist`が単なる変数ではなく、呼び出しサイトで実際のリスト（括弧付きのタプルまたは角括弧付きのベクトル）である場合、`@ncon`マクロが共役呼び出し（例：`conj(A)`）をスキャンし、これを単に`A`に置き換えますが、`ncon`に指定するための共役フラグの一致するリストを構築します。これにより、共役を事前に実行するコストを支払うことなく、テンソルの共役を指定するのがより便利になります。
