```
segmts(n::Int, m::Int; rep = 1, seed = nothing)
segmts(group::Vector, m::Int; rep = 1, seed = nothing)
```

"テストセット" 検証のための観測のセグメントを構築します。

  * `n` : データセット内の観測の総数。サンプリングは 1:`n` 内で実装されています。
  * `group` : 観測のブロックを定義するベクトル (n)。
  * `m` : 各セグメントで返されるテスト観測の数、または `group` が使用される場合のグループ数。

キーワード引数:

  * `rep` : サンプリングの繰り返し回数。
  * `seed` : `Random.MersenneTwister` ジェネレーターのための任意のシード。長さは `rep` と等しくなければなりません。`nothing` の場合、シードは各繰り返しでランダムになります。

各繰り返しのために、関数はモデルを検証するために使用できるテストセットを構築します。

`group` が使用される場合（長さ n のベクトルでなければなりません）、関数は観測の代わりに全体のグループ（= ブロック）をサンプリングします。このようなブロックサンプリングは、データがブロックによって構造化されている場合や、予測する応答がブロック内で相関している場合に必要です。これにより、一般化誤差の過小評価を防ぎます。

関数は `rep` 要素のリスト（ベクトル）を返します。リストの各要素は、サンプリングされた観測のインデックス（1:`n` 内の位置）のベクトルです。

## 例

```julia
using Jchemo 

n = 10 ; m = 3
rep = 4 
segm = segmts(n, m; rep) 
i = 1
segm[i]
segm[i][1]

n = 10 
group = ["A", "B", "C", "D", "E", "A", "B", "C", "D", "E"]    # 観測のブロック
tab(group)  
m = 2 ; rep = 4 
segm = segmts(group, m; rep)
i = 1 
segm[i]
segm[i][1]
group[segm[i][1]]
```
