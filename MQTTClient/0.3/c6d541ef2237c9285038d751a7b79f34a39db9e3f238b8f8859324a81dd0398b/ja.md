```
クライアント
```

JuliaのMQTTクライアントは、デバイスとMQTTブローカー間のネットワーク通信を容易にします。接続の管理、メッセージ処理、通信の状態を維持します。クライアントは、3つの主要なループを通じて動作します。読み取りループはブローカーからの受信メッセージをリッスンし、指定されたハンドラーを使用して処理します。書き込みループは、キューからブローカーにパケットを送信し、ソケットロックを使用してスレッドセーフを確保します。そして、キープアライブループは、接続を維持し、切断を検出するために定期的にブローカーにpingリクエストを送信します。このクライアントは、共有変数のスレッドセーフを確保するために原子操作を使用し、効率的でノンブロッキングな操作のために非同期タスク管理をサポートします。

# フィールド

  * `state::UInt8`: クライアントの状態。
  * `on_msg::TrieNode`: トピックをコールバック関数にマッピングするトライ。
  * `keep_alive::UInt16`: キープアライブ時間（秒）。
  * `last_id::UInt16`: 最後に使用されたパケット識別子。
  * `in_flight::Dict{UInt16, Future}`: パケット識別子を未来にマッピングする辞書。
  * `write_packets::AbstractChannel`: パケットを書き込むためのチャネル。
  * `socket`: ブローカーとの通信に使用されるソケット。
  * `socket_lock`: ソケットへのアクセスを同期するためのロック。
  * `ping_timeout::UInt64`: pingタイムアウト（秒）。
  * `ping_outstanding::Atomic{UInt8}`: 未処理のpingリクエストの数を表す原子カウンター。
  * `last_sent::Atomic{Float64}`: 最後に送信されたパケットのタイムスタンプを表す原子浮動小数点数。
  * `last_received::Atomic{Float64}`: 最後に受信されたパケットのタイムスタンプを表す原子浮動小数点数。

# コンストラクタ

`Client(ping_timeout::UInt64=UInt64(60))`は、指定されたpingタイムアウト（デフォルト: 60秒）で新しい`Client`オブジェクトを構築します。
