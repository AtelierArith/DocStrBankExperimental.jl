```
connect(client::Client, connection::Connection)
```

指定された [`Client`](@ref) および [`Connection`](@ref) オブジェクトを使用して、MQTT ブローカーへの同期接続を確立します。この関数は [`connect_async`](@ref) をラップし、接続プロセスが完了するのを待ちます。

# 引数

  * `client::Client`: MQTT クライアントインスタンス。
  * `connection::Connection`: ブローカーに接続するために使用される接続情報。

# 戻り値

  * 接続プロセスが完了した後の結果。

connect 関数は、MQTT クライアントと MQTT ブローカーの間に接続を確立する責任があります。クライアントの状態を初期化し、必要な通信チャネルを設定し、MQTT プロトコルに従って接続ハンドシェイクを処理します。呼び出されると、connect はまずクライアントの状態とリソースが適切に初期化されていることを確認します。これには、クライアントの状態のリセット、ソケット接続の設定、および通信に必要なチャネルとロックの作成が含まれます。次に、関数は接続を維持し、メッセージが適切に送受信されることを保証するために重要な読み取り、書き込み、およびキープアライブループを管理するために必要な非同期タスクを開始します。

さらに、connect 関数は MQTT プロトコルハンドシェイクの詳細を処理します。クライアント ID、ユーザー資格情報、およびオプションのウィルメッセージなどの詳細を含む CONNECT パケットを構築して送信します。このハンドシェイクプロセスにより、ブローカーはクライアントを認識し、指定されたパラメータに従ってセッションを設定します。同期 connect 関数は接続プロセスが完了するまでブロックされ、非同期操作を直接管理する必要なく接続を確立するための簡単な方法を提供します。これにより、ブローカーに接続してすぐに通信を開始するためのシンプルでブロッキングな呼び出しを必要とするアプリケーションに適しています。

## 例

```julia
client, connection = MakeConnection("127.0.0.1", 1883; client_id="mqtt_client_1")
result = connect(client, connection)
```
