```
aws_s3_meta_request_pause(meta_request, out_resume_token)
```

注意: 一時停止は現在、アップロードリクエストのみサポートされています。進行中のアップロードを一時停止するには、[`aws_s3_meta_request_pause`](@ref)()を呼び出す必要があります。これにより、再開トークンが返されます。このトークンは、一時停止時の操作の状態を照会するために使用できます。一時停止されたアップロードを再開するには、メタリクエストオプション構造体メンバー[`aws_s3_meta_request_options`](@ref).resume*tokenに再開トークンを指定します。アップロードは、同じクライアントまたは異なるクライアントのいずれかから再開できます。アップロードの再開に関するコーナーケースは次のとおりです: - アップロードはMPUではない - AWS*ERROR*UNSUPPORTED*OPERATIONで失敗 - MPUが作成される前に一時停止 - NULL再開トークンが返される。NULL再開トークンはアップロードの再起動に相当します - 部分転送の途中で一時停止 - 再開トークンを返します。新しい部分アップロードのスケジューリングが停止します。 - completeMPUが開始された後に一時停止 - 再開トークンを返します。S3がそのトークンで再開する際に関連するMPU IDを見つけられず、アップロードされた部分の数が合計部分の数と等しい場合、操作はノーオプです。そうでなければ、操作は失敗します。注意: ノーオプの場合、呼び出しは成功し、終了/シャットダウンリクエストコールバックが発火しますが、ヘッダーコールバックは発火しません。注意: キャンセルと同様に、一時停止はすでにフライト中のリクエストをキャンセルせず、部分が一時停止が要求された後に完了する可能性があります。

# 引数

  * `meta_request`: 一時停止されるアップロードの[`aws_s3_meta_request`](@ref)へのポインタ
  * `resume_token`: 再開トークン

# 戻り値

AWS*OP*ERRまたはAWS*OP*SUCCESSのいずれか

### プロトタイプ

```c
int aws_s3_meta_request_pause( struct aws_s3_meta_request *meta_request, struct aws_s3_meta_request_resume_token **out_resume_token);
```
