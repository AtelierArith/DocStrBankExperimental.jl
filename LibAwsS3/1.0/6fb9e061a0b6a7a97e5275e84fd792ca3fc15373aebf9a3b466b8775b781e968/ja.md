```
aws_s3_meta_request_increment_read_window(meta_request, bytes)
```

フロー制御ウィンドウを増加させ、レスポンスデータのダウンロードを続行します。

クライアントが `enable_read_backpressure` を true に設定して作成された場合、各メタリクエストにはレスポンスボディデータがダウンロードされるにつれて縮小するフロー制御ウィンドウがあります（ヘッダーはウィンドウのサイズに影響を与えません）。クライアントの `initial_read_window` は、各メタリクエストのウィンドウの開始サイズを決定します。メタリクエストのフロー制御ウィンドウが 0 に達すると、これ以上のデータはダウンロードされません。`initial_read_window` が 0 の場合、ウィンドウが増加するまでリクエストは開始されません。高いダウンロードスループットを維持するためには、より大きなウィンドウを維持する必要があります。ウィンドウが複数のパーツを保持できるほど大きくない限り、パーツは並行してダウンロードできません。メモリにバッファリングされるデータの量を制限するためには、より小さなウィンドウを維持します。

`enable_read_backpressure` が false の場合、この呼び出しは効果がありません。バックプレッシャーは適用されず、データは可能な限り速くダウンロードされています。

警告: この機能は実験的です。現在、バックプレッシャーは複数のパーツに分割された GetObject リクエストにのみ適用されており、ウィンドウが 0 に達した後でもデータを受け取ることがあります。
