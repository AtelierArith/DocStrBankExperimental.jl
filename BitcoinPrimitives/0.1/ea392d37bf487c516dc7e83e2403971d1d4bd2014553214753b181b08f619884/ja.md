生のトランザクションフォーマットといくつかのピアツーピアネットワークメッセージは、次のデータのバイト数を示すために可変長整数の一種を使用します。Bitcoin Coreコードはこれらの可変長整数をcompactSizeと呼びます。他の多くの文書ではそれらをvar_intまたはvarIntと呼びますが、これは他の可変長整数エンコーディング（例えば、データをディスクにシリアライズするためにBitcoin Coreで使用されるCVarIntクラス）と混同するリスクがあります。トランザクションフォーマットで使用されるため、compactSize符号なし整数のフォーマットはコンセンサスルールの一部です。0から252までの数値に対して、compactSize符号なし整数は通常の符号なし整数のように見えます。0xffffffffffffffffまでの他の数値に対しては、長さを示すためにバイトが数値の前に付加されますが、それ以外は数値はリトルエンディアン順の通常の符号なし整数のように見えます。

|                                       値 | 使用バイト数 |                フォーマット |
| ---------------------------------------:| ------:| ---------------------:|
|                          >= 0 && <= 252 |      1 |               uint8_t |
|                     >= 253 && <= 0xffff |      3 | 0xfdの後に数値をuint16_tとして |
|             >= 0x10000 && <= 0xffffffff |      5 | 0xfeの後に数値をuint32_tとして |
| >= 0x100000000 && <= 0xffffffffffffffff |      9 | 0xffの後に数値をuint64_tとして |

例えば、数値515は0xfd0302としてエンコードされます。
