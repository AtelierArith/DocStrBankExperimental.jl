空間情報は、座標を要素とする単一の次元 `Coord` によって表されます。すなわち、2要素の `SVector(longitude, latitude)` です。各座標は空間内の任意の多角形の中心を表します。パフォーマンスの理由から、各多角形の実際の限界は次元に含まれていません。

[`spaceagg`](@ref) のような統計関数では、座標の入力が **同じ面積をカバーしている** と仮定されています。もしそうでない場合は、カバーされる面積に対応する追加の重みベクトルを提供することができます。

この次元は、緯度範囲によるインデックス付けも可能です。例えば、次のようにできます。

```julia
A # `Coord` 次元を持つ `ClimArray`
A[Coord(Lat(-30..30))]
```

ClimateBase.jl のほとんどの関数は、座標が緯度でソートされていると暗黙的に仮定しています。次のコードでこれを実現できます。

```julia
A # `Coord` 次元を持つ `ClimArray`
coords = gnv(dims(A, Coord))
si = sortperm(coords; by = reverse)
A = A[Coord(si)]
```

**これは [`ncread`](@ref) によって自動的に行われます。**
