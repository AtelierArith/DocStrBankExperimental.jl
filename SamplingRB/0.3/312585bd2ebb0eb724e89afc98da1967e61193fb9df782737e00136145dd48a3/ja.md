```
cutting_planes_cvar(B::Vector{Float64}, alpha::Float64, rel_losses::Array{Float64,2};
                    tol::Float64=1e-6, maxiters::Int=1000, ub_w::Float64=2000., debug::Int=0, normalize::Bool=true)
```

CV@R-alphaリスクバジェットポートフォリオを構築するために、カッティングプレーン法を使用して資産の投資ウェイトを計算します。

これにより、rel_lossesの相対損失のシミュレーションを小さなサブプロブレムに分解し、数千の実現にスケールすることができます。ポートフォリオは、各資産jが総リスクにB[j]で寄与するように構成されており、これはリスクアペタイトとも呼ばれます。

アルゴリズムは、（絶対または相対の）許容誤差tol内で証明可能な最適性ギャップに達するか、maxiters回の反復を経た後に停止します。収束に失敗した場合です。

最初に、アルゴリズムはウェイトのバウンディングボックスが必要です。対数がすでにw > 0を示唆しているため、すべてのウェイトに対して十分に大きな上限ub_wを使用します。アルゴリズムがそのような大きなウェイトを持つ解に収束した場合は、健全性を確認する必要があります。最も一般的なケースは、問題が下に無制限である場合で、たとえばalphaが十分に大きくない場合です。

debug >= 1は、反復番号、ギャップ、およびウェイトの変化を印刷します。debug >= 2は、現在の上限と下限、および試行ポイント（ウェイト、V@R）も印刷します。debug >= 3は、リスクバジェッティング制約と第一段階のステータスも印刷します。

normalize == trueの場合、ウェイトの合計が1になる三つ組（result, weights, V@R-alpha）を返します。
