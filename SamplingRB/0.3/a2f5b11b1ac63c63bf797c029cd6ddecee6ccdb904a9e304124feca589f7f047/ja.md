```
cutting_planes(B::Vector{Float64}, rel_losses::Array{Float64,2},
               risk_measure::AbstractRiskMeasure;
               tol::Float64=1e-6, maxiters::Int=1000, ub_w::Float64=2000., debug::Int=0, normalize::Bool=true)
```

リスクバジェッティングポートフォリオを構築するために、カッティングプレーン法を使用して資産の投資ウェイトを計算します。このポートフォリオは、各資産 j が総リスクに B[j] を寄与するように構成されており、これはリスクアペタイトとも呼ばれます。

リスク測定の評価が効率的であれば、これは相対損失のシミュレーションの数千の実現にスケールします。

アルゴリズムは、（絶対または相対）許容誤差 tol 内で証明可能な最適性ギャップに達するか、maxiters 回の反復後に収束に失敗した場合に停止します。

最初に、アルゴリズムはウェイトのバウンディングボックスが必要です。対数がすでに w > 0 を示唆しているため、すべてのウェイトに対して十分に大きな上限 ub_w を使用します。アルゴリズムがそのような大きなウェイトを持つ解に収束した場合は、健全性を確認する必要があります。最も一般的なケースは、リスク測定が過度に保守的な場合のように、問題が下に無制限であるときです。

debug >= 1 は反復番号、ギャップ、ウェイトの変化を印刷します。debug >= 2 は現在の上限と下限、および試行ウェイトも印刷します。debug >= 3 はリスクバジェッティング制約と第一段階のステータスも印刷します。

normalize == true の場合、ウェイトの合計が1になるタプル (result, weights) を返します。
